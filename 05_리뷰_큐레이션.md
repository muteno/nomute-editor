# 05 · 리뷰·큐레이션 (취향 코퍼스 적립)

> 목적: 매 생성 후 **운영자가 베스트로 고른 것만** 최소·신호 단위로 모아, 나중에 few-shot 취향연료로 쓴다.
> ⚠️ 원칙: **통째 저장 금지.** 단순 카드/이미지/텍스트 전부 저장 = 신호 0. **"선택한 것 + 앵글 + (있으면) 왜"**만.

## 언제 — ③ 피드백 진입 (발사 후, 응답이 세션을 연다)
**② 발사(생성·합성) 완료 후**, 에디터가 **첫 질문 1개만** 띄운다: *"꽤 괜찮은 결과물 있었어?"* (= 아래 Q1)
- **무응답/닫기 = 피드백 없음** (코퍼스 미적립, 그냥 끝).
- **'있다' 응답 시에만** 본 리뷰(Q2~)가 시작된다. → 첫 질문이 진입 게이트, 운영자의 답이 세션을 연다.

## 리뷰 흐름 — 객관식 (Q1·Q2 버튼, 번호는 타이핑)
에디터가 단계별로 묻는다. Claude Code면 **AskUserQuestion(버튼 2~4개)으로 클릭 선택.**

**Q1. 베스트 있어?** → 버튼 `[있다]` `[없다(pass)]`
- **'있다' 누르는 순간 → 카드 보기 띄움**: 생성된 카드를 *번호 + 테마 한 줄* 리스트 + Drive 폴더 링크로 보여줘 운영자가 보고 고르게.
  예) `1 소멸·학교 / 2 선순환 / 3 권한 / 4 균형 / 5 예산삭감 / 6 거버넌스` + [폴더 링크]

**Q2. 뭐가 좋아?** → 버튼 `[이미지]` `[텍스트]` `[둘 다]` `[앵글/훅]` (복수 OK)

**Q3. 몇 번?** → **번호 타이핑** ("3" 또는 "3,5").
- ⚠️ AskUserQuestion은 버튼 2~4개 고정 = **스크롤/5개+ 픽커 없음.** 그래서 카드 수 무관하게 **번호는 타이핑이 제일 깔끔**(한 줄). (정 버튼 원하면 ≤4장=버튼 / 5장+='1-4? / 5-?' 분할.)

**Q4. (선택) 왜 좋았어?** → 한 줄 타이핑, 없으면 스킵.

> ⚠️ "기타/없음" 잡옵션 안 넣음. 운영자는 **[있다] → 카드 보고 → [이미지] → 번호**만. 손 최소.
> 나머지(앵글 한 줄·프롬프트 레시피=카메라·조명·맺힘점·파일명)는 **에디터가 자동 채워** `_corpus.md` 적립 = 운영자는 "잡아주기"만.

## 적립 (운영자가 픽하면)
1. `_산출/_베스트/_corpus.md`에 **엔트리 append** (아래 포맷, 최신 위로).
2. 픽한 **이미지 파일을 `_산출/_베스트/`로 복사**(파일명 그대로).
3. ⚠️ 픽한 항목 + 앵글 + (있으면) 왜만. 나머지 버린다.

## 엔트리 포맷
```
## YYMMDD · {주제}
- 앵글: {에디터 앵글 한 줄}   | 화풍/악센트: {A/B} / {#0FFD02 or 상징색}
### 픽
- 🖼️ 카드{N} 이미지 — 카메라 {S··/H··/O··/L·· 코드} · 조명 {LGT··} · 맺힘점 "{한 줄}" · 파일 {장면NN.png}
  - 왜: {운영자 메모 or 빈칸}
- ✍️ 카드{N} 텍스트 — "{그 카피}"
  - 왜: {메모}
- 🎯 앵글 — "{그 앵글/훅}"
  - 왜: {메모}
```
> 이미지 픽엔 *프롬프트 레시피*(카메라/조명/맺힘점)를 꼭 남긴다 — 그래야 "왜 좋은 그림이 나왔나"가 재현 가능. 이미지 파일만 모으면 신호 0.

## 나중에 — few-shot 활성화 (차별화 점화)
코퍼스가 **~10–15 픽** 쌓이면 → 그 앵글·맺힘점·카피·프롬프트 패턴을 지침/라이브러리에 **"우리 취향 예시(few-shot)"**로 주입 → 단일런이 취향 zone으로 수렴(variance↓, 검수↓). 그 전까진 적립만.
- **이 코퍼스 = 너만의 취향 자산.** 모델·경쟁자가 못 베끼는 백그라운드. 쌓일수록 "딸깍 = 네 취향"에 수렴.
