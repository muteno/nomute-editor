전송 버튼 상태머신

비용이 드는 전송(카드뉴스 발사·합성·생성처럼 러너/API를 쓰는 액션)은 한 흐름으로 고정한다 — 클릭 → 재확인 → 전송(프로그레스) → 완료 → 원복. 버튼이 매 단계 살아 움직이고, 어느 메뉴(comp·ly·k·뉴스)든 색·동작·버튼 안 라벨까지 동일해야 한다. 정본 = viewer 수집함 .sc-pick(arm→firing 게이지→done).

1 라이브 — 직접 눌러보기

버튼을 눌러 흐름을 끝까지 밟아봐. 재확인 상태에서 다시 눌러야 비용이 나가는 전송이 실행된다(오발사 방지).

대기→ 재확인→ 전송 중→ 완료↺ 원복
눌러서 시작 — 첫 클릭은 재확인으로만 바뀜(아직 전송 안 함)

2 5단계 정지 프레임 + 스펙

각 상태의 정확한 색·라벨·지속시간. 게이지는 좌→우 1방향, 완료는 솔리드 채움 + 체크 pop.

STATE 1

대기 (idle)

accent-dim 글래스
border .4
라벨 "전송"

STATE 2

재확인 (arm)

앰버 색변경
3초 무동작→대기 복귀
라벨 "재확인"

STATE 3

전송 (firing)

초록 게이지 좌→우
.72s · 클릭잠금
라벨 "전송 중…"

STATE 4

완료 (done)

솔리드 초록 채움
체크 pop · 1.2s 유지
라벨 "완료"

STATE 5

원복 (revert)

대기로 자동 복귀
다시 누를 수 있음
라벨 "전송"

3 버튼 안 라벨 — 단계별 고정

버튼 내용도 통일. 동사+상태로 한 단어 단위. 메뉴마다 다른 문구 쓰지 말 것.

단계버튼 라벨색지속
대기아이콘 + "전송" / "발사" / "합성" / "생성"accent-dim—
재확인"재확인"arm #ffd93d3s 타임아웃
전송"전송 중…"게이지 .72s실제 요청까지
완료"완료" + ✓solid accent1.2s

대기 라벨만 메뉴 동사로 바뀔 수 있다(전송/발사/합성/생성). 재확인·전송·완료 라벨은 전 메뉴 고정 — 이게 "버튼 안 내용 일치".

4 메뉴가 달라도 같은 버튼

전송 버튼은 메뉴 대표색을 따르지 않는다. 비용·위험 신호(초록↔앰버)는 전 도구 공통이라야 손이 기억한다. 메뉴색은 헤더·아이콘에만.

comp · 카드뉴스
ly · 영상 자막
k · 영상 프롬프트

대기 라벨만 다름(합성/전송/생성). 누르면 셋 다 똑같이 앰버 재확인 → 초록 게이지 → 완료 → 원복.

5 비용 전송 vs 단순 액션 — 구분

재확인 게이트는 비용·되돌리기 어려운 액션에만. 복사·미리보기 같은 공짜 액션에 끼우면 피로해진다.

비용 전송 (재확인 O)

러너 발사 · 카드 합성 · Gemini 생성 · API 호출 — 돈/시간이 나가거나 외부로 발사됨.

클릭 → 재확인 → 게이지 → 완료 → 원복

단순 액션 (재확인 X)

복사 · 파일 선택 · 미리보기 토글 — 공짜·즉시·되돌리기 쉬움.

클릭 → 즉시 ✓ → (1.5s) 원복

일회성(PICK 등)은 예외: 완료 후 원복하지 않고 소진 상태(회색 비활성)로 잠근다 — 같은 항목 중복 발사 방지.

AFTER +

완전판 — 실패·부분·멈춤·소진까지

위 1~5는 happy-path만 그렸다. 실코드(viewer/index.html)엔 확인 게이트(짧음)와 작업 수명주기(긺)가 별개로 돌고, 성공 외에 실패·부분·멈춤·흉터·소진이 1급 상태다. 게다가 진행 표시가 미결정 shine과 결정형 3세그 두 종류다. 이걸 전부 그린 교정판.

6 두 축으로 분리 — 확인 게이트 ↔ 작업 수명주기

기존 카드의 오류: 확인 게이지(.72s 코스메틱)를 "전송 진행"으로 오표기. 실제 생성은 분 단위 async라 둘은 다른 축이다.

A · 확인 게이트 짧음 · 오발사 방지 · 코스메틱
대기→ 재확인(앰버·3s)→ 게이지 .72s → 발사
B · 작업 수명주기 긺 · 서버 상태 반영 · 폴링
Picking…(진행)→ PICKED/done ↘일부만 생성됨 ↘Failed →소진(잠금)

핵심 교정: 작업 완료는 원복 안 함. PICKED는 소진(회색·비활성)으로 잠그고 탭하면 피드 이동(실코드 L505). "1.2s 후 원복"은 단순 액션(복사)에만.

7 라이브 — 성공 / 부분 / 실패 분기

발사 후 서버 결과에 따라 세 갈래. 버튼/바가 각각 어떻게 사는지 직접 돌려봐.

작업 진행 + 결과 분기
시나리오를 골라 발사 — 미결정 Picking… → 결정형 3세그 → 결과
요약프롬프트이미지

8 진행바 2종 — 미결정 vs 결정형

끝을 예측 못 하면 미결정 shine(좌→우 무한 빛쓸기), 단계를 셀 수 있으면 결정형 세그. 섞지 말 것.

미결정 (indeterminate)

대기열·분석처럼 ETA 모름. 무한 shine 1.2s. 라벨 Picking…. 비활성.

PICKING…

결정형 (determinate)

요약→프롬프트→이미지 3단. 완료 세그 채움, 진행 세그 빗금 펄스, 실패 세그 노랑+빨강 흉터.

✓진행대기

흉터(scar): 한 번이라도 깨진 단계는 성공·재시도 후에도 세그에 빨강 표식 영속(실코드 failedOnce L1596). "이 작업은 과거에 깨진 적 있음" 신뢰 신호.

9 실패·소진 = 죽은 버튼 아님 (진입점)

Failed는 disabled가 아니라 풀폭 위험 칩 + 컨텍스트 메뉴(롱프레스: 진단 다운로드·전문 입력). PICKED는 소진이지만 탭하면 피드로.

Failed 칩

노랑 빗금(위험 텍스처)+빨강 라벨. 풀폭. 롱프레스→실패사유/전문입력.

PICKED 소진

회색·테두리만. 중복 발사 잠금. 탭→해당 피드로 이동.

◑ 일부만 생성됨(fired_partial): 발사됐으나 이미지 미완. 재발사 버튼 노출. 18분째 미완이면 "인증 만료·렌더 실패" 경고로 강조(실코드 L2083).

10 라벨 사전 — 실코드 문구 1:1

버튼 안 라벨을 실코드와 일치시킨다. 내가 지어낸 문구(빨강 취소선)는 폐기, 실코드 문구(초록)로.

상태실코드 라벨(정본)비고
진행(미결정)전송 중… → Picking…대문자 P, 말줄임 …
완료·소진완료 → PICKED전부 대문자, 탭=피드
실패Failed / 실패(시간초과)stuck=시간초과 명시
부분일부만 생성됨재발사 유도 문구 동반

11 접근성 — 빠졌던 행

상태가 색·모션으로만 전달되면 스크린리더·운동장애 사용자가 놓친다. 상태마다 보조 채널을 박는다.

상황보조 채널
재확인 3s 타임아웃aria-live="polite"로 "다시 누르세요" 안내 · 타임아웃은 prefers-reduced-motion일 때 연장/해제
Picking…/firing 중aria-busy="true" + 라벨 텍스트 갱신(색만 X)
Failedrole="alert" 또는 live 영역에 실패 사유 · 빗금색만으로 의미 전달 금지
완료/PICKED체크 아이콘에 aria-label · 소진 버튼은 aria-disabled(탭 이동은 유지)
