노뮤트 UI 구성도 · 텍스트

부속 설명 규약

본문(텍스트 위계)이 아닌 UI를 거드는 보조 텍스트 4종 — label · sub · hint · status. 매 화면에서 크기·색·위치가 제각각이면 "어디까지가 설명이고 어디부터 내용인지"가 흐려진다. viewer 실사용을 정본으로 4종의 역할·톤·정렬을 못박는다. 공통 원칙 = 전부 좌측정렬, 본문보다 작고 mut, 활성/라이브일 때만 --accent.

1 보조 텍스트 4종

각 종은 역할이 1:1로 갈린다. 헷갈리면 "이건 무엇을 거드나"로 판정 — 칸 이름이면 label, 화면 부연이면 sub, 입력 안내면 hint, 진행/결과면 status.

2 어디에 두나 — 좌측정렬 기준

보조 텍스트는 거드는 대상의 좌측 라인에 맞춰 시작한다. 가운데정렬은 빈 상태(empty)·모달 안내처럼 "내용이 없을 때"만. 입력·항목 곁의 설명은 항상 좌측.

3 해야 / 하지 말 것

✓ 이렇게

  • 4종 모두 본문보다 작게 + mut. 활성/라이브만 accent.
  • label·hint·sub = 좌측정렬(거드는 칸 라인).
  • status = 진행 중이면 pulse, 결과는 배지 3색(SUCC·처리·FAIL).
  • 같은 역할엔 같은 종 — 칸 이름은 늘 label.

✕ 이러지 말 것

  • hint를 가운데정렬(입력 옆 설명은 좌측).
  • label·sub에 강조색 상시 칠하기(강조 인플레).
  • status를 본문 크기로 키워 내용과 혼동.
  • sub와 hint를 섞어 — 화면 부연(sub) ≠ 입력 안내(hint).
