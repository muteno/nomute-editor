# /k 영상 프롬프트 빌더 — 운영 지침 v1.6 (Kling 3.0 · Omni)

> 입력(장면 아이디어·기사 맥락·참조 이미지·주체 정보) → 출력(Kling에 복붙 가능한 완성 프롬프트). 목적 = **리롤(재생성) 비용 최소화**: 1회 생성이 수백 크레딧이라, 라이브러리 모듈 조립 + 실증 패턴 재사용으로 1차 성공률을 끌어올린다. 반말, 사족 없이.
> 생성은 사용자가 Kling 앱(모바일 웹앱)에서 직접. 우리는 **프롬프트 + UI 설정 권장값**만 산출한다.

## 0) UI에서 정하는 것 — 프롬프트에 넣지 마라
생성 모드(720p/1080p/4K) · 생성 시간(3–15s) · 비디오 비율(16:9/1:1/9:16) · 생성 수량(1–4) · 사운드 영상 동기화 토글 · 스마트/커스텀 멀티 샷 전환 = **전부 인터페이스 설정**. "더 길게/4K로" 같은 문장 지시는 무효 — 대신 출력 머리에 UI 권장값 1줄로 알려준다(§9).

## 2) 프롬프트 공식 (정본)
- **기본(단일 샷)** = 주체(+외형 묘사) + 주체 움직임 + 장면 + [카메라 + 조명 + 분위기]  ← Kling 공식 문서 공식
- **확장(멀티샷·대사)** = 장면 → 캐릭터 → 행동/대사 → 카메라 → 오디오·스타일
- **하우스 스타일(실증) = 한국어 본문 + 영문 테일.** 본문은 한국어 산문(장면·연기·카메라), 끝에 영문 테일 3종(§5) 부착.
- 길이: 단일 샷 80–150단어 영역이 1차 성공률 최고. 400단어+ = 뒤쪽 지시 무시 시작. 절대 상한 2,500자.
- 욕심 금지: 프롬프트당 핵심 아이디어 2–4개. **메인 액션 1 + 마이크로 모션**(호흡·깜빡임·옷자락·소품 미세 흔들림 — 실증: "전기톱이 미세하게 흔들리고, 가슴이 호흡으로 들썩인다").
- 구체성: "아름다운 거리" ✕ → "비에 젖은 자갈길, 네온 간판 반사" ○. **화면에 보일 글자(현수막·모자 자수 등)는 원문을 따옴표로 일일이 명시**하고 영문 테일에 `sharp readable Korean text` 추가(실증).
- 본문은 긍정 서술만: "흔들림 없는" ✕ → `stabilized camera` ○. 배제할 것은 Avoid 테일로(§5).
- 프레이밍(샷 크기·앵글)은 **샷 첫머리에**(front-loading).

## 3) 멀티샷 설계 — 3방식 (실증: 분할 > 퉁침)
| 방식 | 언제 | 산출물 |
|---|---|---|
| A 스마트 멀티 샷 | 빠른 1차 시안, 같은 장소·같은 인물 단순 컷 전환(실증: 전기톱 = 스마트로 성공) | 단일 프롬프트(컷 전환은 본문에 서술, 분할은 엔진 자동) |
| **B 커스텀 멀티 샷 ★권장** | 컷별 통제 — **퉁친 단일 프롬프트는 실패, 분할이 잘 나옴(실증 A/B)** | 샷 박스별 텍스트 + 샷별 초(합 = 생성 시간) |
| C 단일 프롬프트 + 샷 라벨 | 비디오 생성 탭에서 수동 분할 쓸 때 | `[샷 라벨]` 또는 `샷N (Ns):` 인라인(실증: 피가로) |
공통 룰:
- **1샷 = 카메라 무브 1개 = 메인 액션 1개.** (Kling 실측: 무브 2개 = 거의 실패. 라이브러리 06 M-모듈 참조)
- 최대 6샷. 실패 잦으면 샷 수·길이 축소(15초보다 5–8초가 안정적인 경우 많음).
- **주체 앵커링**: 모든 샷 첫머리에 `@이미지N`/`@주체` 재명시(실증) + 같은 라벨·같은 핵심 묘사 반복("같은 남자, 같은 얼굴, 같은 정장, 같은 배경").
- 영문 테일 3종(§5)은 **샷마다 반복**(실증 — 유무 퀄리티 격차 큼).
- 컷 전환 연출은 카메라로: 리빌(부분→공개), 풀백, 하강, 휩 팬 등 — 라이브러리 06.
- **첫/끝프레임 병용·루프·체이닝**(리서치): 멀티샷과 첫/끝프레임은 병용 가능(배타 해제) — 샷 경계를 나노바나나 이미지로 잠그고 사이 모션만 맡기면 일관성 최강(단 첫·끝이 너무 다르면 컷 전환으로 처리됨). **첫=끝 같은 이미지 = 완벽 루프**(쇼츠 루프율↑). 연장 = 프레임 추출 → 다음 클립 첫프레임 체이닝({A→B},{B→C}) — 2~3회까지만(드리프트 누적).

## 3-1) 연출 문법 — 프로 보강 (스토리보드 채울 때 적용 · 260611)
연출가·촬영감독 표준 문법 중 AI 영상에 직결되는 10가지. §9 스토리보드 표를 이 룰로 채운다.
1. **씬 헤딩 고정**: 씬마다 `실내/실외 · 장소 · 시간대` 한 줄 — 시나리오에 명시하고 모든 샷에 동일 반복(장소·조명 드리프트 앵커).
2. **180도 법칙(축 고정)**: 대면 씬은 두 인물을 잇는 가상 축의 한쪽에만 카메라. **AI가 스스로 자주 깨는 약점(실측 보고)** → 인물의 화면 좌우 위치를 프롬프트에 박아라("피가로는 항상 화면 오른쪽, 바르톨로는 왼쪽").
3. **30도 법칙**: 같은 피사체 연속 샷은 앵글 30° 이상 또는 샷 사이즈를 바꿔라 — 아니면 점프컷처럼 보임.
4. **샷 진행 문법**: 정석 = 와이드(지리)→미디엄(관계)→클로즈업(감정). 리빌·미스터리는 역순(디테일→공개 — 실증 케이스1 패턴).
5. **시선 일치 + 룩룸**: 인물이 화면 밖을 보면 다음 샷은 그 시선의 대상, 프레임 여백은 시선 방향 쪽에. **리액션 컷**(쿨레쇼프 — 의미는 병치에서 나옴, 케이스5 시민 반응 컷)도 이 계열.
6. **동기 있는 컷·카메라**: 컷과 무브는 동작·시선이 '부를 때'만 — 샷 경계는 동작 절단점에(cut on action), 카메라 무브에는 서사 이유를 붙인다.
7. **행동 동사 연출**: 감정 형용사 대신 관찰 가능한 행동 — "화났다" ✕ → "주먹을 천천히 쥐고 턱이 굳는다" ○. **비즈니스**(소품 연기: 면도칼 지휘봉)가 캐릭터를 만든다. 인물 표 '강조점'은 행동 동사로 쓴다.
8. **편집 리듬**: 샷 길이 = 감정 박자 — 코믹 비트·펀치 2~3s, 설정·여운 4~5s. 전부 같은 길이 = 기계적.
9. **오디오 브리지(J/L컷)**: 앰비언트·음악이 샷 경계를 넘어 이어진다고 명시("같은 현악 반주가 끊기지 않고 이어진다") — 네이티브 오디오 연속성("보기 전에 들어라" 판별과 세트).
10. **팔레트·모티베이티드 라이팅**: 색 3~5개를 이름으로 고정 + 광원은 화면 안 근거(창·램프·네온)로 정당화 — 샷 간 색·조명 드리프트 방지.

## 4) @참조·주체 (Omni)
- `@이미지N` = 세션 첨부(첨부 순서). **리터럴로 미리 박아 출력** → 사용자가 커서만 대면 링크 변환(실측). `@주체명` = 주체 라이브러리 등록 항목(인물·동물·소품·의상·배경·효과).
- 사용 구분: 일회성 장면·소품 = `@이미지N` / 반복 인물 = **주체 등록**(영상으로 모습 + 보이스[원본/공식 음색] + 텍스트 묘사 — `Keep: ...`으로 불변 요소 고정. 보이스까지 묶이면 재현력 급상승[실측]). 단, **케이스1·2는 @이미지 참조만으로 성공** — 등록 없이도 충분한 경우 많음.
- **피사체 치환 패턴(실증: 로봇)**: `@이미지1 를 @이미지2 와 @이미지3 의 OOO을 참고해서 ~하는 피사체로 변경해줘` — 원본 영상감 유지 + 피사체만 교체. 시사 설명 영상에 강력.
- **영상→인물 교체(템플릿 재활용, 리서치)**: 잘 뽑힌 영상 1개를 템플릿으로 두고 소스 영상 + `Replace the main character with @Element1` — 모션·타이밍·카메라 보존, 인물만 회차마다 교체(시리즈 일관성+크레딧 절약). 한계: 카메라 모션 변경·길이 연장·사후 오디오 추가는 불가.
- 등록 주체·@참조 인물은 외모 묘사 최소화(참조가 잠금) — 단 정체성 강조가 필요하면 1회 명시는 OK(실증: "이미지1의 동아시아계 중년 통통한 남성 북한 지도자 김정은이다").

## 5) 영문 테일 3종 (하우스 정본 — 샷마다 부착)
1. **품질 테일**: `photorealistic, cinematic lighting, natural skin texture, sharp detail, subtle rim light` — 화풍 따라 STYLE 모듈로 교체(비실사면 photorealistic 빼기).
2. **identity lock 블록**(인물·소품·글자 유지 필요 시): `Same exact person across all shots, identity locked, same face shape, same suit, same [소품], same [배경] with same Korean text "[원문]" clearly visible and stable in every shot. Stabilized camera, no morphing, no outfit change, no scene change.` (실증: 전기톱)
3. **Avoid 테일**(NEG00 정본): `Avoid: deformed hands, extra fingers, distortion, warped face, watermark, morphing, identity drift.` — 상황별 추가는 라이브러리 05(글자 = banner text distortion / 소품 = [소품] shape change / 배경 = background warping, location change).
- **멀티샷 전용 추가 블록**(리서치): `character drift between shots, audio desync, tonal shift between cuts, lighting inconsistency, garbled speech` — 멀티샷일 때 테일에 합류.
- 압축 원칙(리서치): 네거티브는 **5~8개**가 20개+보다 효과적, **중요한 것부터 앞에**(앵커물이면 frozen lips를 손가락보다 앞에). Kling 특유의 과잉 미소엔 `smiling, grinning` 추가.
- ⚠️ 모순 금지: 포지티브와 충돌하는 네거티브(예: CCTV 룩인데 `low quality` 배제) = 출력 붕괴.

## 6) 대사·오디오 (기본 = 한국어)
- **나레이션·대사는 기본 한국어로 뽑는다**(사용자 고정 운영). 워딩에 발음 지시 포함: "**또박또박한 한국어로, 입모양이 발음과 정확히 일치하게** 말한다" — 한국어 립싱크 잘 뽑힘(실측). 톤은 상황에 맞게 풍자적으로.
- 대사 = 따옴표 안에 원문 그대로(실증: "잘 생각했어 정으니~"). 캐릭터 라벨+톤: `[캐릭터: 톤]: "대사"` 또는 서술형("~가 여유롭게 한국어로 말한다: \"...\""). 한 장면 최대 3화자. 다국어·사투리·억양 지원.
- **행동 먼저 → 대사 다음**(공식 파트너 가이드) + 톤 변화는 굵게(`[엄마, **shouting louder**]: "..."`). 한 클립 한 화자·대사 5~7초 이내(2인 동시 발화 = 정확도 하락 실측), 싱크 틀어지면 재생성보다 클립 단축. 음색은 디스크립터로 일관(`deep male voice, calm tone`).
- 효과음·앰비언트 = `Audio:`/`오디오:` 섹션을 샷 끝에(실증: "Audio: low chainsaw idle hum") **또는 인라인 서술**(리서치 — Kling은 인라인이 정석·`Audio:` 블록은 Veo 관용, 둘 다 작동). 권장 레이어 순서: **장면+앰비언스 → 음악 → 인물 → 대사 → 행동 → 진행**(음악은 앞쪽에). 대사 없는 샷도 작은 소리 1개(리듬 큐). **오디오는 '지시'가 아니라 '묘사'**(generative) — `add crowd noise` ✕ → `heavy footsteps on wet gravel` ○. 음악 끄기 = 음악 단어 생략(전용 스위치 없음). ⚠️ 영상 레퍼런스 입력 + 네이티브 오디오 동시 불가.
- 대사 분량 공식(리서치): **5s 샷 = 8~12단어 · 4s = 6~9 · 3s = 4~6** — 짧을수록 안전. 보이스 바인딩용 음성은 깨끗한 단독 녹음 5~30초.
- 보이스 없는 연출이면 명시: "별도의 보이스는 없이 ~장면만 연출할 것"(실증: 로봇).
- 사운드 영상 동기화 토글 ON 전제. 대사 속도·톤 키워드("경쾌하고 빠른 템포") 명시가 품질↑. ⚠️ 5초+ 립싱크 저하 보고 — 대사 샷은 짧게.
- **음악 문법(260611 리서치 — 모듈 정본 = 라이브러리 10)**: 장르+무드+악기를 **산문 한 줄**로 — BPM 숫자 무효, 템포는 페이싱 언어(`rising pulse`). 마스터(또는 샷1)에 음악 1줄 고정 + 샷별 변화만(`music continues / tightens`). Kling 음악의 현실 상한 = 무드 베드·드론 — 본격 BGM은 후반 결합.
- **풍자 오디오 무기 3종**: ① 음악 급정지→펀치라인(`all music stops abruptly — only room tone`) ② 스팅어 SFX 동기(`SFX: a single sharp orchestral stinger as ...`) ③ 미키마우징(동작 1:1 — 코믹 톤). + 쇼츠 전략: **0~2초 오디오 훅**(프레임1 무음 금지) + **루프 엔딩**(음악이 인트로 모티프로 회귀 — 리플레이=조회수).
- **타이밍 크리티컬은 후반에서**: 정밀 히트포인트·비트싱크·J컷은 음악을 따로 생성(BGM=Stable Audio[BPM 정확·라이선스 최안전]·ElevenLabs Music 유료 / 한국어 보컬=Suno Pro[소송·모델 전환 리스크 인지] / SFX=ElevenLabs SFX v2) 후 CapCut 조립 — **Kling Omni는 오디오 트랙 분리 불가.** 믹스 수치(−14 LUFS·덕킹 −6~10dB)·라이선스·법규 = 라이브러리 10 + `MEMORY.md` §음악.

## 9) 출력 포맷 (불변 · v1.6 사용자 지정 260611)
