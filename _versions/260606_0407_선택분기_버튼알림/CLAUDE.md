# 노뮤트(no_mute) 플랫폼 — 마스터 라우터 (CLAUDE.md)

> 이 파일은 매 세션 자동 로드된다. 이 레포는 **여러 앱이 든 플랫폼**이고, 이 파일은 어느 앱으로 갈지 정하는 **얇은 라우터**다. 각 앱의 실제 두뇌는 `apps/<앱>/` 안에 있고 **진입할 때만 온디맨드로 로드**한다(평소엔 이 라우터만 가볍게 뜸). **반말로, 사족 없이.**

## 🧭 앱 목록 + 라우팅
| 앱 | 상태 | 진입 | 두뇌 |
|---|---|---|---|
| **뉴스 에디터** | 가동중 | 기사·이슈·주제·사진·리서치 (또는 `/news`) | `apps/news/` |
| **썸네일 제작** | 가동중 | `/th` (문구 + post/reels + 이미지) | `apps/thumbnail/` |

**라우팅 규칙 (위에서부터 판정)**
1. ⛔ **0순위 — 수정 모드**: 입력이 `git`/`GIT`/`Git`/`깃`(이거나 그 말로 시작)이면 → 아래 [✂️ 수정 모드]. 앱·기사 일절 안 다룸. (플랫폼 공통)
2. **`/news`·`/th` = `.claude/skills/`의 실제 명령어**(웹 자동완성·첫 메시지로 진입). `/news` → 뉴스 에디터(`apps/news/00`+`01` 로드) / `/th` → 썸네일 제작(`apps/thumbnail/00_지침_v22.19`+`MEMORY` 로드). **명시 명령이 1차·정확 진입로**(오분류 0).
3. **명령 없이 내용만 오면**: 앱이 2개라 추측은 오분류 위험 → 짧게 **"어느 앱? `/news`(기사→카드뉴스) / `/th`(썸네일 제작)"** 되묻는다. (단, 딱 봐도 긴 뉴스 기사 원문이면 `/news`로 바로 진입해도 됨 — 그 외엔 되묻기.)
4. ⚠️ **앱 전환 시**: 이전 앱은 닫힌 것으로 보고 **활성 앱 룰만** 따른다. 다른 앱으로 넘어갈 땐 `/clear` 권장(컨텍스트 깔끔·비용↓).

> ※ **정본 우선순위**: 플랫폼 공용 불변(아래 §기틀 보호·§수정 모드·§모델)의 **정본은 이 라우터**다. `apps/news/00`은 분리 전 마스터의 전체 사본이라 같은 내용을 일부 포함하지만, 충돌 시 **라우터가 우선**(00의 플랫폼 중복분은 추후 정리). 뉴스 *콘텐츠/파이프라인* 룰의 정본은 `apps/news/`.

## ⚠️ 기틀 보호 — 구조 변경은 반드시 사용자에게 묻는다 (모델 맘대로 절대 금지)
이 빌드의 **기틀**(앱↔라우터 분리 · 라이브러리↔지침 분리 · 3단계 파이프라인 · 라이브러리 참조 방식 · 출력 포맷 골격 · INVARIANTS)을 바꾸는 일 — 특히 **라이브러리를 지침에 다시 인라인**하거나, 분리를 해체하거나, 단계를 합치는 것 — 은 **스스로 결정하지 말고, 사용자에게 확실히 묻고 승인받은 뒤에만 한다.**
- 왜: 모델은 '효율화·정합성 개선'한답시고 구조를 맘대로 손대고, 그 *조용한 누락*이 품질을 복리로 갉는다(모델은 좋아졌는데 출력은 나빠지는 '모서리 깎임'). 그걸 막는 게 이 프로젝트의 핵심.
- 주변부(문구·임계값 등 무른 부분) 갱신은 자유. **기틀만은 반드시 확인 후.**

## 🗂 저장소·백업 원칙 + ✂️ 수정 모드 (운영 고정)
- **단일 원본 = 이 GitHub 레포(`muteno/nomute-editor`).** 모든 수정은 여기서만. OneDrive·로컬·다운로드 사본은 폐기(동기화 안 함).
- **⛔⛔ 수정 모드 트리거 = `git`(대소문자 무관 `git`/`GIT`/`Git`) 또는 `깃`.** 입력이 이 단어 하나거나 이 말로 시작하면 → **앱(뉴스 등) 절대 시작 금지.** 즉시 **✂️ 수정 모드**로 전환해 시스템 파일(라우터 CLAUDE.md·apps/* 지침·라이브러리 등)을 편집한다. 기사·이슈·카드뉴스는 이때 일절 안 만든다. (`git` 뒤에 지시가 붙어 있으면 그게 수정 지시.)
- **✂️ 수정 모드 절차(반드시 이 순서):** ⓪ **편집 전 `git fetch origin main` → 최신 main 기준으로 시작**(오래된 복제본 위 작업 금지) → ① 고칠 파일 현재본을 `_versions/{yymmdd_HHmm}_{라벨}/`에 **백업** → ② 수정 → ③ `git add·commit·push`(작업 브랜치) → ④ **PR 생성 → 즉시 `main`에 머지**(draft로 방치 금지 = 이게 '라이브 확정') → ⑤ `git fetch` 후 **origin/main 해시·내용 일치 검증 보고**. ⚠️ **기틀 변경이면 ② 전에 사용자 확인.** ⚠️ **머지 직전 main이 그새 앞섰으면 재동기화(rebase)·충돌 확인 후 머지; 충돌이면 STOP+사용자 보고, `force` 금지.**
- **⚠️ 동시 세션 동기화 (사고 방지 핵심):** `main`이 단일 진실원본 — 클라우드는 세션마다 `main`을 새로 복제한다. 그래서 **각 수정은 위 절차로 그때그때 `main`에 머지**(브랜치·draft에 방치하면 새 세션이 옛 룰을 봄). 시스템 파일 수정은 가급적 **한 번에 한 세션**; 부득이 동시면 머지 직전 최신 main 재동기화 필수. **이미 열려 있던 세션은 옛 복제본** → 최신 룰 쓰려면 새로 열거나 `git fetch`로 재동기화.
- **롤백** = 사용자가 "그 버전으로" 한마디 → 해당 `_versions/` 폴더에서 복원.
- **실행** = 폰/PC 어디서든 *이 레포를 열고 기사 던지기*(기사 던지면 라우터가 뉴스 에디터로 진입). 로컬 `,` 트리거는 폐지.
