## 2) 프롬프트 공식 (정본)
**🎯 15초 서사 원칙 (v3.0 — A/B 실측: 구조·락보다 이게 결과를 갈랐다)**
> 대전제(사용자 가설 채택 260612): **프롬프트를 읽는 모델도 '인물이 어떤 의도로 움직이는지' 이해돼야 연기가 나온다** — "xyz에 뭘 배치한다"만 아는 구현과 "이게 무엇을 표현하고 어떤 의미를 담는지" 알고 하는 구현은 퀄리티가 갈린다. 같은 악보라도 의미를 아는 가수가 부르면 깊이가 다른 것과 동일(사용자). 의도 없는 프롬프트 = 동작 나열 = 무의미한 연기(실증: 같은 면도 장면, 의도 내장본만 성공).
1. **로그라인 의무**: 시나리오 첫 줄 = 이 영상의 **한 문장**(예: "피가로가 면도를 위장해 바르톨로 몰래 쪽지를 빼돌린다"). **모든 샷이 이 문장에 복무하는지 검사 — 복무 안 하면 삭제**(반실증: 뜬금 노래 엔딩).
2. **15초 = 사건 1개 · 질문 1개**("들킬까?") — 사건 병렬 금지(반실증: 전경 면도 + 후경 밀회 동시 = 의도 전달 실패). 샷들은 같은 사건의 단계: 설정 → 실행 → 위기 → 해소.
3. **행위 주체 1명 + 리액터 ≤2**(3인 커버 실측) — 음모/행동의 주체와 화면 주인공을 일치시킬 것.
4. **why 내장 동작**: 모든 동작에 목적 절 — "시야를 가리려고 고개를 뒤로 젖힌다"(to block his view). what만 나열하면 모델이 연기의 의도를 모름.
5. **핵심 소품 클로즈업 의무**: 스토리를 운반하는 소품(쪽지 등)은 손/소품 클로즈업 샷을 배정 — 관객과 모델 둘 다에게 "이게 핵심"을 알림(실증: [CLOSE-UP ON HANDS]).
6. **부위 명시 · 과장 부사 금지**: 동작 대상 부위를 콕("턱에 거품" ○ / "얼굴에 산처럼" ✕ — 머리통 크림 사태의 직접 원인).
7. **상태 묘사 다이어트**: 배치·팔레트·조명 서술이 사건 서술보다 길면 실패 신호 — 비트가 주, 상태는 종.
- **기본(단일 샷)** = 주체(+외형 묘사) + 주체 움직임 + 장면 + [카메라 + 조명 + 분위기]  ← Kling 공식 문서 공식
- **확장(멀티샷·대사)** = 장면 → 캐릭터 → 행동/대사 → 카메라 → 오디오·스타일
- **하우스 스타일(실증) = 한국어 본문 + 영문 테일.** 본문은 한국어 산문(장면·연기·카메라), 끝에 영문 테일 3종(§5) 부착.
- 길이: 단일 샷 80–150단어 = **안전 기본값**(1차 성공률 실증). 단 **3.0은 구조화된 상세를 벌하지 않는다**(파트너·상위 가이드 수렴 260707) — 문제는 길이가 아니라 *구조 없는 나열(word salad)*: 문단·샷 라벨로 구조화된 장문은 유효, "400단어+ 뒤쪽 무시"는 비구조 덩어리 기준. 절대 상한 2,500자. (구 근거 CLIP 77토큰은 이미지 확산모델용 — Kling 3.0 영상 인코더엔 미적용.)
- 욕심 금지: 프롬프트당 핵심 아이디어 2–4개. **메인 액션 1 + 마이크로 모션**(호흡·깜빡임·옷자락·소품 미세 흔들림 — 실증: "전기톱이 미세하게 흔들리고, 가슴이 호흡으로 들썩인다").
- 구체성: "아름다운 거리" ✕ → "비에 젖은 자갈길, 네온 간판 반사" ○. **화면에 보일 글자(현수막·모자 자수 등)는 원문을 따옴표로 일일이 명시**하고 영문 테일에 `sharp readable Korean text` 추가(실증). **단 in-frame은 짧은 리터럴(슬로건·자수·간판 몇 글자)만** — 긴 글자(뉴스 티커·로어서드·자막·그래프 라벨) = 생성단 금지 → 후반 합성(GT-01/14·SF-08 — garble 사고 방지 · 260707 규칙 단일화).
- 본문은 긍정 서술만: "흔들림 없는" ✕ → `stabilized camera` ○. 배제할 것은 Avoid 테일로(§5).
- 프레이밍(샷 크기·앵글)은 **샷 첫머리에**(front-loading).

## 3-1) 연출 문법 — 프로 보강 (스토리보드 채울 때 적용 · 260611)
연출가·촬영감독 표준 문법 중 AI 영상에 직결되는 10가지. §9 스토리보드 표를 이 룰로 채운다.
1. **씬 헤딩 고정**: 씬마다 `실내/실외 · 장소 · 시간대` 한 줄 — 시나리오에 명시하고 모든 샷에 동일 반복(장소·조명 드리프트 앵커).
2. **180도 법칙(축 고정)**: 대면 씬은 두 인물을 잇는 가상 축의 한쪽에만 카메라. **AI가 스스로 자주 깨는 약점(실측 보고)** → 인물의 화면 좌우 위치를 프롬프트에 박아라("피가로는 항상 화면 오른쪽, 바르톨로는 왼쪽").
3. **30도 법칙**: 같은 피사체 연속 샷은 앵글 30° 이상 또는 샷 사이즈를 바꿔라 — 아니면 점프컷처럼 보임.
4. **샷 진행 문법**: 정석 = 와이드(지리)→미디엄(관계)→클로즈업(감정). 리빌·미스터리는 역순(디테일→공개 — 실증 케이스1 패턴).
5. **시선 일치 + 룩룸**: 인물이 화면 밖을 보면 다음 샷은 그 시선의 대상, 프레임 여백은 시선 방향 쪽에. **리액션 컷**(쿨레쇼프 — 의미는 병치에서 나옴, 케이스5 시민 반응 컷)도 이 계열.
6. **동기 있는 컷·카메라**: 컷과 무브는 동작·시선이 '부를 때'만 — 샷 경계는 동작 절단점에(cut on action), 카메라 무브에는 서사 이유를 붙인다.
7. **행동 동사 연출**: 감정 형용사 대신 관찰 가능한 행동 — "화났다" ✕ → "주먹을 천천히 쥐고 턱이 굳는다" ○. **비즈니스**(소품 연기: 면도칼 지휘봉)가 캐릭터를 만든다. 인물 표 '강조점'은 행동 동사로 쓴다.
8. **편집 리듬**: 샷 길이 = 감정 박자 — 코믹 비트·펀치 2~3s, 설정·여운 4~5s. 전부 같은 길이 = 기계적.
9. **오디오 브리지(J/L컷)**: 앰비언트·음악이 샷 경계를 넘어 이어진다고 명시("같은 현악 반주가 끊기지 않고 이어진다") — 네이티브 오디오 연속성("보기 전에 들어라" 판별과 세트).
10. **팔레트·모티베이티드 라이팅**: 색 3~5개를 이름으로 고정 + 광원은 화면 안 근거(창·램프·네온)로 정당화 — 샷 간 색·조명 드리프트 방지.
11. **물리 성립 = §3-2 단일 로직으로 검사** — 동선(이동은 비트로, 아니면 정지 명시)·거리(안 닿으면 신호/데드드롭 — 리액터는 시선·표정만으로 충분)·시선 방향(서사에 걸리면 명시)·소품 소재 추적은 전부 §3-2 그리기 검사의 파생이다. 개별 사례를 룰로 늘리지 말 것(3차전 실증: 미성립 물리 = 퀀텀 점프).
- **확장 모듈(260611 리서치) = 라이브러리 11**(애니 연기 AN·그림자 SH·고급 조명 LC): 인물 샷 기본 탑재 = **마이크로모션 리스트(AN-03)** + 클로즈업엔 **마이크로 표정(AN-10)** / 멀티샷 기본 = **조명 연속성 블록(LC-01)** + 팔레트 앵커(LC-03) / 합성·치환 피사체 = **접지 그림자(SH-02)** + 단일 광원·방향 쌍(SH-01) / 카툰 물리·스쿼시는 ⚪실험 태그 — 문두 스타일 앵커 필수, 결과 MEMORY 적립.

## 3-2) 🗺 물리 성립 원칙 (v3.2 — 단일 로직 · 사례 나열 금지)
**로직 한 줄**: 서사가 기대는 물리적 사실(누가 어디에 · 무엇이 누구 손에 · 누가 무엇을 본다)은 **프롬프트 안에서 성립해야 한다 — 적히지 않은 물리는 모델이 임의로 메꾼다**(그 산물이 퀀텀 점프·시선 붕괴·소품 순간이동).
**검사 한 줄(그리기 검사)**: 프롬프트만 읽고 **무대를 종이에 그릴 수 있는가** — 못 그리는 지점 = 모델이 메꿀 지점 = 결함 후보. **그 지점만** 채운다(전부 좌표로 깔지 말 것 — 행위 서술에 녹이는 게 최상: "턱을 들어 올리며 손질" → 시선↑·사각이 자연 유도. 과명시 = 경직·토큰 비용).
**미확정 — 추가 테스트 로직**: '최소 명시 수준'(어디까지 적어야 충분한가)은 실측으로만 좁혀짐 — 결함 발생 시 역검사: 그리기 검사에서 그 지점이 그려졌었나? **안 그려졌으면 명시 부족(프롬프트 결함), 그려졌는데 깨졌으면 모델 한계**(CASES에 구분 기록 — 이 구분이 누적되면 최소 명시 수준이 수렴).
**물리 다이내믹 서술(260707 리서치 — 무대 물리와 별개 축)**: 3.0 물리엔진은 동역학을 **능동동사**로 쓸 때 접지·무게를 계산한다 — 보행 = `heel-first, visible weight transfer`(발 미끄러짐·부유 방지 · 다수 보고) · **손은 오브젝트에 앵커**(`fingers grip the cup edge` — 빈 공간의 자유로운 손 = 모핑 1원인, "She moves her hands" ✕) · 팔 동작은 보폭 리듬에 연동 · 낙하·충돌·회전은 결과 물리까지 한 절에(`glass shatters on impact`).

## 4) @참조·주체 (Omni)
- `@이미지N` = 세션 첨부(첨부 순서). **리터럴로 미리 박아 출력** → 사용자가 커서만 대면 링크 변환(실측). `@주체명` = 주체 라이브러리 등록 항목(인물·동물·소품·의상·배경·효과).
- 사용 구분: 일회성 장면·소품 = `@이미지N` / 반복 인물 = **주체 등록**(영상으로 모습 + 보이스[원본/공식 음색] + 텍스트 묘사 — `Keep: ...`으로 불변 요소 고정. 보이스까지 묶이면 재현력 급상승[실측]). 단, **케이스1·2는 @이미지 참조만으로 성공** — 등록 없이도 충분한 경우 많음.

- **피사체 치환 패턴(실증: 로봇)**: `@이미지1 를 @이미지2 와 @이미지3 의 OOO을 참고해서 ~하는 피사체로 변경해줘` — 원본 영상감 유지 + 피사체만 교체. 시사 설명 영상에 강력.

- 등록 주체·@참조 인물은 외모 묘사 최소화(참조가 잠금) — 단 정체성 강조가 필요하면 1회 명시는 OK(실증: "이미지1의 동아시아계 중년 통통한 남성 북한 지도자 김정은이다").
- **참조 스펙(공식 260707)**: 이미지 = 각 변 ≥300px·≤10MB·jpg/png / 참조 영상 = 3–10초·≤200MB·≤2K / **element(주체)용 영상 = 3–8초**(모습+음성 자동 추출 — 별도 보이스로 덮어쓰기 가능) / **단독 보이스 업로드 = 5–30초**(음색 추출 최소 3초 · 깨끗한 배경·보통 말속도·중립 톤 권장 — 영상 3–8초를 보이스 스펙으로 혼동 금지 · 260707 정정). 표기 문법 = 웹앱 `@` / API(fal·replicate) = `<<<image_N>>>` 꺾쇠 — 웹앱 밖 확장 시 문법이 전혀 다름(혼동 금지). ⚠️ 텍스트로 미리 박은 `@`의 커서 칩 변환은 노뮤트 실측 고유(공식 무기재) — Kling 에디터 업데이트마다 재검증.
