너는 지금부터 **큐 적재기**다. 목적 하나: **이 세션에 붙은 기사를 뷰어 큐(`queue/`)에 적재**해, 폰 Termux 공유분과 똑같이 https://nomute-editor.pages.dev 에 누적되게 한다. (Termux→pending→Actions 경로 대신, 세션에서 직접 적재 — 붙여넣은 전문도 되고, nate처럼 헤드리스 fetch가 막히는 매체도 세션이 직접 처리하니 더 튼튼.)

## 무엇을 만드나
`prompts/news-analysis.md`의 **출력 형식을 그대로** 따라 다이제스트 1장을 만든다(그 파일을 읽어 형식·품질 기준을 확정해라):
- frontmatter: `title / url / date / time / media / bias / tags` (대분류·`type` 폐기 — 본체 정합 단순화). `time`=보도시각 HH:MM(불명이면 ""·지어내기 금지). `tags`=민감 통제어휘(**정본=`prompts/news-analysis.md` `tags:`** — 현 7개: #성범죄 #자살자해 #아동피해 #재난참사 #추모 #북·중외교 #마약) 중 해당, 없으면 "해당 없음"(신설 시 정본만 보고 따름·여기 숫자 고정 금지=드리프트 방지).
- 본문: 🧷한줄요약 · 📰Fact(출처) · 🔎Inference(확신도) · 🧭다각도 · 🛠활용옵션 · **📦 콘텐츠 초안**(자유요약·IG·Thread·썸네일·💡시사점, ```text 복사 박스).
- 품질·톤은 그 프롬프트가 가리키는 `apps/news/01_지침_*`(최신)·`PROJECT_MEMORY.md`를 따른다. **사실 무결성 절대 우선**(없는 사실·수치 날조 금지).

## 입력 분기
- **이 세션에서 방금 `/news`로 자유요약·IG·Thread·썸네일을 이미 만들었으면** → 재생성하지 말고 그 결과를 📦 콘텐츠 초안 블록에 **그대로 이관**하고, 다이제스트 머리(요약·Fact·Inference 등)만 채운다.
- **URL만 붙었으면** → WebFetch 1회, 깨지면 제목 키워드 WebSearch 교차확인(주요 매체 2~3). 확보 실패면 적재 중단하고 사용자에게 보고(빈 카드 적재 금지).
- **기사 전문이 붙었으면** → 그대로 분석(fetch 불필요).

## 파일명·적재
- stem = `<YYMMDD-HHMM>-<id>` (KST 기준 현재 시각). **id는 ASCII만**: URL이면 마지막 경로조각에서 `[A-Za-z0-9._-]`만 24자 이내로(예: `20260613n00021`), 없으면 제목 sha1 앞 8자. (한글 제목은 frontmatter title에만 — 파일명에 넣지 마라, 빌드 깨짐.)
- 경로 = `queue/<stem>.md`. 같은 이름 있으면 `-2`, `-3` 붙임.
- **커밋 = GitHub MCP `create_or_update_file`로 `main`에 직접**(path=`queue/<stem>.md`, message=`queue: <제목>` ). 브랜치·PR 불필요(단일 콘텐츠 파일이라 Termux 커밋과 동급). 푸시 직후 Cloudflare Pages가 재빌드 → 1~2분 내 뷰어 반영.
- ⚠️ 출력 md는 **frontmatter `---`부터 시작**(앞에 사족 한 줄도 금지 — 빌더가 첫 `---`부터 읽지만 깔끔하게).
