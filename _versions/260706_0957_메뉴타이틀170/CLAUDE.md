# 노뮤트(no_mute) 플랫폼 — 마스터 라우터 (CLAUDE.md)

> 이 파일은 매 세션 자동 로드된다. 이 레포는 **여러 앱이 든 플랫폼**이고, 이 파일은 어느 앱으로 갈지 정하는 **얇은 라우터**다. 각 앱의 실제 두뇌는 `apps/<앱>/`에 있고 **진입할 때만 온디맨드 로드**한다. **반말로, 사족 없이. 항상 내가 틀릴 수 있다고 가정하고 최선을 다하라.**

## 0. 이 문서 읽는 법 + 문서 표기 규약

### 1) 절 공통 틀
- a) 각 절은 **요지 → 규칙 → 예외 → 강제(기계 게이트) → 정본(상세 위치)** 순으로 적는다.
- b) `§이름` = 이 문서 안 다른 절 참조(기존 문서·훅·스킬이 쓰는 이름 그대로 유지).

### 2) 문서 표기 규약 (운영자 정본 — 이 문서·신규 문서 공통)
- a) **계층 넘버링 4단**: 큰 정리 `1. 2. 3.` → 중간 정리 `1) 2) 3)` → 그다음 `a) b) c)` → 그다음 `가) 나) 다)`. (대한민국 법령 체계 준용.)
- b) **괄호**: 대괄호 `{}` > 중괄호 `[]` > 소괄호 `()` — 겹칠 때 바깥부터 `{ [ ( ) ] }`. 한 겹이면 소괄호만.
- c) **따옴표·강조 특수기호 = 백틱과 별표만** — 낫표·겹낫표·꺾쇠 등 다른 인용부호 금지. (단 코드·경로·정규식 원문은 예외.)
- d) **날짜 스탬프(yymmdd) = 본문에 안 박음** — 결정 날짜·경위는 `docs/작업이력.md`가 원장. 파일명·브랜치명·백업 폴더명 등 식별자 속 날짜는 유지.

## 1. 핵심명령 (§핵심명령 · 레포 절대명령 · 최상위 불변)

**제1 핵심명령 — 모든 디자인은 `루트 디자인 토큰` 계승 또는 갱신, 둘 뿐.** 웹앱 UI의 *모든* 디자인 값(색·간격·radius·blur·곡선·모션·그림자·크기·타이포)은 토큰을 **(계승)하거나 (갱신)한다 — 둘 중 하나로만**. raw 값 임의 창작·제각각 재설계 절대 금지.

### 1) 진입점·대상
- a) **디자인 기틀 단일 진입점 = `규칙2번` `docs/디자인기틀_SSOT.md`** — 이 규칙1은 규칙2번을 *절대적으로 따른다*. 값[`:root`+`design-tokens.lock`]·규칙[§핵심명령·§디자인·nomute-design 스킬·SessionStart 훅]·컴포넌트[CII·가이드북]·확립본[브랜드컬러·모달셸팔레트·팔레트개편·UIUX기틀 사인오프]·거울[tokens.css·base.css·build_design_mirror]·게이트[check_refs·design_gate]의 **정본 위치는 전부 규칙2 인덱스에서 찾는다**. **디자인 작업 착수 전 규칙2부터 편다.** 새 디자인 기틀 파일 = 규칙2에 행 추가(누락 = 위반).
- b) `루트 디자인 토큰` = **값 SSOT = `viewer/index.html`의 `:root`** — 기존 문서의 `값 SSOT`·`:root 토큰`·check_refs `디자인 토큰`과 동일물·별칭(새 개념 아님).
- c) *컴포넌트*(버튼·모달·아이콘)는 토큰이 아니라 `docs/CII_컴포넌트계승인덱스.md` 정본 셀렉터를 계승 = **별개 축**.
- d) **우선순위는 디자인 *내용* 축 한정**: 이 절은 raw 금지·계승/갱신 2택의 최상위 선언일 뿐, §라우팅(0순위 git)·§작업표준 실행절차(HTML보고서·평의회·작업이력)·§기틀검증을 대체·완화하지 않는다(별개 축 = 중첩 적용). 값·절차·예외·기틀판정의 실체 정본 = §디자인·§기틀검증·CII·check_refs — 구체 충돌 시 그쪽이 우선.

### 2) 계승·갱신 2택
- a) **(계승) = 항상 디폴트(안 물음)**: `var(--토큰)`으로 가져다 쓴다. **정확히 맞는 토큰이 없어도 가장 가까운 토큰을 자동 계승**(이 판단도 안 물음 — 재설계·raw 창작만 금지).
- b) **(갱신) = 운영자 승인 필수 · 2티어**: 값 미세조정(기존 토큰 값 조정) = 승인 후 편입. **토큰 신설·삭제·구조 변경 = `:root` 구조 변경 = 기틀** → §기틀 보호 확인 + §기틀검증 교차검증까지가 승인의 일부. 승인 후에만 편입 → `build_design_mirror build` → CII 등재 → `check_refs` → **`design-tokens.lock` 갱신**.
- c) **근접 토큰조차 없어 (갱신)이 불가피할 때만 묻는다**: 갱신(신설)할지 / 어떤 근접 토큰을 계승할지 두 선택지 제시 후 진행. 정확히 안 맞는다고 바로 묻지 마라 — **근접 계승은 자동**(운영자 스트레스 방지 정본).

### 3) 예외 (= 이 명령 대상 아님 · 정당 raw)
- a) 기하·수학값(중앙 `-50%`·원 `50%`·빈 게이지 `0`) · 키프레임 안무 `%`값 · 1회성 광학 보정 px · OS/서드파티 강제(`env()`·`-webkit-*`) · 컴포넌트 로컬 z-index(1~8·드롭다운 20~22) · 아직 토큰 없는 축(box-shadow 등).
- b) **표면별 독립 팔레트**: thumb/ly/k/comp 4뷰어는 색 공유 안 함 — index로 통일(재색칠) = **영구금지**(§디자인).
- b-1) **콘텐츠 산출물 색 상수 = UI 팔레트와 별개 축 — 개편 비동행(영구)**: PIL 렌더 스크립트의 산출물 색[`nomute_overlay/reels2` GREEN=(15,253,2) · `card_news` COLOR_HIGHLIGHT = 형광그린 #0FFD02]은 *발행 콘텐츠*(인스타 포스트·릴스·카드뉴스)의 브랜드 텍스트색이지 웹앱 UI가 아니다. UI 팔레트 개편에 절대 동행 스왑 금지{실측 사고: 260705 터쿼이즈 개편 때 콘텐츠까지 전환 → 운영자 *제작물이 브랜드 컬러로 뒤집어씌어져 콘텐츠 못 씀* → 260706 형광그린 원복}. 콘텐츠 색 변경 = 운영자가 *콘텐츠 색*을 콕 집어 지시할 때만.
- c) 자기완결 템플릿[CSP로 `var()` 불가] → 값 **복사 = 계승**.
- d) 이들은 §디자인 사다리·`check_refs` baseline을 따른다(매 px·%·그림자마다 안 물음).

### 4) 오독 방지 (무엇이 위반이 *아닌지*)
- a) 금지 대상 = *팔레트 밖 새 색을 즉석 raw로 창작·반입하는 행위*(*비슷하니 대충 이 hex*). 아래는 위반 아님:
  - 가) 닫힌 집합 *안*의 의도된 유사색 스펙트럼 공존[예: 편향 5단 파랑#3a6ddb↔스카이시안↔흰↔코랄#ff7a6b↔빨강 = 전부 `:root` 토큰·승인 완료] = 정상 — *근접색 위반*으로 오판해 병합·삭제·축소 금지(다이버징/등급 스케일 = 하우스 표준).
  - 나) 같은 토큰의 alpha/투명도 변주(hover·press·틴트·그라데이션) = 신규 색 아님 = 자유.
  - 다) 닫힌 집합 ≠ 동결 — 진짜 새 색 슬롯이 필요하면 (갱신) 정문으로 넓힌다.

### 5) 강제 경계(정직)·정본
- a) 기계 하드차단 = raw 강조색 · 거울 동기 · `design-tokens.lock` 신토큰(`check_refs`). 그 외 raw(radius·blur·간격·그림자·모션) = WARN+소프트룰[모델 준수 의존 · SessionStart 주입으로 가시성].
- b) **`check_refs` 초록 ≠ 이 절 완전 준수**(게이트 통과를 규칙 이행으로 착각 금지).
- c) 정본: 위치 인덱스 전부 = 규칙2번. 규칙 실체 = §디자인 + `docs/CII_컴포넌트계승인덱스.md` + `구성도/00_가이드북_버튼인터랙션.html`. 거울 전파 = 구조 토큰만(색은 뷰어별 독립 :root). 계약은 SessionStart 디자인훅이 매 세션 기계 주입. 타 레포 복제 = `docs/디자인토큰_제1핵심명령_복제전문.md`[일반형 개념규칙 + 바인딩표 9슬롯].

## 3. 일하는 방식 (플랫폼 공통 · 항상 로드)

### 1) §운영 태도 — 대화·검증·능동 제안
*소프트 룰(모델이 읽고 따름) — 강한 모델일수록 안정적(§모델 연계).*
- a) **한국어·반말·직설·자기 의심.** 응답은 **첫 글자부터 끝까지 한국어**(분석·사고·대답·도구 설명 전부 — 외국어로 시작 금지). 모든 세션·모든 작업에 적용[수정 모드 `git` 포함 — 언어 정본은 여기]. 예외 = 코드·식별자·파일경로·고유명사 등 원문 보존 토막만.
- b) 사족·아부 빼고 **결론부터**. *내가 틀렸을 수 있다*를 기본값으로 **주장 전에 검증**(파일·실행 확인·근거 제시) — 추측을 사실처럼 말하지 않는다(§수정 모드 반영 보장의 *검증 안 된 됐다는 거짓*과 동일 정신). 측정으로 뒤집히면 변명 없이 정정. 사용자 전제도 틀려 보이면 근거로 반박.
- c) **능동 제안 = 디폴트, 침묵이 해명할 일.** 분기·교착·*근데* 신호 끝에 `한 수` 1개(없으면 *없음* — 짜내면 노이즈). 의도 안 좁혀지면 *이거 맞아?*로 목표부터 확정.
- d) **비전문가 보정**: 답만 말고 선택지(A/B/C) 펼쳐 제일 나은 걸 기본값으로 진행(고르라고 멈추지 마 — 사용자는 선택자면 됨). 한 창 2개 이하.
- e) **정직 표식**: 추측·곁가지는 던지되 OK확인/추정/제안/설익음 딱지 필수(표식 없으면 최약등급). `됐다 거짓`은 단정에만 걸리고 제안엔 안 걸림. 몰입 중이면 보류, `ㄴ` 하면 즉시 접어라.

### 2) §작업표준 — 작업 4대 표준 (매 작업 실행방식 최상위 · 라우팅 판정엔 종속 · 운영자 정본)
**요지**: 무슨 작업이든 **착수 전·완료 전에 이 4가지를 확인**한다. 단 §라우팅(0순위 git·앱 진입)이 *먼저* 끝난 다음 도는 *실행 방식* 체크리스트. 중복=드리프트라 a)·c)는 대부분 기존 절 포인터(정본 = §운영 태도·§검증·§분신술)고, 새 규칙은 b) HTML 보고서·d) 누적 원장뿐.

**실질 변경의 정의** = 운영자가 의도해 동작·구조·출력이 바뀌는 것. 봇 자동커밋(scrape·AI판정·thumb·cards)·조회/진단·오타·순수 표기 미세수정 = 실질 변경 아님 → b)·c) 면제(면제는 사람 주도 산출에만 적용). 단 **큐레이션 임계(`FAST_MAX_H`·cross 등) 변경은 출력이 바뀌므로 실질 변경 = 면제 아님**[§curation 기틀급 — `docs/curation-algorithm.md`]. (작업 대상 라이브 = https://nomute-editor.pages.dev/)

- a) **의도 명확화 = 착수 전 필수 · `왜`부터 3단.** 지시의 *표면 문구*에 바로 반응하지 마라 — {왜 시켰지?(목적) → 목적 달성 방향? → 그 방향의 최선 방법?} 3단으로 *근본·전체*를 해결한다. 콕 집은 한 줄만 땜질하면 옆이 틀어져 왕복·피로 유발. 예: *빈공간 없애줘*(표면) → 목적=*팝업이 깔끔하길* → 방향=*세로 리듬 전체 일관* → 빈공간+간격+정렬을 *같이* 본다. 불분명하면 추측 말고 {처한 문제 상황 / 해결하려는 문제}를 한 줄로 되짚어 재확정 후 착수(분기면 §선택질문 버튼). 단 입력이 `git`/`깃`이면 라우팅 0순위 선행 — 되묻지 말고 수정 모드 진입(수정 지시 *자체*가 모호할 때만 적용).
- b) **결과 제시 = 쉬운 설명 + HTML 보고서 (정본=여기).** 실질 변경·산출은 쉽게 설명하고 **전/후 비교·장단점·효율성·품질저하 리스크** 5요소를 담아 `docs/reports/{yymmdd}_{라벨}.html` **단일파일 HTML**[인라인 CSS·SVG·`<meta name="color-scheme" content="dark">`·시스템폰트 폴백 = 폰에서 자기완결로 열림]로 만들어 `SendUserFile`로 보낸다. 색은 §디자인 토큰값 **복사** 인라인[보고서 = 일회성 스냅샷 = viewer 동기화 의무 없음·`check_refs` 비대상]. 재사용은 직전본 복제(재설계 0). *작은 변경은 채팅 마크다운 표로 갈음 가능.*
- c) **검증 = 적대적 평의회 후 머지 (절차 정본 = §분신술·§검증).** 실질 *코드·구조* 변경 = **10인 평의회 상시** · 국소 뷰어 손질 = 자가검증 · 10인 미만 감축은 **사용자 승인 후에만** · 순수 룰·문구·산문 = `check_refs`+원격 read-back 자가검증 · 순수 시각/목업 = 평의회 완전 면제 · **시각 제안·미리보기는 PNG 아닌 HTML로.** 완료 후 운영자 보고(통지)→머지[즉시 main = §수정 모드 · 기틀이면 §기틀 보호 사전확인 선행].
- d) **누적 = 작업이력 원장 + 개선 우선사항 (정본=`docs/작업이력.md`).** 완료 작업은 **append-only 누적**(날짜·요지·전후·결정·검증결과), 실행 개선점은 같은 문서 §우선사항에 박아 다음 세션이 더 잘하게 한다(역할 경계·비대 cap·중복차단은 원장 머리말이 정함).

## 4. 저장소·백업 원칙 + 수정 모드 (§수정 모드 · 운영 고정)

### 1) 원칙
- a) **단일 원본 = 이 GitHub 레포(`muteno/nomute-editor`).** 모든 수정은 여기서만. OneDrive·로컬·다운로드 사본은 폐기(동기화 안 함).
- b) **수정 모드 트리거 = `git`(대소문자 무관) 또는 `깃`.** 입력이 이 단어 하나거나 이 말로 시작 → 앱 절대 시작 금지, 즉시 수정 모드로 시스템 파일(라우터·apps/* 지침·라이브러리 등) 편집. 기사·이슈·카드뉴스 일절 안 만듦. (`git` 뒤에 붙은 말 = 수정 지시.)
- c) **실행** = 폰/PC 어디서든 이 레포 열고 기사 던지기(라우터가 뉴스 에디터로 진입). 로컬 `,` 트리거는 폐지.
- d) **롤백** = *그 버전으로* 한마디 → 해당 `_versions/` 폴더에서 복원.

### 2) 수정 모드 절차 (반드시 이 순서 · 구 표기 0~5단계와 동일)
- a) 편집 전 `git fetch origin main` → **최신 main 기준으로 시작**(오래된 복제본 위 작업 금지).
- b) 고칠 파일 현재본을 `_versions/{yymmdd_HHmm}_{라벨}/`에 **백업**.
- c) 수정. (**기틀 변경이면 수정 전에 사용자 확인** = §기틀 보호.)
- d) `python3 shared/check_refs.py` 통과 확인[참조 실존·버전 일치 기계 점검 · 실패면 고치고 재실행] 후 `git add·commit·push`(작업 브랜치).
- e) **PR 생성 → 즉시 main 머지**(draft 방치 금지 = 이게 `라이브 확정`).
- f) `git fetch` 후 **`git show origin/main:<파일>`로 원격 실내용 직접 읽어 검증·보고**(해시+핵심 변경 줄 실측 · 화면·로컬만 보고 `됐다` 금지).
- g) **머지 충돌 처리**: 머지 직전 main이 앞섰거나 squash로 갈렸으면 `git merge origin/main -X ours` 흡수(또는 rebase)·충돌 확인 후 머지. 진짜 충돌 = STOP+사용자 보고. `force` 안 씀. **단 `-X ours`는 *코드 충돌* 한정** — `docs/작업이력.md` 등 append-only 문서는 `-X ours`가 다른 세션 추가분을 조용히 덮어 데이터 손실 → `-X` 없이 머지 후 충돌 마커 *수동 정리*로 양쪽 보존(시각순).

### 3) 반영 보장 (불변·최우선)
- a) 변경 요청 = **main 반영 + 실측 검증까지가 `완료`**. 완료 선언은 `git show origin/main:<파일>` 확인 후에만.
- b) 머지가 막히면(force 차단·non-ff·충돌) 우회로 끝까지 반영[흡수 머지 등 · append-only 문서는 수동 보존].
- c) 그래도 안 되면 **`됐다` 절대 금지 → 즉시 STOP + 사용자 보고.** *검증 안 된 완료 보고는 거짓 — 이 원칙이 모든 편의·속도보다 우선.*

### 4) 동시 세션 동기화 (사고 방지 핵심)
- a) main = 단일 진실원본 — 클라우드는 세션마다 main을 새로 복제. 각 수정은 **그때그때 main에 머지**(브랜치·draft 방치 = 새 세션이 옛 룰을 봄).
- b) 시스템 파일 수정은 가급적 한 번에 한 세션 · 부득이 동시면 머지 직전 최신 main 재동기화 필수. 이미 열려 있던 세션 = 옛 복제본 → 새로 열거나 `git fetch` 재동기화.

## 5. 기틀 보호·검증

### 1) §기틀 보호 — 구조 변경은 반드시 사용자에게 묻는다 (모델 맘대로 절대 금지)
- a) **대상(기틀)**: 앱↔라우터 분리 · 라이브러리↔지침 분리 · 3단계 파이프라인 · 라이브러리 참조 방식 · 출력 포맷 골격 · INVARIANTS. 특히 라이브러리 재인라인·분리 해체·단계 합치기 = **스스로 결정 금지, 운영자 승인 후에만.**
- b) 왜: 모델은 `효율화·정합성 개선`한답시고 구조를 손대고, 그 *조용한 누락*이 품질을 복리로 갉는다(`모서리 깎임`). 그걸 막는 게 이 프로젝트의 핵심.
- c) 주변부(문구·임계값 등)는 자유. **기틀만은 반드시 확인 후.**

### 2) §기틀검증 — 기틀 변경 = 최상위(Opus) 5명 다중 교차검증 필수 (정본)
*물부터 잘못되면 모든 음식이 망한다 — 뿌리는 토큰보다 정확이 우선. §기틀 보호와 한 쌍.*
- a) **대상(=§기틀 보호 동일 범위)**: 라우터 CLAUDE.md 구조 · 플랫폼 공용 불변(수정 모드·모델·기틀 보호·명명표준) · 환경 Setup script/env 변수 · 라이브러리 SSOT 구조·build 파이프라인 · 앱 지침의 출력 포맷 골격/INVARIANTS. (주변부 = 제외·자유.)
- b) **절차**: §기틀 보호 사용자 확인 *먼저* → 확정 전 **독립 Opus 5명 병렬 발사**(동일 후보 적대적 감사·레포 실측) → 수렴 확인 — **전원 APPROVE 아니면 STOP·재설계**(이견은 다수결 맹종 말고 근거로 해소·기각 사유 기록) → 통과 후에만 §수정 모드 절차 전체로 확정.
- c) **한계(정직)**: 소프트 룰(기계 게이트 아님). 교차검증은 ground-truth 검증가능 영역(구조·스크립트·참조정합)에서 값을 함 — 순수 문구·톤은 효용 작음.

### 3) §검증 — 다앵글 적대적 Opus 4.8·effort max 만장일치 (검증 처리 정본)
**요지**: 사용자가 검증 요청(*검증해줘/verify/문제 없는지* 등)하거나 내가 수정 모드 작업을 끝내 능동 자가검증할 때 — git 작업 전부에 이 표준 적용. *인원 티어 = 아래 b)의 가)~마) — 구 표기 원문자 티어(외부 문서가 §검증 뒤에 붙이는 동그라미 a~e)와 같은 순서로 1:1 매핑(가=구a·나=구b·다=구c·라=구d·마=구e).*
- a) **모델·노력**: 검증가 전원 **Opus 4.8(`claude-opus-4-8`)·effort max**. 약한 모델·저노력 금지.
- b) **인원 티어**:
  - 가) **상시·기본 = 자가검증** — 모든 수정 모드 작업은 머지 전 `check_refs`+원격 read-back으로 안 잊고 자가검증(평의회 소환 안 함 · 옛 5인 상시 폐지).
  - 나) **10인 승격** = 사용자 명시 검증 요청 또는 대상이 기틀. **운영자 표준: 실질 *코드·구조* 변경 = 10인 상시 운용.** 순수 룰·문구·산문 = `check_refs`+read-back 자가검증.
  - 다) **국소 뷰어 손질**{단일 `viewer/*` UI/UX 조정 · 동작·구조·파이프라인 영향 작음 · 기틀 아님[`:root`·배너 크로스페이드·파이프라인이면 10인]} = 자가검증으로 끊는다.
  - 라) 10인 미만 감축 = **사용자에게 물어(§선택질문) 명시 승인 후에만**(무응답·미승인 = 10인 유지 · 임의 감축 금지).
  - 마) **순수 시각/목업 = 평의회 완전 면제**(동작·로직·상태·데이터 안 건드리는 순수 외형) → 자가 시각확인[렌더·HTML 직접 보기·check_refs 게이트]으로 갈음. 경계 = 로직·동작·상태·데이터가 한 줄이라도 끼면 면제 아님(`:root` 토큰·이벤트 핸들러·렌더 분기·파이프라인 = 동작). **시각 제안·미리보기 = 기본은 바로 반영 후 라이브 확인**(HTML 시안 남발 금지 — 매번 시안 = 운영자 피로 · 폰 브라우저는 시스템폰트 폴백 실패로 한글이 두부로 깨져 무용[실측 260705]). 확신 있는 변경은 시안 없이 반영. 시안이 꼭 필요한 경우만[대규모 구조·새 인터랙션·되돌리기 어려운 변경] HTML로 주되(PNG는 왜곡·실제 CSS/애니/탭 확인 불가라 여전히 금지) **한글 깨짐 방지 필수**[웹폰트 인라인 임베드 또는 `font-family` 첫 항목에 한글 시스템폰트 포함·`sans-serif` 우선 폴백].
- c) 어느 쪽이든 **각기 다른 앵글**로 적대적 감사(메커니즘·호환성·헤더/보안·모바일·메모리·에러처리·다운스트림·버전·통합·총괄 등). 추측 금지 = 레포·소스·라이브 **실측** 근거.
- d) **READ-ONLY = 하드 계약 (over-step 재발방지 정본)**: 평의회/분신술 에이전트는 **분석·보고만 — 레포 1바이트도 못 바꿈**. 전면 금지 = `git add/commit/push/branch/merge`·PR 생성/머지(`mcp__github__*` 쓰기툴)·`Edit/Write/NotebookEdit`·파일 생성/삭제. 산출 = 텍스트(평결·근거·제안)만. **메인 루프가 레포 변경의 유일 행위자.** 강제 3중:
  - 가) 소환 프롬프트 첫 줄에 계약 그대로: `READ-ONLY: 어떤 git/파일 변경·커밋·브랜치·PR·머지도 하지 마라. 발견은 텍스트로만 보고하라. 모든 변경은 메인이 한다`
  - 나) **읽기전용 에이전트타입 = 기본·필수** — 평의회·검증·분신술·아이데이션은 예외 없이 `subagent_type=Explore`(Edit/Write 미보유 = 기계 차단). `general-purpose`(전권)는 메인이 파일 만들 실작업에만 — 읽기·평결 에이전트엔 금지. 정직: Explore도 Bash로 git은 가능 → 잔여 경로는 가)+다)가 막음(3중 병행·단일 의존 없음).
  - 다) 사후 점검 — 발사 후 `git status`/`git branch`/`git log`로 rogue 브랜치·커밋·PR 확인. 에이전트가 변경·머지 시도/실행 = 계약 위반 → 즉시 운영자 보고 + 산출 무효 처리. {근거: 과거 분신술 중 에이전트 1명이 READ-ONLY를 어기고 브랜치·커밋·PR#1080 머지까지 자율 실행 = 검토 우회 — 이 3중이 그 방지턱[Write 미보유였다면 원천 차단됐다].}
- e) **실행 보장 — 묘사 말고 실제 소환**: 말만 하지 말고 반드시 실제 `Agent` 툴(Explore·opus·READ-ONLY 계약)로 N명{명시·기틀·실질 코드/구조 = 10인} 병렬 소환. 머릿속·서술 대체 금지(과거 안 돌리고 넘어간 전례의 방지턱).
- f) **만장일치 루프 (정본)**: 평결 = `OK/주의/불가`. N인 발사 → 평결 종합 → 문제 전부 메인이 직접 수정 → 재검증(필요 시 재발사) → 전원 OK(만장일치)까지 루프 → `git show origin/main:<파일>` 원격 실측 → 그제서야 *검증 완료* 보고. **어느 단계든 안 끝났으면 `됐다` 절대 금지.**
- g) **기틀이면 둘 다**: 이 검증(10인) + §기틀 보호 사용자 확인 모두.
- h) **한계(정직)**: 소프트 룰. 검증가능 영역에서 값을 함 — 순수 문구·톤은 `check_refs`+read-back 자가검증이 낫다.

### 4) §분신술 — 다앵글 적대적 10인 평의회 (피드백·검증·효율·품질·아이데이션 · 운영자 정본)
- a) 운영자 호출어 = **`분신술`**(자주 씀·매우 중요). 입력에 있으면 → **Opus 4.8·effort max·adaptive thinking** 에이전트 **10명**을 실제 `Agent` 툴로 병렬 소환[`subagent_type=Explore` 읽기전용 · READ-ONLY 계약 프롬프트].
- b) 각자 다른 앵글+적대적 시각으로 그때까지의 전 절차를 훑어 5가지: 피드백 · 검증(실측·근거) · 효율성 · 품질 · 아이데이션(개선·대안).
- c) §검증과의 관계 = 10인 적대 검증의 **명명된 상위판**(검증+효율+품질+아이데이션 = 평의회). 절차·READ-ONLY·만장일치 루프·read-back 실측 전부 §검증 상속. 변경·머지는 오직 메인.
- d) **묘사 말고 실제 소환 (절대)** — 반드시 10명 실제 병렬 발사 + 발사 후 rogue 사후점검.
- e) 언제·효용: 운영자가 `분신술` 외칠 때(주로 큰 변경·기틀·세션 마무리 점검·아이데이션 필요 시). 값은 검증가능 영역(구조·스크립트·동작·참조정합·실측)에서 큼 · 순수 문구·톤은 작음(그건 check_refs+read-back). **출력** = 10인 앵글별 핵심 → 수렴 평결(OK/주의/불가) → 메인 조치 목록 → 아이데이션 베스트픽.

### 2) §기록SSOT — 기록 SSOT 지도 (압축·컨텍스트 증발 대비 · 정본 · 항상 로드)

### 1) 큐레이션 착수 전 필독
- a) **수집함 큐레이션(누적/최신 분류·정렬·신호 = `scFast`·`scScore`·`fastScore`·`gate_judge`·`breaking_judge`·`report_count`) 손대기 전 반드시 `docs/curation-algorithm.md` 최상단 §방향 `전체 방향 한 장` 먼저 읽기(운영자 명시).** 신호 5종{cross·연속보도·burst·AI경중·[속보]태그}·두 컬럼·속보판정·랭킹곱 = 방향 정본. 방향 망각한 채 불만마다 entry/임계만 국소패치 = 도돌이표(실측 교훈). 새 세션·압축 후 수집함 작업이면 이 문서부터.

- c) **누적 ≠ 긴급 = 두 축 (정본=curation §방향)**: **자리(시간 칼럼)** = 신규(나이<4h)/누적(≥4h) — 경계 `FAST_MAX_H=4` 단일상수[최신진입·누적진입·핀·토스트·배지 전부 결정 · 화면 `3시간 전`까지 최신·`4시간 전`부터 누적]. **사건(배지)** = 긴급=속보 = `isBreaking` = AI 2층[`breaking_judge` 급발?(방금 터진 대형·다수피해·전국주목 — 개별 형사 선고·사법결과 제외·엄선) AND `gate_judge` 경중 grade≥2(선고도 동일 엄선)]. 긴급 사건은 나이 들면 *자리만* 최신→누적 이동[개념 불변 · 배지 긴급(<4h)↔이슈(4h~24h)·1일 이상 배지없음]. **누적 진입 = 4h + {cross≥8 OR 긴급(breaking) OR 연속보도 보조진입(cross≥4 AND report_count≥6 AND grade≠0 · grade1구제)}**[8매체+ 엄선 · grade 재정의: grade=긴급탐지지 경중 아님 → 랭킹·진입은 발행량 우선(gradeW floor 0.05→0.5·CROSS_POW 1.6→1.3·FOLLOW_W 0.2→0.5·긴급부스트 ×1.5) · 연속보도 보조진입 = 지속보도+양질이면 cross<8도 합류(cross 완화 아닌 3중 게이트) · 긴급이었던 건 cross<8이어도 4h 후 누적 합류 — 최신서 졸업하면 사라지던 구멍 차단·일반 누적 점수로 랭킹]. **연속보도(`report_count`→`scFollow`) = 제일 무거운 가점**(또 보도될수록 상승·양 칼럼 곱). 라벨 최종확정: **칼럼=누적/신규 · 배지=긴급(breaking·신규<4h)/이슈 = 조건식 `([cross≥10 AND grade(null 또는 ≥2) AND !badgeJunk] OR aged-breaking) AND 4h≤나이<24h`(신규엔 안 뜸·1일 이상 없음)** — 칼럼과 배지 이름 분리(옛 최신/속보/누적 폐기·정본 docs §8).

### 4) 새 세션 안 헤매기 (할 때마다 고생 방지)
- a) 작업 첫 손 = `git fetch origin main` — 라이브 파이프라인은 main에서만 돈다(`ref: main`) · 브랜치/draft는 라이브 아님.
- b) 요약 지침 ≠ 카드 지침 해시[파일집합 다름: 요약=01+MEMORY, 카드=00+01+02+MEMORY] — 두 도장 값이 다른 게 정상.
- c) `text_done`(텍스트·프롬프트, 이미지 0) ≠ `done`(이미지).
- d) 자동은 텍스트까지만, 이미지는 운영자 슛.

## 9. 인프라·공통 UX

### 1) 합성기/제작기 deps = 7일 캐시 필수 + 네트워크 op 타임아웃 (플랫폼 인프라 · UX 안정)
**요지**: 제작 워크플로(`*-make.yml`)가 매 런 무거운 deps(폰트·ffmpeg·pip·모델)를 재설치하다 네트워크 스톨하면 무한 행→cancel→산출 없음 = *되다 안되다*(실측: thumb-make 20분 행). 두 가지 필수:
- a) **7일 캐시 필수**: 무거운 deps는 반드시 `.github/actions/runner-setup` 캐시로 — pip(`~/.cache/pip`)·whisper(`~/.cache/huggingface`)·apt .deb(`~/.cache/nomute-apt`·폰트/ffmpeg). 워크플로는 `with: { setup, apt: '<패키지>', whisper: 'true' }`만 → 첫 런만 다운·이후 오프라인 적중(7일). 새 제작 앱·새 deps도 `setup.sh` 생설치 말고 runner-setup 캐시 경로(`apt` 입력)로(재다운 0).
- b) **네트워크 op 타임아웃 필수**: `setup.sh`의 `apt-get`/`pip`/prefetch 등 모든 네트워크 명령은 `timeout`(+1회 재시도)로 감싼다 → 무한 행 대신 빠른 실패(`set -e`)·재시도 시 캐시 적중 성공. + `*-make.yml` Setup에 `timeout-minutes` 백스톱(20분 cancel 차단).
- c) 정본 = `runner-setup/action.yml`(apt .deb 캐시·오프라인 dpkg) + 각 `apps/*/setup.sh`(timeout 폴백) + `*-make.yml`(`timeout-minutes`). 라이브: thumb·comp(`fonts-noto-cjk`)·ly(`ffmpeg`). 클라 폴 타임아웃(제작 3분·#458)과 한 쌍.

### 2) §미디어첨부 — 미디어 첨부 입력 (플랫폼 공통)
