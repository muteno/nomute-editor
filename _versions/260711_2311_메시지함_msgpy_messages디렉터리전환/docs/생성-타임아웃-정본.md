# 생성기 대기·타임아웃·재시도 정본 (SSOT · 260619)

> 모든 생성기(썸네일·합성·영상·자막·뉴스카드·픽)의 **프론트 대기 budget ↔ 백엔드 타임아웃 ↔ 실패신호 ↔ 재시도**를 한 곳에 못박는다. 코드에 흩어진 값이 드리프트(프론트는 실패라는데 백엔드는 계속 돎)하지 않게. 값 바꾸면 **여기도 같이 갱신**.

## 🧭 핵심 원칙
- **백엔드 claude 콜은 전부 `timeout 900`(15분)** = 죽은 콜 강제종료 *천장*(보통 생성은 1~5분). analyze·ask·cardmake·kmake·lymake·revise 동일.
- **프론트 budget = 백스톱**(워크플로가 흔적 없이 증발한 드문 경우만 대비). 천장(15~30분)이 아니라 **현실 생성시간**에 맞춰 컴팩트.
- **진짜 실패는 budget이 아니라 `error.log`/상태로 즉시 표시** — budget 길어도 평소엔 그만큼 안 기다림.
  - `error.log` 빠른실패 읽는 폼: **`/k`·`/ly`**(둘 다 같은 폴더 `error.log` 폴 → 뜨면 즉시 `❌`). 썸네일·합성은 PIL이라 error.log 없이 budget만.
- **재시도**: 폼은 **메인 생성버튼 재클릭**(실패 후에도 활성), 카드는 **전용 재시도 버튼**. 재클릭 시 `_curStop`이 이전 폴을 멈춰 상태표시 충돌·중복 폴 차단(`/k`·`/ly`·`/comp`).

## 📣 분석 도구 시스템성 실패 = 사용자 알림(프로필 점등 · 260619)
claude(분석) 호출이 **시스템성 실패**(인증=키·토큰 만료/무효 · 쿼터·레이트리밋)면 — 단발 콘텐츠 실패와 달리 *전 분석이 다 막힘* → 사용자에게 **작동할 때까지 매 접속 알림**.
- **판정**: `shared/claude_health.sh` `claude_health_update <out> <stderr>` — claude가 응답하면(출력 有)=시스템 정상 → 경고 해제 / 출력 비고 stderr가 인증·쿼터 패턴 → 경고 박음. (콘텐츠 실패·타임아웃은 무변경 = 오발 X.)
- **채널**: `shared/msg.py set/clear claude-down` → `viewer/messages.json`(키 `claude-down` 1건 upsert/clear) → 프론트 `loadMessages`(매 로드+120s폴)가 **프로필 점등(hasmsg)+배지+msgpop**. 성공 1건이면 자동 해제("작동할 때까지").
- **배선**: `analyze.sh`·`ask.sh`(메뉴1 요약요청)·`cardmake.sh` 가 claude 호출 직후 `claude_health_update` 1줄 + 워크플로 커밋에 `viewer/messages.json` 포함(`-X theirs` 안전). 주 경로=analyze/ask가 먼저 잡음.

- 백엔드 timeout: `.github/scripts/{analyze,ask,cardmake,kmake,lymake,revise}.sh`(전부 `timeout 900`)
- 워크플로 cap: `.github/workflows/{thumb,comp,k,ly}-make.yml`·`news-{analyze,ask,revise}.yml`·`card-make.yml`
- error.log 빠른실패: `kmake.sh`/`lymake.sh`가 `viewer/{k_out,ly_out}/<id>/error.log` 작성 → `k.html`/`ly.html` poll이 `errPath` 폴

## 🔧 260619 변경 이력
- **카드 프롬프팅 25분→15분**: `cardmake.sh` `timeout 1500→900`(프론트 genStuck 15분과 정합 — 백엔드가 25분까지 슬롯 붙들던 15~25분 낭비 제거).
- **/k·/ly·comp budget 컴팩트**: 백엔드 천장이 아닌 현실시간 기준(k 600/900·ly 720/1200·comp 600→420). error.log가 실패를 잡으니 budget은 백스톱.
- **/k error.log 빠른실패 추가**: `kmake.sh`는 실패 시 error.log를 쓰는데 `k.html`이 안 읽어 진짜 실패도 budget 다 기다리던 것 → `/ly`처럼 즉시 표시.
- **재시도 폴 정리**: `/k`·`/ly`·`/comp`에 `_curStop` — 재클릭(재시도) 시 이전 폴 중단(상태표시 충돌·중복 폴 차단).
- **"다시 시도" 안내**: comp·k·ly 시간초과 문구에 "위 버튼으로 다시 시도해줘" 추가(재시도=메인버튼 명확화).

> ⚠️ 미래 세션: 어느 한 쪽(프론트 budget·백엔드 timeout)만 바꾸면 다시 어긋난다. **둘을 같이 보고 이 표를 갱신**할 것.
