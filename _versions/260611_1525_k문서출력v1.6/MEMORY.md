## UI 고정 사실 (260611 실측 스크린샷 기준)
- 사용 환경: **모바일 웹앱**(안드/iOS 네이티브 앱은 환경별 입력 과정이 미묘하게 달라 웹앱 사용. 주체 등록 과정도 앱 환경별 상이).
- 탭: 이미지 생성 / 비디오 생성 / 모션 컨트롤 / 디지털 캐릭터. Omni는 별도 진입(결과 화면 표기 "Kling Omni · 영상 O1").
- UI 설정(프롬프트 제외 대상): 생성 모드 720p/1080p(VIP)/4K(VIP) · 생성 시간 3–15s 슬라이더 · 비율 16:9/1:1/9:16(+Omni '스마트') · 생성 수량 1–4 · 사운드 영상 동기화 토글 · 스마트/커스텀 멀티 샷.
- 비디오 생성 탭 = 스타트/엔드 프레임 이미지(엔드 선택). "주체를 바인딩하여 생성 결과의 일관성을 높여보세요" 옵션 + 인물 방향영상/방향이미지 일치 모드 + 안면 요소 바인딩.
- 커스텀 멀티 샷 = 샷 박스별 텍스트 + 초(↑↓) + 순서 드래그·삭제·＋샷. 안내문: "맞춤형 스토리보드를 사용하려면, 현재 패널에 완전한 프롬프트를 입력하고 제출하여 생성하십시오."
- `@` 입력 드롭다운 = ①주체 라이브러리에서(등록 주체) ②이미지1·이미지2(세션 첨부, 첨부 순서). **`@이미지1`을 텍스트로 미리 박아두면 커서 접촉만으로 해시태그식 링크 변환**(웹앱 실측) → 산출물에 리터럴로 포함시키는 근거.
- 주체 라이브러리: 내 주체/공식 주체, 분류 = 인물·동물·소품·의상·배경·효과(+즐겨찾기). 바디 편집 = 이름·분류·텍스트 묘사(예: "Older man, blonde combover, navy suit, red tie. **Keep: flag pin.**")·스마트 설명(자동 생성)·음색(무음색/비디오 원본 음성/Voice 199/공식 William·Laura·Catherine·Owen 등).
- 주체 등록 3요소 = 영상으로 모습 + 보이스 + 텍스트 묘사. **보이스까지 묶이면 인물 재현 거의 됨**(실측). 생성 후 화면: 립싱크 · **프레임 추출**(→ 다음 생성 소스로 재투입 가능) · 프롬프트 변경 · 재설정.
- 크레딧 실측: 1회 생성 450–528 크레딧 수준 — 리롤 비용 큼 = 이 앱의 존재 이유.

## 사용자 실측 (260611)
- **Omni ≫ 3.0**: "영화 같은 움직임 + 카메라 전환 많으면 확실히 Omni. 원본 이미지를 거의 유지하는 장면이면 3.0도 괜찮음(포지티브/네거티브 잘 쓰면)" — 3.0 비디오 생성 탭 단독은 정적 스톡류 외 활용도 낮다는 판정.
- **분할 > 퉁침**: 같은 소재를 커스텀 멀티샷으로 분할하면 잘 나오고, 한 덩어리 프롬프트로 퉁치면 잘 안 나옴(서재 김정은 = 실패 사례).
- **영문 테일 유무 = 퀄리티 격차 매우 큼**: `photorealistic, ...` 품질 테일 + `Avoid: ...` 네거티브 테일을 넣은 것과 안 넣은 것 차이 확연.
- **한국어**: 프롬프팅 한국어가 오히려 잘 먹히는 경향(중국 엔진 추정, 확실치 않음 — 불확실 유지). 한국어 대사 워딩+입모양 잘 뽑힘 → **나레이션·대사 기본 한국어 + "또박또박, 입모양 일치" 워딩**(사용자 고정 운영). 영문 가이드들은 영어 입력+Auto Prompt 변환 권장 — 사용자 실측과 병기(하우스는 한국어 본문+영문 테일).
- **검열**: 시진핑 등 중국 정치 민감 소재만 절대 불가. 그 외 대부분 통과(트럼프·김정은 가능 — 실증).
- **정밀 루트**: 나노바나나(제미나이) 1차 합성 → 스타트 프레임/@참조 시작이 정밀.
- **풍자 운영**: 정치인 활용 + AI 표기 + 모자이크(후처리) + 화풍 치환. 실사 에디토리얼 필수 아님 — 이야기를 풀어 각색하면 더 잘 나옴.
- **빠른 동작 실패**: 인물 방향영상 일치 + 안면 바인딩까지 해도 랩 퍼포먼스(입모양·리듬 동작) 유지 안 됨(3.0 탭) — §미해결.
