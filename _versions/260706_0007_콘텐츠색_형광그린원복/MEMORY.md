# 카드뉴스 합성기 — 프로젝트 메모리

## 목적 & 맥락

세웅은 한국어 카드뉴스(카드뉴스) 이미지 합성 제작 파이프라인을 운영 중. 핵심 목적은 배경 이미지 위에 텍스트 오버레이를 효율적으로 합성하는 것 — 저널리즘·소셜 뉴스 콘텐츠용. 성공 기준은 빠르고 정확한 합성 + 올바른 타이포그래피·맞춤법·강조 렌더링 + 불필요한 확인 단계 최소화.

부차적·별개 프로젝트로 한국어 숏폼 자막 생성기(NO MUTE / 노미트)가 있으며, 카드뉴스 합성기와 **엄격히 분리**. 두 프로젝트 혼합은 명시적으로 금지.

## 현재 상태

카드뉴스 합성기는 **v3**, 스크립트 `260415_203533_card_news.py` 사용. 다양한 뉴스 주제에 걸쳐 매일 실제 운영 중.

### 핵심 v3 스펙
- 캔버스: 1080×1350, Noto Sans CJK KR Bold 54px, 줄 높이 88px
- 색상: 기본 `#FFF` / 강조 `#00EED2`(터쿼이즈 · 260705)
- 상수: `START_X = MARGIN_LEFT = 87`, `START_Y = 888`, `MAX_WIDTH = 937`, `SAFE_BOTTOM = 1310`, `MARGIN_RIGHT = 56`, `MARGIN_BOTTOM = 234`
- 자동 줄바꿈 없음; 폰트 축소 없음
- 스마트 크롭은 비-4:5 이미지만 (정확히 4:5 → 리사이즈만, 크롭 0%)
- 출력: `/mnt/user-data/outputs/` KST 타임스탬프 파일명

### 두 가지 입력 모드
1. 채팅 직접 업로드
2. 드라이브 자동 조회: 트리거 키워드 "드라이브 합성" / "카드뉴스 드라이브" → Prompt 폴더 ID `1jQBoDqnDk5-fw51tCdDLD_cuDBAJp3kf`, 서브폴더 구조 `yymmdd_hhmmss_주제/` 안에 `.md`와 `card_N_scene_1.png`가 평면 배치. 다운로드는 `curl -sL "https://drive.google.com/uc?id={ID}&export=download"`

## 핵심 학습 & 원칙

- **맞춤법 교정**: 합성 전 인라인 자동 적용, 명시 표기 형식 `✏ 'X' → 'Y' 보정`. 자주 나오는 교정: `이였다→이었다`, `아니였다→아니었다`, `그제서야→그제야`, `그 중→그중`, `안돼→안 돼`, `누워있었다→누워 있었다`, 조사/보조용언 띄어쓰기 규칙.
- **비표준 표현**: 의도적 언어유희·스타일 선택으로 확인되면 보존하고 메모만, 교정하지 않음.
- **편집 리라이트**: 세웅은 오버플로우 줄을 Claude의 리라이트로 받기보다 직접 다시 쓰는 걸 선호. 오버플로우 옵션을 픽셀 측정값과 함께 제시하고 세웅이 선택하거나 교정 텍스트를 직접 제공하게 함.
- **이미지-텍스트 매칭(배치)**: 항상 `<uploaded_files>` 블록 첨부 순서 사용, 알파벳 파일명 정렬 절대 금지. 파일명과 내용이 불일치할 수 있음 (예: `03.jpg`가 4번째 첨부일 수 있음).
- **실행 원칙**: 실제 도구 호출 전에 불가능 선언 금지. 세웅이 접근/권한 있다고 하면 바로 진행 — A/B/C 대안 나열 금지, 실패 사유 사전 탐색 금지.
- **따옴표 들여쓰기**: 런타임 `font.getbbox()` 기반 x 오프셋 시프트, 공백 패딩 아님. 14종 따옴표 지원, 각자 픽셀 폭 있음 (Approach 섹션 참조).
- **그라데이션/블라인더**: 상수는 Python 함수에 하드코딩, 메모리에 없음 — 메모리 변경은 블라인더 출력에 영향 없음.

## 접근 & 패턴

### 표준 카드별 워크플로우 (검증 통과 시 확인 불필요)
1. 텍스트 세트 파싱
2. `check_line_widths()` 실행 — `(들여쓰기 + 텍스트) > 937px`면 중단, 오버플로우 제안 모드 진입
3. 맞춤법 교정 인라인 적용 (`✏` 표기)
4. 이미지 경로·텍스트 줄·KST 출력 경로로 `generate()` 호출
5. `present_files`로 전달

### 오버플로우 제안 모드 (937px 초과 시)
- 합성 중단
- 1안(약어/조사/어미 축소), 2안(짧은 동의어), 3안(의미 단위 줄 분할, 최대 5줄) 제시
- 각 안에 픽셀 측정값 포함한 구체 예시
- 세웅이 선택하거나 교정 텍스트 제공 후 재개

### 따옴표 자동 들여쓰기 (14자 지원)
- 페어드 6종 (스택 push/pop): `" "` U+201C/D 31px, `' '` U+2018/9 18px, `「」` U+300C/D 54px, `『』` U+300E/F 54px, `«»` U+00AB/BB 29px, `‹›` U+2039/3A 18px
- 토글 2종: `"` U+0022 31px, `'` U+0027 18px
- 들여쓰기된 줄도 동일 937px MAX_WIDTH 제약 적용

### 배치 모드 (Case 3)
- 이미지 N장 + 텍스트 N세트 (빈 줄 1개 `\n\n`로 구분)
- `<uploaded_files>` 블록 순서로 매칭 — 파일명 기준 금지
- 이미지 수 ≠ 텍스트 세트 수면 중단
- 출력: `YYMMDD_HHMMSS_card_N.jpg`, 배치 공통 TS
- `generate()` 직접 import해서 1프로세스 안에서 N번 호출

### 비대칭/모호성 처리
- 짝 안 맞는 별표 → 합성 전 의도 확인
- 유니코드/ASCII 따옴표 혼용 → 합성 전 정규화
- 비표준 띄어쓰기 → 명백한 오류 아니면 의도로 간주

### 세웅의 작업 스타일
- 간결·지시적 한국어, 설명 최소
- 이미지+텍스트 함께 전송; 수정은 교정 텍스트를 새 입력으로 재전송
- 변경은 카드 번호·줄 번호로 지정 (예: "8 3줄 1안")
- 레이아웃·맞춤법 검증 통과 시 확인 프롬프트 원치 않음 — 바로 합성
- 간결·임팩트 있는 한국어 카피, 저널리즘 구조 선호; 능동적으로 자기 편집

## 도구 & 리소스

### 파일 네이밍 규칙 (모든 파일 유형)
`YYMMDD_HHMMSS_원본파일명.확장자` — 모든 출력에 적용 (jpg, png, md, py, docx 등)
- 예: `260302_183052_IMG_9821.jpg`, `260302_183100_card_news.py`
- 타임스탬프 생성: `TZ=Asia/Seoul date +"%y%m%d_%H%M%S"`

### 전달 규칙
모든 프로젝트 지침·프롬프트·프롬프트 수정은 `present_files`로 완전한 파일로 전달 — 부분 diff나 인라인 변경 요약만으로는 금지.

### 스크립트 위치
- 프로젝트 소스: `/mnt/project/260415_203533_card_news.py`
- 런타임 복사본: `/home/claude/card_news.py`
- 출력: `/mnt/user-data/outputs/`
- 업로드: `/mnt/user-data/uploads/`

### 폰트
`NotoSansCJK-Bold.ttc` (size 54, index 1) — 경로 `/usr/share/fonts/opentype/noto/NotoSansCJK-Bold.ttc`

### 폭 검증
`card_news.py` 모듈의 `check_line_widths()`, `_strip_emphasis()`; 런타임 픽셀 측정은 `font.getbbox()`

### 드라이브 연동
Zapier Google Drive 커넥터; 폴더 리스팅은 `'FOLDER_ID' in parents and trashed = false` 커스텀 쿼리; `.md`·이미지 모두 직접 curl 다운로드 패턴

### 스마트 크롭 체인
MediaPipe 얼굴 → 포즈 → 에지 검출 fallback; subprocess 인자는 Python 리스트로 (셸 문자열 보간 아님) — 따옴표/별표 이스케이핑 이슈 회피
