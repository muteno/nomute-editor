# 드라이브싱크 ↔ 갤러리 폰 파이프라인 플레이북 (v1.1 · 양방향)

> **용도**: PC에서 받은 미디어(주로 X/트위터 영상·이미지)를 **구글 드라이브 → 폰 갤러리**로 자동으로 흘려보내고(하행 §1), **폰 갤러리에 새로 생긴 것(카메라·스크린샷·저장 이미지)을 드라이브로 자동으로 올려보내는**(상행 §1-2) 양방향 파이프라인의 정본 문서. 끊겼을 때 이 문서 하나로 진단·복구한다.
> **작성 계기**: 2026-07-02 "드라이브가 떨어져 나감" 복구(§7 사례). 이 파이프라인은 **폰 안에만 있고 레포엔 한 줄도 없어서** 폰/PC 갈면 통째로 날아가던 것을 여기 박음. 상행(역방향)은 2026-07-13 운영자 요청("갤러리에 올린 것도 자동 업로드 · 오고간 건 패스")으로 추가.
> **범위**: 이 문서는 **드라이브 ↔ 갤러리 양방향 미러링**을 다룬다. 기사 수집 큐(공유 → pending → Actions)는 별도 = `docs/큐파이프라인_폰재구축플레이북_v1.0.md`. 둘은 서로 안 건드리고 한 폰에서 공존한다.

## 0. 전체 구조 (한 장)

```
[PC A]  다운로더(Seal/gallery-dl) ─┐
[PC B]  다운로더(Seal/gallery-dl) ─┤→  Google Drive "내 드라이브/Shared"
                                    │        (드라이브 데스크톱 "내 드라이브 스트림"으로 저장)
                                    ▼
                        ┌───────────────────────────────┐
                        │  폰 Termux: drive-gallery-sync.sh │  ← rclone gdrive:Shared
                        │  (Tasker가 3분마다 + 잠금해제마다 실행) │
                        └───────────────────────────────┘
                                    ▼
                     /sdcard/Pictures/DriveSync  (LOCAL)
                                    ▼
                        termux-media-scan -r  → 갤럭시 갤러리
```

**핵심 불변**
- 폰은 **rclone `gdrive:` = 내 드라이브 루트**만 본다. → PC가 파일을 넣을 곳은 **반드시 "내 드라이브/Shared"**.
- **⚠️ "컴퓨터 백업"(드라이브 데스크톱 백업)에 넣으면 rclone이 못 본다** = 안 내려옴 = "떨어져 나감"(§7 이번 원인). 두 PC 다 **"내 드라이브 스트림"**으로 저장해야 한다.
- Autosync 앱 **안 씀**(실시간 동기화 안 돼서 버림). 순수 **Termux + Tasker**. `.tmp.drivedownload`/`.tmp.driveupload` 폴더가 보이면 옛 Autosync 잔재다(무시·삭제 가능).
- **역방향(상행) = 갤러리 → `gdrive:Shared/FromPhone`** — `drive-gallery-upsync.sh`(§1-2)가 같은 Tasker 트리거(3분마다 + 잠금해제)로 돈다. **오고간 적 있는 파일은 양쪽 다 패스**: 하행이 내려준 건 안 올리고, 상행이 올린 건 안 내려받는다(원장 공유 = `~/.drivesync.seen` · §1-2 루프 차단).

## 1. 폰 스크립트 — `~/.termux/tasker/drive-gallery-sync.sh` (전문)

```sh
#!/data/data/com.termux/files/usr/bin/sh
REMOTE="gdrive:Shared"
LOCAL="/sdcard/Pictures/DriveSync"
SEEN="$HOME/.drivesync.seen"
NOW="$HOME/.drivesync.now"
NEW="$HOME/.drivesync.new"
LOG="$HOME/.drivesync.log"
STAMP="$HOME/.drivesync.last"
T=$(date +%s)
if [ -f "$STAMP" ] && [ $((T - $(cat "$STAMP"))) -lt 120 ]; then exit 0; fi
echo "$T" > "$STAMP"
mkdir -p "$LOCAL"; touch "$SEEN"
[ -s "$SEEN" ] || echo "__init__" > "$SEEN"
rclone lsf -R --files-only "$REMOTE" > "$NOW" 2>>"$LOG" || exit 1
awk 'FNR==NR{s[$0]=1;next} !($0 in s)' "$SEEN" "$NOW" > "$NEW"
if [ -s "$NEW" ]; then
  rclone copy "$REMOTE" "$LOCAL" --files-from "$NEW" --inplace \
    --no-traverse --transfers 4 --log-file "$LOG" --log-level INFO
  cat "$NEW" >> "$SEEN"
  termux-media-scan -r "$LOCAL"
fi
```

**동작 요약**
- `STAMP` 120초 디바운스 — Tasker가 자주 때려도 2분 안이면 그냥 종료(중복 실행 방지).
- `rclone lsf -R gdrive:Shared` 로 원격 파일 목록(`NOW`) → `SEEN`(이미 훑은 목록)과 **diff** → 새 파일만 `NEW`.
- `NEW` 있으면 그것만 `rclone copy` → `SEEN`에 추가 → `termux-media-scan`으로 갤러리 반영.
- 즉 **한 번 SEEN에 등록되면 다시 안 받는다** → SEEN 오염이 대표 고장(§4·§7).

재설치(폰 갈았을 때):
```bash
mkdir -p ~/.termux/tasker
cat > ~/.termux/tasker/drive-gallery-sync.sh << 'EOF'
# ↑ 위 전문 그대로 붙여넣기
EOF
chmod +x ~/.termux/tasker/drive-gallery-sync.sh
```

## 1-2. 역방향(상행) 폰 스크립트 — `~/.termux/tasker/drive-gallery-upsync.sh` (2026-07-13 추가)

> **목적**: 폰 갤러리에 **새로 생긴** 사진·영상(카메라·스크린샷·저장 이미지)을 `gdrive:Shared/FromPhone`으로 자동 업로드 → PC가 드라이브에서 바로 집어감 = **폰에서 파일 수동 전송 소멸**(클립보드 텍스트 제외).
> **레포 정본 = `docs/drive-gallery-upsync.sh`** (아래 전문은 그 사본 — 수정 시 두 곳 같이).

**"오고간 적 있는 건 패스" — 4중 중복·루프 차단**
1. **하행 착지점 제외**: `/sdcard/Pictures/DriveSync`는 스캔 자체를 안 함(내려받은 걸 도로 안 올림).
2. **오고간 이름 대조**: 업로드 전에 `rclone lsf`(원격 현재 목록) + `~/.drivesync.seen`(하행 원장)을 합쳐, **같은 이름이 이미 있으면 업로드 안 하고 등록만**(예: 내려받은 파일을 갤러리 다른 폴더로 복사해도 안 올라감).
3. **선등록 루프 차단**: 업로드 직전에 그 원격 경로(`FromPhone/파일명`)를 하행 원장 `~/.drivesync.seen`에 **먼저 등록** → 하행 스크립트가 새 파일로 착각해 도로 내려받는 루프 원천 차단(실패 시 자동 롤백). 하행은 lsf를 뜬 **뒤에** 원장을 읽으므로 이 순서가 레이스에도 안전.
4. **자기 원장**: 올린 로컬 경로는 `~/.upsync.seen`에 등록 → 한 번 처리한 파일은 다시 안 봄(하행 SEEN과 같은 방식).

**첫 실행 = 베이스라인**: 현재 갤러리 **전량을 '본 것'으로 등록만 하고 업로드 0** → 설치 이후 새로 생긴 파일만 올라간다(옛 사진첩 통째 업로드 폭탄 없음). `~/.upsync.log`에 `baseline: N files` 확인.

**조절 노브**(스크립트 상단 또는 `~/.upsync.conf`로 덮어쓰기): `REMOTE`(기본 `gdrive:Shared`) · `UP_SUB`(착지 하위폴더 `FromPhone` · 빈값=루트) · `SRC_DIRS`(기본 `/sdcard/DCIM /sdcard/Pictures`) · `EXCLUDES`(기본 DriveSync) · `EXTS`(이미지+영상 14종) · `MAX_MB`(0=무제한) · `WIFI_ONLY`(0=항상 · 1=와이파이만).

```sh
#!/data/data/com.termux/files/usr/bin/sh
# drive-gallery-upsync.sh — 갤러리 → gdrive:Shared 역방향 자동 업로드 (레포 정본 = docs/drive-gallery-upsync.sh)
REMOTE="gdrive:Shared"                    # 풀러와 같은 공간(내 드라이브/Shared)
UP_SUB="FromPhone"                        # 업로드 착지 하위 폴더(빈값 = Shared 루트)
SRC_DIRS="/sdcard/DCIM /sdcard/Pictures"  # 갤러리 스캔 대상(공백 구분)
EXCLUDES="/sdcard/Pictures/DriveSync"     # 풀러 착지점 = 재업로드 금지(공백 구분 복수 가능)
EXTS='jpg|jpeg|png|gif|webp|heic|heif|dng|mp4|mov|webm|mkv|3gp|m4v'
MAX_MB=0                                  # 0=무제한 · N=N MB 초과 파일 스킵(로그 남김)
WIFI_ONLY=0                               # 1=와이파이에서만 업로드(termux-api 필요)
UPSEEN="$HOME/.upsync.seen"               # 업로드축 원장(로컬 경로)
PULLSEEN="$HOME/.drivesync.seen"          # 풀러 원장(원격 경로) = 양방향 공용 열쇠
LOG="$HOME/.upsync.log"
STAMP="$HOME/.upsync.last"
LIST="$HOME/.upsync.scan"; NEWL="$HOME/.upsync.new"; RNOW="$HOME/.upsync.rnow"; TRAV="$HOME/.upsync.trav"
[ -f "$HOME/.upsync.conf" ] && . "$HOME/.upsync.conf"   # 값 덮어쓰기용(선택)

# 0) 120초 디바운스(풀러와 동일 관례) + 로그 로테이션
T=$(date +%s)
if [ -f "$STAMP" ] && [ $((T - $(cat "$STAMP"))) -lt 120 ]; then exit 0; fi
echo "$T" > "$STAMP"
[ -f "$LOG" ] && [ "$(wc -l < "$LOG")" -gt 2000 ] && { tail -n 1000 "$LOG" > "$LOG.t" && mv "$LOG.t" "$LOG"; }

# 1) 스캔(로컬만·네트워크 0): 대상 확장자, 숨김(.trashed-* 등 닷파일) 제외, EXCLUDES 제외
{ for d in $SRC_DIRS; do [ -d "$d" ] && find "$d" -type f -not -path '*/.*' 2>/dev/null; done; } \
  | grep -iE "\.($EXTS)\$" > "$LIST" || true
for ex in $EXCLUDES; do grep -vF "$ex/" "$LIST" > "$LIST.t"; mv "$LIST.t" "$LIST"; done

# 2) 첫 실행 베이스라인: 전량 등록만·업로드 0 (기존 사진첩 통째 업로드 폭탄 방지)
touch "$UPSEEN"
if [ ! -s "$UPSEEN" ]; then
  { echo "__init__"; cat "$LIST"; } > "$UPSEEN"
  echo "$(date '+%F %T') baseline: $(wc -l < "$LIST") files registered, no upload" >> "$LOG"
  exit 0
fi

# 3) 새 파일만 추출(풀러와 같은 awk diff — UPSEEN에 없는 로컬 경로)
awk 'FNR==NR{s[$0]=1;next} !($0 in s)' "$UPSEEN" "$LIST" > "$NEWL"
[ -s "$NEWL" ] || exit 0

# 4) 와이파이 게이트(옵션) — 원장 소비 전에 빠져나가야 나중에 다시 시도됨
if [ "$WIFI_ONLY" = "1" ] && command -v termux-wifi-connectioninfo >/dev/null 2>&1; then
  termux-wifi-connectioninfo 2>/dev/null | grep -q '"supplicant_state": *"COMPLETED"' \
    || { echo "$(date '+%F %T') skip: wifi only" >> "$LOG"; exit 0; }
fi

# 5) 원격 현황 1회 조회 → '오고간 이름' 집합 = 원격 목록 + 풀러 원장
rclone lsf -R --files-only "$REMOTE" > "$RNOW" 2>>"$LOG" || exit 1
touch "$PULLSEEN"
cat "$RNOW" "$PULLSEEN" > "$TRAV"

# 6) 새 파일 처리: 오고간 이름 = 패스(원장 등록만) / 아니면 업로드
while IFS= read -r f; do
  [ -f "$f" ] || { echo "$f" >> "$UPSEEN"; continue; }   # 스캔 후 삭제된 파일 = 등록만
  b=$(basename "$f")
  if [ -n "$UP_SUB" ]; then rpath="$UP_SUB/$b"; else rpath="$b"; fi
  # 오고간 적 있는 이름(원격 존재 or 풀러가 내려준 것) → 패스
  if grep -qxF "$b" "$TRAV" || grep -qF "/$b" "$TRAV"; then
    echo "$f" >> "$UPSEEN"
    grep -qxF "$rpath" "$PULLSEEN" || echo "$rpath" >> "$PULLSEEN"
    echo "$(date '+%F %T') PASS(traveled) $b" >> "$LOG"
    continue
  fi
  # 크기 상한(옵션)
  if [ "$MAX_MB" -gt 0 ]; then
    sz=$(wc -c < "$f" 2>/dev/null || echo 0)
    if [ "$sz" -gt $((MAX_MB*1048576)) ]; then
      echo "$(date '+%F %T') SKIP big(${sz}B) $f" >> "$LOG"; echo "$f" >> "$UPSEEN"; continue
    fi
  fi
  # 풀러 원장에 '선등록' → 업로드 완료 직후 풀러 lsf에 떠도 재다운 안 함(루프 차단)
  echo "$rpath" >> "$PULLSEEN"
  if rclone copyto "$f" "$REMOTE/$rpath" --inplace --log-file "$LOG" --log-level INFO; then
    echo "$f" >> "$UPSEEN"
    echo "$b" >> "$TRAV"                                  # 같은 런 내 동명 중복 차단
    echo "$(date '+%F %T') UP $b" >> "$LOG"
  else
    grep -vxF "$rpath" "$PULLSEEN" > "$PULLSEEN.t"; mv "$PULLSEEN.t" "$PULLSEEN"   # 선등록 롤백
    echo "$(date '+%F %T') FAIL $b (다음 런 재시도)" >> "$LOG"
  fi
done < "$NEWL"
```

**동작 요약**
- 하행 스크립트와 같은 골격(디바운스 120s → 목록 → `SEEN` awk diff → 처리 → 원장 등록). **새 파일이 없으면 네트워크 호출 0**으로 즉시 종료(3분 폴링 부담 없음).
- 업로드는 파일별 `rclone copyto` → `Shared/FromPhone/파일명`으로 착지(PC에서 폰발 파일만 모아 보임). `UP_SUB=""`로 비우면 Shared 루트 직행.
- rclone 리모트·auth는 하행과 **동일한 `gdrive:` 하나**를 그대로 씀(§2 · 단 scope가 `drive.readonly`면 업로드 불가 → `drive` 전체 권한이어야 함).
- 하행 스크립트(`drive-gallery-sync.sh`)는 **한 줄도 안 바꿈** — 루프 차단은 전부 상행이 원장 선등록으로 해결.

재설치(폰 갈았을 때):
```bash
mkdir -p ~/.termux/tasker
cat > ~/.termux/tasker/drive-gallery-upsync.sh << 'EOF'
# ↑ 위 전문 그대로 붙여넣기 (레포 있으면: cp docs/drive-gallery-upsync.sh ~/.termux/tasker/)
EOF
chmod +x ~/.termux/tasker/drive-gallery-upsync.sh
bash ~/.termux/tasker/drive-gallery-upsync.sh   # 1회 수동 실행 = 베이스라인 등록
tail -3 ~/.upsync.log                            # "baseline: N files" 확인
```

## 2. rclone 설정 (폰)

- 리모트 이름 = **`gdrive`**, 백엔드 = Google Drive. `gdrive:` 루트 = **내 드라이브**.
- 스크립트가 보는 경로 = `gdrive:Shared` = **내 드라이브/Shared**.
- 최초/재설정: `rclone config` → `n`(new) → 이름 `gdrive` → `drive` → 클라이언트ID 비움 → scope는 `drive`(전체) 또는 `drive.readonly`(읽기만 해도 됨, 폰은 내려받기만 하니까) → 브라우저 인증.
- **토큰 만료 재연결**: `rclone config reconnect gdrive:` (브라우저 동의 다시).

## 4. PC 설정 (두 PC 통일 = 재발 방지의 핵심)

**목표**: 두 PC가 **같은 폴더 `내 드라이브/Shared`** 에 저장 → 폰 rclone이 한 곳만 보면 됨.

각 PC에서:
1. **Google Drive for Desktop** 설치 → 로그인.
2. 설정에서 그 다운로드 폴더를 **"이 컴퓨터 폴더 백업"에서 빼고**, **"내 드라이브" 스트리밍**을 켠다 → 탐색기에 `G:\내 드라이브\Shared`(가상 드라이브)가 보이게.
3. 다운로더(Seal / gallery-dl 등) **저장 위치를 `G:\내 드라이브\Shared`로 지정**.
4. 확인: 그 PC에서 파일 하나 저장 → 폰에서 `rclone lsf gdrive:Shared`에 뜨면 성공.

**⚠️⚠️ 절대 규칙 — "컴퓨터 백업" 금지.** 드라이브 데스크톱의 "이 컴퓨터 폴더 백업"으로 저장하면 파일이 **드라이브 "컴퓨터" 탭**(내 드라이브 아님)으로 가고, **rclone `gdrive:`(내 드라이브)는 그걸 못 본다** → 폰에 안 내려옴. 이게 2026-07-02 "떨어져 나감"의 근본 원인이었다(두 PC가 각각 컴퓨터-백업으로 저장해 `내컴퓨터/Shared`·`내컴퓨터/다운로드2/Shared`로 갈라짐). **두 PC 다 반드시 "내 드라이브 스트림".**

옛 컴퓨터-백업 잔재(`내컴퓨터/Shared`, `다운로드2/Shared`)는 백업 해제하거나 삭제해 헷갈림을 없앤다.

## 5. 진단 명령 모음 (끊겼을 때 순서대로)

```bash
# ① rclone auth·리모트 살아있나 + 내 드라이브 루트 확인
rclone lsf gdrive:
#   → $Temp/ Shared/ 처럼 목록 뜨면 auth 정상.
#   → 'token'·'401'·'couldn't find' 에러면 → rclone config reconnect gdrive:

# ② 새 파일이 내 드라이브/Shared에 실제로 있나 (핵심 판별)
rclone lsf -R gdrive:Shared
#   → 최근 파일 뜨면 = 자리 정상(Ⓐ). 비었거나 옛것만 = PC가 컴퓨터-백업에 저장 중(Ⓑ) → §4로.

# ③ 스크립트 수동 실행 (에러/동작 확인)
bash ~/.termux/tasker/drive-gallery-sync.sh

# ④ 로컬에 내려왔나
ls -la /sdcard/Pictures/DriveSync

# ⑤ 로그·상태 파일
tail -20 ~/.drivesync.log
cat ~/.drivesync.seen      # 이미 훑은 목록(오염 확인)

# ⑥ 역방향(상행) 수동 실행·로그 (§1-2)
bash ~/.termux/tasker/drive-gallery-upsync.sh
tail -20 ~/.upsync.log     # UP=올림 · PASS(traveled)=오고간 것 패스 · baseline=첫 등록
cat ~/.upsync.seen         # 업로드축 원장(로컬 경로)

# ⑦ 갤러리 새 파일이 드라이브에 실제로 올라왔나
rclone lsf -R gdrive:Shared/FromPhone
```

## 6. 흔한 고장 → 복구 (트러블슈팅)

| 증상 | 원인 | 처치 |
|---|---|---|
| **"드라이브가 떨어져 나감" / 새 파일이 폰에 안 옴** | PC가 **컴퓨터 백업**에 저장 → rclone(내 드라이브)이 못 봄 | 파일을 `내 드라이브/Shared`로 이동 + PC를 **"내 드라이브 스트림"**으로(§4) |
| **드라이브엔 파일 있는데 갤러리엔 없음** | `.drivesync.seen`에 이미 등록됐는데 실제 복사가 안 된 채 "본 걸로" 처리됨(SEEN 오염) | `echo "__init__" > ~/.drivesync.seen` → `bash ~/.termux/tasker/drive-gallery-sync.sh` (밀린 것 재-pull) |
| `rclone lsf gdrive:` 에서 auth 에러 | rclone 토큰 만료·취소 | `rclone config reconnect gdrive:` (브라우저 동의) |
| 폴더는 뜨는데 `gdrive:Shared`가 빔 | 파일이 두 PC 컴퓨터-백업으로 갈라짐 | §4 통일 |
| 갤러리에 안 뜸(파일은 `/sdcard/Pictures/DriveSync`에 있음) | media-scan 누락 | `termux-media-scan -r /sdcard/Pictures/DriveSync` |
| Tasker가 안 돎 | allow-external-apps 주석 / 배터리 최적화 | 큐 플레이북 §3-1 + Termux 배터리 최적화 제외 |
| **갤러리 새 파일이 드라이브에 안 올라감** | ① 첫 실행 직후(베이스라인은 등록만) ② `.upsync.seen`에 이미 등록 ③ 같은 이름이 드라이브에 이미 있어 PASS ④ rclone scope가 `drive.readonly`(업로드 권한 없음) | `tail ~/.upsync.log`에서 UP/PASS/FAIL 판별 → 밀린 파일만 살리려면 `.upsync.seen`에서 **그 경로 줄만 삭제** 후 재실행 · scope 문제면 `rclone config reconnect gdrive:`에서 `drive` 전체 권한 |
| **올린 게 도로 폰에 내려옴(중복 1회)** | 극단 레이스 or `.drivesync.seen`에서 `FromPhone/…` 줄이 지워짐 | 중복 파일 삭제로 끝(1회성·무해). `.drivesync.seen`에 `FromPhone/파일명` 줄 존재 확인·없으면 수동 추가 |
| **첫 실행에 옛 사진 전부 올라갈까 걱정** | 안 올라감 — 첫 실행 = 베이스라인(전량 등록만·업로드 0) | `~/.upsync.log`의 `baseline: N files` 줄 확인 |

**업로드축(.upsync.seen) 리셋 주의**: 파일을 지우면 다음 실행이 다시 **베이스라인**(현재 전량 등록만) — 업로드 폭탄은 없지만 **아직 안 올라간 새 파일까지 '본 것'으로 삼켜진다**. 밀린 것 살릴 땐 리셋 말고 해당 경로 줄만 삭제.

**SEEN 리셋 주의**: `echo "__init__" > ~/.drivesync.seen` 하면 다음 실행에 `gdrive:Shared`의 **현재 전량**을 다시 받는다. 원격에 파일이 많으면 그만큼 복사되니, 밀린 것 따라잡기용 **1회성**으로만. 평소엔 건드리지 말 것(새 것만 자동으로 옴).

## 7. 복구 사례 로그

### 2026-07-02 — "드라이브가 떨어져 나감" 복구
- **증상**: PC에서 받은 X 영상이 폰 갤러리로 안 옴. "드라이브가 떨어져 나간 것 같다."
- **진단 경로**:
  1. `drive-gallery-sync.sh`는 `gdrive:Shared`(내 드라이브/Shared)를 봄 — 스크립트·rclone auth 정상.
  2. 실제 파일은 **"컴퓨터" 탭(드라이브 데스크톱 백업)**에 있었음 — `내컴퓨터/Shared` + `내컴퓨터/다운로드2/Shared` 로 **두 PC가 각각 백업해 갈라짐**. rclone(내 드라이브)은 이걸 못 봄.
  3. 이후 파일을 `내 드라이브/Shared`로 되돌리니 `rclone lsf -R gdrive:Shared`에 3개 영상 다 보임(자리 정상).
  4. 그런데도 갤러리 안 옴 → `.drivesync.seen`에 그 3개가 **이미 등록**돼 있었음(끊겼던 시절에 seen엔 찍혔는데 복사는 실패). 스크립트가 "새 거 없음"으로 스킵.
- **조치**: `echo "__init__" > ~/.drivesync.seen` → `bash ~/.termux/tasker/drive-gallery-sync.sh` → 3개 영상 복사·media-scan → **갤러리 반영 확인.**
- **재발 방지**: 두 PC 다 **"내 드라이브 스트림"**으로 저장하도록 통일(§4). 컴퓨터-백업 금지 규칙 박음.
- **교훈**: 이 파이프라인이 레포에 없어서 매번 폰에서 재추적 → 이 문서로 SSOT화.
