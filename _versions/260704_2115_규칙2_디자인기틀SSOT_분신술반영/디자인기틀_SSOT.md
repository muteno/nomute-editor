# 노뮤트 디자인 기틀 SSOT 인덱스 — 「규칙2번」 (단일 진입점 · 운영자 정본 260704)

> **이 파일 = 규칙2번.** 디자인 기틀의 *모든* 정본 위치를 한 곳에 모은 **단일 진입점 인덱스**다. **CLAUDE.md §🔒 제1 핵심명령(규칙1)이 이 파일을 *절대적으로 따른다*** — 디자인 관련 무엇이든(값·규칙·컴포넌트·확립본·거울·게이트) **정본 위치는 여기서 찾는다.** 지금까지 §🔒·§🎨·SessionStart 훅·CII·확립본에 *흩어져 있던* 참조를 여기로 집약했다.
>
> ⚠️ **이 인덱스는 *포인터만* 담는다 — 값·규칙의 실체(hex·px·셀렉터 본문)를 복붙하지 마라**(복붙 = 드리프트원). 실체는 각 정본 파일에 있고, 여기는 "어디에 있는지"만 가리킨다. 새 디자인 기틀 파일이 생기면 이 인덱스에 **행을 추가**한다(누락 = 규칙2 위반).

## 🧭 위계 한 줄
**값(`:root`) → 규칙(§🔒·§🎨) → 컴포넌트(CII) → 확립본(사인오프) → 거울(build 파생) → 게이트(check_refs 강제).** 구체 충돌 시 **값(`viewer/index.html :root`)이 최종 정본.** 계승이 디폴트(안 물음), 갱신은 운영자 승인(§🔒 ①②③).

## 🔧 유지 규칙 (이 인덱스 자체의 불변)
1. **새 디자인 기틀 파일 = 이 인덱스에 행 추가**(누락 = 규칙2 위반 = §🔒 위반).
2. **포인터만 — 실체(hex·px·셀렉터) 복붙 금지**(드리프트원). 값은 `:root`, 규칙은 §🎨, 컴포넌트는 CII에.
3. **확립본은 append-only**(새 사인오프 = 새 확립본 파일 + 여기 §4 행 추가 · 옛 확립본 삭제 금지 = 시점 증거 보존).
4. 이 인덱스 수정 = 디자인 기틀 = §기틀 보호 + §🧪(§🔒 규칙2 편입은 운영자 승인).
