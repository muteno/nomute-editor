- c) **백엔드 = ly_burn.py 단일 컴포지터**(평의회2 260710): 한 ffmpeg 파이프{(트림)→컷 select→크롭→스케일→fps→패드→ASS} + 음량 후처리(비디오 copy). **자막은 최종 캔버스(PlayRes) 기준 렌더 = 검정 여백 위에도 앉음**(릴스 문법). 편집기 축 전부 결측 = 순수 ly 경로와 동일(회귀 0 — 로컬 9케이스 실측).

## 2. 캡·예산 (기틀 — 완화 = 운영자 확인)
- a) 영상 길이 = **10분(600초) 원본 기준 정직 거절**(편집기는 영상이 곧 산출 — ly의 fail-soft와 달리 선게이트 exit 1 → error.log). 업로드 ≤30MB(conv 통일 · ly 20MB보다 상향).
- b) **60fps 보간 = 예산 가드**: est = 길이×60×0.30s/프레임×(해상도/1080×1920) ≤ **900초** — 초과 = 보간만 건너뛰고 note(다른 처리는 진행 = 잡 살림 · conv의 die와 의도적 상이). 인코딩 백스톱 = 900+est×1.5s · 컴포즈 스텝 45분 · 잡 70분.
- c) 뷰어 폴 = 예산 2100s(하드 ×2 = 4200s = 잡 캡) · 재개 게이트 24h(pend `nm_edit_pend`).

## 3. 알려진 한계 (정직 · v1)
- a) 트림×자막 동시 = 지원(260711 리맵) — 단 STT·의역은 **전체 영상** 기준으로 돌고 번인만 구간 반영(구간 밖 의역 = 비용 낭비 소폭 · 트림 선반영 STT는 후속).
- b) **컷 편집 = 자막 필요**(무음 판정 원천 = STT 전사) — 자막 없이 컷만은 불가(시트에 사유 표기).
- c) 다시 입히기(reburn)·상세 자막 편집기·자막 텍스트 뷰 = 편집기 v1 미탑재(자막 생성기 탭 그대로 사용 — 같은 `ly_out/<id>` 계약이라 후속 접합 쉬움).
- d) 크롭 위치 미리보기(드래그 프레임) = v1 미탑재(위치 슬라이더만 · conv 탭엔 있음 — 후속 이식).
- e) 컷 + 60fps 보간 동시 = 컷 이음매에서 보간 워블 가능(minterpolate 장면전환 감지가 완화 · 원리 한계).
- f) 예상 소요 = 근사(60fps는 파일이면 길이 기반·URL이면 중간값) — 실측은 대기 캡션 경과 초.
