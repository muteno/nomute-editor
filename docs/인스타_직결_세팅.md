# 인스타 직결 세팅 — 운영자 1회 가이드 (Meta Instagram API · Instagram Login)

> `/insta` 자동 수집(`insta-fetch`)을 켜는 1회 세팅. 약 20분 · 무료 · **앱 심사 불필요**(본인 계정 전용 = Standard Access). 끝나면 매일 06:25 KST 자동 수집 + Actions 수동 발사. ⚠️ 메뉴 문구는 Meta UI 개편에 따라 조금 다를 수 있다(취지는 동일 — 화면이 크게 다르면 그 화면 스샷 던져주면 짚어줌).

## 1. 전제 (이미 충족했을 가능성 높음)
1) 인스타 계정 = 프로페셔널(크리에이터 또는 비즈니스). **앱에서 인사이트 화면이 보이면 이미 충족.**
2) 페이스북 페이지 연동 불필요(Instagram Login 경로의 장점).

## 2. Meta 개발자 앱 만들기
1) https://developers.facebook.com → 로그인 → `내 앱(My Apps)` → `앱 만들기(Create App)`.
2) 사용 사례 = **Instagram** 계열 선택{문구 예: *Instagram API 사용* / *Manage everything on your Instagram account*} → 앱 이름 자유(예: `nomute-insta`) → 생성.
3) 좌측 메뉴 `Instagram` → **`API setup with Instagram business login`**(인스타그램 로그인 API 설정) 화면 진입.

## 3. 계정 연결 + 토큰 발급
1) 그 화면 **`Generate access tokens`** 단계 → `계정 추가(Add account)` → no_mute 인스타 로그인·수락.
2) 권한(스코프)은 읽기 전부 체크 — 최소 `instagram_business_basic` + `instagram_business_manage_insights`.
3) `Generate token` → 뜨는 **긴 토큰 문자열 복사**(= 60일 유효 장수명 토큰 · 화면 밖으로 새지 않게 주의).
4) (선택) 같은 화면의 **Instagram account ID**(숫자)도 복사 — 없어도 동작(`me`로 자동 해석).

## 4. GitHub 등록
1) 이 레포 → `Settings` → `Secrets and variables` → `Actions` → `New repository secret`:
   - Name = `IG_ACCESS_TOKEN` · Value = 3-3) 토큰. **(필수 · 이것 하나면 됨)**
2) (선택) 같은 화면 `Variables` 탭 → `New repository variable`: Name = `IG_USER_ID` · Value = 3-4) 숫자 ID.

## 5. 첫 수집 확인
1) 레포 `Actions` 탭 → `insta-fetch` → `Run workflow` 수동 발사.
2) 초록 완료 + `apps/insta/data/`에 {`insights_daily.jsonl` · `media_latest.json` · `audience.json`} 생기면 성공 — 이후 매일 06:25 KST 자동.
3) `/insta` 새 세션 진입 = 이 데이터를 1차 원료로 자동 사용(스크린샷 불요 · 지침 §2-0).

## 6. 토큰 만료·재발급 (60일 주기)
1) 장수명 토큰 = 60일. 관측 50일 경과 시 Actions 로그 warning · 만료되면 런이 빨간 실패(`::error` 토큰 만료 의심).
2) 재발급 = §3 화면에서 `Generate token` 다시 → §4-1) 시크릿 **값만 교체**. (약 2분)
3) 무개입 자동 갱신은 후속 결정 대기 — {가) repo variable에 롤링 토큰 저장 = 편하지만 시크릿 대비 노출면↑ / 나) PAT 경유 시크릿 갱신 = 세팅 1회 추가} 중 운영자가 고르면 배선.

## 7. 안전·한계 (정직)
1) **토큰 = 계정 열람 권한** — 레포 파일·채팅에 절대 붙이지 말고 시크릿에만. 스크립트는 토큰 원문을 어디에도 저장 안 함(`token_meta.json` = sha256 해시 꼬리 12자만).
2) 수집 = 본인 계정만. 경쟁계정 인사이트 불가 · 비공식 스크래핑 금지 불변(지침 §8-3).
3) 인구통계(`follower_demographics`) = 팔로워 100명 이상부터 제공(Meta 제한) · 지표는 Meta가 개폐할 수 있어 수집기는 낱개 폴백으로 자가 적응(죽은 지표 = `dropped` 기록).
4) 과거 이력은 조회 범위 제한이 있어 **연결 시점부터 일별로 쌓는 구조** — 빨리 켤수록 시계열이 길어진다.
