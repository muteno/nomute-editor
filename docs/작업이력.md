> **목적**: 완료 작업을 **append-only로 누적**(컨텍스트 증발·압축에도 안 잃게)하고, 실행 방식 개선점을 **§2 우선사항**에 박아 다음 세션이 더 고효율·고품질로 일하게 한다. 정본 = 이 파일(라우터 §🎯·§🧠가 가리킴). HTML 보고서 산출물은 `docs/reports/`.
>
> **역할 경계(중복 차단 · 드리프트 방지)**: 수집함 큐레이션 결정 = `docs/curation-algorithm.md §8`이 정본(여기엔 "§8에 머지됨" 한 줄 링크만, 전문 재기록 금지) · 세션 서사·디버그 로그 = `docs/_세션로그_*` · **이 원장 = 완료 작업 *인덱스***(전후·결정·검증·산출 한 항목). 같은 결정을 두 곳에 전문 기록하지 않는다.
> **비대 cap·충돌 완화**: §1이 ~50항목 넘으면 6개월+ 지난 항목을 `docs/작업이력_archive_{연}.md`로 롤오버(한 줄 요약만 남김). 동시세션 머지충돌 줄이려 **날짜 헤더(`### YYMMDD`) 블록 단위 append**(다른 날 = 다른 줄). 새 항목은 **최신 헤더 블록 안**에 추가.
>
> **한 항목 양식**: `라벨` → ① 의도(문제상황/해결목표) · ② 변경(전→후) · ③ 효과·리스크 · ④ 검증(평의회 평결) · ⑤ 산출(PR·보고서). ⑤의 PR·평결은 **머지 직전** 실값으로 채운다(미확정 박지 말 것 = §2).
