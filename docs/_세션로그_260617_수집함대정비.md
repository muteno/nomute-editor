## 0. 한 줄 요약
운영자와 집중 세션으로 **수집함(누적/신속) 큐레이션 알고리즘 대정비.** 구조 신호만으론 보도자료 vs 사건을 못 가른다는 걸 *실측으로 입증* → **AI 경중 게이트(Claude grade 0~3)** 도입 → 신속/누적 모델 확정 → 속보 판정(다수동시 + [속보]태그 → AI 검증) → 보수성·SSOT·도돌이표방지 원칙을 문서에 박음.

---

## 1. 전체 방향 (정본 = docs §★ · 여기 재수록)
**목표**: RSS 다매체 수집 → 내용·키워드 *최소*로 중요·양질 가려 큐레이션. **보수적**(의심시 조임).

**핵심 신호 5종 (곱으로 직조):**
1. **다매체(cross)** — 여러 곳 보도 = 검증.
2. **연속보도(report_count)** — 후속 스크랩(15분 뒤)에 또 보도되면 +1 = 양질·중요 핵심 → **상단**(`scFollow`). 끊기면 하강. (보도자료=한번 깔고 끊김 / 사건·양질=계속).
3. **동시성(burst)** — 다수 동시 = 속보 신호.
4. **AI 경중(grade 0~3)** — 보도자료/잡음(0)·경미(1) 침몰, 대형(3) 부상(`gradeW`). 구조가 못 가르는 것 보완.
5. **[속보]·[단독] 태그** — 1~2매체여도 확률↑ → AI 내용확인.

**두 컬럼:**
- **누적(좌) = 9시뉴스감** — 다매체+연속보도(오래돼도 OK). 활동시각(`last_report`) 시간감쇠, `crossW`(10+ 추가 로그가점), 대표=메이저(`breaking_pick`).
- **신속(우/최신) = 최신(<6h)** + (속보 OR 2+매체 양질), 잡음(grade 0/1)만 제외. 연속보도·품질 상단. 오래되면 누적行.

**이슈 vs 속보 (개념)**: 이슈=넓음 / **속보 = 이슈 중 <3h + breaking 확정.** 배지 🚨속보(빨강·<3h)/⚡이슈(노랑·3h+ aged).
**속보 판정**: ① 다수 동시(burst≥3) ② [속보]/[상보] 태그 → **둘 다 breaking_judge(Claude)가 내용 검증** → 확정. (단일매체 중대사건도 AI로 속보 가능.)
**랭킹 = 점수 곱**: `crossW × 시간decay(활동) × scFollow(연속보도) × gradeW(AI품질)` (+ 최신 속보 핀).

## 2. 라이브 메커니즘 (코드 위치)
- **분류·정렬·배지**: `viewer/index.html` — `scFast`(신속 진입=breaking OR cross≥2&grade∉{0,1}) · `scScore`(누적) · `fastScore`(신속) · `scFollow`(연속보도 가점) · `gradeW`(AI 경중) · `crossW`(고cross) · `scReport`(활동시각 decay) · `pinFresh`(최신속보 핀) · `breakingNow`(토스트=<6h) · sc-brk 배지(🚨속보/⚡이슈).
- **신호 산출**: `scraper/to_candidates.py` — `cross`·`burst`·`arts`(cluster_size)·`report_count`(arts 증가=연속보도)·`last_report`·`breaking_candidate`(burst≥3 OR [속보]태그)·대표 media/title=`breaking_pick`(메이저).
- **AI 검증**: `.github/scripts/breaking_judge.py`(속보 0/1 판정) + `.github/scripts/gate_judge.py`(경중 0~3 채점·청크40·런당상한80·최신먼저·해시도장) → `breaking-judge.yml` 한 워크플로(구독 OAuth).
- **데이터 누적**: `scraper/snapshot.py` → `scraper/obs/`(events.jsonl + 일별 델타 = 사건 cross/burst 궤적 영구).
- **검증 도구**: `scraper/hypothesis_lab.py`(신호 AUC·알람 정밀도).

## 4. 핵심 원칙 (문서에 박음)
1. **보수성 (docs §1)**: 의심시 알고리즘 *조이는 쪽*. cross 임계 낮춤·피드 무분별 추가로 커버리지 안 쫓음. **RSS 미수집 = 대부분 정상**(다수매체 미검증=일부러 배제). 네이트 등 외부 포털이 더 띄워도 좌편향·낚시·단일출처 때문 → "구멍"으로 단정해 풀지 말 것.
2. **SSOT 즉시 갱신 (CLAUDE.md · "제일 중요")**: 개념·룰 바뀌면 *즉시* `docs/curation-algorithm.md`(§★·§8) 반영. 코드만 고치면 다음 세션 드리프트.
3. **도돌이표 방지**: 방향 망각한 채 불만마다 entry/임계만 국소패치 = 도돌이표. 수집함 손대기 전 §★ 먼저 읽기(CLAUDE.md 포인터).
4. **기틀 보수 유지**: 신호 구조·두 컬럼·AI검증 2층은 보수적으로. 불만 하나에 기틀 안 흔듦.

## 5. 실측·검증 결과
- **구조 신호 한계 입증**: cross·burst·꼬리·report_count·seen_count 전부 사건 vs 보도자료 분리 **AUC 0.5~0.59(못 가름)**. cross≥4 알람 89% 오발령. → AI 게이트 필수임을 숫자로 증명.
- **AI 게이트 정확도 검증됨**(31건 샘플): grade 0 = 보도자료(현대카드·세븐일레븐·환율·스포츠) 정확 침몰 / grade 3 = 대형(계엄 구속·중학생 사망사고) 부상. **구조로 못 갈랐던 걸 AI가 정확히 가름.**
- **꼬리(last_seen−first_seen) 신호 오작동**: 큰 사건도 늦게 잡으면 꼬리 0 → 활동시각(last_report)·report_count로 대체.
- **네이트 비교**: 톱 진짜뉴스 11개 중 6개 미수집(잠수함·168명·촉법·만취·지명수배) → *대부분 정상*(보수매체 미보도=신호 / 좌편향·단일출처). 보수성 원칙으로 수용.
