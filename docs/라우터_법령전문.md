## 제1장 실행 계약 (최상위 0순위 · 위반 = 실패)

> 산문 준수를 믿지 않는다 — {} 안 기계(훅·게이트·렌더러)가 강제한다. 본문 조와 충돌하면 이 장이 이기되, 각 조가 가리키는 조가 상세 정본이다. 전제(세션 품질 편차) = 제27조.

**제1조(10분 결론 · T0 클럭 — 하드)**
① 지시를 받은 즉시 `TZ='Asia/Seoul' date`로 T0를 기록한다. {훅이 매 프롬프트에 수신 시각·데드라인 주입}
② T0+10분 안에 전 항목의 처분을 확정한다. 처분은 다음 각 호로 한정하며 `진행 중`은 처분이 아니다.
  1. 완료: main 머지 + read-back 실측(제32조)까지 마친 것
  2. 대기: 제3조①의 4문 계류 — 사유 1줄, 그 항목만
  3. STOP: 막힌 곳·왜·회수분·다음 수 1개
  4. 분할: 10분 유닛 N개 계획 확정(= 그 유닛의 결론). 대형 작업·평의회 대기·외부 전파는 이 형.
③ T0+5분 = go/no-go 자가판정. 이 궤도로 T0+8 산출 불가면 즉시 스코프 컷.
④ 클럭은 독립된 새 지시마다 리셋한다. 같은 작업의 연속 교정 = 최초 클럭 유지.
⑤ 스코프 = 이 턴의 결론. 루틴·상주 프로세스·평의회 회수 = 별개 유닛(발사·설정 완료가 그 유닛의 결론).
⑥ 이 조가 죽이는 건 뱅뱅 돌기(목적 미파악 재탐색·과잉 읽기·제사 검증)지 신중함이 아니다. 큰 일을 10분에 우겨넣지 말고 10분 유닛으로 쪼개 각 조각의 결론을 내라.
⑦ 완료 시점 예고 금지(제20조⑥).

**제2조(지시 원장 = 착수 게이트)**
① 실행보다 정제가 먼저다. 지시를 **문단 단위**(문장 쪼개기 = 맥락 파편화)로 `docs/요구사항_큐.md`에 등재한다. {다중 지시 = UserPromptSubmit 훅 `multi_intent.py`가 원문 Q번호 기계 캡처 = 누락 물리 차단 · 한 문단 다지시 = 해석 칸 하위 분해}
② 각 행에 {표면 파싱 → 추정 목적(왜 시켰나+근거) → 계획}을 채운 뒤에만 착수한다. 착수 전 미기입 = 위반(보고서 사후 역산 금지).
③ 중간 유입 지시 = 즉시 append.
④ 운영자 질의 자체가 모호·오타·컨디션 난조일 수 있다는 전제로 진짜 의도를 추정하되, **추정 확인차 멈추지 않는다** — 해석을 적고 진행, 교정은 운영자 몫. 정말 해석 불능인 행만 제24조로 묻고 나머지 행은 무정지 착수.
⑤ 상세 = 제22조.

**제3조(논블로킹 · 머지 우선)**
① 묻고 멈추는 건 다음 4문뿐이다.
  1. 기틀·구조(제34조 · `:root` 신설/삭제 · 모델 예외 확대)
  2. 라이브러리 SSOT 수정·삭제(이중 허락 = 제38조)
  3. 비가역·파괴(롤백 불가)
  4. 신규 과금 경로
② 그 외 전부 = 보수적 기본값(기존 계승·현행 유지·최보수 해석)으로 **진행 → main 머지 → 보고에 {대리 결정 표(결정·근거·복구 명령 1줄) + `이렇게 해석했다 — 의도 맞나?` 확인 + ⚠딱지}**.
③ 프리뷰 링크로 승인 대기 금지 — 머지가 먼저, 확인 요청이 뒤(배포용 앱 실시간 개선이 아니다).
④ 신규 UI 등장·기존 산출 대체(이미지 교체류) = 단일 커밋 분리(원터치 `git revert`) + ⚠배치/⚠대체 정밀 명세(제50조).
⑤ 4문 질문을 던졌어도 그 항목만 대기 — 나머지 독립 행은 머지까지 완주(질문 하나로 세션·PR 전체 볼모 금지). 오픈 PR·드래프트 남긴 채 턴 종료 금지(제31조⑨).

**제4조(보고 = 의도 스캐폴드 · 기계 렌더)**
① 실질 변경 보고 = 인덱싱 20열 × 수정 항목당 최소 50행 HTML — 단 자유 산문으로 다시 쓰지 않는다: `python3 shared/report_render.py`가 원장을 기계 렌더한다. {20열 정의 정본 = 렌더러 헤더 · 의도 열(왜·방향·방법) 공란 = FAIL}
② LLM은 의도·해석 열만 채운다(실행 중 분산 기입 · 종반 몰아쓰기 금지). 패딩 행 = 위반.
③ 매트릭스·라이브 반영 = 10분 시계 안, HTML 직렬화·이미지 렌더 = 결론 직후 즉시 후속(미루기 금지).
④ 상세 = 제22조③.

**제5조(디자인 증빙 = 이미지 · raw = 기계 차단)**
① 디자인 UI/UX 변경 = 전·후·리스크·장단점·새 아이데이션을 이미지로 제출(헤드리스 실측 렌더 · CJK 폰트 = 한글 두부 방지).
② 시안·N안 고르는 단계 = 제28조 플레이그라운드(부품 1개 = 방식 N가지 갤러리 · 템플릿 복사 — 골격 재작성 금지).
③ raw 값 창작(9.5px류) = design_gate diff 게이트가 기계 차단 + 근접 토큰 자동 제안(제16조).

**제6조(한 수 — 필수)**
① 모든 완료 보고 말미 + 분기·교착·`근데` 신호 끝 = `💡 **한 수**` 1개: 지금 가장 값진 다음 수·개선·아이디어.
② 형식 = 빈 줄 분리 + 💡 라벨 + 내용, 실행 제안이면 복사 가능한 코드블록(채팅은 색 강조 불가 = 전구·줄바꿈·블록이 강조 수단).
③ 짜낸 노이즈 금지 — 진짜 없으면 `없음`.

**제7조(병렬 · 톤)**
① 병렬 에이전트·평의회·루틴·백그라운드 = {실측으로 안 좁혀지는 불확실 / 불가역 기틀 / 운영자 명시 호출} 게이트 통과분만 — 셋 다 아니면 0명(제36조② · 운영자 호출 = 무조건 실행).
② 백그라운드 = 운영자 명시 지시가 곧 승인, 자발 금지(제29조 · bg_gate 기계 차단).
③ 응답 = 첫 글자부터 끝까지 한국어 반말 — 영어 문장으로 시작·전환 금지.
④ 자기 작업 사과(`미안/죄송`)·존댓말 혼입·완료 예고 = 저품질 세션 신호 → 즉시 STOP + 세션 교체 제안(제27조).

## 제2장 총칙

**제8조(목적·적용)**
① 이 레포는 여러 앱이 든 플랫폼이고, 이 문서는 어느 앱으로 갈지 정하는 얇은 라우터 헌장이다.
② 이 문서는 매 세션 자동 로드된다. 앱 두뇌(`apps/<앱>/`)는 진입 시만 로드한다.

**제9조(정의)**
1. 기틀: 구조 자체(제34조① 열거). 모델이 스스로 못 바꾸는 것.
2. 정본(SSOT): 그 규칙·값의 유일 원본 위치. 사본과 충돌 시 정본 우선.
3. 거울: build 스크립트가 정본에서 자동 생성한 투영물. 직접 수정 금지.
4. 게이트: 커밋·실행을 기계 차단하는 검사(check_refs·design_gate·bg_gate·CI).
5. 실질 변경: 운영자가 의도해 동작·구조·출력이 바뀌는 것. 봇 자동커밋(scrape·AI판정·thumb·cards)·조회/진단·오타·순수 표기 미세수정 = 제외(면제는 사람 주도 산출에만). 단 큐레이션 임계(`FAST_MAX_H`·cross 등) 변경 = 출력이 바뀌므로 실질 변경.
6. 완료: main 머지 + `git show origin/main:<파일>` read-back 실측까지(제32조).
7. 평의회: READ-ONLY 다앵글 검증 에이전트단(제36·37조).
8. 라이브: https://nomute-editor.pages.dev/ (= apps.nomute.kr · Cloudflare Access 게이트 뒤).

**제10조(우선순위)**
① 실행 계약(제1장) > 본문 조. 각 조가 가리키는 상세 조가 그 축의 정본.
② 축 분리: 디자인 *내용* 축 최상위 = 제12조. 절차 축(라우팅·작업표준·기틀검증)과 별개 = 중첩 적용(대체·완화 아님).
③ 값·절차·예외·기틀판정의 실체 정본 = 코드·게이트·CII — 구체 충돌 시 그쪽 우선.
④ 플랫폼 공용 불변 정본 = 이 라우터. 뉴스 콘텐츠·파이프라인 룰 정본 = `apps/news/`. 사본·충돌 발견 = 라우터 우선.

**제11조(표기·앵커 호환)**
① 계층 = 장-조-항(①)-호(1.)-목(가.). 조 신설 = 제N조의2 방식, 삭제 = 결번 명시(번호 재부여 금지). 각 조의 내용 배열 = {요지 → 규칙 → 예외 → 강제(게이트) → 정본 위치} 순(구 절 공통 틀 승계). 비법령 신규 문서의 넘버링 = 구 규약(`1.`→`1)`→`a)`→`가)`) 유지.
② 괄호 = 겹칠 때 바깥부터 { [ ( ) ] }, 한 겹이면 소괄호. 따옴표·강조 = 백틱과 별표만(코드·경로·정규식 원문 예외).
③ 본문 날짜 스탬프(yymmdd) 금지 — 결정 경위 = `docs/작업이력.md` 원장. 파일명·브랜치명·백업 폴더명 등 식별자 속 날짜 = 유지.
④ 구 §앵커 = 조 제목·장의 별칭으로 존속(§디자인=제8장 · §수정 모드=제6장 · §검증=제36조 등 각 조 제목 병기). 구 이모지 앵커 = {🎨=디자인 ✅=검증 🥷=분신술 🧪=기틀검증 🔒=기틀 보호 📐=표기표준 🤖=모델 ✂️=수정 모드 🔔=선택질문 🖼=미디어첨부 📰=파이프라인 🩺=일일점검 🧭=라우팅 🧠=기록SSOT 🎯=작업표준 🚨=속보(제67조)}. 구 `N절`·`절대명령 N`·하위기호(`§작업표준 e-1`류) 참조 = 해설원장 매핑표로 해석. 구 실행 계약 1~7 = 제1~7조 동일 순서(번호 호환).

## 제3장 핵심명령 (§핵심명령 · 디자인 내용 축 최상위 불변)

**제12조(제1 핵심명령)**
① 웹앱 UI의 모든 디자인 값(색·간격·radius·blur·곡선·모션·그림자·크기·타이포)은 루트 디자인 토큰을 **(계승)하거나 (갱신)한다 — 둘 중 하나로만**. raw 값 임의 창작·제각각 재설계 금지.
② 디자인 기틀 단일 진입점 = 규칙2번 `docs/디자인기틀_SSOT.md` — 값·규칙·컴포넌트·확립본·거울·게이트의 정본 위치는 전부 규칙2 인덱스에서 찾는다. **디자인 작업 착수 전 규칙2부터 편다.** 새 디자인 기틀 파일 = 규칙2에 행 추가(누락 = 위반).
③ 루트 디자인 토큰 = 값 SSOT = `viewer/index.html`의 `:root`(기존 문서의 `값 SSOT`·`:root 토큰`과 동일물).
④ 컴포넌트(버튼·모달·아이콘) = 토큰이 아니라 `docs/CII_컴포넌트계승인덱스.md` 정본 셀렉터 계승 = 별개 축(제43조).

**제13조(계승·갱신 2택)**
① (계승) = 항상 디폴트(안 물음): `var(--토큰)`. 정확히 맞는 토큰이 없어도 **가장 가까운 토큰을 자동 계승**(이 판단도 안 물음 — 재설계·raw 창작만 금지).
② (갱신) = 운영자 승인 필수 · 2티어: 값 미세조정 = 승인 후 편입 / **토큰 신설·삭제·구조 변경 = `:root` 구조 변경 = 기틀** → 제34조 확인 + 제35조 교차검증이 승인의 일부.
③ 승인 후 편입 체인: `:root` 편입 → `build_design_mirror build`(거울) → CII 등재 → `check_refs` → `design-tokens.lock` 갱신.
④ 근접 토큰조차 없어 (갱신)이 불가피할 때만 묻는다: {신설할지 / 어떤 근접 토큰을 계승할지} 2선택지 제시 후 진행. 정확히 안 맞는다고 바로 묻지 마라(근접 계승 = 자동 = 운영자 스트레스 방지).

**제14조(정당 raw — 이 명령 대상 아님)**
1. 기하·수학값(중앙 `-50%`·원 `50%`·빈 게이지 `0`) · 키프레임 안무 % · 1회성 광학 보정 px · OS/서드파티 강제(`env()`·`-webkit-*`) · 컴포넌트 로컬 z-index(1~8·드롭다운 20~22) · 아직 토큰 없는 축(box-shadow 등)
2. 표면별 독립 팔레트: thumb/ly/k/comp 4뷰어는 색 공유 안 함 — index로 통일(재색칠) = **영구금지**(제42조②).
3. 콘텐츠 산출물 색 상수 = UI 팔레트와 별개 축(영구 비동행): `nomute_overlay/reels2` GREEN=(15,253,2) · `card_news` 형광그린 #0FFD02(`COLOR_HIGHLIGHT`)는 발행 콘텐츠 브랜드색 — UI 팔레트 개편에 절대 동행 스왑 금지. 변경 = 운영자가 *콘텐츠 색*을 콕 집어 지시할 때만.
4. 자기완결 템플릿(CSP로 `var()` 불가) = 값 **복사 = 계승**.
5. 이들은 사다리·`check_refs` baseline을 따른다(매 px·%마다 안 물음).

**제15조(오독 방지 — 위반이 아닌 것)**
① 금지 대상 = 팔레트 밖 새 색을 즉석 raw로 창작·반입하는 행위. 다음은 위반 아님:
  1. 닫힌 집합 안 의도된 유사색 스펙트럼 공존(편향 5단 파랑↔스카이시안↔흰↔코랄↔빨강 등) = 정상 — 근접색 위반으로 오판해 병합·삭제·축소 금지(다이버징/등급 스케일 = 하우스 표준).
  2. 같은 토큰의 alpha/투명도 변주(hover·press·틴트·그라데이션) = 신규 색 아님 = 자유.
  3. 닫힌 집합 ≠ 동결 — 진짜 새 색 슬롯 필요 = (갱신) 정문으로 넓힌다.

**제16조(강제 게이트)**
① 하드차단(rc=1) = raw 강조색{`accent_raw`(rgba(0,238,210)·`accent_hex`(#00EED2)} · 구 코어 그린 재유입{`legacy_green`(rgba(15,253,2·#0FFD02)} · 초록 워시(`green_wash`) · 거울 비동기 · `design-tokens.lock` 신토큰.
② design_gate diff 게이트: Edit/Write로 *추가된 줄*의 font-size·letter-spacing·radius·gap·padding·margin·blur() raw = 즉시 차단(exit 2) + 가장 가까운 토큰 자동 제안. 기존분 관용(diff 기반 = 상쇄 게이밍 면역). `/* raw-ok: 사유 */` 동일 줄 주석 = 광학 보정 유일 통로. 제외 = 순수 흑/백·`:root`·`@keyframes`·`<script>`(자기완결 템플릿)·env()·calc()·음수 margin.
③ 잔존 기존 raw·토큰 없는 축(그림자·모션) = 소프트룰. **`check_refs` 초록 ≠ 이 장 완전 준수.**
④ 이 계약은 SessionStart 디자인훅이 매 세션 기계 주입. 타 레포 복제 = `docs/디자인토큰_제1핵심명령_복제전문.md`(일반형 규칙+바인딩표 9슬롯).

## 제6장 수정 모드 (§수정 모드 · 운영 고정)

**제30조(원칙)**
① 단일 원본 = 이 GitHub 레포(`muteno/nomute-editor`). 모든 수정 여기서만. OneDrive·로컬·다운로드 사본 = 폐기(동기화 안 함).
② 트리거 = `git`(대소문자 무관)/`깃` 단어거나 그 말로 시작 → 앱 절대 시작 금지, 즉시 수정 모드로 시스템 파일 편집(뒤에 붙은 말 = 수정 지시). 기사·이슈·카드뉴스 일절 안 만듦.
③ 실행 = 폰/PC 어디서든 레포 열고 기사 던지기(로컬 `,` 트리거 폐지). 롤백 = `그 버전으로` 한마디 → `_versions/` 복원.

**제31조(절차 — 반드시 이 순서)**
① `git fetch origin main` → 최신 main 기준 시작(오래된 복제본 위 작업 금지).
② 백업: 고칠 파일 현재본 → `_versions/{yymmdd_HHmm}_{라벨}/`.
③ 수정. 기틀 변경이면 수정 전 사용자 확인(제34조).
④ `python3 shared/check_refs.py` 통과 확인(실패 = 고치고 재실행) 후 add·commit·push(작업 브랜치).
⑤ PR 생성 → **즉시 main 머지**(draft 방치 금지 = 이게 라이브 확정).
⑥ fetch 후 `git show origin/main:<파일>`로 원격 실내용 직접 읽어 검증·보고(해시+핵심 변경 줄 실측 · 화면·로컬만 보고 `됐다` 금지).
⑦ 충돌 처리: 머지 직전 main이 앞섰거나 squash로 갈렸으면 `git merge origin/main -X ours` 흡수(또는 rebase). 진짜 충돌 = STOP+보고. `force` 안 씀. 단 `-X ours` = **코드 충돌 한정** — `docs/작업이력.md` 등 append-only 문서는 다른 세션 추가분을 덮으므로 `-X` 없이 머지 후 충돌 마커 수동 정리(양쪽 보존·시각순).
⑧ 머지 직전 = 최신 origin/main 재동기화로 타 세션 충돌·역행 점검. 머지 후 = 완료 보고에 운영자 직접 확인 요청(라이브/결과 확인 포인트 1줄) 명시 — 자가검증 대체 아님·그 위의 마지막 단계.
⑨ 턴 종료 전 검사: 1. 내가 연 미머지 PR = 그 턴 안 머지 또는 보고 맨 위 `⚠ 미머지 PR #N — 사유 1줄`(조용히 종료 = 위반) 2. 4문 질문 항목만 대기(`⚠ 답 대기 N건` 최상단 표기)·독립 나머지는 계속 진행·머지.

**제32조(반영 보장 — 불변·최우선)**
① 변경 요청의 `완료` = main 반영 + 실측 검증까지. 완료 선언 = `git show origin/main:<파일>` 확인 후에만.
② 머지가 막히면(force 차단·non-ff·충돌) 우회로 끝까지 반영(흡수 머지 등 · append-only = 수동 보존).
③ 그래도 안 되면 `됐다` 절대 금지 → 즉시 STOP + 보고. 검증 안 된 완료 보고 = 거짓 — 이 원칙이 모든 편의·속도보다 우선.

**제33조(동시 세션 동기화)**
① main = 단일 진실원본 — 클라우드는 세션마다 새로 복제. 각 수정 = 그때그때 main 머지(브랜치·draft 방치 = 새 세션이 옛 룰을 봄).
② 시스템 파일 수정 = 가급적 한 번에 한 세션. 부득이 동시면 머지 직전 최신 main 재동기화 필수. 이미 열려 있던 세션 = 옛 복제본 → 새로 열거나 fetch 재동기화.

## 제7장 기틀 보호·검증

**제34조(기틀 보호 · §기틀 보호 — 구조 변경 = 반드시 묻는다)**
① 대상(기틀) = 앱↔라우터 분리 · 라이브러리↔지침 분리 · 3단계 파이프라인 · 라이브러리 참조 방식 · 출력 포맷 골격 · INVARIANTS · `:root` 구조 · 모델 예외 목록. 특히 라이브러리 재인라인·분리 해체·단계 합치기 = 스스로 결정 금지, 운영자 승인 후에만.
② 모델의 `효율화·정합성 개선` 명목 구조 손대기와 그 조용한 누락이 품질을 복리로 갉는다(모서리 깎임) — 이를 막는 게 이 프로젝트의 핵심.
③ 주변부(문구·임계값) = 자유. 단 큐레이션 임계 = 실질 변경(제9조5호) · 기존 라이브 값 추출(토큰화) = 픽셀 회귀 0 = 자유.

**제35조(기틀검증 · §기틀검증 — 기틀 변경 = Opus 5인 교차검증)**
① 대상 = 제34조① 동일 범위 + 라우터 구조 · 플랫폼 공용 불변(수정 모드·모델·기틀 보호·표기표준) · 환경 Setup script/env 변수 · 라이브러리 SSOT 구조·build 파이프라인 · 앱 지침의 출력 포맷 골격/INVARIANTS. (주변부 = 제외·자유.)
② 절차 = 제34조 사용자 확인 *먼저* → 확정 전 독립 Opus 5명 병렬(동일 후보 적대적 감사·레포 실측) → 수렴 확인 — **전원 APPROVE 아니면 STOP·재설계**(이견 = 다수결 맹종 말고 근거로 해소·기각 사유 기록) → 통과 후에만 제31조 절차 전체로 확정.
③ 한계(정직) = 소프트 룰. 교차검증은 ground-truth 영역(구조·스크립트·참조정합)에서 값함 — 순수 문구·톤은 효용 작음.

**제36조(검증 표준 · §검증)** — 검증 요청·수정 모드 자가검증 전부 적용.
① 검증가 전원 opus 4.8(`claude-opus-4-8`)·effort max. 약한 모델·저노력 금지.
② 인원 티어:
  1. 상시·기본 = 자가검증: 모든 수정 모드 작업은 머지 전 `check_refs`+원격 read-back(옛 5인 상시 폐지).
  2. 10인 평의회 = 진입 게이트 3조건 중 하나일 때만: {실측으로 안 좁혀지는 진짜 불확실 / 되돌리기 어려운 기틀·파이프라인 구조(제35조 동반) / 운영자 명시(`분신술`·`검증해줘`)}. 셋 다 아니면 평의회 금지 → 자가+CI로 끊는다(자명한 버그·회귀 = 건당 분~십분).
  3. **제사 금지**: 평의회는 결정을 가르는 도구지, 이미 완성한 결과물을 둘러앉아 추인하는 사후 의식이 아니다. 검증이 목적이면 자가+CI가 그 일을 한다.
  4. 국소 뷰어 손질(단일 `viewer/*` UI/UX·동작·구조 영향 작음·기틀 아님) = 자가로 끊는다.
  5. 10인 미만 감축 = 제24조로 명시 승인 후에만(무응답 = 10인 유지).
  6. 순수 시각/목업 = 평의회 완전 면제 → 자가 시각확인. 경계 = 로직·동작·상태·데이터 한 줄이라도 끼면 면제 아님(`:root`·핸들러·렌더 분기·파이프라인·전환 애니메이션[배너 크로스페이드] = 동작 = 국소여도 티어 재판정). 시각 제안 기본 = 바로 반영 후 라이브 확인(시안 남발 금지 — 폰 PNG = 한글 두부 실측). 시안 필요 시(대규모 구조·새 인터랙션·비가역) = 제28조 HTML(PNG 금지·한글 폰트 폴백 필수).
③ 어느 쪽이든 각기 다른 앵글 적대 감사(메커니즘·호환성·헤더/보안·모바일·메모리·에러처리·다운스트림·버전·통합·총괄). 추측 금지 = 레포·소스·라이브 실측 근거.
④ READ-ONLY 하드 계약: 평의회/분신술 에이전트 = 분석·보고만 — 레포 1바이트도 못 바꿈. 전면 금지 = git add/commit/push/branch/merge·PR 생성/머지·Edit/Write/NotebookEdit·파일 생성/삭제. 산출 = 텍스트만. 메인 루프가 레포 변경의 유일 행위자. 강제 3중:
  1. 소환 프롬프트 첫 줄: `READ-ONLY: 어떤 git/파일 변경·커밋·브랜치·PR·머지도 하지 마라. 발견은 텍스트로만 보고하라. 모든 변경은 메인이 한다`
  2. 읽기전용 에이전트타입 필수 — 평의회·검증·분신술·아이데이션 = 예외 없이 `subagent_type=Explore`(Edit/Write 미보유 = 기계 차단). `general-purpose`(전권) = 읽기·평결 에이전트엔 금지.
  3. 발사 후 `git status`/`branch`/`log`로 rogue 브랜치·커밋·PR 사후 점검. 위반 = 즉시 보고 + 산출 무효.
⑤ 실행 보장: 말만 하지 말고 반드시 실제 `Agent` 툴로 N명 병렬 소환(머릿속·서술 대체 금지).
⑥ 만장일치 루프: 평결 = OK/주의/불가 → 문제 전부 메인이 직접 수정 → 재검증(필요 시 재발사) → 전원 OK까지 루프 → read-back 실측 → 그제서야 `검증 완료`. 어느 단계든 미완 = `됐다` 금지.
⑦ 기틀이면 이 조 + 제34조 둘 다.
⑧ 한계 = 소프트 룰. 순수 문구·톤 = check_refs+read-back 자가검증이 낫다.

**제37조(분신술 · §분신술)**
① 운영자 호출어 = `분신술`. 입력에 있으면 opus 4.8·effort max·adaptive thinking 10명을 실제 Agent 툴로 병렬 소환(`subagent_type=Explore`·READ-ONLY 계약 프롬프트).
② 각자 다른 앵글+적대 시각으로 전 절차를 훑어 5종: 피드백 · 검증(실측) · 효율성 · 품질 · 아이데이션.
③ = 제36조 10인의 명명된 상위판. 절차·READ-ONLY·만장일치 루프·read-back 전부 상속. 변경·머지 = 오직 메인.
④ 묘사 말고 실제 소환(절대) + 발사 후 rogue 사후점검. 출력 = 앵글별 핵심 → 수렴 평결(OK/주의/불가) → 메인 조치 목록 → 아이데이션 베스트픽.
⑤ fable 평의회: 1. 중요 영역의 미적지근한(애매·판단 갈리는) 부분 = fable 교차검증 디폴트 2. fable = 고비용 → 시간 상한 보통 10분(도달 = 회수분으로 수렴·미회수 앵글 = 메인 실측 갈음) 3. 발사 전 운영자 고지(무엇을·몇 명·왜 1줄).

**제38조(라이브러리 SSOT 보호)**
① /k 단일 정본 = `apps/k/library/archive_media_master.tsv`(SSOT) → 유닛(01a~e·02~47)은 `shared/build_library.py` 자동 생성·투영. 편집 = SSOT에서만 · 유닛 직접 수정 금지(정합 = check_refs 강제).
② SSOT 기존 항목 수정(덮어쓰기)·삭제 = 사용자 이중 허락{1차 승인 → 2차 재확인 = 서로 다른 두 번} 필수. 새 ID 추가·순서 정리 = 무권한 OK. `정리·효율화` 명목 임의 수정·삭제 절대 금지(불응·모호 = STOP). 실무 절차 = `apps/k/00_지침_에디터_클링` §10.
③ 아카이브는 조용한 누락이 가장 치명 — 별도 잠금의 이유.

② 색 = 기존 색 유지가 절대 원칙:

  2. 새 색 창작(임의 raw·새 토큰) = 게이트가 기계 차단(진짜 새 슬롯 = (갱신) 정문 = 기틀 승인). 기존 색 제거·`무채색으로` 류 모호 지시 = 멈춰 묻지 말고 최보수 해석(형제·셸토큰 정렬)으로 진행 → 보고에 ⚠대체 명세 + `이렇게 해석했다 — 의도 맞나?`(제3조②).

**제41조(산문 최소 · 빈 상태)**
① 컨트롤 옆 사용법 설명 문구(힌트 줄·안내 캡션) = 기본 넣지 않음. 버튼 라벨·상태색(on=accent/off=mut)·픽토그램·비활성 dim이 어포던스로 설명. 표기 허용 = 단위(분·px)·파괴적 동작 경고·접근성 라벨(aria·SR 전용 = 삭제 대상 아님)만. 어포던스 부족 = 문구가 아니라 컴포넌트를 고친다. 신규 UI도 계승.
