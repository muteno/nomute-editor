## ★ 전체 방향 한 장 (도돌이표 방지 — 국소 패치 전 *반드시* 읽기 · 260617)
> 매 불만마다 entry/임계만 만지면 도돌이표. 아래 *방향*에 맞춰서만 고친다.

**목표**: RSS 다매체 수집 → 내용·키워드 *최소*로 **중요·양질**을 가려 누적/최신에 큐레이션. **보수적**(의심시 조임, §1).

**핵심 신호 5종 (서로 직조 — 곱으로 합쳐짐):**
1. **다매체(`cross`)** — 여러 곳이 보도 = 검증된 뉴스.
2. **연속보도(`report_count`)** — *후속 스크랩(15분 뒤)에 또 보도되면 +1* = **양질·중요의 핵심 신호 → 상단**. 끊기면 안 오름 → 하강. (보도자료=한 번 깔고 끊김 / 양질·사건=계속 보도). **뷰어 `scFollow`로 양 컬럼 점수에 곱 — 이미 구현.**
3. **동시성(`burst`)** — 다수 매체 동시 = **속보** 신호.
4. **AI 경중(`grade` 0~3)** — 구조가 못 가르는 **보도자료/잡음(0)·경미(1)를 침몰**, 대형(3) 부상. 품질 필터(`gradeW`).
5. **`[속보]`·`[단독]` 태그** — 속보/특종 확률↑ → **AI 내용 확인**(1~2매체여도). *([단독]은 게이트 surfaced 포함 · [속보]/[상보]/긴급 태그 = breaking 후보로 AI 검증 — 구현됨 260617.)*

**두 컬럼:**
- **누적(좌·옛 '누적') = 9시뉴스감** — **진입 = 발생 4h+ AND (cross≥8[8개+ 매체] OR 긴급[breaking])** (260617: 5→8 엄선 · ⚠️ **긴급이었던 건 cross<8이어도 4h 후 누적 합류** = 최신서 졸업하면 사라지는 구멍 차단, 일반 누적 점수로 랭킹). 넓게 다뤄진 검증 누적만 쌓음(최신서 4h 지나 내려온 것 포함). **점수=cross 볼록가점 × 발행기준 시간 가속감점 × 연속보도 × AI경중**(§5·260617 재설계). 오래될수록 *가속* 하강, 단 다매체+계속보도면 상단 유지.
- **최신(우·옛 화면 '최신') = 신선(<4h)** + (**속보** OR **2+매체 양질**). 잡음(grade 0/1)만 제외 — 미채점·주목·대형·속보 포함. **연속보도·품질일수록 상단**(scFollow·gradeW). **4h+ 면 누적行.** ⚠️ 경계=`FAST_MAX_H`(4) **단일상수** — 최신진입·누적진입·속보핀·토스트·배지(🚨속보↔⚡누적) 전부 이거 하나서 결정(드리프트 차단). **∴ 최신엔 ⚡누적 안 뜸**(다 <4h=🚨속보) / ⚡누적는 4h~24h 전용(1일↑ 배지없음). (화면 '3시간 전'까지 최신 · '4시간 전'부터 누적.)

**누적 vs 긴급(속보) — 두 축 (개념 · 260617 재정립 = 합본 C)**: ⚠️ **누적 ≠ 긴급. 서로 다른 축이다.** **누적 = *자리*(시간 칼럼)** = 넓은 누적가 쌓이는 ≥4h 칼럼(옛 '누적'). **긴급=속보 = *사건 속성*(배지)** = `breaking` 확정 사건. **긴급 사건은 나이 들면 *자리만* 최신(<4h)→누적(≥4h)로 이동 — 개념은 끝까지 긴급**(예전 '속보→누적 변신' 서술은 *위치 이동*을 개념변화로 오기록한 것). 배지: **🚨속보**(빨강 = breaking AND <4h, 신선) / **⚡누적**(노랑 = breaking 4h~24h aged, 누적칼럼 거주 · **1일↑은 배지없음**). ⚠️ "누적" = 칼럼 이름 + 4h~24h aged-breaking 배지 라벨 둘 다(운영자 260617 — 맥락 구분). ⚠️ **긴급(속보) = breaking AND 경중 grade≥2(또는 미채점 null)** — 경중 게이트가 grade 0/1(잡음·경미)로 깐 건 breaking이어도 긴급 아님(배지·카드글로우·핀·최신 전부 박탈 → 제 점수로 침몰). 단일 헬퍼 `isBreaking`로 통일. *(breaking_judge='급발?' ↔ gate='경중?' 2층 불일치면 경중이 긴급 라벨 거부.)*
**속보 판정**: ① 다수 동시(burst↑) = 속보 / ② 1~2매체 `[속보]` 태그 → AI 내용확인 → 속보. (단일매체 중대사건도 AI 분석으로 속보 가능.)
**랭킹(260617 재설계 — 두 가속 곡선)**: **누적 = `cross^1.6`(볼록 가점·다매체 가속) × `시간 가속감점`(발행기준 Hill·~12h 후 가속 침몰) × `scFollow`(연속보도) × `gradeW`(AI품질).** 최신 = `(burst+breaking) × 신선decay × scFollow × gradeW` (+속보 핀). → *다수동시+최신+계속보도+양질이 자동 상단 / 오래되면 가속 하강.* (옛 누적의 활동시각 decay는 last_report가 균일해 시간축이 죽던 것 실측·폐기 → §5·§8.)

## 2.5 ★핵심 아키텍처 (260616) — 일반 알고리즘(코드) + 특이케이스(AI 검증)
순수 코드로 다 풀면 키워드·패턴 지옥. **AI를 게이트에 끼우면 훨씬 단순** — 구조 신호가 *후보만* 추리고, 애매한 결정은 Claude가 내용 읽어 *사람 대신* 경중 판정.

**① 일반 알고리즘 (코드·무료·항상 작동):** 구조 신호로 누적/최신 분류·정렬.
다수매체(cross) → **연속 보도 가점** → 최신순 최상단 → 임계 넘으면 알람 후보.

**② 특이 케이스 핸들러 (AI·타겟 과금·따로 만듦):** 코드가 못 가르는 것만 Claude가 내용 1콜로 경중 판정.
- **케이스 A — 빠른 단독 특종**: `[단독]` 헤더 + 단일 매체 1건만 수집 → cross 낮아 일반 알고리즘선 묻힘 → **Claude가 깐다**(언론고시 본 기자라 `[단독]` 헛소리 안 씀 = 값어치). 디스패치류 연예특종은 출처로 라우팅 가능.
- **케이스 B — 다수 매체 동시 분사**: 한 시점에 다수가 쫙 보도 → 보도자료(홍보)냐 누가 봐도 엄중한 사안이냐 구조론 모름 → **Claude가 내용 읽어 판정 → 엄중하면 알람.**

**이미 절반 있음**: `breaking_judge.py`(burst≥3 → Claude가 사고/재난 판정) = 정확히 이 패턴. **확장만** 하면 됨(처음부터 안 짬).

✅ **구현됨(260616)** = `.github/scripts/gate_judge.py`: 노출 후보(cross≥3 or 속보후보 or `[단독]`)를 Claude가 **0~3 경중 채점**(0 비뉴스/보도자료 · 1 경미 · 2 주목 · 3 대형). RUBRIC 해시 도장(`grade_rubric` = 드리프트 차단)·청크 120·self-gate(`--count`)·`breaking-judge.yml` 동거(구독 OAuth 재사용). 뷰어 `gradeW`(0→×0.05 침몰 · 3→×1.8 부상 · 미채점→×1 중립)를 최신·누적 점수에 곱 + 알람 grade≥2 게이트. 구조 신호(AUC 0.5~0.59) 한계를 AI 내용판정이 보완.

## 3. 신호 (전부 내용·키워드 0 · 현 데이터)
- `cross` = 보도 매체 수
- `burst` = 15분 내 동시 보도 매체 최대 (동시성)
- `first_seen` / `last_seen` = 처음 본 시각 / **마지막으로 cross가 자란** 시각
- **성장 꼬리 = `last_seen − first_seen`** = 얼마나 오래 새 매체가 계속 붙었나 (= 연속 보도)
- `seen_count` = 수집 사이클마다 등장 횟수. ⚠️ **현재 결함**: 정체 등장(보도자료가 피드에 그냥 남아있는 것)도 +1 → 보도자료 오가점. **개선 = `grow_count`(cross가 *실제로 자란* 사이클 수)로 바꿔야 연속 보도를 정확히 잼.**

## 4. 최신 (속보 캐처) — 규칙 (260616 운영자 확정)
- **피드 진입**: `cross≥2` (2개 이상 매체가 같은 내용 보도).
- **속보 알람**: `cross≥4` (스크랩 시 4개 이상 매체). **단, 순수 cross 알람은 89% 오발령(§7.5) → AI 게이트(§2.5 케이스 B) 통과분만 실제 알람.**
- **하강(피드서 내림)**: 후속 보도 있나? → **후속 있으면 가점=유지, 없으면 계속 하강**(한 번 깔고 죽는 보도자료 = 즉시 침몰).
- 🔴 **현재 한계(실험 §7.5)**: "후속"을 `last_seen−first_seen`(꼬리)으로 재면 **틀림** — 큰 사건은 우리가 보기 전 이미 만렙이라 꼬리 0 → "후속 없음" 오판(미·이란 종전·추락사망이 하강 대상으로 잘못 찍힘). **꼬리 = '사건 지속'이 아니라 '우리가 일찍 잡았나'를 잼.** 진짜 후속 = **매 스크랩 cross 시계열 기록**해야 정확.

## 5. 누적 (9시뉴스 · 옛 '누적') — 규칙 (260617 운영자 재설계 · 두 가속 곡선)
- **점수 = `cross^CROSS_POW`(볼록·다매체 가속) × `timeAcc`(발행기준 Hill 가속감점) × `scFollow`(연속보도) × `gradeW`(AI경중).** 정본 코드 = `viewer/index.html`(`crossConvex·timeAcc·scScore`).
- **① cross 볼록**(`CROSS_POW=1.6`): 매체수 늘수록 단계 가점 *가속* — 5→7→9→11 = Δ9.8·11.5·13.1(5^1.6≈13·11^1.6≈48). 다수 동시보도 = 상단. (잡음·보도자료는 `gradeW`가 veto해 안전.)
- **② 시간 가속감점**(`timeAcc = 1/(1+(발행age/ACC_T_HALF)^ACC_T_POW)`, `18·2.5`): 신선엔 ~1, ~12h 넘으면 *가속* 침몰(8→10h Δ.071 < 10→12h Δ.079). 발행(`scTs`=화면 'N시간 전') 기준. 값: 6h=.94·12h=.73·18h=.50·24h=.33·48h=.08.
- ⚠️ **옛 '활동시각(`last_report`) decay' 폐기(실측 교체)**: 활성 사이클선 거의 모든 클러스터가 매 스크랩 새 기사 붙어 `last_report`가 0~2h로 *균일* → 시간축이 죽어 발행 20·40h가 랭킹에 무영향("뒤죽박죽")이었음(실데이터 확인). 발행 기준으로 복원하고, '계속 보도되는 가치'는 `scFollow`(report_count)가 *따로* 가점 — 역할 분리.
- 화두 큰 것 = 많은 매체 동시 + 최근 + 계속 보도 + 양질. 보도자료(grade0·연속0)는 바닥.
- ⚠️ **breaking 강제핀 없음(260617)**: 누적 칼럼은 *순수 scScore 순* — breaking이어도 상단 강제고정 안 함. 옛 `pinB`가 오래된 긴급을 신선 고득점 위에 박아 시간 가속감점을 무력화하던 것 차단(실측: 밀양 28h·score9.5가 신선 score71 위 점유). ⚡누적 배지는 유지(찾기 가능·4h~24h). 신선 속보 핀(`pinFresh`)은 최신 칼럼에만.
