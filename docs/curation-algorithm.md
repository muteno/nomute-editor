2. **연속보도(`report_count`)** — *후속 스크랩(15분 뒤)에 또 보도되면 +1* = **양질·중요의 핵심 신호 → 상단**. 끊기면 안 오름 → 하강. (보도자료=한 번 깔고 끊김 / 양질·사건=계속 보도). **뷰어 `scFollow`로 양 컬럼 점수에 곱 — 이미 구현.**

4. **AI 경중(`grade` 0~3)** — ⚠️ **재정의(260628·§8): grade = *긴급탐지* 도구지 *경중* 도구 아님. 경중·랭킹은 발행량(1·2·3 신호)이 정함.** grade는 긴급축(isBreaking g≥2·isAlert g≥3)만 + 랭킹엔 *약한 잡음감점*(`gradeW` floor 0.5 = grade0/1 ×0.5/0.7·대형3 ×1.8)만. 옛 ×0.05 곱셈지배(문화 grade2·3=0건·대세 1/20 침몰)는 폐기. 정형잡음은 `JUNK_HEAD`가 별도 컷.

**두 컬럼:**

**① 일반 알고리즘 (코드·무료·항상 작동):** 구조 신호로 누적/신규 분류·정렬.

**② 특이 케이스 핸들러 (AI·타겟 과금·따로 만듦):** 코드가 못 가르는 것만 Claude가 내용 1콜로 경중 판정.

## 3. 신호 (전부 내용·키워드 0 · 현 데이터)
- `cross` = 보도 매체 수
- `burst` = 15분 내 동시 보도 매체 최대 (동시성)
- `first_seen` / `last_seen` = 처음 본 시각 / **마지막으로 cross가 자란** 시각
- **성장 꼬리 = `last_seen − first_seen`** = 얼마나 오래 새 매체가 계속 붙었나 (= 연속 보도). ⚠️ **폐기(§7.5)**: 관측 타이밍 의존이라 큰 사건은 꼬리 0으로 오판 → 신호서 제외.
- `seen_count` = 수집 사이클마다 등장 횟수. ⚠️ **폐기·대체 완료(§7·§8)**: `report_count`(arts가 실제 자란 사이클 수)로 교체 — 정체 등장(보도자료 잔존)도 +1하던 결함 해소.

## 4. 신규 (속보 캐처) — 규칙 (260616 운영자 확정)

## 5. 누적 (9시뉴스 · 옛 '누적') — 규칙 (260617 운영자 재설계 · 두 가속 곡선)
