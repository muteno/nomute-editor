## 1. 대원칙 — APK 서명 패밀리 통일 ⚠️ (이번 구축 최대 함정)

Termux 패밀리(본체·:API·:Widget·:Tasker)는 **전부 같은 서명의 빌드**여야 한다. 본체가 GitHub 릴리즈 debug 빌드면 부속 앱도 전부 GitHub 릴리즈 `+github.debug.apk`. **F-Droid판과 절대 혼용 금지** — 서명 키가 달라 "앱이 설치되지 않음"으로 거부된다(에러 메시지가 원인을 안 알려줘서 한참 헤맴).

- 이번 구축 기준: 본체 = `termux-app_v0.11..._arm64-v8a.apk` (GitHub debug)
- 새 폰에서는 **본체를 뭘로 깔지부터 정하고**, 나머지를 같은 출처로 따라가면 된다. F-Droid로 통일해도 되지만, 그 경우 4개 전부 F-Droid.
- (선택) Play 프로텍트의 "보안 위험 앱 차단"이 설치를 막으면 일시 해제.

### 2-1. 부속 APK 다운로드 (Termux 안에서)
```bash

cd ~/storage/downloads   # 먼저 termux-setup-storage 1회 실행 필요(§3)
```

- **`termux-open`은 무반응 전적 있음** → 믿지 말고 **내 파일 앱 → 다운로드 → APK 탭 → 설치**가 정답 루트.

### 3-1. termux.properties — 외부 앱 허용 ⚠️ (주석 함정)
Tasker가 Termux 명령을 실행하려면 이 스위치가 **실제로 켜져** 있어야 한다. 기본 파일에는 `# allow-external-apps = true`처럼 **주석 처리된 줄만** 있어서 켜진 걸로 착각하기 쉬움 — 이게 "플러그인 오류: 누락, 비활성화, 내보내지지 않음 또는 권한 수신기 없음" 에러의 직접 원인이었다.

```bash
grep allow-external ~/.termux/termux.properties 2>/dev/null
# '#' 없는 줄이 안 보이면:
echo "allow-external-apps = true" >> ~/.termux/termux.properties
termux-reload-settings
grep allow-external ~/.termux/termux.properties   # 주석 없는 줄 확인
```
(참고: `file-share-receiver=termux-file-editor` 줄도 추가해 봤으나 **효과 미검증** — 없어도 파이프라인 동작에 지장 없음.)

## 4. GitHub PAT (폰 → 리포 push 인증)

1. github.com → Settings → Developer settings → **Fine-grained tokens** → Generate
2. Repository access: **Only select repositories → nomute-editor**
3. Permissions: **Contents → Read and write** (그 외 전부 No access)
4. Expiration: 90일 → 발급 토큰을 **Proton Pass에 저장**
5. 폰에서 첫 push 때 Username = `muteno`, Password = **PAT** 입력 → credential.helper store가 영구 기억
6. **갱신 캘린더 등록**: 발급일 + ~85일 (이번 사이클: 2026-09 초)

검증:
```bash
cd ~/nomute-editor
git commit --allow-empty -m "phone auth test" && git push
# "main -> main" 출력이면 인증 완료. 테스트 커밋은 남아도 무해.
```

## 5. 스크립트 설치 (전문 — 복붙)

### 5-1. 미디어 다운로더 보존/개명 (기존 다운로더가 있는 폰만)
기존에 `~/bin/termux-url-opener`가 영상 다운로더(gallery-dl 기반)였다면:
```bash
cp ~/bin/termux-url-opener ~/bin/url-opener.bak     # 백업
mv ~/bin/termux-url-opener ~/bin/media-handler      # 다운로더는 media-handler로 개명
```
새 폰이라 다운로더가 없으면: media-handler 자리는 비워도 됨 — 단 §5-2의 미디어 분기 case 블록을 지우거나, 더미를 만들어 둔다(`echo '#!/data/data/com.termux/files/usr/bin/bash' > ~/bin/media-handler; chmod +x ~/bin/media-handler`).

> **260719 알림 래퍼**: `~/bin/media-handler` = **알림 래퍼**(실물 정본 = 레포 `docs/media-notify.sh`)가 됐고, 원본 다운로더는 `~/bin/media-handler.real`로 개명 보존. 래퍼 = 다운로드 전후 갤러리 폴더(`~/storage/dcim/nomute_image`) diff → 새 파일 있으면 "**다운로드 완료** · [게시자] 파일명 외 N건"(v5 260719 — 게시자 = 표시명(닉네임) 확정 · `yt-dlp --print uploader` 조회 15s · **이모지·기호 자동 제거**(유니코드 정제 · 통째 이모지 닉 = 시간 폴백) · 조회 실패/이미지 전용 글 = [hh:mm AM/PM] 폴백 · `/i/status` 링크라 URL 추출 불가 = 조회가 정공법) / 새 파일 0이면 "**다운로드 실패** · [hh:mm AM/PM] 링크"(시각 = KST 강제) + stdin 폐쇄로 원본의 'Enter로 종료' 대기 소멸(Tasker 백그라운드 경로 안전). 알림 문구 = 스크립트 상단 '문구 칸' 변수(타이틀 -t·내용 -c 자유 조정). 설치/재설치 = media-notify.sh 머리의 curl 한 줄(원본 보존 가드 포함 · 멱등).

### 5-2. 핵심 처리기 queue-handler
```bash
mkdir -p ~/bin
cat > ~/bin/queue-handler << 'EOF'
#!/data/data/com.termux/files/usr/bin/bash
# 공유 텍스트는 공백으로 인자 분할될 수 있어 "$*"로 전체 재결합(§5-5 보강 흡수)
INPUT="$*"

# 입력이 존재하는 파일이면(텍스트 공유 경로) 내용물로 치환
[ -f "$INPUT" ] && INPUT="$(cat "$INPUT")"

# 텍스트 안에서 첫 번째 http(s) URL 추출 + 한글 본문량(전문 붙여넣기 판정용)
URL=$(echo "$INPUT" | grep -oE 'https?://[^ "'"'"'<>]+' | head -1)
HANGUL=$(printf '%s' "$INPUT" | LC_ALL=C grep -oE $'[\xea-\xed]' | wc -l)   # 한글 UTF-8 lead 바이트≈글자수(로케일 무관)

notify(){ termux-notification -t "$1" -c "$2" 2>/dev/null || true; }
logline(){ echo "$(date '+%y-%m-%d %H:%M:%S') | $1 | ${LINE1:-$URL}" >> ~/nomute-queue.log; }

cd ~/nomute-editor || { notify "큐 실패" "리포 폴더 없음"; logline "NO_REPO"; exit 1; }
source shared/url_guard.sh 2>/dev/null || is_article_url(){ return 0; }   # 포털 루트 가드 SSOT(없으면 통과)

# ⚠️ 증거보존(260620 분신술 ⓐ①): 공유분은 *먼저* 레포 밖 ~/nomute-pending-failed/ 에 저장한 뒤 push.
#    push 5회 실패해도 백업은 git reset --hard 에 증발 안 함 → 다음 공유 때 자동 재시도로 합류(silent-drop 차단).
FAILDIR="$HOME/nomute-pending-failed"
mkdir -p "$FAILDIR"
push_file(){   # $1=백업파일 · $2=최대시도(기본5) → 0=성공 / 1=네트워크·push / 2=commit.
  local src="$1" maxtry="${2:-5}" name; name="$(basename "$src")"; local try
  for try in $(seq 1 "$maxtry"); do
    git fetch -q origin main || return 1
    git reset -q --hard origin/main          # 레포 안만 영향 — FAILDIR 백업은 무사
    mkdir -p pending; cp "$src" "pending/$name"; git add pending
    git diff --cached --quiet && return 0     # 이미 main에 동일내용(부분성공) = 반영됨 → 성공처리(rc2 무한 DEFERRED 차단)
    git -c user.name=muteno-phone -c user.email=phone@nomute commit -qm "queue: $(head -n1 "$src")" 2>/dev/null || return 2
    if git push -q origin HEAD:main; then return 0; fi
    [ "$try" = "$maxtry" ] || sleep $((try * 2))   # 마지막 시도 뒤엔 sleep 생략(off-by-one)
  done
  return 1
}

if [ "$HANGUL" -ge 200 ]; then
  # ── 전문 붙여넣기 경로 (운영자가 기사 페이지 '전체선택→공유') ──
  # 막힌 매체(403)·JS렌더·페이월 *전부* 우회 — 본문이 이미 손에 있으니 fetch 안 함.
  # line1 = paste:<해시>(원문 URL 없음 → dedup·식별용 합성 id). 안의 트위터 등 링크는 무시(media 라우팅 안 함).
  LINE1="paste:$(printf '%s' "$INPUT" | sha1sum | cut -c1-12)"
  BODY="$(printf '%s' "$INPUT" | head -c 20000)"   # iconv 불필요 — 분석기가 정리(전문경로 = libiconv 의존 0)
else
  # ── URL 경로 ──
  [ -z "$URL" ] && { notify "큐 실패" "URL/전문 못 찾음"; logline "NO_URL"; exit 1; }
  # 포털/도메인 루트 가드 — 기사 경로 없는 'https://m.daum.net/' 류(공유 중 경로 잘림)는 큐에 넣어봤자
  #   분석기가 failed 격리 → 여기서 막고 즉시 알림(재공유·전문붙여넣기 유도). 전문 붙여넣기 경로는 무관.
  is_article_url "$URL" || { notify "큐 거부 ⚠️" "기사 주소가 아니라 포털 홈($URL) 같아요 — 기사 페이지에서 다시 공유하거나 전체선택→전문 공유"; logline "ROOT_URL"; exit 1; }
  LINE1="$URL"
  # 미디어 사이트 → 기존 다운로더
  case "$URL" in
    *x.com*|*twitter.com*|*instagram.com*|*youtube.com*|*youtu.be*|*tiktok.com*)
      exec ~/bin/media-handler "$URL" ;;
  esac
  # 폰 선-fetch (근본 우회 · 260618) — 클라우드 러너는 조선·동아·연합·중앙 등에 IP기반 403 이지만
  #   폰(가정용 KR IP)은 200 이다. 본문을 폰에서 미리 긁어 pending '# body:' 로 동봉하면 분석기가
  #   클라우드 fetch 없이 그대로 쓴다(403 우회). repo 의 fetch_article.sh 재사용. timeout 20 + || true.
  #   ⚠️ URL 경로 전제: §3 `python libiconv`(없으면 선-fetch 빈값 → 클라우드 폴백). 전문경로는 무관.
  git fetch -q origin main && git reset -q --hard origin/main   # repo 최신화(fetch_article.sh 최신)
  BODY="$(timeout 20 bash .github/scripts/fetch_article.sh "$URL" 2>/dev/null || true)"
fi
# 새 공유분을 *먼저* 레포 밖 백업으로 저장(절대 안 잃음) → 백업디렉터리 전체를 오래된 순으로 push.
# main 은 자동 파이프라인(분 단위 커밋)으로 분주 → push non-ff 경쟁이 실패 1순위. push_file 이 매 시도
# 재fetch·재reset·재push(2·4·6·8s 백오프, 5회)로 흡수. 실패분은 FAILDIR 에 남아 다음 공유 때 자동 재시도.
NEWF="$FAILDIR/$(date +%y%m%d-%H%M%S)-$RANDOM.txt"
if [ -n "${BODY//[$' \t\r\n']/}" ]; then
  printf '%s\n# body:\n%s\n' "$LINE1" "$BODY" > "$NEWF"   # 본문(선-fetch/붙여넣기) '# body:' 동봉
else
  echo "$LINE1" > "$NEWF"                                  # 본문 없음(JS렌더·페이월) → line1만(클라우드 폴백)
fi
shopt -s nullglob
new_ok=0; left=0
for src in "$FAILDIR"/*.txt; do            # glob = 파일명 사전순(YYMMDD 접두) = 오래된 순(FIFO)
  if [ "$src" = "$NEWF" ]; then push_file "$src" 5; else push_file "$src" 1; fi   # 방금 것=풀재시도 / 백로그=1회(전경 행 방지)
  rc=$?
  if [ "$rc" = 0 ]; then rm -f "$src"; [ "$src" = "$NEWF" ] && new_ok=1
  else left=$((left + 1)); fi
done
shopt -u nullglob

if [ "$new_ok" = 1 ] && [ "$left" = 0 ]; then
  notify "큐 등록됨 ✅" "$LINE1"; logline "OK"
elif [ "$new_ok" = 1 ]; then
  notify "큐 등록됨 ✅ (이전 ${left}건 보류)" "$LINE1"; logline "OK_LEFT:$left"
else
  notify "큐 보류 ⏳ ${left}건" "네트워크 실패 — 보존됨, 다음 공유 때 자동 재시도(데이터 안 잃음)"; logline "DEFERRED:$left"
fi
EOF
chmod +x ~/bin/queue-handler
```

> 🔁 **기존 폰도 재배포 필요 (260620 분신술 ⓐ① · 중요):** 증거보존(`~/nomute-pending-failed/` 선저장+드레인)은 **이 §5 핸들러 블록을 폰에 다시 붙여넣어야** 적용된다. *이미 쓰던 폰*은 옛 `~/bin/queue-handler`(push 5회 실패 시 미푸시 로컬커밋이 다음 `git reset --hard`에 증발하는 silent-drop) 그대로이므로, 위 `cat > ~/bin/queue-handler … EOF` 한 블록을 Termux에 재실행해 **갈아끼운다**(레포만 머지하면 폰엔 반영 안 됨). 폰 교체뿐 아니라 *이 갱신 1회*도 재배포 대상.
>
> 🛡 **포털 루트 가드 추가 (260623):** 핸들러에 `source shared/url_guard.sh` + URL 진입 전 `is_article_url` 가드를 넣었다 — `https://m.daum.net/`처럼 기사 경로 없는 포털 홈이 공유되면 큐에 안 넣고 즉시 "큐 거부 ⚠️" 알림(재공유 유도). **폰 단은 위 블록 재배포해야 적용**되지만, 분석기(`.github/scripts/analyze.sh`)에도 같은 가드가 있어 **레포 머지만으로 루트 URL 둔갑 카드는 원천 차단**된다(폰 미재배포여도 안전망 작동).

> ⚠️ **'공유했는데 안 떨어짐' 진단 (260618 갱신):** ① **새 1순위 = push 경쟁(non-ff).** main 이 자동 파이프라인으로 분 단위 커밋돼(예: 하루 50+커밋) fetch~push 사이에 또 앞서가면 단발 push 가 거부된다 → ❌ 빈번·성공 드묾. 위 핸들러의 **재시도 루프(매 시도 재fetch·재reset·재push, 5회 백오프)** 가 이를 흡수한다. ② **구 1순위 = 폰 로컬 갈림**(옛 `git pull -q` 가 조용히 실패) — `fetch + reset --hard` 자가치유로 이미 차단됨. 알람을 꺼뒀어도 `tail ~/nomute-queue.log` 한 줄이면 매 시도의 OK/OK_LEFT/DEFERRED 가 보인다. ③ **증거보존(260620 분신술 ⓐ①)**: 공유분은 *먼저* `~/nomute-pending-failed/` 에 저장된 뒤 push 된다 → **5회 실패해도 데이터는 안 사라지고**(`DEFERRED`) 다음 공유 때 자동 재시도(`OK_LEFT:N` = 방금 건 성공·이전 N건 아직 보류). 옛 버전의 *미푸시 로컬커밋이 다음 reset 에 증발하던 silent-drop* 은 이걸로 차단. 백업이 안 빠지면(계속 DEFERRED) `ls ~/nomute-pending-failed/` 로 잔존 확인 후 네트워크/PAT 점검.
>
> ⚠️ **네이트(`news.nate.com`) 등 아그리게이터 링크는 공유해도 카드가 안 나온다** — 분석 단계 WebFetch 가 본문을 인코딩 깨짐으로 못 읽어 모델이 날조를 거부하고 `pending/failed/` 로 격리한다. **언론사 원문 직링크**(예: 매일신문·연합뉴스 기사 URL)를 공유하면 정상 처리된다.

### 5-3. Termux 입구 래퍼 2종 (URL 직공유 / 파일 수신)


### 5-4. 클립보드 큐잉 — 명령어 q + 홈 위젯
```bash
cat > ~/bin/q << 'EOF'
#!/data/data/com.termux/files/usr/bin/bash
~/bin/queue-handler "$(termux-clipboard-get)"
EOF
chmod +x ~/bin/q

cp ~/bin/q ~/.shortcuts/큐등록

```
홈 화면 길게 누름 → 위젯 → **Termux:Widget** 배치 → 목록에서 `큐등록`.

### 5-5. Tasker 연동 스크립트 (⚠️ 반드시 `"$*"`)
Termux:Tasker는 `~/.termux/tasker/` 안의 스크립트만 실행 가능. 공유 텍스트가 공백으로 인자 분할될 수 있어 `"$1"`이 아니라 **`"$*"`**(전체 재결합)를 쓴다 — 이게 1차 구축 때의 보강 포인트.
```bash
mkdir -p ~/.termux/tasker
cat > ~/.termux/tasker/queue-from-tasker << 'EOF'
#!/data/data/com.termux/files/usr/bin/bash
~/bin/queue-handler "$*"
EOF
chmod +x ~/.termux/tasker/queue-from-tasker
```

## 6. Tasker 세팅 (텍스트 공유 입구)

### 6-1. Task 만들기
1. Tasker → **작업(Tasks)** 탭 → ➕ → 이름 `자동화` (이름 자유)
2. ➕ → **플러그인 → Termux:Tasker** → 구성(연필)
   - **Executable**: `queue-from-tasker`
   - **Arguments**: `"%rs_text"` ← **따옴표 포함해서** 입력 (공백 분할 방지)
   - **Wait for result**: 해제 (백그라운드 push라 기다릴 필요 없음 · 타임아웃 10초 기본 무방)
3. 처음 실행 시 "플러그인 오류 …권한 수신기 없음" 에러가 뜨면 → §3-1 allow-external-apps 확인 + 안드로이드 설정 → Tasker 앱 권한에서 Termux 명령 실행 허용.

### 6-2. 프로필 만들기 — 공유시트 등록의 핵심
Tasker 6.6.20에는 구버전의 "Show in Share Menu" Task 토글이 **없다**(속성·⋮·길게누름 어디에도 — 헛수고 전적 있음). 대신 **이벤트 프로필**로 등록한다:

1. **프로필** 탭 → ➕ → **이벤트**
2. 카테고리에서 **앱** → **Received Share** 선택
   - ⚠️ 검색창에 "공유"라고 쳐도 **안 나온다** — 이벤트명이 영어 그대로 + 카테고리가 '앱'이라서. 카테고리로 직접 들어갈 것.
3. 설정 화면(필터들)은 전부 기본값/빈칸 → 뒤로
4. 연결할 작업 목록에서 **자동화** 선택
5. 홈으로 나가며 적용 · 프로필 토글 **ON** 확인
- 공유 텍스트는 `%rs_text` 변수로 들어옴 (이벤트 변수 패밀리 = `%rs_*`: rs_text·rs_subject·rs_mime_type·rs_package_name·rs_all_extras 등 — 설정 화면 '출력 변수' 목록에서 확인 가능)
- (팁) Tasker 설정의 **Direct Share Targets**를 켜면 공유시트에 트리거가 바로 떠서 탭 한 번 절약.
- (정리) Tasker의 상시 알림 "Useless Notification"은 탭해서 꺼도 기능 무관.

## 10. 버전 고정에 대한 메모

본 문서의 버전·URL은 2026-06 구축 시점 기준. 운영자 판단으로는 이 도구군(Termux/Tasker)은 AI 패러다임 전환기라 큰 버전 변동 없이 유지될 가능성이 높다고 보지만, 재구축 시점에 릴리즈 페이지에서 최신 `+github.debug` 파일명만 확인해 URL의 버전 숫자를 바꾸면 나머지 절차는 동일하다. **절차(서명 통일 → 본체 → 부속 → 스크립트 → Tasker 프로필)가 본체고, 버전 숫자는 소모품.**
