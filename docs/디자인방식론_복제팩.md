# 디자인 방식론 복제팩 — 타 레포 전달 (값 축과 분리)

> **용도**: 디자인 시스템이 있는 레포에 디자인 *방식론*(레포 달라도 통하는 계승·통일·픽토그램 규율)을 심는다. 마스터 라우터 복제팩(`기틀_복제팩_마스터라우터.md`)과 **나란히 동봉**한다.
>
> **3축 분리**:
> - **값 축**(토큰 hex·팔레트·컴포넌트 인벤토리 실물) = `docs/디자인토큰_제1핵심명령_복제전문.md`(별도) — 반입 금지 대상.
> - **방식론 축**(값 아닌 계승·통일 규율) = **이 문서**.
> - **공통 UI 로직**(어포던스·flicker·자동완성·모달 뒤로가기) = **이 문서 §9**(디자인 시스템 없어도 적용 · 구 마스터 라우터 v1 템플릿에서 이관).
>
> **경계 판정 한 줄**: 그 문장에서 특정 hex·토큰명·정본 셀렉터·컴포넌트 실물을 지워도 규칙이 온전히 성립하면 = **방식론(이 문서)** · 지우면 껍데기만 남으면(그 값이 규칙의 실체) = **값 축(복제전문)**.
>
> **이식**: 새 레포에 이 파일을 넣고 세션에 "이 레포 CLAUDE.md(또는 디자인 지침)에 이식해줘 — 값·토큰명·셀렉터는 이 레포 실측 사례로만 쓰고 원칙은 일반형으로" 지시. 값 축이 필요하면 복제전문도 같이 전달.

## 0. 문서 표기
- a) 넘버링 4단(`1.`→`1)`→`a)`→`가)`) · 괄호 `{ [ ( ) ] }` · 따옴표·강조 = 백틱과 별표만 · 본문 날짜 스탬프 금지(경위는 원장에).

## 1. §총칙 — 계승이 디폴트
- a) 새 UI/UX는 새로 디자인하지 말고 **기존 토큰·컴포넌트를 계승**한다. 계승이 디폴트라 매번 "기존 거 계승" 선언 안 함 — 말해야 하는 건 예외뿐. 일관성이 품질이고, 재설계 반복은 드리프트와 운영자 피로를 부른다.
- b) **계승 소스 = 생김새 아닌 배치 맥락(형제 유무·표면)으로 고른다**: 여러 개 나란한 바를 전제한 텍스트-온리 칩을 형제 없이 단독 배치하면 어포던스 0의 고아 라벨이 된다 → 그 표면의 버튼 패밀리(배경·테두리·눌림 있는)를 계승해 어포던스 확보.

## 2. §픽토그램 — 표지판 도형은 SVG · 정렬은 시각중심 실측
- a) **표지판성 도형(상하좌우 화살표·삼각형·X·체크)은 유니코드 문자로 찍지 마라** → SVG path 픽토그램만. 폰트 글리프는 원/박스 정중앙에서 편심돼 정렬이 깨진다(대칭 viewBox path만 정중앙 보장). 문자 렌더 허용 = 순수 텍스트·숫자뿐. 문자·SVG 혼용 금지, 잔존 문자 도형은 발견 즉시 교체. 방향성 아이콘(↗ ↻ ▶)은 viewBox 편심을 측정해 보정하거나 대칭 viewBox로 재정의.
- b) **아이콘 정렬 기준 = 셀(버튼 박스) 아니라 그 안 픽토그램(path) 시각중심**. 정렬 지배 값들(컨/헤더/행 padding·셀폭)은 연쇄 계산이라 하나 바꾸면 전부 재검증(짝 미갱신 = 어긋남). 검증은 눈짐작·스크린샷 금지 → path getBBox 시각중심을 절대좌표로 환산해 픽셀 실측, 1px 초과일 때만 손댐(서브픽셀 잔여는 홀짝 반올림이라 불가피). {선례 이식팩이 있으면 이 절에 편입/링크.}

## 3. §색 규율 — 기존 유지 · 무채색 크롬 (값=팔레트는 복제전문)
- a) **기존에 이미 있는 색은 그대로 유지가 원칙**. 새 색 창작이나 기존 색 제거는 임의 판단 금지 → 운영자 확인.
- b) **단 '통일'은 예외 = 묻지 말고 즉시**: 팔레트 밖으로 튄 아웃라이어(상속 leak·제각각 raw·형제와 어긋난 틴트)를 기존/형제 색에 맞춰 정렬하는 건 계승 디폴트. 확인이 걸리는 건 팔레트에 없는 새 색 창작이나, 의도된 색을 없애 통일을 깨는 경우뿐.
- c) **무채색 크롬**: 중립이어야 하는 팝업·메뉴·드롭다운은 무채색 — 색 틴트·뒤 배경색 비침 금지(배경·테두리·hover·포커스·입력칸 전부 중립). 글래스(blur)는 유지하되 충분히 불투명 + 필요시 채도 0으로 뒤 색까지 차단. 예외 = 색이 '의미'인 요소(상태색·on-state·콘텐츠). 규칙은 크롬 한정.

## 4. §인터랙션 — 반복 인터랙션 = 단일 표준 + 위임 1핸들러
- a) 반복되는 인터랙션(다운로드 버튼, 아이콘 누름 모션 등)은 표면마다 새로 만들지 말고 **단일 표준 클래스 + 위임(캡처) 1핸들러**로 처리한다. 새 인스턴스는 표준만 붙이면 색·효과·모션을 자동 상속 — 개별 창작은 드리프트. 값 정본은 한 곳, 위치·크기만 컨텍스트 클래스로 변주. (완료 표식 같은 영속 상태도 같은 표준에 귀속.)

## 5. §통일 — 최외곽/형제 표면을 한 값으로
- a) '통일'은 **최외곽/형제 표면을 한 값으로 맞추는 작업**이다. 각 뷰의 제일 바깥 박스는 단일 값으로 통일(뷰마다 다른 값 창작 금지), 내부 중첩 박스는 각자 값 유지(패딩 인셋이라 코너 충돌 없음). 한 행에 같은 성격의 요소가 두 스타일로 혼재 금지. 반복 요소(닫기 등)의 제각각 크기도 점진 통일. 별도 레이어(모달 등)는 별도 통일값 허용.

## 6. §머리·타이포 위계
- a) 머리·제목 위계는 **전역 통일된 소수 계층**으로: 대주제(최상위 제목) → 부제(무기호 강조 = 굵게/강조색만) → 소머리(불릿 `•` 하나로 전역 통일). 크기·굵기·색은 리터럴이 아니라 **토큰으로 박아** 하나 고치면 전파.
- b) **헤더·타이틀·부제·버튼 라벨은 텍스트 단독 또는 픽토그램 단독** — 한 줄에 아이콘+글자 공존 금지(상태 변화는 교체). 외부링크 등 어포던스는 장식 pseudo-element SVG로만 덧댐. 어포던스를 통일·제거할 땐 대체를 먼저 마련 — 그냥 없애 기능 저하 금지.

## 7. §기하 — 뷰포트 @media 분기 금지 (앵커가 px 고정이면)
- a) 뷰포트별 `@media` 분기는 앵커 요소가 px 고정일 때 금지 — 종속값(스크림·오프셋 등)도 단일값이어야 정합. 한쪽 뷰포트만 조정하면 다른 쪽이 방치돼 회귀한다. 분기는 앵커 자체가 뷰포트 가변일 때만.

## 8. §승인 경계 — 신규 UI 요소 '화면 등장' = 배치 사전 승인 (기틀 보호의 디자인판)
- a) 라이브 화면에 **새로 생기는 요소**(새 버튼·칩·배지·토글)는 컴포넌트 계승과 **별개 축** — 어디에·어떤 모양으로 등장하는지는 운영자 결정. **기능을 한 줄로 승인받았어도 그건 기능 승인이지 배치 승인이 아니다.** 반영 전 HTML 시안(§디자인제안 = 플레이그라운드) 또는 §선택질문(위치·모양 명시)으로 확인 후 머지. 스코프 = '새로 등장'하는 요소만 — 기존 요소 계승·수정·통일은 안 물음.

## 9. §공통 UI 로직 — 디자인 시스템 없어도 적용 (값 아닌 UI 로직 · 구 마스터 라우터 v1 §공통 UI 방식 이관)
- a) **어포던스가 설명한다**: 컨트롤 옆 사용법 설명 산문(힌트 줄·안내 캡션)은 기본값 = 넣지 않음 — 라벨·상태색·픽토그램·비활성 dim이 스스로 설명하게. 필요한 단위·경고·접근성 라벨만. 빈 상태(빈 리스트·패널)는 필러/설명 문자열 금지, 조용한 공백이 디폴트(예외 = 사용자 행위 피드백 1줄: 결과 없음·불러오는 중·에러). 설명 넣고 싶어지면 = 어포던스 부족 신호 → 문구가 아니라 컴포넌트를 고친다.
- b) **번쩍임(flicker) 방지**: ① 입력칸 담은 컨테이너·오버레이엔 backdrop-filter 금지(리페인트마다 재샘플 = 모바일 GPU 번쩍) ② 아이콘 갱신은 innerHTML 직접 재대입 말고 가드 세터 경유(SVG 노드 재생성이 타이핑마다 번쩍) ③ 비동기 처리중 표시는 전체화면 딤 대신 타이틀 옆 인라인 점모션(필터/트랜스폼 조상이 fixed 오버레이를 모달 안에 가두는 함정 회피).
- c) **자유입력칸 = OS 자동완성/자동교정 끔이 디폴트**(속성 4종 세트) — 안 끄면 모바일 OS가 비번·카드·주소 바를 붙여 오입력·번잡. 맞춤법은 브라우저 아닌 앱 자체 로직이 처리.
- d) **모달 = history pushState/popstate로 뒤로가기 흡수**: 열 때 history push, popstate로 닫기 → 폰 뒤로가기 = 직전 화면 복귀(앱 이탈 아님). X/백드롭/ESC로 닫으면 push 항목 정리.

## 10. 이식 시 유의
- a) **값 vs 방식론 경계**(서두 판정)를 매 항목에 적용 — 특정 hex·토큰명·셀렉터·컴포넌트 실물은 이 문서에 넣지 말고 그 레포 실측 사례로만 인용, 원칙은 일반형.
- b) SSOT↔거울/파생물 관계(거울은 파생·충돌 시 SSOT가 정본)의 *메커니즘*(락·거울 빌더)은 값 축(복제전문)이 담당 — 이 문서엔 "SSOT 우선·거울은 파생" 한 줄만.
- c) 픽토그램 정렬 등 이미 추출된 일반형 선례 이식팩이 있으면 이 문서의 해당 절에 편입/링크한다.
