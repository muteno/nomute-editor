## 0. 기존 인스타 토큰 겸용 여부 (운영자 260718 "인스타 API가 인스타 전용 아니고 메타였는데 못 끌어와?")
- 메타 API는 경로가 2개: **① Instagram Login**(현 인스타 직결 세팅 = 인스타_직결_세팅.md 1행 · graph.instagram.com 전용 · 페이지 접근 불가) / **② Facebook Login**(graph.facebook.com · 페이지+인스타 겸용).
- 현 `IG_ACCESS_TOKEN`은 ①로 발급된 토큰이라 **원칙상 페이지 인사이트는 못 읽는다**. 단 fb_fetch.py에 **겸용 프로브** 내장(260718): `FB_PAGE_TOKEN` 미등록 + `IG_ACCESS_TOKEN` 존재 시 `me/accounts`로 페이지 토큰 자동 수급을 1회 시도 — ②경로 토큰이면 **추가 등록 0으로 즉시 연동**(FB_PAGE_ID도 자동), ①이면 로그에 "인스타 전용 판정"을 남기고 종전 no-op(무해).
- 판별 확인 = Actions fb-fetch 수동 발사 로그(§5) — "겸용 판정 · 자동 연동"이 뜨면 §1~3 생략, "인스타 전용 판정"이면 §1~3 진행.
- **실측 판정(260718 런 29648587638)**: 현 IG 토큰 = **인스타 전용 확정**(HTTP 400 거절) → §1~3 필요. 단 아래 **1값 온보딩**으로 등록은 `FB_PAGE_TOKEN` 하나면 끝.
- **1값 온보딩(260718 "끌어와서 지속 반영")**: `FB_PAGE_TOKEN`만 등록하면 페이지 ID는 코드가 자동 해석(유저 토큰 = me/accounts로 페이지 토큰 자동 교체 · 페이지 토큰 = me 직독) — `FB_PAGE_ID` 변수는 특정 페이지를 **고정 지정**하고 싶을 때만.

## 3. 토큰 발급 → 장기 교환 (Meta 앱 = 인스타 직결 때 만든 앱 재사용)
1) 앱 대시보드 https://developers.facebook.com/apps — 앱이 없으면 인스타 문서 §2 절차로 1개 생성 후 재사용.
2) 그래프 API 탐색기 https://developers.facebook.com/tools/explorer 접속 → 우측에서 앱 선택 → §2 권한 체크 → 토큰 종류 = **User Token이 아니라 Page Access Token**(노뮤트 페이지 선택) → `Generate Access Token`. 이때 뜨는 토큰 = **단기**(수 시간).
3) **장기 토큰 교환(60일+)** — 단기 토큰을 아래 URL로 교환(브라우저 주소창 GET이면 끝 · 앱ID·앱시크릿 = 앱 대시보드 `설정 → 기본 설정`):
```
https://graph.facebook.com/v21.0/oauth/access_token?grant_type=fb_exchange_token&client_id={앱ID}&client_secret={앱시크릿}&fb_exchange_token={단기토큰}
```
   응답 JSON의 `access_token` = 장기 토큰 → §1 `FB_PAGE_TOKEN` 시크릿 값으로 등록. ⚠ 토큰 = 페이지 열람 권한 — 레포 파일·채팅에 절대 붙이지 말고 시크릿에만(인스타 문서 §7 동일 수칙).
4) 페이지 숫자 ID 확인(선택 — 1값 온보딩이 자동 해석 · 고정 지정용):
```
https://graph.facebook.com/v21.0/me/accounts?access_token={토큰}
```
   응답 `data[].id` = `FB_PAGE_ID` — 등록 선택(미등록 = 코드 자동 해석 · 1값 온보딩 260718).
