# 페이스북 페이지 직결 세팅 (채널 요약 1-2 · 운영자 260718 Q155)

> 채널 요약(메뉴4)의 **1-2 페이스북** 탭 데이터 수집(`.github/scripts/fb_fetch.py` → `viewer/fb_data.json`).
> 시크릿 미등록 = 조용한 스캐폴드(수집 스킵 · 뷰어 = 세그만 표시·본문 조용한 공백) — 아래 2값 등록 즉시 다음 run부터 점등.
> 자매 문서 = docs/인스타_직결_세팅.md(같은 Meta 앱 재사용 가능).

## 등록 2값 (GitHub 레포 → Settings → Secrets and variables → Actions)

| 구분 | 이름 | 값 |
|---|---|---|
| **Secret** | `FB_PAGE_TOKEN` | 페이스북 **페이지 액세스 토큰**(장기 · 아래 발급) |
| **Variable** | `FB_PAGE_ID` | 노뮤트 페이스북 **페이지 숫자 ID** |

## 발급 절차 (Meta 앱 = 인스타 직결 때 만든 앱 재사용)

1. https://developers.facebook.com/tools/explorer/ 접속 → 앱 선택.
2. 권한 추가: `pages_show_list` · `pages_read_engagement` · `read_insights` → **Page Access Token** 선택(노뮤트 페이지). (pages_show_list = 4번 me/accounts 페이지 목록 조회용 — Explorer가 페이지 토큰 발급 때 자동 부여하는 경우도 있으나 명시가 안전 · 페이블 검증단 260718)
3. 장기 토큰 교환(60일+): ⚠ 인스타 직결(docs/인스타_직결_세팅.md)은 Instagram Login 'Generate token'이 60일 토큰을 **직접** 주므로 이 교환 단계가 없다 — **페이스북 페이지 토큰만** 아래 교환이 필요(구 "인스타 §토큰 교환과 동일 곡선" 표기 = 실존하지 않는 절 참조라 정정 · 페이블 검증단 260718):
   `https://graph.facebook.com/v21.0/oauth/access_token?grant_type=fb_exchange_token&client_id={앱ID}&client_secret={앱시크릿}&fb_exchange_token={단기토큰}`
4. 페이지 ID 확인: `https://graph.facebook.com/v21.0/me/accounts?access_token={토큰}` 응답의 `id`.

## 수집 스케줄·검증

- 스케줄 = `insta-fetch.yml` 동승(3시간마다 · 별도 크론 0) — 커밋도 같은 `git_land` 착지.
- 즉시 실측 = Actions → insta-fetch → Run workflow(dispatch) → 로그 `fb-fetch: OK` 확인 → 앱 채널 요약 → `1-2 페이스북`.
- 메트릭(page_impressions 계열)은 Graph 버전에 따라 개폐 — 스크립트가 메트릭별 fail-soft라 죽은 메트릭은 '—'로 비고 나머지는 수집.
