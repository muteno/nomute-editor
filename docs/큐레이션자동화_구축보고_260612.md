## 1. 한 줄 결론
첨부 스펙대로 큐레이션 자동화 파이프라인 구축 완료. **인증을 API 키 → 구독 OAuth 토큰(계정 동적 스위칭)으로 전환**했고, run #1~3 디버깅을 거쳐 실기사 분석 카드가 프로덕션 뷰어에 라이브로 떠 있음.

## 2. 아키텍처
```
[폰: 기사 공유]
   │ Termux(queue-news) → pending/YYMMDD-HHMMSS.txt (URL 한 줄) → git push(main)
   ▼
[GitHub Actions: news-analyze]   트리거 = pending/** push (+ 수동 dispatch)
   │ 1) Claude Code 헤드리스(claude -p, claude-opus-4-8)로 URL별 분석
   │    - 인증 = 구독 OAuth 토큰(아래 §4) / 도구 = WebFetch,WebSearch만
   │    - 분석 기준 = prompts/news-analysis.md (apps/news 에디터 지침에 종속)
   │ 2) 결과 md → queue/YYMMDD-HHMM-기사ID.md (파일명 ASCII 한정)
   │ 3) 처리한 pending 삭제 / 실패는 pending/failed/ 로 격리(+.log)
   │ 4) 분석 직후 즉시 커밋·push (if: always — 분석물 보존 최우선)
   ▼
[Cloudflare Pages]  queue/ 커밋마다 자동 재배포
   │ build: node build-viewer.mjs / output: viewer
   ▼
[뷰어]  최신순 리스트·검색·날짜필터·클릭 시 md 렌더
   ▼
[운영자]  심화할 기사 선택 → 클라우드 세션 /news 풀 파이프라인(콘텐츠화)
```
> 이 자동화는 **큐레이션(분류·다이제스트)** 단계. 풀 카드뉴스(콘텐츠화)는 운영자가 고른 것만 별도 세션에서.

## 3. 산출물 (파일)
| 경로 | 역할 |
|---|---|
| `pending/`·`pending/failed/`·`queue/` | 입력함 / 실패 격리 / 결과 md |
| `.github/workflows/news-analyze.yml` | 워크플로(트리거·인증·커밋백·concurrency·루프방지) |
| `.github/scripts/analyze.sh` | URL별 분석 루프·실패 격리·ASCII 파일명 |
| `prompts/news-analysis.md` | 큐레이션 다이제스트 프롬프트(에디터 지침 종속) |
| `build-viewer.mjs` (무의존) + `viewer/index.html` | 정적 뷰어 빌드·페이지 |
| `docs/news-pipeline.md`·`docs/termux-share.sh` | 설정 문서·Termux 스크립트 |

**다이제스트 구조**: frontmatter(title/url/date/media/type/bias/tags) + 🧷한줄요약 + 📰Fact(출처) + 🔎Inference(확신도) + 🧭다각도 해석 + 🛠활용 옵션(앵글 후보·편향 게이지).

## 4. 인증 — 구독 OAuth 토큰 + 계정 스위칭
- API 키 미사용. CLI는 `CLAUDE_CODE_OAUTH_TOKEN` env로 인증.
- 시크릿 2개: `CLAUDE_CODE_OAUTH_TOKEN_MUTENO`(기본), `CLAUDE_CODE_OAUTH_TOKEN_EMS1130G`(스위칭).
- 동적 선택(공식 문법, 검증됨):
  ```
  secrets[format('CLAUDE_CODE_OAUTH_TOKEN_{0}', inputs.account || vars.ACTIVE_ACCOUNT || 'MUTENO')]
  ```
- **상시 전환** = 리포 Variables `ACTIVE_ACCOUNT` 값 변경(MUTENO/EMS1130G) / **1회성** = Run workflow `account` 입력.

## 7. 운영 안전장치
- 무한루프 방지: 트리거 `paths: pending/**` + GITHUB_TOKEN 푸시 비재트리거(이중).
- 동시 실행: `concurrency` 순차. 실패 격리: per-file(한 건 실패해도 나머지 진행).
- 품질 추종: 프롬프트가 워크플로 하드코딩이 아니라 `apps/news/` 최신 지침을 읽음 → 에디터 개선이 큐레이션에 자동 반영.

## 8. 잔여/참고
- **EMS1130G 계정**: 미실증(현재 토큰 사용량 한도 참 — 정상). MUTENO로 라이브 검증 완료. 한도 회복 후 Run workflow account=EMS1130G로 1회 확인.
- 같은 URL 2회 투입 시 카드 2장(중복) 생성됨 — 중복 제거는 미구현(필요 시 url 기준 dedup 추가 가능).
- nate처럼 봇 차단 + URL에 제목 없는 매체는 검색 폴백이 약해 실패 가능 — 사실 무결성 우선이라 실패가 안전한 기본값.
