## 0. 전체 구조 (한 장)

```
[PC A]  다운로더(Seal/gallery-dl) ─┐
[PC B]  다운로더(Seal/gallery-dl) ─┤→  Google Drive "내 드라이브/Shared"
                                    │        (드라이브 데스크톱 "내 드라이브 스트림"으로 저장)
                                    ▼
                        ┌───────────────────────────────┐
                        │  폰 Termux: drive-gallery-sync.sh │  ← rclone gdrive:Shared
                        │  (Tasker가 3분마다 + 잠금해제마다 실행) │
                        └───────────────────────────────┘
                                    ▼
                     /sdcard/Pictures/DriveSync  (LOCAL)
                                    ▼
                        termux-media-scan -r  → 갤럭시 갤러리
```

**핵심 불변**
- 폰은 **rclone `gdrive:` = 내 드라이브 루트**만 본다. → PC가 파일을 넣을 곳은 **반드시 "내 드라이브/Shared"**.
- **⚠️ "컴퓨터 백업"(드라이브 데스크톱 백업)에 넣으면 rclone이 못 본다** = 안 내려옴 = "떨어져 나감"(§7 이번 원인). 두 PC 다 **"내 드라이브 스트림"**으로 저장해야 한다.
- Autosync 앱 **안 씀**(실시간 동기화 안 돼서 버림). 순수 **Termux + Tasker**. `.tmp.drivedownload`/`.tmp.driveupload` 폴더가 보이면 옛 Autosync 잔재다(무시·삭제 가능).

## 1. 폰 스크립트 — `~/.termux/tasker/drive-gallery-sync.sh` (전문)

```sh
#!/data/data/com.termux/files/usr/bin/sh
REMOTE="gdrive:Shared"
LOCAL="/sdcard/Pictures/DriveSync"
SEEN="$HOME/.drivesync.seen"
NOW="$HOME/.drivesync.now"
NEW="$HOME/.drivesync.new"
LOG="$HOME/.drivesync.log"
STAMP="$HOME/.drivesync.last"
T=$(date +%s)
if [ -f "$STAMP" ] && [ $((T - $(cat "$STAMP"))) -lt 120 ]; then exit 0; fi
echo "$T" > "$STAMP"
mkdir -p "$LOCAL"; touch "$SEEN"
[ -s "$SEEN" ] || echo "__init__" > "$SEEN"
rclone lsf -R --files-only "$REMOTE" > "$NOW" 2>>"$LOG" || exit 1
awk 'FNR==NR{s[$0]=1;next} !($0 in s)' "$SEEN" "$NOW" > "$NEW"
if [ -s "$NEW" ]; then
  if rclone copy "$REMOTE" "$LOCAL" --files-from "$NEW" --inplace \
      --transfers 4 --log-file "$LOG" --log-level INFO; then
    cat "$NEW" >> "$SEEN"
  else
    termux-notification --id drivesync-fail -t "드라이브싱크 실패 ⚠️" \
      -c "복사 실패 — 다음 실행 때 자동 재시도 (tail ~/.drivesync.log)" 2>/dev/null || true
  fi
  termux-media-scan -r "$LOCAL"
fi
[ -x "$HOME/.termux/tasker/drive-camera-up.sh" ] && sh "$HOME/.termux/tasker/drive-camera-up.sh"
exit 0
```

**동작 요약**
- `STAMP` 120초 디바운스 — Tasker가 자주 때려도 2분 안이면 그냥 종료(중복 실행 방지). **수동 테스트 전엔 `rm -f ~/.drivesync.last`**(안 지우면 조용히 그냥 종료 = 고장으로 착각).
- `rclone lsf -R gdrive:Shared` 로 원격 파일 목록(`NOW`) → `SEEN`(이미 훑은 목록)과 **diff** → 새 파일만 `NEW`.
- `NEW` 있으면 그것만 `rclone copy` → **복사 성공시에만** `SEEN`에 추가(v1.1 · 실패 배치 = 다음 실행 자동 재시도 + `--id drivesync-fail` 알림 1장 갱신) → `termux-media-scan`으로 갤러리 반영.
- 즉 **한 번 SEEN에 등록되면 다시 안 받는다** → SEEN 오염이 대표 고장(§4·§7). v1.0은 복사가 **실패해도** SEEN에 넣어 그 배치가 영구 실종됐다(260719 실증) — v1.1이 봉합.
- v1.1에서 `--no-traverse` 제거(260719): 파일별 존재 조회가 대량 배치·특정 파일명에서 구글 400(`couldn't list directory`)을 유발 — 전체 목록 순회(매 3분 도는 lsf와 동일 방식·동일 비용)로 회피.
- (v1.2) 말미에 `drive-camera-up.sh` 체인 — 역방향 업로드(§1-b)가 같은 Tasker 트리거로 돈다(업로드 스크립트 없으면 그냥 통과).

재설치(폰 갈았을 때 · 정본 = curl 한 줄 — 복붙 붙여넣기는 bracketed paste 깨짐 전과가 있어 비권장):
```bash
mkdir -p ~/.termux/tasker
curl -fsSL https://raw.githubusercontent.com/muteno/nomute-editor/main/docs/drive-gallery-sync.sh -o ~/.termux/tasker/drive-gallery-sync.sh && chmod +x ~/.termux/tasker/drive-gallery-sync.sh && sh -n ~/.termux/tasker/drive-gallery-sync.sh && echo 설치OK
```

## 1-b. 역방향 업로드 — `~/.termux/tasker/drive-camera-up.sh` (전문 · v1.2 신설)

설치·교체(정본 = curl 한 줄 · 실물 = `docs/drive-camera-up.sh`):
```bash
curl -fsSL https://raw.githubusercontent.com/muteno/nomute-editor/main/docs/drive-camera-up.sh -o ~/.termux/tasker/drive-camera-up.sh && chmod +x ~/.termux/tasker/drive-camera-up.sh && sh -n ~/.termux/tasker/drive-camera-up.sh && echo 설치OK
```

```sh
#!/data/data/com.termux/files/usr/bin/sh
REMOTE_BASE="gdrive:Shared"
SRCS="/sdcard/DCIM/Camera /sdcard/DCIM/Screenshots /sdcard/Pictures/Screenshots"
WIFI_ONLY=0
UPSEEN="$HOME/.driveup.seen"
DSEEN="$HOME/.drivesync.seen"
UNOW="$HOME/.driveup.now"
UNEW="$HOME/.driveup.new"
LOG="$HOME/.driveup.log"
STAMP="$HOME/.driveup.last"
LOCK="$HOME/.driveup.lock"
T=$(date +%s)
if [ -f "$STAMP" ] && [ $((T - $(cat "$STAMP"))) -lt 120 ]; then exit 0; fi
echo "$T" > "$STAMP"
if ! mkdir "$LOCK" 2>/dev/null; then
  A=$((T - $(stat -c %Y "$LOCK" 2>/dev/null || echo 0)))
  [ "$A" -lt 7200 ] && exit 0
  rmdir "$LOCK" 2>/dev/null; mkdir "$LOCK" 2>/dev/null || exit 0
fi
trap 'rmdir "$LOCK" 2>/dev/null' EXIT
: > "$UNOW"
for SRC in $SRCS; do
  [ -d "$SRC" ] || continue
  ls "$SRC" | while IFS= read -r f; do
    [ -f "$SRC/$f" ] && printf '%s\n' "$SRC/$f"
  done >> "$UNOW"
done
[ -s "$UNOW" ] || exit 0
if [ ! -s "$UPSEEN" ]; then
  cp "$UNOW" "$UPSEEN"
  termux-notification --id driveup -t "갤러리 업로드 기준선 설정 ✅" \
    -c "지금부터 새로 생기는 사진·영상만 드라이브로 올라감" 2>/dev/null || true
  exit 0
fi
awk 'FNR==NR{s[$0]=1;next} !($0 in s)' "$UPSEEN" "$UNOW" > "$UNEW"
[ -s "$UNEW" ] || exit 0
if [ "$WIFI_ONLY" = 1 ]; then
  termux-wifi-connectioninfo 2>/dev/null | grep -q '"supplicant_state": "COMPLETED"' || exit 0
fi
FAIL=0
OKN=0
OKF=""
for SRC in $SRCS; do
  [ -d "$SRC" ] || continue
  grep "^$SRC/" "$UNEW" | sed "s|^$SRC/||" > "$UNEW.d"
  [ -s "$UNEW.d" ] || continue
  cat "$UNEW.d" >> "$DSEEN"
  if rclone copy "$SRC" "$REMOTE_BASE" --files-from "$UNEW.d" --inplace \
      --transfers 2 --log-file "$LOG" --log-level INFO; then
    sed "s|^|$SRC/|" "$UNEW.d" >> "$UPSEEN"
    OKN=$((OKN + $(wc -l < "$UNEW.d")))
    [ -z "$OKF" ] && OKF=$(head -1 "$UNEW.d")
  else
    FAIL=1
  fi
done
if [ "$OKN" -gt 0 ]; then
  MSG="$OKF"
  [ "$OKN" -gt 1 ] && MSG="$OKF 외 $((OKN-1))건"
  termux-notification --id driveup-ok -t "공유 완료: ${OKN}건 ✅" \
    -c "$MSG → 드라이브 Shared" 2>/dev/null || true
fi
[ "$FAIL" = 1 ] && termux-notification --id driveup-fail -t "갤러리 업로드 실패 ⚠️" \
  -c "다음 실행 때 자동 재시도 (tail ~/.driveup.log)" 2>/dev/null || true
exit 0
```

**동작 요약**
- 대상 = `SRCS`(카메라·스크린샷 — 폴더 추가/삭제 = 이 변수 한 줄) → **`내 드라이브/Shared` 바닥(평평)** 업로드(운영자 260719 확정 · 구 하위폴더 구조 폐지 — 이름 충돌은 카메라 파일명 형식상 희박 · 트리거 = §1 말미 체인이라 Tasker 프로필 신설 0).
- **첫 실행 = 업로드 0건, 기준선만 설정**: 현재 파일 전부를 "이미 처리"로 등록 → 이후 **새로 생기는 파일만** 올라간다. "기준선 설정 ✅" 알림 = 가동 신호.
- §1과 같은 규율: 업로드 **성공시에만** `~/.driveup.seen` 등록(실패 = 다음 실행 자동 재시도 + `--id driveup-fail` 알림 1장). 단방향 카피 = 폰에서 지워도 드라이브엔 남는다(백업 성격). **성공 알림(v1.2.2)** = 실제 올린 게 있을 때만 "공유 완료: N건 ✅ · 첫 파일명 외 N건"(`--id driveup-ok` 고정 = 무스팸 · 새 파일 없는 틱 = 무알림).
- **에코 차단**: 올리기 전에 그 경로들을 다운싱크 장부(`~/.drivesync.seen`)에 **선등록** — 다운싱크가 방금 올린 내 사진을 도로 받아 갤러리 중복을 만드는 루프를 원천 차단.
- **Wi-Fi 가드 기본 OFF**(`WIFI_ONLY=0` · 운영자 260719 "LTE에서도 되게"): LTE에서도 즉시 올라간다(영상 포함 = 데이터 소모 주의). 아끼려면 `WIFI_ONLY=1`(termux-wifi-connectioninfo = Termux:API로 Wi-Fi일 때만 업로드).
- **동시 실행 락(v1.2.3)**: 대용량 영상이 3분 틱을 넘겨도 다음 틱이 겹쳐 돌지 않음(mkdir 락 · 겹침 = 조용히 양보 · 죽은 락 2시간 자동 회수) — 드라이브 동명 중복 방지. **영상 = 처음부터 포함**(확장자 필터 없음). 감시 폴더 확장(화면녹화 `DCIM/Screen recordings`[공백 경로라 보강 필요]·다운로드 영상 `DCIM/nomute_image`)은 운영자 지시 시.
- 옛 사진 백필(선택 · 1회 · Wi-Fi에서): 다운싱크 장부 선등록 → 통째 카피.
```bash
ls /sdcard/DCIM/Camera >> ~/.drivesync.seen
rclone copy /sdcard/DCIM/Camera gdrive:Shared --inplace -v
```
- 카메라롤이 수천 장 규모를 넘으면 매 3분 lsf 목록·장부도 같이 커진다 — 그때는 Camera를 Shared 밖 최상위 폴더로 빼는 개편 여지(현 규모 무문제).

## 2. rclone 설정 (폰)

- 리모트 이름 = **`gdrive`**, 백엔드 = Google Drive. `gdrive:` 루트 = **내 드라이브**.
- 스크립트가 보는 경로 = `gdrive:Shared` = **내 드라이브/Shared**.
- 최초/재설정: `rclone config` → `n`(new) → 이름 `gdrive` → `drive` → 클라이언트ID 비움 → scope는 **`drive`(전체) 필수**(v1.2 역방향 업로드부터 — 구 안내의 `drive.readonly`는 다운 전용 시절 것, readonly면 업로드 403) → 브라우저 인증.
- **업로드 권한(스코프) 확인**: `rclone config show gdrive: | grep scope` — `drive.readonly`로 나오면 `rclone config update gdrive: scope drive` → `rclone config reconnect gdrive:`(브라우저 동의 1회). scope 줄이 아예 없으면 기본 = 전체(통과).
- **토큰 만료 재연결**: `rclone config reconnect gdrive:` (브라우저 동의 다시).
- ⚠️ **(260719 예고) rclone 공용 client_id 은퇴 임박**: 현 리모트 = rclone 공용 클라이언트ID(설정 때 비움) — rclone NOTICE 실측 "retired during **2026**". **어느 날 다운·업 전 파이프가 일제히 죽으면 1순위 이것.** 대비 = 자체 client_id 발급(https://rclone.org/drive/#making-your-own-client-id) 후 `rclone config update gdrive: client_id <ID> client_secret <SECRET>` → `rclone config reconnect gdrive:`(동의 1회). ⚠ OAuth 동의화면은 **'프로덕션' 게시**로 — 테스트 모드 = 토큰 7일 만료 함정.

## 4. PC 설정 (두 PC 통일 = 재발 방지의 핵심)

**목표**: 두 PC가 **같은 폴더 `내 드라이브/Shared`** 에 저장 → 폰 rclone이 한 곳만 보면 됨.

각 PC에서:
1. **Google Drive for Desktop** 설치 → 로그인.
2. 설정에서 그 다운로드 폴더를 **"이 컴퓨터 폴더 백업"에서 빼고**, **"내 드라이브" 스트리밍**을 켠다 → 탐색기에 `G:\내 드라이브\Shared`(가상 드라이브)가 보이게.
3. 다운로더(Seal / gallery-dl 등) **저장 위치를 `G:\내 드라이브\Shared`로 지정**.
4. 확인: 그 PC에서 파일 하나 저장 → 폰에서 `rclone lsf gdrive:Shared`에 뜨면 성공.

**⚠️⚠️ 절대 규칙 — "컴퓨터 백업" 금지.** 드라이브 데스크톱의 "이 컴퓨터 폴더 백업"으로 저장하면 파일이 **드라이브 "컴퓨터" 탭**(내 드라이브 아님)으로 가고, **rclone `gdrive:`(내 드라이브)는 그걸 못 본다** → 폰에 안 내려옴. 이게 2026-07-02 "떨어져 나감"의 근본 원인이었다(두 PC가 각각 컴퓨터-백업으로 저장해 `내컴퓨터/Shared`·`내컴퓨터/다운로드2/Shared`로 갈라짐). **두 PC 다 반드시 "내 드라이브 스트림".**

옛 컴퓨터-백업 잔재(`내컴퓨터/Shared`, `다운로드2/Shared`)는 백업 해제하거나 삭제해 헷갈림을 없앤다.

## 5. 진단 명령 모음 (끊겼을 때 순서대로)

```bash
# ⓪ 10초 판별 — 범인 반 가르기 (260719 신설)
[ -f ~/.drivesync.last ] && echo "$(( ($(date +%s) - $(cat ~/.drivesync.last)) / 60 ))분 전 마지막 실행" || echo "실행 기록 없음"
tail -20 ~/.drivesync.log
#   → "3분 안" = Tasker·폰 축 정상 → 범인은 드라이브/rclone 쪽(아래 ①~⑤).
#   → 한참 전/기록 없음 = Tasker가 안 쏨 → §6 마지막 행(allow-external-apps·배터리).
#   ⚠️ 스크립트 수동 실행 전 `rm -f ~/.drivesync.last` — 120초 디바운스 탓에 안 지우면 조용히 종료된다.
#   역방향(업로드 §1-b) 축 = tail -20 ~/.driveup.log · 장부 ~/.driveup.seen · 스탬프 ~/.driveup.last(동일 120초 디바운스)

# ① rclone auth·리모트 살아있나 + 내 드라이브 루트 확인
rclone lsf gdrive:
#   → $Temp/ Shared/ 처럼 목록 뜨면 auth 정상.
#   → 'token'·'401'·'couldn't find' 에러면 → rclone config reconnect gdrive:

# ② 새 파일이 내 드라이브/Shared에 실제로 있나 (핵심 판별)
rclone lsf -R gdrive:Shared
#   → 최근 파일 뜨면 = 자리 정상(Ⓐ). 비었거나 옛것만 = PC가 컴퓨터-백업에 저장 중(Ⓑ) → §4로.

# ③ 스크립트 수동 실행 (에러/동작 확인)
bash ~/.termux/tasker/drive-gallery-sync.sh

# ④ 로컬에 내려왔나
ls -la /sdcard/Pictures/DriveSync

# ⑤ 로그·상태 파일
tail -20 ~/.drivesync.log
cat ~/.drivesync.seen      # 이미 훑은 목록(오염 확인)
```

## 6. 흔한 고장 → 복구 (트러블슈팅)

| 증상 | 원인 | 처치 |
|---|---|---|
| **"드라이브가 떨어져 나감" / 새 파일이 폰에 안 옴** | PC가 **컴퓨터 백업**에 저장 → rclone(내 드라이브)이 못 봄 | 파일을 `내 드라이브/Shared`로 이동 + PC를 **"내 드라이브 스트림"**으로(§4) |
| **드라이브엔 파일 있는데 갤러리엔 없음** | `.drivesync.seen`에 이미 등록됐는데 실제 복사가 안 된 채 "본 걸로" 처리됨(SEEN 오염) | `echo "__init__" > ~/.drivesync.seen` → `bash ~/.termux/tasker/drive-gallery-sync.sh` (밀린 것 재-pull) |
| **로그에 `Failed to copy … 400`(couldn't list directory) 뜬 무렵 파일들만 실종** | `--no-traverse` 파일별 조회가 대량 배치·특정 파일명에서 구글 400 + v1.0이 실패 배치도 SEEN 등록 → 영구 스킵(§7 사례 2) | ① 구멍 메우기: `rclone copy gdrive:Shared /sdcard/Pictures/DriveSync --max-age 48h --inplace -v` + `termux-media-scan -r /sdcard/Pictures/DriveSync` ② §1 v1.1 스크립트로 교체 |
| `rclone lsf gdrive:` 에서 auth 에러 | rclone 토큰 만료·취소 | `rclone config reconnect gdrive:` (브라우저 동의) |
| 폴더는 뜨는데 `gdrive:Shared`가 빔 | 파일이 두 PC 컴퓨터-백업으로 갈라짐 | §4 통일 |
| 갤러리에 안 뜸(파일은 `/sdcard/Pictures/DriveSync`에 있음) | media-scan 누락 | `termux-media-scan -r /sdcard/Pictures/DriveSync` |
| Tasker가 안 돎 | allow-external-apps 주석 / 배터리 최적화 | 큐 플레이북 §3-1 + Termux 배터리 최적화 제외 |

**SEEN 리셋 주의**: `echo "__init__" > ~/.drivesync.seen` 하면 다음 실행에 `gdrive:Shared`의 **현재 전량**을 다시 받는다. 원격에 파일이 많으면 그만큼 복사되니, 밀린 것 따라잡기용 **1회성**으로만. 평소엔 건드리지 말 것(새 것만 자동으로 옴).

## 7. 복구 사례 로그

### 2026-07-02 — "드라이브가 떨어져 나감" 복구
- **증상**: PC에서 받은 X 영상이 폰 갤러리로 안 옴. "드라이브가 떨어져 나간 것 같다."
- **진단 경로**:
  1. `drive-gallery-sync.sh`는 `gdrive:Shared`(내 드라이브/Shared)를 봄 — 스크립트·rclone auth 정상.
  2. 실제 파일은 **"컴퓨터" 탭(드라이브 데스크톱 백업)**에 있었음 — `내컴퓨터/Shared` + `내컴퓨터/다운로드2/Shared` 로 **두 PC가 각각 백업해 갈라짐**. rclone(내 드라이브)은 이걸 못 봄.
  3. 이후 파일을 `내 드라이브/Shared`로 되돌리니 `rclone lsf -R gdrive:Shared`에 3개 영상 다 보임(자리 정상).
  4. 그런데도 갤러리 안 옴 → `.drivesync.seen`에 그 3개가 **이미 등록**돼 있었음(끊겼던 시절에 seen엔 찍혔는데 복사는 실패). 스크립트가 "새 거 없음"으로 스킵.
- **조치**: `echo "__init__" > ~/.drivesync.seen` → `bash ~/.termux/tasker/drive-gallery-sync.sh` → 3개 영상 복사·media-scan → **갤러리 반영 확인.**
- **재발 방지**: 두 PC 다 **"내 드라이브 스트림"**으로 저장하도록 통일(§4). 컴퓨터-백업 금지 규칙 박음.
- **교훈**: 이 파이프라인이 레포에 없어서 매번 폰에서 재추적 → 이 문서로 SSOT화.

### 2026-07-19 — "또 멈췄다" = SEEN 오염 재발 (이번엔 구글 400 · v1.1 개정 계기)
- **증상**: 드라이브에 넣은 파일이 폰에 안 옴. §5 ⓪ 실측 = "2분 전 마지막 실행"(Tasker·폰 축 정상) + 로그 07:44 카피 성공 1건(auth 정상).
- **원인**: 06:45 실행에서 `rclone copy`가 `googleapi: 400 (couldn't list directory)`로 실패했는데, v1.0 스크립트가 **실패한 배치도 `SEEN`에 등록** → 그 배치 영구 스킵(260702와 같은 SEEN 오염 — 이번 유발원만 다름). 400 자체는 `--no-traverse`의 파일별 존재 조회가 **대량 배치**에서 구글 쿼리를 깨뜨린 것 — 사후 실측: 문제 배치 = `Scroll Reverser.app` 맥 앱 폴더 통째 업로드(내부 수백 파일 = 수백 부모폴더 일괄 조회 = 쿼리 폭발 추정 · 특수문자 아님). 전체 목록 lsf는 매 3분 정상, `--no-traverse` 없는 복구 카피도 무에러 통과(아래 복구 실측).
- **조치**: ① `--max-age 48h` 수동 카피로 구멍 메움(§6 새 행) ② 스크립트 v1.1 교체 = 성공시에만 SEEN + `--no-traverse` 제거 + 실패 시 `termux-notification --id drivesync-fail` 알림 1장(같은 id 갱신 = 3분마다 쌓임 방지).
- **복구 실측(당일)**: `--max-age 48h` 카피 = 2파일 회수(ScrollReverser-1.9.zip 등) · 400 무재발 · media-scan 962파일. ⚠ `--max-age`는 **mtime 기준**이라 원본 mtime이 오래된 파일(앱 번들 내부 등)은 그물 밖 — 그런 건 §6 SEEN 리셋 or 폴더 지정 카피(`rclone copy "gdrive:Shared/<폴더>" "/sdcard/Pictures/DriveSync/<폴더>" --inplace`)로. 로그 시각 = UTC(KST−9 · 예: 06:45 UTC = 15:45 KST).
- **교훈**: "실패도 본 걸로 치는" 낙관 기록이 조용한 영구 실종을 만든다 — 상태 기록(SEEN)은 성공 확인 뒤에만. ⓪ 판별(마지막 실행 N분 전) 신설로 폰/드라이브 축을 10초에 이등분.
