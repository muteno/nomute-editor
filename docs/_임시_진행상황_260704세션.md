### 1. 화면잠금 ON/OFF/변경 PIN 인증 (보안 · 진행 예정)
**운영자 요구**:
- ON(OFF→ON): **새 PIN 설정** 입력창(소형 PIN 슬롯 팝업, lockpop 바로 위에)
- 변경: 마찬가지 새 PIN 설정
- OFF(ON→OFF): **기존 PIN 입력(인증)** 필요
- **OFF→ON 시 항상 새 PIN 재부여**(기존 PIN 있어도) · **설정 완료되면 기존 PIN 삭제(날림)**

**현재 코드** (`viewer/index.html`):
- `lockOnBtn`(5627~): PIN 인증 **없이** nm_lock_on 토글 (문제)
- `lockChgBtn`(5650~): 텍스트 input(`lockPinIn`) 2단(verify→new1→new2→armed) PIN 설정/변경
- 보안 = `lkHash`(SHA-256 `+':nmlock'`)·`nm_lock_pin`(해시)·5회잠금(`nm_lock_fails`/`nm_lock_until`·LK_MAX·LK_OUT_MS)·마스터 `LK_MASTER_H`(898900)
- 소형 PIN 슬롯 = `#pindlg`(발행식 동그라미·`openPinDlg(title,sub)`+`_awaitPin()` Promise<6자리|null>) 재사용 가능

**계획**:
- `lkAskNewPin()`: openPinDlg('새 PIN 설정')→_awaitPin()→(2단 확인: 1회차 저장→openPinDlg '다시 입력'→일치 확인)→lkHash→nm_lock_pin 저장
- `lkVerifyOldPin()`: openPinDlg('PIN 확인')→_awaitPin()→lkHash→lkPinH()==h(또는 마스터) 검증·5회잠금 반영
- lockOnBtn: OFF→ON = lkAskNewPin() 성공 시 nm_lock_on=1 / ON→OFF = lkVerifyOldPin() 성공 시 nm_lock_on=0 + nm_lock_pin 삭제
- lockChgBtn: lkAskNewPin()(기존 인증 후 or 바로 재설정 — 운영자 "변경도 마찬가지"=새 설정)
- 기존 lockChgBtn 텍스트input 2단 로직(5637-5696) → 팝업 방식으로 교체
- ⚠️ 불변: lkHash·5회잠금·마스터키·화면잠금 해제(lkFire) 로직
- ⚠️ #pindlg는 발행본 잠금과 공유(모달이라 동시 안 열림) · _awaitPin 값만 받아 화면잠금이 lkHash
