플랫폼 ①

Cloudflare 대시보드

딱 1번 · 약 3분
왜 필요: apps.nomute.kr 전체가 비번(Access) 뒤라, 지금은 발행 링크 /s/… 도 로그인 벽에 막혀. /s/* 경로만 뚫어줘야 남한테 보여줄 수 있어. (예전에 아이콘 우회 만든 거랑 완전 동일한 방식)
  1. 화면: 로그인one.dash.cloudflare.com 접속 → 왼쪽 메뉴 Zero Trust 클릭
  2. 화면: Zero TrustAccess → Applications 로 이동
  3. 화면: ApplicationsAdd an application → Self-hosted 선택
  4. 화면: 앱 설정 (도메인)도메인 칸에 → subdomain apps · domain nomute.kr · Path s/*  (이름은 아무거나, 예: 발행본 공개)
  5. 화면: 정책(Policy)정책 추가 → Action Bypass · Include Everyone  (이름 예: 공개)
  6. 화면: 저장Save → 끝. (즉시 적용 · 배포 기다릴 것 없음)
⚠️ 딱 하나 주의: Path 는 정확히 s/* 로. * 나 api/* 처럼 넓게 잡으면 본체·API 까지 뚫려서 우회 구멍이 생겨. /api/* 는 비번 뒤에 그대로 둬야 함(발행·삭제는 로그인된 나만).
플랫폼 ②

apps.nomute.kr 웹앱 — 발행하기

평소 사용
  1. 화면: 요약 리더 (기사 요약 열었을 때)전에 다운로드였던 자리 → 이제 [공유] 버튼. 누르면 발행 설정 팝업이 뜸.
  2. 화면: 발행 설정 팝업 공개범위: 링크가 있는 누구나 허용한 사람만 비공개
    · ‘허용한 사람만’ 은 아직 미구현이라 비활성(회색)
  3. 화면: 발행 설정 팝업 (계속)만료 1일3일7일  ·  PIN 4자리(선택 — 넣으면 링크 열 때 핀 요구)
  4. 화면: 발행 버튼[발행] 누르면 게이지바 차오르고 → 끝나면 링크 자동 복사 + “✓ 발행됨” 표시. 그 링크 붙여넣어서 공유하면 됨.
💡 PIN 걸었을 때: 복사되는 링크엔 핀이 안 들어가(보안). 링크는 링크대로 보내고 PIN 4자리는 따로 알려줘야 상대가 열 수 있어.
플랫폼 ②

apps.nomute.kr 웹앱 — 발행본 관리

확인·삭제
어디서 보나: 뭘 발행했는지, 언제 만료되는지 한 곳에서 관리. 대기열이랑 똑같은 UI(2차 팝업)라 익숙할 거야.
  1. 화면: 프로필 메뉴상단 프로필 눌러 메뉴 열기 → ‘발행본’ 항목(옆에 발행 개수 숫자)
  2. 화면: 발행본 팝업 (2차)발행한 것들이 최신순 목록으로. 각 줄에 제목 · 만료 D-N · 🔒(핀걸림 표시)
  3. 화면: 발행본 팝업 (각 항목)항목마다 링크 복사 / 삭제(휴지통). 삭제하면 그 링크 즉시 못 열림(비공개).
검증 ③

제대로 됐나 · 우회 막혔나 테스트

1회 확인
목표: 남들은 발행한 것만 보이고, 이 링크로 본체·다른 발행본엔 못 들어가는지 확인.
  1. 화면: 시크릿창(로그아웃 상태)발행 링크 apps.nomute.kr/s/… 를 시크릿/다른 기기에서 열기 → 요약이 보이면 정상(비번 안 물음).
  2. 화면: 우회 시도그 상태에서 apps.nomute.kr 딴 주소로 가보기 → 로그인 벽이 떠야 정상(본체 우회 차단 확인).
  3. 화면: 만료 / 핀기간 지나면 “만료된 링크” 안내만 뜸 · 핀 걸린 건 핀 입력창이 먼저 뜸.
혹시 안 되면: 링크에서 로그인 벽이 뜨면 → 플랫폼① Path가 s/* 맞는지 재확인. 반대로 apps.nomute.kr 본체가 비번 없이 열리면 → 우회 정책 범위가 너무 넓은 것(즉시 알려줘).
① Cloudflare /s/* 우회 (1회) → ② 웹앱: 요약 → 공유 → 발행 → ③ 시크릿창에서 링크 확인
