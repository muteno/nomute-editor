한 줄 요약

AI 썸네일의 재생성 버튼(↻ 새로고침)이 지금까지 깜깜이 재추첨(똑같은 프롬프트로 랜덤)만 했어. 이걸 연필(수정) 버튼으로 바꾸고, 탭하면 코멘트 팝업(뉴스요약 수정 팝업과 같은 모양)이 떠서 "더 어둡게 · 차를 파란색으로" 같은 지시를 이미지 재생성에 반영하게 했어. 보내면 버튼 게이지가 차고, 이미지엔 '재생성 중' 진행바가 떠(1~2분 비동기). 코멘트를 비우면 기존처럼 재추첨.

장 / 단점 · 효율성 · 품질저하 리스크

장점

  • 원하는 방향으로 이미지를 지시해 고칠 수 있음(깜깜이 재추첨 탈피).
  • UI는 기존 revdlg·게이지·카드 진행표시 전부 계승(재설계 0 = 일관성).
  • 연필=팝업 간접화로 오탭 과금 위험이 구조적으로 제거(전엔 버튼서 바로 발사).

효율성

  • 재생성은 그 화풍 1장만 Gemini 호출(검색·다른 화풍 보존 = 재과금 0).
  • 코멘트는 기존 요약(a.body) 재수집 없이 프롬프트에만 얹음 = 추가 비용 없음.
  • 자동 배치(news-analyze) 경로는 영향 0(빈 wish → 프롬프트 바이트 동일).

품질저하 리스크 & 방어

  • 프롬프트 인젝션(자유입력이 안전가드 흔들기): "가능한 범위에서 반영"으로 완화 + 안전 규칙을 프롬프트 앞(구도)·뒤(최후미 재천명)로 감쌈 + 제어문자 제거·500자 캡(프론트+백엔드 대칭).
  • 비동기 조기완료 오판(260630 클래스): 완료 판정을 fresh 소스(DATA)에서 스냅 → 재발사 stale 방지.
  • 상태 잔상 오발사: 기사 이동·화풍 전환 시 코멘트 리셋(같은 화풍 재진입만 초안 보존).
  • 기존 태세(무인증 /api/thumbredo·rate-limit 없음)는 이번 변경이 악화하지 않음 — 운영자 수용(암호게이트 추후 일괄). 새 이미지는 6분 초과해도 10분 타임아웃+안내 후 동기경로로 자동 반영.
