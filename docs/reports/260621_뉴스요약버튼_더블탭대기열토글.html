📰 뉴스요약 버튼 — 「연달아 두번 = 대기열」 + 「떠있을 때 한번 더 = 닫기」

날짜 260621 · 대상 nomute-editor.pages.dev · 파일 viewer/index.html

요청: "길게 누르면 대기열 뜨는데(유지) 연달아 두번 누르면 대기열 뜨게, 떠있는 상태에서 한번 더 누르면 닫히게"

검증: 적대적 Opus 4.8·effort MAX 10인 평의회 → 핵심 3동작 ✅ 만장일치 · ⚠️ 지적 4건 전부 반영(키보드·오발·문서·정밀)

? 한눈에 — 뭘 한 건가 (쉬운 설명)

하단 뉴스 요약 버튼으로 "처리 대기열" 팝업을 여는 길이 하나 더 생겼다. 원래는 꾹 누르기(롱프레스)·우클릭·작은 배지 탭으로만 열렸는데, 이제 빠르게 톡톡 두 번이면 바로 뜬다. 그리고 떠 있을 때 같은 버튼을 한 번 더 톡 하면 닫힌다(토글).

한 번만 톡 하면 예전처럼 그냥 피드 탭으로 이동한다 — 평소 사용감은 그대로다.

👆👆
연달아 두번 탭 (≤0.4초)NEW → 대기열 열기
👆
떠 있을 때 한 번 탭NEW → 대기열 닫기
⏱️
꾹 누르기 600ms유지 · PC 우클릭유지 · 배지 탭유지 → 열기
📲
한 번만 톡유지 → 피드 탭 이동(평소 그대로)

⇄ 전 / 후 비교

전 (Before)
  • 대기열 열기 = 롱프레스 600ms / 우클릭 / 배지 탭뿐
  • 닫기 = 작은 ✕ · 바깥 탭(버튼으로 닫는 길 불명확)
  • 빠른 더블탭은 아무 동작 없음(두 번째 탭도 그냥 피드 이동)
  • 피드 버튼에 touch-action 없음 → 더블탭 시 줌·지연 위험
후 (After)
  • 열기 = 위 3가지 + 연달아 두번 탭(가장 직관적인 길 추가)
  • 닫기 = ✕·바깥 탭 + 버튼 한 번 더 탭(토글로 명확)
  • 한 번 탭은 그대로 피드 이동(주 기능 무손상)
  • touch-action:manipulation → 모바일 더블탭 신뢰성↑(ed-chip 패턴 계승)

± 장점 / 단점

  • 발견성↑ — 숨은 롱프레스 말고 누구나 떠올리는 "톡톡 두 번"으로 열림.
  • 주 기능 무손상 — 단일 탭=피드 이동은 지연 없이 즉시(첫 탭을 미루지 않는 설계).
  • 기존 길 전부 보존 — 롱프레스·우클릭·배지·✕·바깥탭 모두 그대로.
  • 최소 침습 — 위임 클릭 핸들러 안에 분기만 추가(새 리스너 0·메모리 0).
  • 더블탭 첫 탭의 부수효과 — 첫 탭이 피드 리로드+맨위 스크롤을 한 뒤 두 번째 탭이 연다(의도된 "첫 탭=평소대로" 결과 · 무해·저렴).
  • 느린 렌더 기기서 더블탭 타이밍 — 두 번째 탭이 0.4초를 넘기면 안 열릴 수 있음(실패해도 피드에 머묾·배지/롱프레스로 폴백).

⚡ 효율성

항목내용
코드 증분JS 분기 5줄 + 상태변수 1개(lastFeedTap) + CSS 1속성. 새 이벤트 리스너·타이머·네트워크콜 0.
런타임 비용탭당 Date.now() 비교 1회. 대기열 조회(api/pending)는 열 때 1회만(연타 스팸 없음·qGen 세대토큰이 stale 응답 폐기).
파이프라인 영향0 — 뷰어 읽기전용 UX. 수집·분석·카드 파이프라인 무관.
번들/배포단일 viewer/index.html 수정. _headers no-cache라 머지 후 하드새로고침 없이 반영.

🛡 품질저하 리스크 & 차단

리스크평가 · 차단
롱프레스/우클릭 회귀✅ 없음 lpFired 가드가 토글 분기보다 먼저 실행 → 롱프레스 동작 불변(평의회 2 만장).
두 클릭 리스너 충돌(연 걸 바로 닫음)✅ 없음 문서 바깥탭 핸들러는 #bnav 내부 클릭이면 가드로 무시 → 새로 연 대기열 유지(평의회 3).
키보드 Enter 오발/연타 깜빡임✅ 차단 더블탭 열기를 e.detail(포인터=≥1, 키보드=0)로 게이트 → Enter는 평범한 탭전환(평의회 8 반영).
피드→타탭→피드 0.4초 내 오발✅ 차단 다른 탭 누르면 lastFeedTap=0 리셋(평의회 1 반영).
모바일 더블탭 줌·300ms 지연✅ 차단 touch-action:manipulation(ed-chip·sc-fail 선례 계승) — 롱프레스는 무영향(둘 다 쓰는 Failed칩이 증명).
JS 스코프/TDZ/구문✅ 안전 check_refs JS구문 통과 + 클릭은 파싱 후 실행이라 lastFeedTap 초기화 완료(평의회 6).
문서 드리프트✅ 동기화 docs/news-pipeline.md 제스처 설명 갱신(480→600+신규동작·평의회 9).

평의회 평결: 핵심 3동작 전원 ✅, 블로커(❌) 0. ⚠️로 들어온 개선 4건(키보드·오발·문서·주석정밀) 전부 반영. "롱프레스 후 첫탭이 안 닫힌다"는 우려는 qpop이 <dialog>가 아닌 <div>라 포커스 이동·blur가 없음을 실측 확인 → 실제로는 정상(추가 수정 불요).

{ } 변경 핵심 (코드)

위임된 #bnav 클릭 핸들러의 피드 버튼 분기:

  • 떠있으면 닫기: if (!qpop.hidden) { qpop.hidden = true; … return; }
  • 연달아 두번이면 열기: if (e.detail && Date.now() - lastFeedTap < 400) { openQueue(); return; }
  • 첫 탭: lastFeedTap = Date.now() 기록 후 평소대로 showTab('feed').
  • 타 탭: lastFeedTap = 0 리셋.
  • CSS: 피드 버튼에 touch-action:manipulation.
