역방향 동기화 — 폰 갤러리(사진·캡처) → GDRIVE

내려받기(GDRIVE→폰)의 짝. 캡처·사진이 알아서 GDRIVE로 올라가면, 구글 드라이브 데스크톱이 그걸 PC로 내려주니 갤러리·GDRIVE가 폰↔PC 공용 창고가 됨. · 20260715_162515 KST · v1

폰 갤러리(캡처·사진) ──rclone copy──▶ GDRIVE 공유폴더 ──구글 드라이브 자동──▶ PC 폴더
✅ PC쪽은 할 일 없음. 사진·캡처는 그냥 파일이라, 네 PC의 구글 드라이브 데스크톱이 그 폴더를 자동으로 동기화해서 캡처가 알아서 PC에 나타나. 우리가 세팅할 건 폰 → GDRIVE 한 구간뿐.

0 값 확인 (제일 중요)

스크립트에 넣을 rclone 이름 · GDRIVE 폴더를 알아낸다. 터먹스에서:

rclone listremotes            # 이름 확인(콜론 뺀 게 REMOTE, 보통 gdrive)
rclone lsd gdrive:            # GDRIVE 폴더 목록 — 내려받기가 쓰던 그 폴더가 GDRIVE_DIR

그리고 올릴 갤러리 폴더 확인(캡처·카메라):

ls /storage/emulated/0/DCIM/          # Screenshots(캡처) · Camera(사진) 있는지
# 캡처가 Pictures 밑이면: ls /storage/emulated/0/Pictures/
아직 ~/storage 접근이 안 되면 termux-setup-storage 한 번(팝업 허용) 후 다시.

1 스크립트 설치 둘 중 아무거나

A. 리포에서 뽑기 (폰 로컬 안 건드림 — 추천):

cd ~/nomute-editor
git fetch origin claude/clipboard-photos-sync-scg4zn
git show origin/claude/clipboard-photos-sync-scg4zn:sync/gdrive-up.sh > ~/bin/gdrive-up
chmod +x ~/bin/gdrive-up

B. 그냥 복붙:

mkdir -p ~/bin; cat > ~/bin/gdrive-up << 'EOF'
#!/data/data/com.termux/files/usr/bin/bash
export TZ='Asia/Seoul'
# ── 채울 값 ──
REMOTE="gdrive"
GDRIVE_DIR="__여기_채워__"
PHONE_DIRS=(
  "/storage/emulated/0/DCIM/Screenshots"   # 캡처
  "/storage/emulated/0/DCIM/Camera"        # 사진
)
# ────────────
MODE="${1:-all}"                     # all=사진+클립 · clip=클립만(위젯)
CLIP_STATE="$HOME/.gdrive_clip_last"; LOG="$HOME/gdrive-up.log"
notify(){ termux-notification -t "$1" -c "$2" 2>/dev/null || true; }
log(){ echo "$(date '+%y-%m-%d %H:%M:%S') | $1" >> "$LOG"; }
case "$GDRIVE_DIR" in *__여기_채워__*) notify "설정 미완료 ⚙️" "REMOTE·GDRIVE_DIR 채워"; log "SETUP_INCOMPLETE"; exit 1;; esac
command -v rclone >/dev/null 2>&1 || { notify "업로드 실패 ❌" "rclone 없음"; log "NO_RCLONE"; exit 1; }
if [ "$MODE" != "clip" ]; then
  for d in "${PHONE_DIRS[@]}"; do
    [ -d "$d" ] || { log "SKIP_NODIR $d"; continue; }
    if rclone copy "$d" "$REMOTE:$GDRIVE_DIR" --exclude "clip_*.txt" --transfers 4 --checkers 8 --min-age 10s --log-file "$LOG" --log-level INFO; then
      log "PHOTO_OK $d"; else notify "사진 업로드 실패 ❌" "tail ~/gdrive-up.log"; log "PHOTO_FAIL $d"; fi
  done
fi
CLIP="$(termux-clipboard-get 2>/dev/null)"
if [ -n "$CLIP" ] && [ "$CLIP" != "$(cat "$CLIP_STATE" 2>/dev/null)" ]; then
  TS="$(date +%Y%m%d_%H%M%S)"; TMP="$HOME/clip_$TS.txt"; printf '%s\n' "$CLIP" > "$TMP"
  if rclone copy "$TMP" "$REMOTE:$GDRIVE_DIR" --log-file "$LOG" --log-level INFO; then
    printf '%s' "$CLIP" > "$CLIP_STATE"; notify "클립보드 올림 ✅" "clip_$TS.txt"; log "CLIP_OK clip_$TS.txt"
  else notify "클립보드 업로드 실패 ❌" "tail ~/gdrive-up.log"; log "CLIP_FAIL"; fi
  rm -f "$TMP"
fi
EOF
chmod +x ~/bin/gdrive-up

2 값 채우기

nano ~/bin/gdrive-up   # REMOTE / GDRIVE_DIR / PHONE_DIRS 확인·수정. 저장 Ctrl+O→Enter→Ctrl+X
GDRIVE_DIR의 __여기_채워__는 꼭 실제 폴더로(안 채우면 안전장치가 실행을 막음). PHONE_DIRS는 캡처·사진 폴더가 맞는지 0번에서 본 경로로 맞춰 — 폴더 더 넣어도 됨(한 줄씩).

3 손으로 테스트

~/bin/gdrive-up          # 직접 한 번
tail ~/gdrive-up.log     # PHOTO_OK 보이면 성공

→ 캡처 하나 찍고 다시 돌려서, GDRIVE 그 폴더 + PC의 구글 드라이브 폴더에 그 캡처가 나타나면 전 구간 관통 🎉

4 Tasker 자동 실행 (잠금해제 / 3분마다)

이미 내려받기를 도는 그 Tasker 작업에 액션 하나만 더 — 받고 나서 바로 올림.

mkdir -p ~/.termux/tasker && cp ~/bin/gdrive-up ~/.termux/tasker/gdrive-up && chmod +x ~/.termux/tasker/gdrive-up

Tasker에서 내려받기 Task 열기 → ➕ → 플러그인 → Termux:Tasker → Executable gdrive-up · Arguments 비움 · Wait for result 해제. (또는 내려받기 스크립트 맨 끝에 ~/bin/gdrive-up 한 줄.)

잠금해제·3분 트리거는 내려받기가 이미 가진 걸 재활용. 새 트리거 X.

5 PC에서 받기 — 할 일 없음 ✨

PC에 구글 드라이브 데스크톱이 깔려 있으니, GDRIVE 그 폴더가 PC에 자동 동기화돼. 폰서 캡처하면 몇 초 뒤 PC의 내 드라이브\… 폴더에 그 이미지가 그대로 생김. 별도 스크립트·설정 없음.

혹시 폰↔PC 폴더가 안 맞으면: PC 구글 드라이브 설정에서 그 폴더가 '동기화 대상'인지, '스트리밍'이면 오프라인 사용으로 바꾸면 로컬에 실물 파일로 내려옴.

＋ (옵션) 텍스트도 즉시 — 홈 위젯 없어도 됨

가끔 텍스트도 PC로 바로 넘기고 싶으면(뉴스 요약 등), 복사 후 위젯 한 번 탭 → clip_시각.txt로 GDRIVE행. 위젯 설치:

mkdir -p ~/.shortcuts
cat > ~/.shortcuts/클립올리기 << 'EOF'
#!/data/data/com.termux/files/usr/bin/bash
~/bin/gdrive-up clip
EOF
chmod +x ~/.shortcuts/클립올리기

홈 길게 눌러 → 위젯 → Termux:Widget → 클립올리기. 텍스트는 파일로 가니 PC에선 그 .txt를 열어 보면 됨(클립보드 자동주입까지는 이번 범위 밖 — 필요해지면 그때).

⚠️ 알아둘 것

무한 왕복 걱정 X. 받기·올리기 둘 다 copy라 이미 있는 건 스킵. 폰서 지워도 드라이브 원본은 유지. 단 GDRIVE에서 지운 사진이 폰에 남아 있으면 다시 올라올 수 있음 — 지울 땐 양쪽, 또는 올릴 폴더를 캡처/카메라 위주로.

추적. 매 실행이 ~/gdrive-up.log에 남아 — tail ~/gdrive-up.log로 OK/FAIL·어느 폴더 올렸는지 확인.
