역방향 동기화 — 폰 갤러리 + 클립보드 → GDRIVE

내려받기(GDRIVE→폰)의 짝. 이미 rclone 승인은 끝났으니 값 3개 채우고 → 스크립트 설치 → Tasker에 한 줄이면 끝. · 20260715_162515 KST · v1

폰 갤러리 폴더 + 클립보드 ──rclone copy──▶ GDRIVE 공유폴더 이미 올린 건 자동 스킵 · 폰서 지워도 드라이브 유지

0 값 3개 확인 (제일 중요)

스크립트에 넣을 rclone 이름 · GDRIVE 폴더 · 폰 폴더 3개를 먼저 알아낸다. 터먹스에서:

rclone listremotes

→ 나온 이름에서 콜론(:) 뺀 것이 REMOTE (보통 gdrive).

rclone lsd gdrive:            # ← 위에서 나온 이름:  으로. GDRIVE 폴더 목록이 뜬다
rclone lsd gdrive:내가찾는폴더  # 하위 폴더면 한 단계씩 들어가 전체 경로 확인

→ 내려받기가 쓰던 그 폴더의 전체 경로가 GDRIVE_DIR (예: 공유/폰).

ls ~/storage/dcim ~/storage/shared/Download 2>/dev/null   # 갤러리 폴더 후보 훑기

→ 내려받은 사진이 들어오던 폴더가 PHONE_DIR (예: /storage/emulated/0/DCIM/GDrive). Files 앱에서 폴더 ⋮ → 상세정보로 경로 확인 가능.

아직 ~/storage가 없으면 termux-setup-storage 한 번 실행(팝업 허용) 후 다시.

1 스크립트 설치 둘 중 아무거나

A. 리포에서 뽑기 (폰 로컬 안 건드림 — 추천):

cd ~/nomute-editor
git fetch origin claude/clipboard-photos-sync-scg4zn
git show origin/claude/clipboard-photos-sync-scg4zn:sync/gdrive-up.sh > ~/bin/gdrive-up
chmod +x ~/bin/gdrive-up

B. 그냥 복붙 (git 신경 쓰기 싫으면 이거 통째로):

mkdir -p ~/bin; cat > ~/bin/gdrive-up << 'EOF'
#!/data/data/com.termux/files/usr/bin/bash
export TZ='Asia/Seoul'
# ── 채울 값 3개 ──
REMOTE="gdrive"
GDRIVE_DIR="__여기_채워__"
PHONE_DIR="__여기_채워__"
# ────────────────
CLIP_STATE="$HOME/.gdrive_clip_last"; LOG="$HOME/gdrive-up.log"
notify(){ termux-notification -t "$1" -c "$2" 2>/dev/null || true; }
log(){ echo "$(date '+%y-%m-%d %H:%M:%S') | $1" >> "$LOG"; }
case "$GDRIVE_DIR$PHONE_DIR" in *__여기_채워__*) notify "설정 미완료 ⚙️" "gdrive-up 상단 값 채워"; log "SETUP_INCOMPLETE"; exit 1;; esac
command -v rclone >/dev/null 2>&1 || { notify "업로드 실패 ❌" "rclone 없음"; log "NO_RCLONE"; exit 1; }
if [ -d "$PHONE_DIR" ]; then
  if rclone copy "$PHONE_DIR" "$REMOTE:$GDRIVE_DIR" --exclude "clip_*.txt" --transfers 4 --checkers 8 --min-age 10s --log-file "$LOG" --log-level INFO; then
    log "PHOTO_OK"; else notify "사진 업로드 실패 ❌" "tail ~/gdrive-up.log"; log "PHOTO_FAIL"; fi
else log "NO_PHONE_DIR $PHONE_DIR"; fi
CLIP="$(termux-clipboard-get 2>/dev/null)"
if [ -n "$CLIP" ] && [ "$CLIP" != "$(cat "$CLIP_STATE" 2>/dev/null)" ]; then
  TS="$(date +%Y%m%d_%H%M%S)"; TMP="$HOME/clip_$TS.txt"; printf '%s\n' "$CLIP" > "$TMP"
  if rclone copy "$TMP" "$REMOTE:$GDRIVE_DIR" --log-file "$LOG" --log-level INFO; then
    printf '%s' "$CLIP" > "$CLIP_STATE"; notify "클립보드 올림 ✅" "clip_$TS.txt"; log "CLIP_OK clip_$TS.txt"
  else notify "클립보드 업로드 실패 ❌" "tail ~/gdrive-up.log"; log "CLIP_FAIL"; fi
  rm -f "$TMP"
fi
EOF
chmod +x ~/bin/gdrive-up

2 값 3개 채우기

nano ~/bin/gdrive-up      # 상단 REMOTE / GDRIVE_DIR / PHONE_DIR 를 0번 값으로. 저장 Ctrl+O → Enter → Ctrl+X
A로 설치했으면 REMOTE도 확인해서 맞춰. B로 복붙했으면 __여기_채워__ 2곳을 꼭 실제 값으로 (안 채우면 안전장치가 실행을 막음).

3 손으로 테스트

~/bin/gdrive-up          # 직접 한 번 실행
tail ~/gdrive-up.log     # PHOTO_OK / CLIP_OK 보이면 성공

→ GDRIVE 그 폴더에 사진들이 올라왔는지, 클립보드에 텍스트 복사해 두고 다시 돌리면 clip_…​.txt가 생기는지 눈으로 확인.

4 Tasker 자동 실행 (잠금해제 / 3분마다)

이미 내려받기를 도는 그 Tasker 작업(Task)에 액션 하나만 더 붙이면 끝 — 받고 나서 바로 올림.

방법 A · Tasker에서 (권장)
기존 내려받기 Task 열기 → ➕ → 플러그인 → Termux:Tasker → 구성:
· Executable: gdrive-up   · Arguments: 비움   · Wait for result: 해제
※ 그러려면 아래 한 줄로 tasker 폴더에도 복사(권한 필요):
mkdir -p ~/.termux/tasker && cp ~/bin/gdrive-up ~/.termux/tasker/gdrive-up && chmod +x ~/.termux/tasker/gdrive-up
방법 B · 내려받기 스크립트 끝에 한 줄
내려받기 스크립트가 하나로 돼 있으면, 그 맨 끝에 이 줄 추가가 제일 간단:
~/bin/gdrive-up      # ← 내려받기 다 한 뒤 이어서 올리기
잠금해제·3분 트리거는 내려받기가 이미 갖고 있으니 그대로 재활용. 새 트리거 안 만들어도 됨.

⚠️ 알아둘 3가지

1. 클립보드는 텍스트만. 이미지 복사한 건 못 올라가 — 근데 사진은 갤러리 파일로 어차피 챙기니 괜찮아. 텍스트는 clip_시각.txt로 쌓임.

2. 무한 왕복 걱정 X. 받기·올리기 둘 다 copy라 이미 있는 건 스킵. 폰에서 찍은 새 사진만 올라가고, 그게 다음 받기 때 폰에 이미 있으니 다시 안 받아. 단 GDRIVE에서 지운 사진이 폰에 남아 있으면 다시 올라올 수 있음 — 지울 땐 양쪽 다 지우거나, 올릴 폴더를 "새로 찍는 폴더" 위주로 두면 깔끔.

3. 알림 꺼도 추적됨. 매 실행이 ~/gdrive-up.log에 남아 — tail ~/gdrive-up.log 한 줄로 OK/FAIL 확인.
