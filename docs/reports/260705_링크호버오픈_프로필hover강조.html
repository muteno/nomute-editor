하단 네비게이션과 프로필 메뉴의 호버(마우스 올림) 경험을 다듬었어. 두 변경 모두 기존 디자인 토큰 계승(새 색 창작 0)이고, viewer 파일 한 개만 손댄 국소 손질이야. 아래 데모는 실제로 마우스를 올려보면 동작해.
1 링크 버튼 — 0.5초 호버 = 자동 오픈 하단 '링크'에 마우스를 0.5초 이상 올리면 링크 메뉴가 저절로 뜸. 클릭 안 해도 됨.
2 프로필 메뉴 — hover 시 강조색 프로필 메뉴 항목에 마우스를 올리면 초록 강조색으로 바뀜. 링크 메뉴와 동일한 느낌.

① 하단 '링크' = 마우스 0.5초 호버 시 자동 오픈

하단 네비의 링크 버튼(#linkfab)에 마우스를 0.5초 이상 계속 올려두면 링크 메뉴(피드 탐색 팝업)가 자동으로 떠. 클릭해서 여는 기존 방식은 그대로 유지돼.

👇 '링크'에 마우스를 올린 채 0.5초 기다려봐 — 저절로 뜸. 마우스를 떼면(0.5초 전) 취소돼. (터치 화면에선 이 데모의 hover가 없으니 실제 폰에선 탭으로 열려.)

왜 이렇게 했나 · 장단점

  • 마우스 전용 가드: e.pointerType==='mouse'일 때만 호버 타이머가 걸려서, 터치·펜으로 탭할 때 오작동(눌렀는데 딴 게 뜸)이 없어.
  • 기존 로직 재사용: 클릭으로 열 때 쓰던 코드(다른 팝업 닫기→표시→위치잡기)를 그대로 호출 = 열림 결과가 클릭과 100% 동일. 새 경로 안 만듦.
  • 열기만 넣고 닫기·클릭 토글은 안 건드림 = 회귀 위험 최소.
  • 데스크톱에서 호버 아웃해도 자동으로 안 닫힘(닫기는 요청 범위 밖이라 기존 방식 유지). 마우스가 벗어나도 팝업이 남아있는 게 어색하면, '0.5초 뒤 자동 닫기'도 추가할 수 있어 — 원하면 말해줘.

② 프로필 메뉴 항목 = hover 시 강조색

지적한 대로 프로필 메뉴 항목은 마우스를 올려도 흰색 그대로라 뭘 고르는지 잘 안 보였어. 링크 메뉴는 올리면 초록 강조색이 나와서 명확했지. 프로필 메뉴도 링크 메뉴와 똑같이 강조색이 나오게 통일했어.

👇 세 메뉴에 각각 마우스를 올려봐 — 왼쪽(옛 프로필)은 흰색뿐, 가운데(새 프로필)·오른쪽(링크)은 초록 강조색.

왜 이렇게 했나 · 장단점

  • 선택 어포던스: 지금 커서가 어느 항목 위인지 색으로 즉시 보임 = 오클릭 감소.
  • 링크 메뉴와 통일: 둘이 다르게 반응하던 불일치를 없앰. 참고 소스는 .bpop a:hover(링크 메뉴)를 그대로 계승.
  • 기존 토큰만 사용(--accent) = raw 색 창작 0, 브랜드 팔레트 안에서 움직임.
  • 과거 260629에 프로필 메뉴를 일부러 무채색으로 만든 이력이 있어(팝업 크롬 중립화). 이번엔 운영자 요청으로 그 결정을 뒤집은 것 — 경위를 코드 주석에 남겨서 다음 세션이 '무채색 규칙 위반'으로 오판해 되돌리지 않게 했어.

효율성 · 품질 리스크

  • 효율: 순수 CSS hover 색 + 이벤트 리스너 3개(총 +18줄/−3줄). 렌더·네트워크·파이프라인 영향 0. 리플로우 없음.
  • 접근성: 색만 바뀌고 구조·포커스·aria 불변. 키보드 동작 그대로.
  • 알림 토글(ON/OFF)도 .pmenu button에 걸려 OFF일 때 hover하면 초록이 돼(누르면 켜진다는 신호로 자연스러움). 어색하면 이 항목만 제외 가능 — 판단 원하면 알려줘.
