아웃라이어 감쇠하한(OUT) — 압도적 다매체는 하루 안엔 안 묻힌다

수집함 누적 랭킹 · 운영자 지시(cr39 장윤기 스샷) · 라이브 candidates.json 3,421건 실측 시뮬 기반 · 정본 = curation-algorithm.md §5 ②-b·§8 260706

요지 (한 줄)

매체 보도량이 통계적으로 압도적(강건 z ≥ 2.5~3.5)인 카드는, 1일(24h)이 지나기 전까지 시간감쇠가 12h 시점 값 밑으로 안 내려가게 하한을 깔았다. 1일 지나면 종전대로 침몰.

문제 (운영자 제보 그대로)

  • 화면의 사회 · 39 · 이슈(장윤기 증거인멸) = AI 사건묶기 병합 카드(클러스터 3개 = 15+15+9 합산 39 · rc16 · grade2).
  • 23시간 시점 시간감쇠 timeAcc=.15에 눌려 순위 7위 안팎 — 신선한 중형(cr9~13)들이 갱신될 때마다 앞질러 순서가 계속 뒤바뀜 → "그 카드를 봤는지도 모르겠다".
  • 랭킹 버그는 아님(공식대로 정확히 침몰) — 공식에 '압도적 보도량' 개념이 없던 것이 문제.

핵심 발견 — 왜 '그냥 표준편차'로는 안 됐나

풀 전체(병합 후 cross≥8 · n=417)의 단순 평균/표준편차는 아카이브에 남은 과거 고래들(cr74 반도체기금 · cr101 투표함 · cr117 홍명보호)이 꼬리를 부풀려 σ=9.56 → cr39가 z=2.0 = "아웃라이어 아님" 오판.

→ 강건 통계(중앙값 + MAD×1.4826)로 교체: med=14.0 · σr=2.97 → 장윤기 z=7.0 = 압도적 확정. (아웃라이어를 찾는 통계는 아웃라이어에 오염되지 않는 잣대로 재야 한다.)

z는 표시값 39가 아니라 병합 댐핑 랭킹축(_rankCross≈34.9) 기준 — (39−14)/2.97로 검산하면 8.4가 나와 어긋나 보이는 게 정상.

변경 (전 → 후)

전: scScore = cross^1.3 × timeAcc × 연속보도 × AI경중 × [긴급×3]
후: scScore = cross^1.3 × max(timeAcc, 12h값 .560 × z램프) × 연속보도 × AI경중 × [긴급×3]
  • 자격 4중 게이트: ① 강건 z ≥ 2.5(3.5에서 완전 — 선형 램프 = 경계 순위 튐 방지) ② ⚡이슈 배지 자격(issCross) 그대로 상속(cross≥10·grade null‖≥2·시황/PR 정형컷 — 배지도 못 다는 합산 몬스터는 완화도 없음) ③ 연속보도 rc ≥ 6(일회성 대량 신디케이션은 압도적이어도 제외 — "다매체 AND 계속보도") ④ 나이 < 24h(⚡이슈 배지 소멸선·배지와 동일 max[발행,수집] 나이축 — 배지와 완화가 함께 꺼짐).
  • scScore 시간항 전용 — 신호·진입·배지·글로우·표시 cross·신규 칼럼(fastScore)·수집·AI판정 전부 0 변경.
  • 가드: 심야 풀 n<15 = 완화 OFF · σr 하한 2 · 검색 중 통계 동결.

장단점

  • 장점: ① 운영자가 놓치던 '그날의 진짜 대형'이 24h 동안 상단 고정권 — 순서 뒤바뀜(churn)도 해소(하한 걸린 점수는 신선 중형이 못 넘음) ② 발동이 극히 드묾(이 스냅샷 3건 · daily_health 계기판이 0~5 정상·6+ 경보로 상시 감시) = 화면 성격 유지 ③ z 기준이라 절대값 인플레(수집 확대)에 자동 적응.
  • 단점: ① 24h 경계에서 순위가 한 번에 뚝 떨어짐(계단) — 단 배지도 같이 꺼져 시각적으로 일관, 운영자 스펙("1일 넘으면 밀림") ② 상수 5개 추가(복잡도 소폭↑) — 전부 한 블록에 격리·주석 완비.

효율성

  • 추가 연산 = 렌더당 정렬 1회(풀 ~417개 값) + 카드당 산술 몇 번 = 체감 0(기존 renderScrap 대비 <1%).
  • 서버·수집·AI 판정 비용 0 — 순수 뷰어 랭킹층. 추가 네트워크 0.

품질 저하 리스크 (정직)

  • 합산 몬스터 리스크: 사건묶기 오병합으로 cross가 부풀면 완화도 같이 받는다 → 방어 = _rankCross 댐핑(이미 있음) + issCross 정형컷 + group_judge rubric(260703 조이기) 3중. 오병합 자체는 별개 축에서 계속 관리.
  • 신선 대형과의 경합: 완화된 고래가 갓 나온 중형을 누른다 — 의도된 트레이드오프(운영자 지시). 진짜 긴급 신선건은 긴급×3 부스트·신규 칼럼 핀이 따로 있어 안 묻힘.
  • z 경계 플리커: 풀 통계가 매 렌더 변해 z가 2.5 근처서 흔들릴 수 있음 → 램프(2.5~3.5)라 점수가 연속 변화 = 순위 튐 없음(실측 z7.0·8.7은 경계에서 멂).
  • 콜드스타트 소풀 왜곡: n<15면 통계 자체 OFF(보수 원칙 §1 — 풀은 전연령이라 평시 ~470건 = 통상 미발동 안전망).
  • 긴급 중첩: breaking이면서 아웃라이어면 완화(시간항 ~3.7배)×긴급 부스트 3.0이 중첩 — "긴급+압도 = 최상위" 의도이나 계기판 감시 대상.
