3. 조치 ② — 분량: 실측 가드 + 1회 보강 (기본 OFF)

저장 → 실측(IG<600 또는 Thread<390, 자유요약≥800일 때만) → 보강 1콜(자유요약에서 옮겨 담기 · effort max·safe-mode·무도구·480s) → 검증 이식(펜스·면책 대칭·제목 보존·날조 숫자·상한) → 라벨 = 실측 갱신 → lint
  • 검증 실패 블록 = 원본 유지(fail-soft) — 요약 유실 위험 0. 라벨 뻥튀기도 기계가 실측으로 갱신.
  • 보강 콜은 검색 도구 0·단발이라 #1552 타임아웃 원인(검색×max) 비해당 = 네 타임아웃 수리는 그대로 보존, 품질 레버(max)만 압축 구간에 복원.
  • 발동률 실측 42.5%(최근 2주 226건 기준) → ON 시 추가 콜 약 45/주(전체 쿼터 +3.4%).

6. 카나리아 — 네가 할 일 (직접 테스트)

  1. GitHub 레포 Settings → Secrets and variables → Actions → Variables에 SUMMARY_LEN_GUARD = 1 추가(커밋 불요·즉시 적용).
  2. 평소처럼 요약 2~3건 던지기(폰 공유·✨요약요청 아무거나).
