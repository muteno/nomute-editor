헤더 = 백엔드 흰칸(box.jpg), 배경 ON = 기본(nobg.jpg) 추가 → 헤더 1블록이 최대 2장. 자막·오버레이 = OPA 값마다 1장. 저작권/안내문 = 삽입구합성 OFF면 별도 장, ON이면 매칭 포맷 산출물에 병합.

❌ 불가능 · 제약 조합 (QA 포인트)

❌ 헤더 + 사진 합성 — 불가 (사용자 QA 정확)
헤더는 백엔드에서 흰칸(box)/기본(nobg)만 그림 — 이미지 페이로드를 안 받음. 사진은 오직 자막(오버레이)에만 합성된다. 그래서 "헤더에 사진 합성"은 구조적으로 불가.
근거: thumb.js — 헤더 out = box.jpg/nobg.jpg 고정, imageB64 미참조. 발사 items에서 헤더 payload엔 imageB64 없음.
❌ 헤더 + 포스트 오버레이 — 불가(릴스 강제)
부제·제목 중 하나라도 입력되면 effOvFmt()가 오버레이를 릴스 9:16 강제. 흰칸(9:16)과 짝을 맞추기 위함. 헤더가 있으면 포맷 토글은 무시(릴스 고정, dim). → 헤더 + 포스트오버레이 동시 불가.
❌ 반쪽 헤더 (부제 or 제목 하나만)
헤더는 부제+제목 둘 다 있어야 발사. 하나만 채우면 발사 차단 — 제목을 넣어주세요 / 부제를 넣어주세요. (반쪽 헤더 = 흰칸 백엔드 불가)
❌ 자막에 강조(*별표*) 없음
자막(오버레이)은 강조 별표 최소 1개 필수. 없으면 발사 차단 — 강조를 넣어주세요.
❌ 카드뉴스 + 헤더 동시
카드뉴스 ON = 헤더 입력칸 read-only 잠금. 카드=포스트4:5 전제 ↔ 헤더=릴스 강제 = 모순이라 원천 차단. 동시 불가.
❌ 카드뉴스 + 사진 없음
카드뉴스는 배경 사진 필수 — 없으면 이미지를 선택해주세요.
⚠ 사진만 첨부(글자 0) — "보이는데 안 나옴" 착시 (이슈1 진범)
헤더·자막·저작권·안내문이 모두 비면 아무것도 발사 안 됨(차단). 그런데 사진만 붙이면 미리보기엔 '자막' 프레임(사진 배경 미리보기용)이 뜬다 — 발사는 자막 글자가 있어야 되므로 실제로는 미생성. → 발사/백엔드 버그 아님. 헤더+자막을 둘 다 채우면 정상적으로 같이 발사됨(3행).
⚠ 사진 미첨부 + 오버레이만 — ON 고정으로 교정 (이슈2 · 260720 Q292)
이전엔 사진이 없어도 '오버레이만'이 자유 토글이었음(합성 대상이 없는데 OFF 표시 = 거짓 상태). → 사진 없으면 🔒ON 고정·잠금(dim·클릭 무효)으로 교정. 사진 붙이면 다시 토글 가능.

토글 상호작용 요약

• 포맷(포스트/릴스) — 헤더 입력 중이면 릴스 강제(토글 무시).
• 배경(cBg) — 헤더 있을 때만 의미(기본/nobg 추가). 헤더 없으면 무효.
• 오버레이만(ovOnly) — 사진 있을 때만 의미. 사진 없으면 ON 고정(Q292).
• 카드뉴스(A2M) — 헤더 잠금 + 포스트 4:5 + 사진 필수.
• 삽입구 합성(cpMerge) — 저작권·안내문을 매칭 포맷 산출물에 알파합성(별도 장 없앰).
• 미리보기 — 변형 선택칩 제거(Q293) → 우선순위 1개 변형(흰칸>자막>카드)만 표시. 발사는 채운 섹션 전부.
