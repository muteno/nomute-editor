## 항목별 통합 해설

### #1 트렌드 접기 촤르륵 애니 — 완료
`.tgroup`(details) 최상위 단위(AI요약·TOP10·플랫폼 그룹)에 `::details-content{block-size:0→auto·.32s}` + `interpolate-size:allow-keywords`(서브트리 한정으로 전역 부작용 차단) + `content-visibility .32s allow-discrete`로 부드러운 슬라이드. CSS-only 프로그레시브 인핸스먼트라 미지원 브라우저는 종전 스냅으로 회귀0. reduced-motion에서는 `transition:none`. **"필터"라는 요구 문언은 트렌드에 해당 UI가 없어 접기 섹션 `.tgroup`으로 재해석**(피드탭 `.filterpop`은 별개 축·하위 `.trend-sec`엔 미적용).

### #2 좌측 네비 싱크 — 부분(이견)
"좌측 네비"의 유일 실체 = 좌중앙 책갈피 레일 `.trail#trendRail`(좌하단은 장식 픽셀펫). 동작 2방향: (a) 레일 클릭 → 접힌 섹션이면 `el.open=true`로 펼치며(촤르륵 동반) `scrollIntoView` + IO 점등, (b) 스크롤 → `IntersectionObserver` 스크롤스파이가 현재 그룹을 `.on` 점등. **접힘(toggle 리스너) 자체는 `nm_trend_gfold` 저장만 하고 레일을 구동하지 않는다.** 요구 문언 "접을 때 → 네비 싱크"는 코드 인과와 역방향 — 네비 점등은 접힘이 아니라 스크롤 위치가 구동.

### #3 플랫폼 라벨 색 — 완료
`TR_PCOL.gg='var(--cat-eco)'`(#FFE13D 네온 골드레몬)로 구글을 코랄에서 분리(유튜브와 색 구분), 나머지(yt=intl·tk=soc·ig=cul·x=pol) 현행 유지. 260712 플레이그라운드 회신 = 기틀 갱신 승인. tpc/tlr/tsk 3개 렌더러가 동일 `TR_PCOL` 소비.

### #4 TOP 슬라이드 매거진 오버레이 — 완료
20안 중 10안(매거진 오버레이) 채택. 히어로(1위 `.tsk.mag` 16:10 와이드 커버 + 좌측 스크림 + `.mag-ov` 오버레이 제목) + 하단 3열 미니(2·3·4위 `.tsk.mini` 1:1). 순환/스와이프/6초 자동회전은 앨범 스택 로직 그대로 보존하고 레이아웃만 교체(화면밖/탭숨김/12s 인터랙션 홀드 시 정지·reduced-motion 무효).

### #5 #4 확대 — 보류(정상)
이전 패스에서 처리. 대응 코드 변경 없음 = 정상.

### #6 카드 중구난방 해소 — 완료
커버 몰입 오버레이(구 `.imm`·스크림)를 폐지하고 제목/메타를 항상 커버 아래로 통일 = 제목 위치 일관. 업로더 `.tcard-acc` 제거·`.tcard-ago`(N일전) 대체. 조회수는 `vfmt`(≥1e8→억·≥1e4→만 반올림)로 **코드가 강제** 만/억 표기(원시콤마 폐지). **커버 크롭 3/4 통일**: 검증가가 발견한 기존 버그 — `tkL.map(card)`·`stkL.map(card)`·`sreL.map(card)`가 `Array.map`의 3번째 인자(배열)를 `card(t,i,wd)`의 `wd`로 흘려 세로영상(틱톡 인기·구독 틱톡·구독 릴스)이 16:9 가로크롭으로 렌더됐다(쇼츠는 `(t,i)=>card(t,i)`라 3/4). 이게 운영자 #6 "어디는 좌우 자름" 중구난방의 원인 하나 → 세 곳을 `.map((t,i)=>card(t,i))`로 교정해 쇼츠와 동일 3/4 통일(AI영상만 의도적 16:9 유지). 렌더러는 tpc(가로 레일)·tcard(그리드) 2종 잔존 = 제목 위치는 통일·카드 형태는 이원(잔여).

### #7 AI 브리프 개선 — 완료
`sns_brief.sh` = "사장에게 보고하는 비서" 페르소나 프롬프트·핵심 수치 `*별표*` 강조·헤더/번호/이모지 금지·줄바꿈+별표만. `PVER='brief-v2-260712'`로 프롬프트 개정 시 캐시 1회 무효화. 뷰어 `.tbrief-t{line-height:1.85;white-space:pre-line}`로 줄간격·개행 보존, `*별표*`→`<b>`를 텍스트노드로만 분해(XSS 안전). **주의: 만단위 반올림은 프롬프트 지시일 뿐 코드 강제 아님** — 입력 다이제스트는 원시 콤마를 넘기므로 모델 미준수 시 원시숫자 누출 가능(#6의 코드강제 `vfmt`와 성격 다름).

### #8 예타풍 글래스 → 하단네비 + 솔기 — 부분
`.bnav` 하단 네비바에 예타풍 글래스. 회신값 대비 실측:

| 속성 | 회신값(260712 플그) | 코드 실측 | 판정 |
|---|---|---|---|
| blur | 30 | `blur(var(--blur-xl))` = 30px | 일치 |
| 불투명(배경 알파) | .41 | `rgba(0,0,0,.30)` | **불일치**(260704 "네비 30%" 준거) |
| 채도 | 1.95 | `saturate(1.9)` | 근사 |
| 테두리 | .05 | `rgba(255,255,255,.06)` | 근사 |
| 상단 광택 | .30 | `linear-gradient(rgba(255,255,255,.08),transparent 78%)` | 수동 근사 |
| 그림자 | .46 | `0 -10px 30px rgba(0,0,0,.46)` | 일치 |
| 모서리 | 11 | N/A(전폭 바 = 코너 라운드 없음) | N/A |
| 틴트 | 0 | 없음 | 일치 |
| 노이즈 | 0 | 없음 | 일치 |

값 편차는 대부분 "이 느낌으로 하되"(느낌 우선) 허용 범위. **솔기 수정 완료** — `.bnav::before` 광택을 `border-radius:inherit` 전폭 페이드로 넣어 "seam 0"으로 첨부1 "도형 주변 살짝 남은 부분" 원천 차단.

## 운영자 확인 필요 3건

1. **#2 인과 방향** — 요구 "접을 때 좌측 네비 싱크"는 코드 인과와 역방향. 구현된 것 = (a) 스크롤 위치 기반 IO 스크롤스파이 점등, (b) 레일 클릭이 섹션 펼침. 접힘(toggle) 자체는 fold 상태만 저장. "싱크·동시작동" 정의(접힘이 네비를 바꿔야 하는지, 현행 스크롤스파이로 충분한지) 확정 필요.

2. **#8 불투명도 .30 vs .41** — 260704 "네비 30% 투명"(코드 = `.30`) ↔ 260712 플레이그라운드 회신(`.41`) 충돌. 코드는 260704 준거로 `.30` 채택 중. 어느 값이 정본인지 결정 필요(채도/테두리/광택 편차는 느낌 우선으로 허용 가능).

3. **#1 "필터" 해석** — 트렌드 탭에 "필터" UI가 없어 "필터 접기"를 접기 섹션 `.tgroup`(AI요약·TOP10·플랫폼 그룹)으로 재해석. 피드탭 `.filterpop`은 별개 축이며 하위 `.trend-sec`엔 촤르륵 미적용. 이 재해석이 의도와 일치하는지, `.trend-sec`까지 확장할지 확인 필요.
