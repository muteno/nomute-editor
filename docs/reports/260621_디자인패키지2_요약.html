2카테고리별 요약 + 왜 그 상태인지 (실측 근거)

A. 수집함 카드 — 이미 라이브 + 배지만 의도적 분기

테두리 우선순위(긴급 빨강 ＞ 이슈 노랑 ＞ PICK 초록 ＞ 기본)·점등(독립)·1줄 트리아지 버튼 전부 라이브. 근거 = viewer L464~469 주석 "테두리 우선순위(디자인안02·운영자 260621)", .b-brk/.b-iss/.lit-brk/.lit-iss/.sc-pick/.sc-down.

분기 1건: 패키지 README는 배지 = "미니 글래스 칩(안1)". 그런데 repo는 L4234에서 "글래스 칩 폐지·운영자 260621" → 인라인 플랫 텍스트(긴급/이슈)로 감 (참고제안#2의 '안1 플랫 라벨' 채택). ⟹ 패키지대로 글래스칩 되살리면 회귀.

B. 메타 시간 라인 — 이미 라이브 (디자인안04 그대로)

좌측 보도시간 dlReport(≤6h 라임+글로우 / 7~11h 흰 / 12h↑ 회), 우측 스크랩 dlScrap(<30분 NEW / 30분~6h 흰 / 7h↑ 회), 시간 사다리 relAge(방금→30분 전→N시간 전→N일→N주일→N달) — 04번 명세와 1:1. 근거 = viewer L1698~1725 · L299 주석 "디자인안04".

C. 카드만 토글 — 미반영 (진짜 갭)

지금도 옛 iOS식 검정 스위치(.tk 40×23, OFF=rgba(0,0,0,.75) 죽은 검정 · L240~250). 옆 칩들은 글자칩인데 혼자 튐 → 참고제안#3이 지적한 그 문제 그대로. 변형 = A 정제스위치 / B 필터칩(참고제안 추천) / C 세그먼트.

D. 디자인 토큰 before/after — 옛 기준 (적용 부적합)

이 제안은 옛 뷰어(bg #0b0b0c, 강조 외 보조색이 JS에 매장)를 전제로 "그린톤분화·쿨중성·tokens.css 단일화"를 말한다. 그런데 repo는 이미 한참 진화: bg #0a120d(웜그린), :root 풀 토큰셋(blur 4단·on-amber·btn·arm/thumb 분리), build_design_mirror.py+check_refs 닫힌 루프까지. ⟹ before/after 수치를 그대로 넣으면 현 토큰과 충돌. 아이디어(시맨틱 분리)는 이미 repo가 실현했고, 남은 건 없음.

E. 구성도 + 공통규약 — repo가 신버전 · 단 미완 2장

zip의 구성도/base.css는 AUTO-MIRROR 이전 손-스냅샷(토큰 인라인 중복). repo는 이미 AUTO-MIRROR 자동거울 + CI 게이트. ⟹ zip 구성도로 덮으면 후퇴. 단, repo 구성도/index.html에 아직 '예정' 빈칸 2장 살아있음(L65·L68 — '부속설명 규약', '내어쓰기·정렬'). 리뷰요청.md §3가 "viewer 실측으로 채워달라"던 바로 그 칸. = "흔들리지 않게"의 미완 토대.

F. 정합성 감사 — 지적 대부분 해소

01 보고서가 든 값 충돌 4건(fs-h2 14→15·fs-h3 13.5→14·accent-2 생존·radius 4단)은 repo :root에 이미 정답값(--fs-h2:15px ··· --r:24px)으로 들어있음. blur/on-amber/btn 토큰 16개도 라이브. ⟹ 새로 고칠 것 거의 없음(미완 2장 = E와 동일 잔여).

G. 에셋·폰트·뷰어 — 신규 없음

logo·main_background = repo와 md5 동일. repo는 오히려 maskable·scrap/sns 배너·profile까지 더 풍부. 폰트는 repo가 CDN 로드(jsdelivr Pretendard + Google Orbitron)라 zip의 로컬 woff2 불필요. zip viewer는 더 작음(440KB<474KB) = 옛 스냅샷.

패키지 대부분은 이미 라이브라 "재적용"은 무의미/회귀. 아래 셋 중 하나를 골라 반영하고, 그걸 디자인시스템 SSOT(구성도+공통규약+check_refs)에 박아 다음 페이지가 자동 상속하게 만든다.

1구성도 미완 2장 채우기 + 문서 정합 추천 · "흔들림 방지" 토대

'부속설명 규약'·'내어쓰기·정렬'을 viewer 실측값으로 채워 구성도 9장을 완성 + 공통규약.md를 repo 현재와 정합. 화면 변화 0, 미래 페이지 안정성 ↑ — 사용자가 말한 "추가 페이지 만들어도 안 흔들리게"에 정확히 부합. 저위험.
