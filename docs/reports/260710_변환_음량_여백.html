장단점 · 리스크 (정직)

  • 장: 업로드 그대로 한 번에 — 여백은 무손실(안 자름), 음량은 0dB 절대 안 넘음(TP 리미트). 조용한 영상은 크게, 과한 영상은 살짝 내려 영상끼리 체감 음량 동일.
  • 단·리스크: ① 음량 통일 시 오디오 aac 재인코딩 1세대 추가(160k→192k — 음성 콘텐츠 비가청 수준) ② 측정 실패 시 단일패스 폴백 = 근사(1패스) 표기(여전히 정규화·정확도만 소폭↓) ③ 여백 모드는 영상 위치 중앙 고정(위/아래 치우침은 후속) ④ 스테레오 음악 소스는 좌우 통일로 공간감이 줄 수 있음(뉴스 인터뷰엔 무해·의도된 동작).
  • 효율: 음량 후처리 = 비디오 무재인코딩(copy)이라 몇 초~수십 초. 러너 예산·기존 캡(일반 300초·보간 120초) 무변.

같이 확정된 방향 (평의회 10인 · 운영자 버튼 확답)

2단계 통합: ① 지금 = 이 배선(변환 탭) ② 다음 = 새 편집기 탭 신설(자막+변환 통합 · 업로드 1번·1잡) — 메뉴 골격은 레시피+카테고리 vs 레시피+처리 스택 두 안을 만지는 플레이그라운드 시안으로 먼저 보고 결정. 자막 생성기 탭 토글은 편집기로 미룸(중복 UI 방지). 기존 탭은 병존 → 검증 후 정리. 트래킹·영상 프롬프트는 별도 유지.
