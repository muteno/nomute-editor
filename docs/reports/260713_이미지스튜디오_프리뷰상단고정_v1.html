4. 아이데이션 (후속 제안)

  • 비율/모자이크 프리뷰(#iePrev)도 동일 상단고정 — 지금은 크롭 위치를 풀사이즈로 봐야 해서 제외했지만, 원하면 같은 sticky+캡을 붙여 전 모드 일관 가능.
  • 고정 시 글래스 필(.stuck) — 영상 스튜디오는 스크롤이 시작되면 헤더에 글래스 배경을 입힌다. 이미지 프리뷰는 헤더가 없어 생략했지만, 원하면 sentinel+글래스로 "떠 있는 느낌"을 더 줄 수 있음.
  • 높이 캡 미세조정 — 현재 46vh(≤340px). 더 작게(예: 38vh) 원하면 한 값만 바꾸면 됨(운영자 취향 = 플레이그라운드로 튜닝 가능).
재현 한계 각주 — 위 목업은 실제 thumb.html이 아니라 동일 토큰(--bg #0b0d0c·--accent #00EED2 등)으로 재현한 동작 시연용 스키매틱이다(미리보기 박스 = 실제론 첨부 이미지+PIL 텍스트 합성 근사). 실제 반영은 구조 검증(#cpPrev가 <div class="panel"> 직후로 이주·sticky CSS 존재) + check_refs.py RC 0 + 영상 스튜디오와 동일 iframe 스크롤 구조로 확인했다. 라이브 렌더는 Cloudflare Access 게이트 뒤라 세션 curl 불가 = 머지 후 운영자 눈 확인 권장.
확인 포인트 — 이미지 스튜디오(+ → 이미지) 진입 → 사진 첨부·텍스트 입력 → 미리보기가 맨 위에 작게 고정돼 스크롤해도 따라오는지.
