최하단 배경 = 무채색 물결 셰이더 + 부팅 인트로

260705 · viewer/index.html · manifest.json · PR #1666 · §검증 10인 평의회 통과
무채색 회색 물결(명암 0.10) — 실제론 천천히 흐름

무엇을 바꿨나

최하단 배경(.bgfx)에 깔려 있던 정적 실크 이미지(feed_bg.png)를 채도 0 회색 물결이 천천히 흐르는 WebGL 셰이더로 교체했어. 앱 켤 때 부팅화면(웹)에도 같은 물결 + 프로필 아이콘·앱이름을 얹었고, 물결은 부팅부터 앱까지 끊김 없이 연속돼. paper-design/shaders의 mesh-gradient 원본을 그대로 이식하고 색만 우리 --bg 토큰으로 무채색화한 거야.

전 / 후

항목전후
피드 배경feed_bg.png 다크 실크 (정적)회색 물결 셰이더 (은은히 흐름) · feed_bg = 폴백 강등
부팅화면검정 breathing (crefDim)물결 + 프로필 아이콘 + "노뮤트 뉴스 큐레이션"
색—--bg에 흰색 알파 tint (무채색·raw hex 0)
OS splash 배경#0a120d#121212 (물결 톤 이음매)

효율성 · 성능

  • 30fps 프레임 캡 — 앰비언트 배경이라 60/120fps 불필요, 무손실로 GPU/배터리 절반 절감
  • 정지 조건 3중 — 백그라운드(화면 꺼짐)·전체화면 모달(dialog[open]) 가림·reduced-motion 시 렌더 정지
  • 정적 셰이더 아님이지만 가벼움 — antialias 끔, 색은 1회 계산, 프레임당 힙 할당 0
  • OS splash(안드 PWA)는 셰이더 불가(OS 제약) → 배경색만 물결 톤 정렬

10인 평의회 발견 · 수정

회귀 수정 부팅 후 body.booted>*{opacity:1}이 푸터 딤(.foot .55)·알림 pill 숨김(.nm-toast 0)을 덮어써서 고스트 pill이 하단에 눌러붙는 회귀 → :where() 특이도 0으로 근본 해결(headless로 foot=.55 확인)
견고성 컨텍스트 로스트(모바일 GPU 리셋) 핸들러 — 즉시 feed_bg 폴백 + 복원 시 재빌드
성능 30fps 캡 · 모달 가림 정지 · antialias off
전환 스플래시 페이드아웃 delay .12s — 로고 고스트 겹침 완화
정합 border-radius→var(--r-modal) · resize 치수가드 · 딥링크 ::backdrop 억제

품질저하 리스크

  • 주의 부팅게이트 규칙 변경 — 앞으로 body 직속 자식을 새로 추가하면 #wavebg/.bgfx/#bootsplash 화이트리스트를 확인해야 함(누락 시 부팅 중 노출 어긋남). 작업이력에 명시.
  • 주의 mediump 정밀도 — 저사양 폰 장시간 포그라운드 시 물결이 미세하게 끊길 수 있음(highp는 전력 트레이드오프라 유지)
  • 주의 저사양 자동감지 게이트는 없음(WebGL 하드실패만 폴백). OS splash(로고)↔부팅 스플래시(프로필) 이미지가 달라 런치 시 미세 점프. manifest 변경은 iOS 홈화면 재설치해야 반영.
  • 영구 블랙 없음 — 4중 부팅 안전망 유지, headless 부팅 전/후 전환 PASS.
