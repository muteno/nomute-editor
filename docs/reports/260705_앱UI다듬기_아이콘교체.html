1. 전/후 비교 (9건)

#항목전후

2FIX프로필 메뉴 로고헤더에 노뮤트 워드마크 로고로고 삭제 — 새로고침·X 버튼만

4FIX검색·새로고침 픽토터쿼이즈(강조색)네온 골드레몬 --accent-4 #FFE13D — 단일 룰이라 모든 메뉴 공통
5FIX네비 하단 빈 공간라벨이 바 위쪽으로 치우침(위 10.6 / 아래 19.6px) = 하단이 비어 보임라벨 세로중앙(위 15.6 / 아래 14.6px) — 바 높이·＋버튼·라디얼 기하는 그대로
6FIX구 로딩화면 먼저 뜸OS 스플래시가 manifest 아이콘(6/22 구본·초록 배경)으로 그려짐 → 구 화면 먼저 노출아이콘 5종 전부 신프로필로 재생성 + 캐시버스트 + manifest no-cache
7FIXHome·SNS 링크프로필 메뉴에 Home + SNS 아코디언(4링크)삭제 — 메시지·발행본·설정만 남음

9확인앱 제목정상 — <title>·manifest name 모두 "뉴스 큐레이션 : News Curation". 홈화면 아이콘 밑 짧은 라벨만 short_name "노뮤트" 유지(원하면 변경 가능)

3. 장단점 · 효율성

  • 장점: 번쩍임 원인(배경 하드컷)을 제거해 열림·닫힘이 대칭 페이드 — 수정 중 타이핑 번쩍 차단 효과는 기존 그대로 유지. 메뉴 슬림화로 프로필 팝업이 한 눈에.
  • 효율성: 픽토 색·네비 정렬은 각 한 줄 수정(기하 연쇄 0) · SNS 아코디언 JS 제거로 코드 소폭 감량 · 아이콘은 기존 파일 교체라 참조 변경 0.
  • 단점/트레이드오프: 없음에 가까움 — 기능 제거는 운영자 지시분(로고·Home·SNS)뿐.

4. 품질저하 리스크

  • 잔존설치된 PWA의 OS 스플래시: 안드로이드가 설치 시점 아이콘을 캐시 — 크롬이 manifest를 재체크(보통 1~3일)하거나 홈화면에서 지우고 다시 추가하면 즉시 신본. 코드 쪽은 완료.
  • 별건PIN·전광판 등 DIV 앵커 입력팝업의 타이핑 재샘플은 기존 문서화된 후속(이번 건과 무관·변화 없음).
  • 번쩍 픽스는 CSS만 변경(JS 0) — 열림 애니·뒤로가기·최소화 흐름 전부 불변, Playwright로 열림 0→1→닫힘 0 전이 실측.

검증: check_refs 전 게이트 rc=0 · 뷰어 JS 구문 5파일 OK · 콘솔 pageerror 0(로컬 404는 Pages Functions 부재 = 환경 한계) · 머지 후 origin/main read-back 예정.
