결과: 구현 완료·머지 안전 — 기존 뷰어 회귀 0, 10인 평의회 지적 전건 수정, 오프라인 6시나리오 실측 통과.

단 지금은 스캐폴드(잠자는 상태) — 아래 §셋업 5단계(R2 비공개 버킷)를 네가 해줘야 대화가 켜진다. 그 전까지 yeta 메뉴는 열리고 캐릭터 목록·안내만 보인다.

1. 뭘 만들었나 (전/후)

전(어제)후(지금)
진입없음프로필 메뉴 Home 아래 yeta → 캐릭터 선택 → 채팅(폰 뒤로가기 지원)
캐릭터—3종: 데스크(냉철 편집장)·카피(도발 카피라이터)·무디(들어주는 리스너) — 각자 말투 예시 few-shot 내장
대화—보내면 40초~2분 뒤 답장(낙관 버블+"입력 중···" 점모션+10초 폴+답장 웹푸시 딥링크)
기억—답장에 얹는 관계노트(추가 비용 0)로 장기기억 — 100턴 가도 프롬프트 길이 일정
저장—R2 비공개 버킷 전용 — 대화는 레포에 1바이트도 안 감(공개 박제 차단·커밋 스팸 0)

2. 장단점·효율·품질 리스크 (5요소)

  • 장점: 구독 무료(claude -p) · 세션급 opus 4.8 대화 · 기존 컴포넌트 전계승(재설계 0) · 뉴스 파이프라인 완전 격리.
  • 단점(정직): 실시간 아님(턴당 40초~2분 — 인증 구조상 영구 제약) · 셋업 전엔 안 켜짐.
  • 효율: 턴당 쿼터 소비 ~$0.04 상당(effort 무·도구 0) → D4 무제한이어도 하루 100턴 = 본업 대비 오차범위. + 폴오버 최하위 구현: 활성 계정이 주간한도면 챗만 멈추고(서브계정 안 훔침) 뉴스 본업은 정상.
  • 품질 리스크: 말투 붕괴(→ few-shot 상주+관계노트로 방어) · 활성 계정 한도 시 챗 정지 = 의도 동작("고장" 아님).
  • 롤백: Pages 바인딩 제거 또는 pmenu 버튼 한 줄 제거 = 즉시 OFF. 뉴스 경로와 코드 공유 없음.

4. ⚠️ 셋업 — 네가 할 5단계 (이거 해야 켜짐 · 순서 중요)

  1. R2 토큰 스코프 확인: Cloudflare 대시보드 → R2 → Manage R2 API Tokens → 기존 토큰이 Admin Read & Write(계정 전체)인지 확인. "특정 버킷만"이면 새 버킷을 스코프에 추가하거나 계정 스코프로 재발급(그럼 GH Secrets의 R2 키 2개 갱신).
  2. 비공개 버킷 생성: R2 → Create bucket → 이름 예 nomute-yeta → 생성 후 Settings에서 Public access가 OFF인지 꼭 확인(켜지면 대화가 공개로 샌다 — 썸네일 공개 버킷과 절대 혼동 금지).
  3. GitHub Secret: 레포 Settings → Secrets → Actions → New → 이름 YETA_R2_BUCKET, 값 = 버킷명.
  4. Pages 바인딩(마지막에): Workers & Pages → 노뮤트 프로젝트 → Settings → Bindings → Add → R2 bucket → 변수명 YETA_R2(정확히) → 그 버킷 선택 → 저장(다음 배포부터 적용).
  5. 카나리아: 뷰어 → yeta → 데스크에게 메시지 1건 → 1~2분 내 답장 확인(Actions의 yeta-chat 런 초록 확인). 품질 괜찮으면 나한테 "yeta 세이프 승격"이라고 한마디 — YETA_SAFE 켜서 호출당 쿼터를 더 줄인다.

순서를 지키는 이유: 시크릿 없이 바인딩만 먼저 하면 뷰어가 정상처럼 보이는데 답장이 영영 안 오는 반쪽 상태가 된다(평의회⑧).
