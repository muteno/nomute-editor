장단점 · 리스크 (정직)

장점
  • API 키·과금·LLM 0 — 기존 30분 크론에 얹혀 러너 분만 소모
  • 기존 레인 회귀 0(평의회 실측) · 한 스크롤·접기 기억 그대로
  • 실패해도 안 죽음: 계정·소스 단위 fail-soft + 직전분 보존 + wall-clock 예산(240s)
  • 계정 목록 = 뷰어에서 관리(플랫폼당 15 상한·형식 검증 3중)
리스크·한계
  • 비공식 경로 — X·인스타는 언제든 막힐 수 있음(막히면 그 섹션만 직전분 유지·화면 안 깨짐)
  • 인스타 = 차단 리스크 최고(6s 간격·429 시 잔여 중단 보수 운용)
  • X 정렬 = 좋아요(댓글·RT·조회수는 데이터로 동봉 — 조회수는 응답에 거의 없음)
  • 스레드 = 후순위(로그인 없는 조회가 doc_id 회전으로 불안정 — 레퍼런스 도구도 폴백 처리)
  • 서명 커버 URL churn으로 sns_trends.json 커밋 diff 비대 — 눈에 띄면 R2 재호스팅 후속

품질 게이트
