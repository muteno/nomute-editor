요약 셸 — 요약 요청 = 도구 모달 탭 2구분

260712 2차 · 운영자 "이미지 부분하고 동일한 UIUX · 기본 = 요약 요청(뉴스) / 2번째 = 미디어 요약" · 헤드리스 실클릭 18항목 전부 PASS

전 → 후 (이미지 — 후 2장은 실물 스크린샷)

전 — 요약 요청 = 탭 없는 단독 모달(askdlg) · 자료화는 영상 스튜디오 5탭째에 별거
후(실측) — 도구 셸 {헤더·−·✕} + 탭 2개, 기본 = 요약 요청(기존 폼 그대로 이식)
후(실측) — 미디어 요약 탭 = 유튜브 자료화(nb) 폼. 같은 셸 안에서 스왑

요지·장단점·리스크

  • 메커니즘: 탭1은 iframe이 아니라 기존 요약 폼(askwrap)을 셸 안으로 이식(reparent) — 전송·첨부·AI썸네일·실패 재시도 로직을 한 줄도 복제 안 함(드리프트 0). 탭 전환·닫기·재오픈에도 입력값 보존 실측.
  • 진입 단일화: 자료화(미디어 요약) 진입 = 요약 셸로 이동. 영상 스튜디오는 4탭 복귀(복원 = CAP_TABS 1행).
  • 장점 — 라디얼 3버튼(요약·이미지·영상)이 전부 동일 셸 UIUX로 통일 · 실패 재시도/프리필 등 모든 요약 진입로가 자동으로 새 UIUX.
  • 리스크(정직) — 미디어 요약 탭을 떠났다 재클릭하면 nb 폼은 리로드(영상 탭들과 동일 관례 · 진행 중 자료화는 pend 재개가 복원) · 레거시 askdlg 경로는 dormant로 보존(호출 불가 실측).
