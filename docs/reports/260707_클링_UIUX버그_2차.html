260707 · 아이데이션 5인(오퍼스) + 검색 + 페이블 최종 검수 · PR #1830

요지

핵심이제 산출물이 "내가 네 요청을 이렇게 읽었다"(의도 에코)를 첫 줄에 보여주고, 복사한 샷은 회색 표식으로 영속 추적되며, 모달을 닫거나 새로고침해도 결과·입력이 살아있다. 문서 절반이 깨지던 렌더 버그(불균형 펜스)와 조용히 레퍼런스가 빠지던 버그, 타임아웃 즉사 버그도 봉합.

① 전 / 후 비교

항목전후
의도 확인산출물을 다 읽기 전까지 오독 여부를 모름⚙️ 첫 줄 의도 에코 — 의도: {요지} → {해석} 한 줄로 즉시 감지. 유형별 각색 강도(상세 묘사는 운영자 요소 보존·시리즈 1편·첨부 불가 고지)·앵글 기본값(화풍치환)·리터럴 보존·추정 구조화("바꾸려면 이 문구")까지 5종
멀티샷 복붙 왕복복사 체크가 1.5초 뒤 사라져 어느 샷을 붙였는지 기억에 의존복사한 블록 = 회색(mut) 영속 표식(.dl-done 전 앱 기틀 계승 · 재진입해도 생존 · 새로 생성하면 리셋)
결과 생존모달 닫기·새로고침 = 결과 증발(1~3분 재생성)재열람 — 마지막 결과를 24시간 안에 재진입 시 자동 복원("지난 결과 · HH:MM" 라벨)
입력 생존모달 닫으면 쓰던 장면 텍스트 증발드래프트 자동 저장·복원(+화면 이탈 순간 즉시 플러시)
렌더 강건성모델이 닫는 펜스 하나 빠뜨리면 문서 후반 전체가 코드박스로 붕괴불균형 펜스 = 텍스트 강등(하네스 실측: 뒷섹션 정상 렌더)
이중 발사생성 중 재클릭 = 즉시 새 워크플로(경고 문구만)4초 재확인 암 — 1탭 = "생성 중 — 다시 누르면 재발사", 2탭 = 진짜 재발사(오타 수정 길 유지)
타임아웃 회복claude 900s 초과 = 즉시 하드 실패(재시도 0)rc=124 → 계정 강제전환 1회 재시도(ask.sh 패턴 이식)
레퍼런스 생성펜스 태그에 잔여물 있으면 조용히 생략(무경고 no-op)정규식 관용화 — 뷰어 렌더와 동일 관용도

② 장점

  • 신규 화면 요소 0 — 전부 기존 요소의 동작 개선(배치 승인 불요 티어)·하우스 선례 계승(.dl-done·REVISING·armCancelThumb·execCommand 폴백).
  • UX 리서치 정합 — 헤드리스 원샷에서는 사전 프롬프트 개선 루프 대신 출력측 에코백+추정 확정이 정답(Prompt Rewrite 패턴의 의도 보존·가시화 원칙 — uxtigers·shapeof.ai 수렴).
  • 검수 실효 — 페이블 diff 정독에서 결함 1건 추가 발견(재열람 복원 시 복사 표식 리셋)·즉시 봉합. 하네스가 회귀 방지 기준선.

③ 단점 · 한계

  • iOS Safari 붙여넣기 버튼(readText)은 폴백 불가(웹 한계) — 복사 쪽만 폴백 확보.
  • 재열람은 마지막 1건만(히스토리 패널은 신규 요소 = 승인 대기).
  • 의도 에코·리터럴 보존은 프롬프트 지시(소프트) — 모델 준수 확률을 골격+금지 이중으로 높였지만 0은 아님.

④ 효율성

  • 재생성 낭비 제거: 결과 증발(1~3분·ref 최대 15분 재실행)과 입력 재타이핑이 사라짐.
  • 오독 감지가 산출물 첫 줄로 당겨져 리롤 판단이 빨라짐 + "바꾸려면 이 문구" = 재입력 비용 최소화.
  • 타임아웃 계정 전환으로 헛 실패(운영자 재발사) 감소.
