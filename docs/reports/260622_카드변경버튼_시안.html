카드 변경 — 버튼 · 체크박스 시안

디자인 시스템(.mkbtn 글래스 고스트 + 게이지→체크 + ↻ 재생성 + 앰버 재확인) 계승 · 실제 CSS/모션 그대로 · 탭해서 동작 확인

1) 무료 / 유료 토글 — 여기를 바꾸면 아래 버튼들이 같이 반응

유료 = ⓐ 체크박스 ON 또는 ⓑ 이미지 수정 희망 입력. 둘 중 하나라도면 제미나이 재생성(유료).

텍스트
현재: 무료(텍스트만) — 변경 = 게이지 → ✓ 바로 완료

시안 A — 라벨 유지 + ↻ 아이콘(추천)

버튼은 늘 “변경”. 유료면 라벨 앞에 ↻만 붙어 “이건 재생성(유료)”를 알림. 색은 초록 유지(누르기 전엔 평온).

동작

무료: 탭 → 게이지 채움 → ✓.
유료: 탭 → 앰버 “재확인” → 한 번 더 탭 → 게이지 → ✓.

유료일 때 한 번 탭 → 재확인 / 두 번째 탭 → 발사

시안 B — 유료면 라벨도 “재생성”으로

유료 조건이면 라벨이 “변경”→“재생성 ↻”로 바뀜(가장 명시적). 무료면 “변경”.

동작

유료 라벨 = 재생성 + ↻. 탭 → 재확인 → 탭 → 게이지 → ✓.

유료일 때 라벨이 바뀌는지 확인

시안 C — 유료면 버튼 전체가 앰버 톤(상시)

유료 조건이 켜진 순간부터 버튼이 앰버 글래스로(누르기 전에도 “돈 든다”를 색으로 경고). 누르면 재확인 → 발사.

동작

유료 = 버튼 앰버+↻ 상시. 탭 → 진한 앰버 “재확인” → 탭 → 게이지 → ✓. 무료 = 초록.

색만으로 유료를 먼저 경고하는 안

2) 체크박스 통일 — 메뉴1(수집함 확인) 스타일

현재 = OS 기본 네모 체크. 제안 = 수집함 확인 토글의 −→✓ 드로우(초록) 박스 계승. ↑ 위 토글 박스가 그 스타일이야. 이게 “메뉴1 체크박스” 맞아?

box 안에서 ✓가 그려지고, 체크 시 라벨도 초록으로. (디자인 시스템 ACK_MORPH 계승)
