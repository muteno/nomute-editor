이미지 제작 — 기기 간 공유·완료 알림 수리

260711 · 운영자 지시: 미리보기 기기 간 공유 + "배포가 끝나면 알림"(매우중요 기틀) · 검증 생략 지시로 평의회 미소환(자가 실측만)

무엇이 문제였나 (쉬운 설명)

  • 알림이 결과보다 먼저 왔다 — 제작 완료 푸시가 커밋 직후 즉시 발사되는데, 결과 목록(thumb-hist.json)은 Pages 배포(30~90초 뒤)에야 라이브. 그래서 알림을 탭하면 제작 메뉴만 열리고 이미지가 없는 "엥?"이 생겼다. 재로그인이 끼면 딥링크(#done)가 해시라 유실돼 그냥 맨 화면.
  • 기기 간 공유가 앱마다 달랐다 — 썸네일(/1~/4)만 전 기기 이력에 병합되고, 카드뉴스(/5)는 미구현·리사이즈(/7)는 '세션만'이라 다른 기기에서 안 보임. 맨 위 '결과' 미리보기도 그 기기 로컬 전용.

전 → 후

항목전후
완료 알림 시점커밋 직후 즉시(배포 전) → 탭해도 결과 없음 가능배포 게이트 — 커밋의 Cloudflare Pages 체크런 success 확인 후 발송 = 탭 시점엔 결과가 이미 라이브
알림 딥링크#done=<id> 해시 — Access 재로그인에 유실?done=<id> 쿼리(생존) · 구 해시도 호환 파싱
기기 간 이력(이전 제작)/1~/4만+ 카드뉴스(/5 · comp_out 스캔) + 리사이즈(/7 · resize.json 병합 — 낮 '세션만' 번복 확정)
맨 위 '결과' 미리보기(부팅)이 기기 마지막 제작만로컬·전 기기 중 최신 제작(접힌 채 · 12h·기록지우기 컷 준수 · 동률=로컬 승)
기틀 기록—CLAUDE.md §9-3 §완료알림 신설(deploy-then-notify + 쿼리 딥링크 = 신규 제작 앱도 계승)
렌더→R2 업로드→_meta.json 커밋→배포 게이트(체크런 폴링)→완료 푸시 ?done=id→탭 = 이미지 즉시 복원·랜딩

장단점 · 리스크

축내용
품질"알림 탭 = 이미지 바로"가 구조로 보장(재시도 사다리는 폴백 안전망으로 유지). 미리보기·이력이 전 앱·전 기기 일관.
트레이드오프알림 도착이 배포 완료만큼 늦어짐(평시 +30~90초) — 운영자가 의도적으로 선택한 방향("배포가 끝나면 알림").
잔여 리스크① 게이트는 Pages GitHub 연동 체크런에 의존 — 미관측 시 180초 폴백 통과·상한 8분(이땐 클라 사다리가 커버) ② 카드뉴스는 이력 공유만, 수정 연필(src 스냅샷)은 후속 PR ③ 리사이즈 git-폴백 파일명(R2 장애 시)은 id 미추출=그룹 제외·표시만.
효율성러너 점유 +약 0.5~1.5분/제작(공개 레포 = 무료 분) · API 폴링 15초 간격(경량) · 뷰어 추가 fetch 0(기존 thumb-hist 파이프 재사용).
