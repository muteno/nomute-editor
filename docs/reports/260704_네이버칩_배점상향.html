① 소스칩 — 긴 키워드 → 픽토 + 짧은 글자

전 · 후

네이버카페 네이버블로그 지식iN → 카페 블로그 지식iN

실제 소스칩 줄 (커뮤니티와 섞였을 때)

클리앙 카페 블로그 더쿠 지식iN

카페=☕컵(정면) · 블로그=✒️펜 · 지식iN=텍스트 그대로. 네이버끼리 색 동일 통일(옛 짙게/옅게 무게변주·지식iN 점선 폐지) — 초록 틴트가 "네이버"를 신호, 카페↔블로그는 픽토로, 지식iN은 이름으로 구별. 픽토는 모노 라인 SVG(기존 아이콘 세트 톤 계승·이모지 아님).

② 배점 — 네이버 수집 가산점 상향

개당 가점 = min(1, NORM_REF / 그 소스 총수집) · NORM_REF 20 → 75

소스 총수집20건50건75건100건200건400건
전 (NORM_REF 20)100%40%26.7%20%10%5%
후 (NORM_REF 75)100%100%100%75%37.5%18.8%

네가 물어본 수치 — 지금은 100수집 → 20%까지 깎여 있었어(짐작한 50이 아니라 20). 목표대로 100수집 → 75%로 완화했어. 커뮤(대개 <20수집 = 이미 100%)엔 거의 무영향, 뽐뿌(~60)만 부분 상향. 네이버카페(수백 수집)는 75/N로 여전히 물량 방어(예 400수집→18.8%).

③ 장단점 · 효율성 · 품질저하 리스크

장점

  • 칩 라벨 짧아짐 → 소스줄 안 밀림, 한눈에
  • 픽토로 카페/블로그 직관 구별(색은 통일)
  • 네이버 교차 클러스터 저평가 해소(20→75%)
  • SVG 픽토 = 디자인 계약 부합(이모지 금지)

리스크 · 대비

  • 네이버 개당 가점↑ → 물량 재지배 우려
  • ↳ 방어 유지: platform_count 1차정렬(네이버 단독=1→하단) + POSTS_CAP 6(로그상한) 불변
  • 뽐뿌(~60) 가점 상향 = 커뮤 순위 미세 변동
  • ↳ 되돌리기 쉬움: env SOCIAL_NORM_REF 한 값

효율성 · 검증

  • 토큰/과금 0 — 순수 렌더(칩) + 파이썬 스코어(배점). LLM 무관.

  • 검증: check_refs 전체 게이트 통과(rc=0) · social_burst --sample 회귀 없음 · 정규화 수치 결정론적 계산으로 확인.
