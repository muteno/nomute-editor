한 줄 요약 — 사진·썸네일·카드 위에 뜨는 기능 버튼(수정·다운로드·삭제·투표·재생성)의 받침판을 반투명 검정 20%에서 각 화면의 배경 토큰 var(--bg) 완전 불투명으로 전면 통일했다. 좌상단 「포스트 #1」 라벨 배지처럼 솔리드하게 깔려, 밝은 사진 위에서도 버튼이 또렷하게 읽힌다.

위 데모 = 실제 CSS 값 재현(전: rgba(0,0,0,.20)·글래스 rgba(8,15,11,.3)+blur / 후: #121212·#0b0d0c = 각 뷰어 --bg). 실물은 머지 후 라이브에서 그대로.

기틀 감사 결과 (요청: "지금 다 그러고 있나 확인")

  • 기틀 룰(§디자인 n `검정 20%`) 준수 자체는 양호 — 다운로드류(.dlbtn 계열) 8곳 전부 규정값이었음.
  • 발견한 드리프트 1건: 같은 이미지 위 한 줄에 반투명 글래스(.sbtn = 투표·수정·재생성)와 검정 20%(.dlbtn)가 혼재 — 룰 n)의 셀렉터 목록에 .sbtn 계열이 빠져 있던 것. 이번에 전부 룰로 편입·통일.

장점어떤 사진 위에서도 버튼 가시성 보장(운영자 우선순위) · 한 줄 혼재(글래스↔검정) 해소 = 일관성 · 문서(기틀)-코드 정합 복구 · blur 5곳 제거 = 스크롤/리페인트 비용 감소 · hover 단계 룰 6줄 순감
트레이드오프글래스 특유의 '비침 미감' 상실(운영자가 명시적으로 포기 선택) · 데스크톱 hover 배경 미세반응 제거(폰 주사용이라 영향 0 — 누름 scale·아이콘 모션은 그대로) · 어두운 사진에선 플레이트 윤곽이 덜 도드라짐(배지와 동일 거동 · 가시성은 아이콘 색이 담당)
품질저하 리스크낮음 — 순수 CSS 값 변경(로직·상태·파이프라인 0) · 상태색(.voted 투표됨)·전역 .dlbtn 표준(초록 10%)·받음 표식(.dl-done mut) 전부 불변 실측 · 신규 raw 0(라이브 전부 var(--bg) 토큰 · 발행본 템플릿만 +1 hex = baseline 사유 기입)
