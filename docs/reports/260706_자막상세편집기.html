자막 상세 편집기 구축 보고 — Vrew식 조각·단어칩 편집 + SRT

운영자 지시: 스크린샷 디테일 최대 구현 · 기존 창 안 · 질문 없이 완성 · 초등학생 사용성 게이트 · 1분 다국어 각각 실테스트

1. 뭐가 생겼나 (쉬운 설명)

이제 + → 영상 → 자막 생성기에서 자막을 만들면, 결과가 두 탭으로 나온다.

  • 개요 탭(기본) — 지금까지 보던 다듬어진 자막(표 + 복사 블록) 그대로. 1픽셀도 안 바뀜.
  • 상세 탭(신설) — 자막을 조각 카드로 펼쳐서: 단어를 탭 한 번에 빼고, 두 번 탭해 고치고, 조각을 합치고 지우고, 찾아 바꾸고, 완성본을 SRT 파일(캡컷·프리미어에 바로 들어감)로 받는다.
  • 미리보기 — 폰에서 고른 영상이 위에 뜨고, 재생하면 지금 나오는 자막이 밑에 크게 따라온다. 지운 조각은 재생도 건너뛴다(컷 스킵). 자막 크기 슬라이더·재생 속도도 있다.
  • 자동 복원 — 탭을 갔다 와도, 창을 닫았다 열어도 결과와 편집한 내용이 3일간 그대로 살아 있다.

단어 타이밍탭=빼기두 번 탭=고치기스페이스=쪼개기합치기·삭제찾아 바꾸기전체 선택SRT 다운로드컷 스킵자막 크기복원 72h키보드 조작

5. 파이프라인 변화 (additive · 기틀 무접촉)

  • ly_stt.py: 두 번째 인자를 주면 viewer/ly_out/<id>/segments.json을 함께 기록(word 타임스탬프·KST). 인자 없으면 종전과 완전 동일.
  • ly-make.yml: STT 두 스텝에 경로 전달만 추가. 커밋 스텝 무변경 — 의역이 실패해도 전사가 커밋돼 뷰어가 복구 표시(이번 카나리아 ja/zh에서 실증).
  • 무접촉: subs.md 골격 · prompts/ly-make.md · apps/ly 지침 · lymake.sh · 계정 폴오버 · index/k/comp 뷰어.
  • 위험 수용(정직 기록): word 타임스탬프를 켜면 Whisper가 조각 경계를 단어 정렬로 다시 잡는다 → 의역 입력 트랜스크립트의 타임코드가 종전과 미세하게 달라질 수 있음. 카나리아 3개 언어에서 의역 품질 정상 실측 + 경계가 오히려 정밀해져 수용 확정(2회 전사 = 비용 2배라 비채택). 상세 = ly_stt.py 헤더 + 작업이력.

7. 장단점 · 효율성 · 품질저하 리스크

장점

  • 편집이 폰 안에서 끝남(빼기·고치기·합치기·SRT) — 밖으로 복사해 고치던 왕복 제거.
  • 전부 계승 부품(.ed-chip 엔진·.dlbtn·.dl-cbox·.tooltab·nm-svg) = 신규 raw 색 0·신규 hex 0·플랫폼 일관.
  • 편집은 클라 로컬 — 서버 되쓰기 0 = 의역 산출 골격·복사영역 불변(기틀 보호 정합) · 추가 과금 0(구독 쿼터 무증).
  • 의역 실패 시에도 전사 생존(복구 표시) — 이번 카나리아에서 실전 증명.

단점·트레이드오프

  • 상세 탭 = 날것 받아쓰기 기반(의역 조각과 별개 축) — 그래서 기본 탭을 개요(다듬은 자막)로 둠.
  • 미리보기는 이 세션 첨부 파일만(영상 호스팅 안 함 = 저장 비용 0의 대가).
  • 편집본은 이 기기 브라우저에만 저장(기기 간 동기화 없음·최근 8개 잡만 유지).

효율성

  • 파이프라인 추가 비용 = word 타임스탬프 연산(수 분대 영상서 수십 초 수준·LLM 콜 0 증가) + segments.json 약 35~57B/단어(13분 영상 ≈ 130KB 텍스트·gzip 잘 됨).
  • 뷰어 성능: 300조각/3,000칩 기준 — 편집 시 카드 1개만 재빌드·저장 400ms 디바운스로 저가폰 프리즈 제거(평의회 6 재실측 OK).

품질저하 리스크 (수용 기록)

  • word_timestamps 상시화로 의역 입력 타임코드 미세 변동 가능 — 카나리아 3개 언어 의역 품질 정상 실측으로 수용(상세 §5).
  • CJK 실발화의 무공백 재조립·문자 단위 칩은 실사용 1회 확인 항목(합성음 한계로 카나리아 미커버 — 로직은 유닛 검증).
  • word 타임스탬프 자체가 근사치(Whisper 추론 트릭·CJK 경계 모호) — 칩 타이밍은 편집 참고용, SRT 조각 경계는 세그 단위라 안전.

8. 후속(도먼트 — 필요 시 지시)

  • 전체화면 재생 시 캡션 오버레이(현재는 영상 아래 바) · 모두 바꾸기 단발 되돌리기 · lymake error.log 상세화(실패 사유 표면화) · 장영상 word_timestamps 소요 실측 · 내용 삽입 필수 문구 아이말 전환(thumb 통일 재검토 시) · WhisperX 정밀 정렬(단어 ±100ms — 러너 deps 추가 필요).
