어떻게 동작하나 (쉬운 설명)

  • 다운로드 버튼을 누르면 → 눌림(액티브) 효과 후 → 색이 강조색 → 흰색으로 굳어짐 = "이거 이미 받았음" 표시.
  • 버튼이 어떤 이미지인지 URL로 기억(DLED) → 목록이 다시 그려져도 받은 건 계속 흰색.
  • 전체 버튼은 저장이 다 끝난 뒤 흰색으로(진행 중엔 강조색 유지).
  • 기틀 한 곳(.dlbtn.dl-done)만 고쳐서 새 다운로드 버튼도 자동 적용 — 앞으로 색 재설계 불필요.

장단점 · 효율성 · 품질 리스크

  • 장점: 어떤 걸 이미 받았는지 한눈에 → 중복 다운로드·누락 방지. 전 앱(뉴스·썸네일) 다운로드 버튼 일관.
  • 효율성: 값은 --fg·--accent 토큰 재사용, 새 색·raw 값 0(check_refs 디자인 게이트 통과). 로직은 URL Set 한 개 + 클릭당 클래스 1개 = 성능 영향 무시가능.
  • 품질 리스크(작음): ① 개별 버튼 기본색이 흰색→강조색으로 바뀜(전 강조색 통일 요구 반영). ② 브라우저는 '다운로드 완료' 이벤트가 없어 단일 파일은 클릭 시점=완료로 간주(전체는 실제 완료 시점). ③ URL 기억은 세션 단위(새로고침 시 초기화) — 영구 저장은 안 함(과설계 방지).
