릴스 기본(통합) 모드 신설 + 헤더 배경체크 버그픽스

260624 · 대상 viewer/thumb.html · functions/api/thumb.js · 이 보고서는 일회성 스냅샷

🔎 한눈에

썸네일 제작기 릴스(/2)의 모드 버튼이 2개 → 3개로 늘었다. 새로 생긴 "기본"은 누르기만 하면 헤더와 자막을 한 번에 만들어주는 모드다.

  • 전엔 헤더(흰칸+제목)와 오버레이(자막)를 따로따로 두 번 만들어야 했음.
  • 이제 기본 모드에서 부제·제목·자막을 한 화면에 넣고 누르면 → 흰헤더 1장 + 자막(투명도60) 1장이 동시에 나옴(2장).
  • "배경" 체크하면 흰칸 없는 헤더가 한 장 더(총 3장).
  • 기존 헤더 → 2번 / 오버레이 → 3번으로 그대로 살아있음.

💡 한 줄 멘탈모델 = "릴스 기본 = 흰헤더 + 자막60 한 방에." 따로 만들던 두 작업을 한 버튼으로.

📍 1. 모드 버튼 — 전/후

전 (2개)
1번헤더 (기본)
2번오버레이

헤더와 자막을 각각 만들려면 모드 바꿔가며 두 번 작업.

후 (3개)
1번기본 ← 신설·디폴트
2번헤더
3번오버레이

기본 = 헤더+자막 동시. 헤더·오버레이는 단독으로 쓸 때.

📍 2. 기본 모드 = 어떻게 동작

구분내용
입력 (한 화면)부제(흰색) · 제목(초록) · 배경 체크 · 자막(줄바꿈=여러 줄)
자막 투명도60 고정 (오버레이의 60·30·직접입력 → 60 하나로 간소화)
이미지 삽입없음 (빠른 기본 생성용 · 이미지 합성이 필요하면 3번 오버레이로)
강조 *별표*선택 (없어도 제작됨 · 기존 오버레이는 필수)
출력 (기본)흰헤더 + 자막 OPA60 = 2장 동시
출력 (배경 체크)기본(흰칸없음) + 흰헤더 + 자막 OPA60 = 3장

구현 = 헤더 1건 + 자막 오버레이 1건을 동시 발사(경고문·저작권이 포스트+릴스 동시 발사하던 패턴 그대로). ∴ 백엔드·렌더 스크립트는 한 줄도 안 건드림 — 기존 헤더/오버레이 처리를 그대로 두 번 부른 것.

📍 3. 덤 — 헤더 "배경" 체크 버그픽스

기본 모드를 만들다가 기존 헤더 모드의 "배경" 체크가 그동안 작동 안 하던 버그를 발견해 같이 고쳤다.

전 (버그)
증상배경 체크해도 흰칸 1장만

프론트는 체크값(bothBg)을 보냈지만 thumb.js가 그 값을 백엔드로 안 넘겨서(L60 누락) → 워크플로·출력 분기가 항상 꺼져 있었음. = 영구 no-op.

후 (정상)
결과체크 시 흰칸없는 버전도 추가(2장)

thumb.js:60에 bothBg:!!p.bothBg 한 줄 추가 → 워크플로(params.get('bothBg'))·출력 분기와 1:1로 맞물림. 미체크는 동작 동일(무회귀).

📍 4. 분석 (전후·장단점·효율·리스크)

전 → 후
릴스 모드 2개 → 3개.
헤더·자막을 따로 두 번 → 기본 한 버튼으로 동시. 헤더 배경체크 정상화.
장점
· 가장 흔한 조합(흰헤더+자막60)을 한 번에.
· 기존 2·3번 모드 무회귀(그대로 단독 사용 가능).
· 백엔드·렌더 무변경 = 검증면 좁음.
단점/트레이드오프
· 기본 모드는 자막 투명도 60 고정·이미지 없음(빠른 생성에 초점 — 세밀 조정은 3번 오버레이).
· 한 번에 2~3개 잡이 떠 캐러셀에 누적(의도된 동작).
효율·안전장치
· 듀얼발사는 검증된 기존 패턴(경고문·저작권) 재사용.
· 두 잡은 각자 독립 id·R2 경로·폴링(충돌 0).
