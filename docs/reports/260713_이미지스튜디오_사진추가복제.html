2. 장점 / 단점 / 리스크

장점

  • 단일 어휘 — 사진 입구가 편집 탭과 완전 동일 문법(빈 슬롯 픽토·X 삭제) = 학습 0, 드리프트 축 1개 소멸.
  • 오조작 감소 — 구 "재탭=제거"는 미리보기 확인하려다 삭제되는 함정 → X로만 삭제(thumb 검증된 관례).
  • 구분선 의미 복원 — 밑줄이 소머리에 귀속 = 섹션 시작이 한눈에, 이중선·유령선 구조적 불가.
  • 간격 기틀 편입 — raw 12/2px → --sp-2/--sp-1 사다리(하나 고치면 전파).
  • ON/OFF 대칭 — 웹툰화·문구 두 토글이 같은 문법(라벨=의미·값=상태) = §디자인 e 어포던스 원칙.

단점·리스크

  • 슬롯 폭 증가 — 46px 고정 → 행 1/3(thumb 비율). 주문 텍스트칸 폭 감소(전폭→2/3). 완화 = thumb 검증 비율 그대로·모바일 3:1.
  • min-height 미복제 — thumb 104px 대신 wishrow stretch 동기(textarea 3줄 ≈ 유사 높이). 실기기 확인 포인트.
  • 소머리 높이 +4px — 블릿↔밑줄 2→6px(--sp-1). 토큰 정렬 우선 판단.
  • 배지 폐지 — 첨부 체크 ✓ 배지 삭제(원본에 없음). 첨부됨 신호 = 커버 이미지+X(더 강한 신호).
  • 품질저하 리스크 총평 — 로직(다운스케일·refMode·close 정리) 무접촉 = 생성 파이프 영향 0. 표면 전용 변경.
