1. PIN 잠금화면 — 6자리 자동검증

6자리를 다 넣으면 버튼 없이 자동으로 검증(더 빠르게)돼. 맞으면 바로 열림, 틀리면 흔들리며 빨강 점등 → 처음부터. 카드 배경도 없앴고 첫 로드에 입력창이 바로 포커스돼. 아래에서 직접 쳐봐 — 123456=성공 · 898900=마스터 초기화 안내 · 그 외=실패 연출.

PIN으로 잠긴 문서입니다.
✓ 확인됨 — 문서 열림
6자리 = 자동 발사 · 틀리면 흔들림+빨강 · 눈 아이콘=표시 토글

동작 흐름 (260703 갱신)

  • 6자리 입력 → 140ms 뒤 자동 검증(별도 열기 버튼 X · 더 빠르게)
  • 검증 중 = 입력칸 중립 accent 글로우 펄스(‘확인 중’) — 서버가 성공/실패 결정이라 가짜 성공 애니 없음
  • 성공 = 서버 문서 응답 → 페이지 교체 = 자동 오픈 / 실패 = 폼 재응답 → 흔들림 + 빨강 점등 + 처음부터
  • 마스터(898900) = 접속 잠금 초기화 안내 표시(accent 정보 · 흔들림 X · 데모서 쳐봐)
  • 카드 배경 제거 — 글자·입력칸만 흑배경 위에 · 첫 로드 입력창 자동 포커스(탭 어디나 재포커스)
  • 5회 누적 = 10분 잠금 · 마스킹 · 눈 토글 불변

2. 발행본 팝업 — 도형 없는 픽토그램(대기열 계승)

공유·삭제 버튼의 박스(테두리·배경) 제거 → 픽토그램만(뉴스 대기열 상태아이콘의 ‘도형 제거’ 계승). 평소엔 무채색(행마다 전경색 X), 다가갈 때만 공유=초록 / 삭제=빨강. 행 아무 데나 누르면 새 탭+링크복사(기존). 눌림 = 픽토그램 단일 모션(공유 ↗상승 · 삭제 축소)로 정리 — 버튼 눌러도 행 배경 안 번쩍.

발행본Today 2 / 누적 4
행 클릭 = 새 탭+복사 · 공유=↗ · 삭제=hover 빨강 · 잠금=자물쇠
rest 무채색 · hover/active만 의미색 · 히트영역 ≥42px

전 (Before)

  • 공유 = 초록 테두리+배경 박스 · 삭제 = 빨강 박스 → 행마다 전경색이 튐
  • 누름 = 박스 scale + 아이콘 모션 이중 → 어수선

후 (After)

  • 박스 제거 → 픽토그램만(대기열 도형제거 계승)
  • 평소 무채색(--mut), hover/active만 의미색
  • 누름 = 단일 모션(공유 ↗ / 삭제 scale .82) · 액티브 유지

효율 · 품질저하 리스크

효율 PIN 검증 왕복 1회(기존과 동일 · 클릭 1회 절약 = 버튼 삭제). 발행본은 CSS만 정리 · 신규 아이콘 0. LLM·네트워크 추가 0.

품질저하 PIN — 낙관적 unlock 애니 제거로 가짜 성공 사라짐(정직). 자동검증은 fired 가드로 중복발사 차단. 발행본 — 삭제가 평소 무채색이라 위험도 낮아 보일 수 있으나 hover/active에서 빨강 노출 + 잠금본은 PIN 확인 유지(오삭제 방지 불변).
