요지 (전/후)

전: /insta 원료 = 운영자가 인사이트 스크린샷을 직접 찍어 첨부(회차 추적 수동·기간 비교 어려움).
후: 매일 06:25 KST 자동 수집 — 프로필·계정 일지표(조회수·도달·프로필 방문·상호작용)·팔로워 시계열·인구통계·활동 시간대·최근 미디어 25건 개별 인사이트 → apps/insta/data/ 누적 → /insta 진입 즉시 최신 데이터 진단 + 전월 대비 추이 자동.

장단점

장점
  • 스크린샷 업로드 0 · 일별 시계열 자동 누적(연결 시점부터 = 빨리 켤수록 이득)
  • LLM 0콜·stdlib only(설치 0) = 과금 0·러너 캐시 불요
  • 지표 개폐 자가 적응(묶음 실패 → 낱개 폴백·dropped 기록) — Meta의 views 이관류 개폐에 전건 실패 방지
  • 시크릿 미등록 = no-op = 선머지해도 라이브 무영향(활성화 스위치 = 운영자의 시크릿 등록 행위)
단점·한계(정직)
  • 토큰 60일 — 50일 경과 warning·만료 시 빨간 런. 재발급 2분(가이드 §6) · 무개입 자동갱신은 보안 트레이드오프 2갈래 중 운영자 선택 대기
  • 본인 계정만(경쟁계정 인사이트 불가) · 인구통계는 팔로워 100+부터(Meta 제한)
  • 스토리 지표·과거 소급 이력은 제한적 — 연결 후부터 촘촘

효율성 · 품질저하 리스크

효율: 일 1런 ≈ 30여 API 콜·수 초(rate limit 200/h 대비 여유) · 커밋 = 변동 시만.
