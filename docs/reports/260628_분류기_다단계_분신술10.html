🎯 한 장 결론 — 데이터가 방향을 바꿨다

운영자의 0~5열 다단계 테이블은 맞는 추상화다. 하지만 10인 실측이 드러낸 진짜 그림은 다르다:

  • 근본 원인 재발견: gate_judge AI는 cross≥3에만 돈다 → cross-2 후보 1,883건(전체 63%)은 AI가 영영 안 닿는다. 베이비몬스터·지수가 사회로 뜬 건 "AI 판정 전 순간"이 아니라 AI 경로 자체가 없어서. 거기선 키워드가 1차이자 최종.
  • 키워드 테이블은 거의 죽은 자산: 등재 가수 70명 중 라이브에 실제 등장 8명(11%). cross≥2 게이트가 네임스페이스를 이미 극단적으로 좁혀, 등재해도 안 나타난다. 2차 게이트 실발동 3건·3차 0건.
  • 정답: 키워드를 무한 확장하지 말고, gate_judge AI의 *카테고리 판정*만 cross-2로 당겨라(빈칸·셀럽경계 표적 선별). AI는 같은 콜에서 cat을 이미 출력 중이라 한계비용 ≈0.

→ 0~5열 테이블은 "새 엔진"이 아니라 이미 있는 엔진(2차=AMBIG+MUSIC_CTX, 3차=CRIME/JUDICIAL_OVERRIDE)의 데이터 외부화일 뿐. 채울 건 구조가 아니라 데이터 + 게이트 임계 한 칸.

🔬 데이터가 말한 것 (운영자 직관 정정 2건 포함)

🟢 그대로 맞은 직관

  • 방송사 단독 위험 — KBS·MBC·JTBC 비문화 50~100%(사장 부친상·노조·청문회). 2차확인 필수 확정.
  • 도끼 = 사회 多 — 힙합 동반만 문화, CRIME 우선. 운영자 안 그대로.
  • 충돌어 2차게이트 정상작동 — 레이/슈화/하니 사회분류 정확.

🔴 데이터가 뒤집은 직관

  • 화사: '화사하다' 오탐이 문제가 아니라, 진짜 가수 화사인데 분류실패(빈칸)가 문제 → AMBIG에 추가해야(빠져있음).
  • 스포츠경향 등 전용매체: 추가 불가 — 수집기가 클러스터 대표를 종합지로 띄워 전용매체가 노출 0건. 매체규칙은 사실상 사문화 → URL 도메인 화이트리스트가 진짜 레버.
  • [Oh!쎈] 외 태그: 변형([Oh!쎈 이슈/현장])은 OSEN_RE가 이미 다 잡음. 진짜 누수는 [순간포착]·[핫피플]뿐.

영어 약어 = 지뢰밭 (분신 #2·#7 — 반드시 단어경계 분리)

re.I+substring으로 넣으면 즉시 터지는 것: IVE → 9건(survive·live·archive), NCT → sanctuary, IU → FIU, TXT→.txt, ROSE→장미, god→신, TWS→무선이어폰, EXO→exoplanet. → 영문은 한글과 다른 정규식: (?<![A-Za-z])…(?![A-Za-z]) 영문경계 + 위험약어는 2차게이트. 안전군(aespa·NMIXX·ILLIT·STAYC·ENHYPEN·ATEEZ·RIIZE·ZEROBASEONE)만 직등재.

2차 키워드 풀 — 오염 솎아냄 (분신 #3 실측)

빼야: 출연(40%)·예고(60%)·진행(65%)·경기(35%)·골(38%)·부상(57%)·중계(100%) — 정치/경제 오염.   안전: 앨범·신곡·컴백·콘서트·녹화·MC·게스트·시청률·차트·발매·무대(정치무대 오염 0)·뮤비.

✅ 권고 — 지금 / 보류 / AI / 구현

즉시 머지 (저위험·고확신)

  • 그룹 영문 안전군 직등재(단어경계): aespa·NMIXX·ILLIT·STAYC·ENHYPEN·ATEEZ·RIIZE·ZEROBASEONE·THE BOYZ·NCT·BTS 등 + 한글 누락분.
  • 예능 IP 직등재: 무한도전·런닝맨·놀면뭐하니·나는솔로·유퀴즈·흑백요리사·삼시세끼·미운우리새끼 (워크맨·복면가왕·환승만 2차).
  • 가수 추가: 이영지·한로로·타블로 + 화사를 AMBIG에 추가.
  • 태그 +2가점: [순간포착]·[핫피플]·[공식].   레이 즉시 제외.
  • 3차에 "부정선거" 1단어 추가 = 운영자 예시 정확 해결·부작용 0(ROI 최고).

보류 — 데이터가 위험

  • 방송사·소속사 단독 등재 보류(2차 강게이트 설계 후). 충돌어 저빈도(지수·청하·리사·가을)는 등재가치 ≈0.
  • 리센느 영문표기·클래식 기획사 명단 = 운영자 확인 대기.

AI에 위임 (키워드로 불가)

  • 셀럽 + 술어형(침묵·논란·입장·동참) = 맥락 판단 → gate_judge. "유재석 부정선거 침묵"의 '침묵'은 키워드 불가.
  • cross-2 cat 판정을 gate_judge로 당기기(빈칸·셀럽경계 표적). 같은 콜에서 cat 이미 출력 = 한계비용 0. 이게 가장 큰 레버.

구현 (분신 #9 — 레포에 이미 있는 패턴 재사용)

지금 py(to_candidates.py)·js(index.html) 2곳에 정규식 손복붙(주석 "동기" 7회). 72개+다단계로 가면 드리프트 수학적 확정. → 단일 SSOT lexicon/categories.tsv(0~5열) + pools.tsv(2·3차 풀) → build_lexicon.py가 py/js 투영 → check_refs.py 하드게이트. 이건 신규 발명이 아니라 build_library·build_design_mirror·check_sens_vocab 3종 가동 패턴의 재적용.

+ 가시화 무료: SSOT를 한 번 더 투영해 viewer/lexicon.html(티어별 색칠 표) = 항상 정합되는 이 문서의 라이브판.
