미디어 생성물 — 기기·플랫폼 무관 동기화 전수 진단

"이미지·영상 생성물이 하드웨어·진입 플랫폼과 무관하게 모든 사이트에서 동기화돼 보이나?" · 260723 · 오퍼스 3인 교차검증 + 자체 실측

결론: 예 — 완성 생성물은 전 기기 동기화됨 ✓

이미지·카드뉴스·영상·노래·음성·자료의 완성물은 하드웨어(폰/PC/태블릿)·브라우저·진입 URL과 무관하게 같은 목록으로 보이게 배선돼 있음. 3중 독립 감사에서 중대한 반례(하드 카운터) 0건.

단, "보이느냐 = 참"이고 "타 기기서 원본으로 다시 편집/재열람하느냐"는 일부 층에서 부분 반례(아래 5절) — 대부분 설계상 의도이거나 운영자 기인지 건.

1. 무엇을 물었고, 어떻게 확인했나

질문 = 생성한 이미지·영상이 어느 기기·어느 플랫폼에서 접속해도 똑같이 동기화돼 보이는 상태가 지금 되어 있는지 확인.

확인 방법 = 생성 → 저장 → 서버 반영 → 타 기기 표시까지 사슬 전체를 코드·워크플로 실측으로 추적 + 서로 다른 관점의 OPUS 3인 적대적 교차검증(이미지·영상 / 노래·음성·자료·배포 / 반동기화 헌터).

2. 동기화가 도는 사슬 (전 미디어 공통)

① 생성(워크플로 렌더)→ ② 메타 git 커밋·미디어 R2 공개 버킷→ ③ Cloudflare Pages 매 push 자동빌드가 build-viewer.mjs 실행→ ④ 미디어별 서버 인덱스 재생성→ ⑤ no-cache + ?t= 정적 서빙→ ⑥ 각 스튜디오가 fetch → 전 기기 갤러리

핵심 = 완성물은 브라우저(localStorage·IDB)가 아니라 서버(git·R2)에 저장되고, 서버가 만든 목록 파일을 모든 기기가 똑같이 내려받음. 그래서 하드웨어·플랫폼이 달라도 같은 목록. 완료 알림(?done=<id> 딥링크)도 기기 무관 착지.

배포 실행자 = Cloudflare Pages 대시보드 빌드커맨드가 유일(레포엔 package.json·배포 스텝 0 · 워크플로의 build-viewer 언급은 전부 주석). 강증거: news-analyze.yml:100 "Pages가 push마다 node build-viewer.mjs 직접 실행" + articles.json·thumb-hist.json이 gitignore인데 앱이 의존 = 빌드 때만 생성 = 반증 불가. 각 도구의 "배포 후 알림" 게이트가 커밋 SHA의 'Cloudflare Pages' 체크런 success를 폴링(song-make.yml:156-179)해 자동빌드를 전제·확인.

4. 겉보기 우려 → 실측 결론 (전 → 후)

우려 (질문의 전제)

"기기·플랫폼마다 생성물이 다르게/안 보일 수 있다. 브라우저에만 저장돼 폰↔PC가 어긋나는 것 아닌가?"

실측 (3인 교차검증)

완성물은 서버(git·R2)가 진실원본이라 폰·PC·태블릿·다른 브라우저·PWA 어디서 봐도 같은 목록. 브라우저 저장은 진행 중 대기·UI 설정·재편집 원본 캐시뿐 = 완성물 가시성과 무관.

5. 남은 갭 (심각도순 · 대부분 설계 의도/기인지)

갭내용성격
중 이미지 연필복원 원본완성 썸네일은 전 기기 보이나, 타 기기서 그걸 원본 사진으로 재편집하면 설정만 오고 배경은 빗금 참고본(원본 b64 = 기기 IDB 단독 · uploads/ Pages 미서빙).운영자 기인지(260713 리포트에 '별도'로 등재) · 텍스트-only 썸네일 무해
중 k 즉답 프롬프트레퍼런스 없는 k 생성(흔한 경로)은 결과가 서버 파일 0 · localStorage 단독 → 타 기기 미표시(k.html:817 "파일 경로 없음").산출 = 텍스트 프롬프트라 "이미지/영상" 명제 직접 대상 아님 · 서버 보존은 별도 결정
중 침묵 폴백(강건성)노래·음성·자료 목록은 404·파싱실패 시 조용히 빈 목록 유지(에러 무표시). Pages 자동빌드가 깨지면 전 기기서 무신호로 텅 빔 → 운영자가 "없음"과 "고장"을 구별 불가.개선 여지 = 로드 실패 시 구분 표시 배선
하 conv·track 갤러리 부재완료 푸시로 타 기기 도달은 되나, 푸시 소진 후 재열람할 서버 목록 없음(build-viewer가 conv_out/track_out 인덱스 미생성).단발 유틸 성격 = 경계선(의도적 transient 개연)
하 gitignore 위생viewer/ly_out/index.json만 우발 커밋(무관 scrape 커밋에 혼입) · song/voice/nb index는 gitignore 미등재. 정상 빌드 땐 무해하나 빌드 실패 시 stale 서빙 여지.이번에 수리 반영(viewer/*_out/index.json gitignore + ly_out 언트랙)
별개 음성·자료 성공 산출 0동기화 기전은 건전하나, 현재 voice_out·nb_out 잡이 전부 error.log만(성공물 0). 포착된 원인: exit_code 127 · claude: No such file or directory(CLI 미설치). 자료는 어제(260722)까지 반복 실패.동기화 문제 아님 = 생성 파이프라인 장애 · 별도 수리 필요

6. 플랫폼 종속 위험 (의심 · 서버 갤러리는 무관)

  • 배포 랙 30~90초(구조·자가치유) — 만든 기기엔 즉시, 타 기기엔 재빌드·배포 후 반영. 그 창에서만 다름. R2 url은 즉시 공개.
  • iOS Safari 스토리지 축출(의심·미확인) — 로컬 IDB 원본·드래프트가 축출 대상. 서버 동기 갤러리는 무관 · IDB 12h prune이라 노출 짧음.
  • PWA ↔ 브라우저 파티션 분리(의심·미확인) — 같은 기기라도 설치 PWA와 브라우저는 로컬 저장 별개. 서버 fetch 갤러리는 정상 공유, 로컬 캐시·재개 포인터만 안 넘어감.
