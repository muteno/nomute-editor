§3썸네일 — 문제 재정의 (앵글⑩·⑦)

"후킹 약함"은 증상이다. 병명 = 역할 무정의. 레포 실측:

  • 텍스트 훅의 정본 = # 제목 — prompts:106 "맨 위 기사 제목이 곧 썸네일이다" + 뷰어 제목 복사 버튼. 후킹의 절반(카피)은 이미 제목이 담당.
  • 이미지의 실제 역할 = 오버레이 배경 — 뷰어 "+5장 더" 툴팁 스스로 "오버레이 뒤 후킹 배경으로 검색". 실전 = 검색 실사진 + 수동 오버레이 생성기(thumb_out 283건/18일 = 일 16건, AI-OFF 기간에도 하루도 안 빠짐).
  • AI 썸네일은 세 역할 어느 쪽에도 최적화 안 됨: 단독 대표컷(→훅 텍스트 필요한데 NO_TITLE 봉쇄) / 오버레이 배경(→조용한 영역·9:16 필요한데 full-bleed 단일초점·4:5 고정) / 카드1 표지(→별도 파이프). 검색 실사진에 밀리는 게 합리적 귀결.
  • 운영자 타임라인이 판정을 새겨놨다(앵글⑦): 260621 Gemini 월 지출캡 429 사고 → 260622 "임시 OFF"(직접 계기는 비용 사고, 8일 꺼놔도 무지장 = 진짜 신호) → 260630 2화풍·1K로 조건부 재가동 → 260702 no_thumb 토글 + 연필 wish 개편 = "자동 전량"에서 "필요할 때 지시해 뽑는 물건"으로 취급 이동 중.
  • 불만의 화석 3개가 같은 방향: GOVERNING "스톡사진풍 도피 금지"·주석 "'밥 먹는 그림'화 방지"·260621 암시룰 제거("충돌·분노 순화 방지·운영자 요청") + 실 wish 「전체적으로 더 어둡고 비 내리는 밤 분위기로」 — 전부 "밋밋하고 안전하게 도피한다".
