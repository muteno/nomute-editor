발견지금 상태 — scale 14개 임의값 + 모션 3종 혼재

현재 눌림 scale이 .55 .8 .82 .85 .88 .9 .92 .94 .95 .955 .96 .97 .98 .99 14가지로 흩어져 있어. 다만 이건 의도된 그라데이션(요소가 클수록 덜 줄임 = 물리적으로 자연 — CII 문서화됨)이라 *하나로 합치면 오히려 퇴행*(카드가 작은 아이콘처럼 확 줄면 어색). 진짜 문제 = ① 값이 임의로 14개라 "하나 바꾸면 다 바뀜"이 안 됨 ② 모션 종류 혼재(plain scale vs 회전 vs 키프레임 grow vs 픽토만 축소) ③ 중첩버그(이미지 위 버튼에 ✓오버레이가 자기 아이콘과 겹침 — 최근 수정).

제안 C중첩버그 안전 규칙 (최근 수정 계승)

버그: 이미지 위·투명 버튼에 ✓팝 오버레이(.dl-ack)를 깔면 버튼 자기 아이콘이 z 아래로 비쳐 둘 다 보임(중첩).

규칙: ① 플레이트 있는 버튼만 ✓오버레이(overflow:hidden으로 가둠) ② 픽토-온리·이미지 위 버튼은 ✓오버레이 끄고 scale만(.dl-ack{display:none} · 이미 수정됨).

✗ 잘못(중첩) ← 누르면 ↓와 ✓가 겹쳐 보임
✓ 올바름(scale만) ← 깔끔한 눌림만

반영 방식"하나 바꾸면 다 바뀜" + 원복 안전

① :root에 --press-xs/s/m/l/pico 5토큰 추가(viewer index → 거울로 4뷰어) → ② 14개 임의 scale을 var(--press-*)로 치환(tier 매핑) → ③ check_refs에 "raw scale 신규 금지" 경고(아이콘 SSOT 게이트와 같은 자기강제).

원복 안전: 토큰은 :root 5줄 + 치환이라, 별로면 git revert 1방. 픽셀도 tier 매핑이 기존값과 ±0.02라 회귀 미미.
