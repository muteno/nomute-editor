장점
  • 입력 순서 = 시선 순서: 사진(원료) → 옵션 → 미리보기(결과) → 발사. 합성 탭과 동일 멘탈 모델(슬롯 컴포넌트 = 공용 #imgBox 그대로 이동 = 상태·핸들러·복원 로직 무손실).
  • 사진 게이트(.ie-off)로 빈 발사·헛조작 원천 차단 — dim 값은 .hlock(.go:disabled) 계승 = 신규 raw 0.

단점·리스크
  • 사진 첨부 시 슬롯 썸네일(4:5 크롭)과 하단 미리보기가 이미지 2회 노출 — 정보 축은 다르지만(첨부 확인 vs 변환 시안) 세로 길이 증가. 과하면 슬롯 축소 후속.
  • 슬롯 DOM 이동이 탭 전환마다 왕복 — 카드뉴스(app5 도먼트·포스트 토글) 복귀 지점(insertBefore .gowrap)이 발사행 구조 변경 시 짝 재확인 필요.
  • app8 검출 후 하단 미리보기 자동 숨김(이중 노출 방지) — 검출 실패 시엔 원본 미리보기 복귀로 방어.
효율성

새 아이데이션

  • 하단 미리보기에 비율 칩 오버레이(9:16·1:1 등) 탭 = 즉시 전환 — 순환 탭보다 비교가 빨라짐(순환값 문법과 병행 가능).
