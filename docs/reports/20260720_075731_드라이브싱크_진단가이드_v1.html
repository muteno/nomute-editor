010초 판별 — 범인 반 가르기
마지막으로 언제 돌았는지 + 로그 꼬리:
[ -f ~/.drivesync.last ] && echo "$(( ($(date +%s) - $(cat ~/.drivesync.last)) / 60 ))분 전 마지막 실행" || echo "실행 기록 없음"
tail -20 ~/.drivesync.log
A. "3분 안"으로 나옴
폰·Tasker는 무죄 → 범인은 드라이브/rclone 쪽 = 1번으로.
B. 한참 전 / 기록 없음
Tasker가 안 쏘는 것 = 4번으로.
로그 시각은 UTC(한국시간 −9). 아침 7시 사고가 로그엔 22:xx로 찍혀 있어.
1인증 생사 (이번 1순위 용의자)
rclone lsf gdrive:
정상이면Shared/ 같은 폴더 목록이 뜬다 → 2번으로. (노란 NOTICE "client_id … retired during 2026"은 에러가 아니라 예고 경고 — 무시하고 그 다음 줄을 본다)
NOTICE만 뜨고 다음 줄이 안 나옴 (= 지금 폰 화면)
공용 출입증(client_id)이 이미 막히기 시작한 증상(응답 지연·재시도 루프). 1분 기다려 최종 출력 확인 → 폴더 뜨면 2번 · 에러 뜨면 아래 처방 · 계속 무반응이면 Ctrl+C 후 한 번 재실행 — 또 멈추면 자체 client_id 전환(아래 노란 박스)이 정답.
token · 401 · unauthorized 에러가 뜨면
토큰 만료 — 재연결(브라우저 동의 1회): 
rclone config reconnect gdrive:
reconnect까지 invalid_client류로 실패하면 = rclone 공용 출입증(client_id) 은퇴 확정(어제 예고한 폭탄). 이때는 자체 client_id 발급(15분 1회성)이 유일한 길: rclone.org 발급 절차 → 발급 후 아래 2줄(<ID>·<SECRET> 교체). 어려우면 채팅에 "가이드 ㄱ" — 단계별로 만들어줄게.
rclone config update gdrive client_id <ID> client_secret <SECRET>
rclone config reconnect gdrive:
2파일이 제자리(내 드라이브/Shared)에 있나
rclone lsf -R gdrive:Shared | tail -20
오늘 넣은 파일이 안 보임
PC가 또 "컴퓨터 백업"으로 저장 중(전과 2회) — 드라이브 웹 좌측 [컴퓨터] 탭에 새 Shared가 생겼는지 확인 → 두 PC 모두 "내 드라이브 스트림" + 백업 탭에서 그 폴더 제거(플레이북 §4).
파일은 보이는데 갤러리에 없음
3번으로.
3그저께 밤 대이동 후폭풍 (2순위)
컴퓨터탭 잔재를 Shared로 옮긴 게 전부 "새 파일"로 잡혀, 매번 그 큰 덩어리를 물고 늘어질 수 있어:
wc -l ~/.drivesync.new; tail -5 ~/.drivesync.log
NEW가 수백 건이면 — 옛 이동분은 폰에 받을 게 아니니 전량 장부 등록 + 최근 24시간 것만 회수:
rclone lsf -R --files-only gdrive:Shared >> ~/.drivesync.seen
rclone copy gdrive:Shared /sdcard/Pictures/DriveSync --max-age 24h --inplace -v
termux-media-scan -r /sdcard/Pictures/DriveSync
⚠ 지금 상태에서 echo "__init__" > ~/.drivesync.seen 리셋은 금지 — 이동분 전체를 폰으로 재다운로드하는 폭탄이 된다.
4Tasker 축 (0번이 "한참 전"일 때)
손으로 직접 한 번 돌려보기(디바운스 지우고):
rm -f ~/.drivesync.last && bash ~/.termux/tasker/drive-gallery-sync.sh; echo rc=$?
수동은 되는데 자동만 안 됨
Termux 배터리 최적화 제외 + allow-external-apps 확인(큐 플레이북 §3-1) + Tasker 프로필 2개(3분마다·잠금해제) 살아있는지.
수동도 에러
에러 문구 그대로 채팅에 붙여줘 — 바로 다음 처방 줄게.
5마지막 한 줄 — v1.4 업그레이드 (원인 불문 무조건 실행)
오늘 고친 버전: 앞으로 연결이 죽으면 9분 안에 "드라이브싱크 실패" 알림이 떠서, 이렇게 조용히 멈추는 일이 다시는 안 생겨. 스크립트가 깨졌던 경우의 재설치 겸용.
