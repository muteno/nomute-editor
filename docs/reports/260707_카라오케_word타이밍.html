자막 번인 카라오케 타이밍 수정

영상 자막(/ly) · .github/scripts/ly_burn.py · 260707

🐞 증상

"가라오케가 제대로 동작 안 하고 어절 단위로 쭉쭉 이어진다." 카라오케 채움(하이라이트)이 실제 말하는 속도와 무관하게 기계적으로 균등하게 흘러감.

원인 — 이미 있는 데이터를 버리고 있었다

STT(Whisper)가 각 단어의 정확한 발화 시각을 segments.json에 이미 뽑아두는데, 번인 스크립트가 그걸 쓰지 않고 세그먼트 시간을 글자수로 균등분배하고 있었어.

해결

  • 발화 곡선에 태우기 — STT word 타임스탬프로 "세그 내 실제 발화 진행 곡선"을 만들고 자막 어절을 그 곡선에 태워 채움 시간을 산출.
  • 리드인 대기 — 자막이 뜨고 첫 단어 발화 전 침묵 구간엔 채우지 않고 대기(\k) → 발화 시작에 정확히 맞춰 채우기 시작.
  • 안전 폴백 — word 타임스탬프가 없으면(SRT·텍스트 입력) 기존 글자수 방식 그대로 → 회귀 0.

장단점 · 효율성

장점 카라오케가 실제 노래방처럼 말에 맞춰 채워짐 · 데이터 흐름/파이프라인 변경 0(이미 있는 파일을 additive 소비) · LLM 호출 0(과금 무관) · 폴백으로 기존 동작 완전 보존.

효율 번인 시간 사실상 불변(채움값 계산만 바뀜) · 최악 병적 입력에서도 0.25초 미만 · ffmpeg 인코딩 대비 무시 가능.

한계 (정직)

  • 한국어 원본 자막 = 단어와 어절이 거의 1:1 → 실발화에 정확 수렴(주 사용 경로).
  • 외국어 → 한국어 의역 = 어순이 뒤집히면 좌→우 카라오케의 본질적 한계라 균등과 대등(자막이 뜨는 시점 자체는 정확 → 최악도 "채움 스윕이 살짝 어긋남"이지 틀린 자막 아님).
  • AI가 자막 조각을 잘게 나눈 경계에 단어가 걸치면 소수 조각에서 ≤0.6초 어긋남(미관) — 후속 개선 후보.

품질 저하 리스크 = 낮음

기존 동작(글자수 방식)은 word 데이터가 없을 때 비트 단위로 그대로 보존(20만 퍼즈 확인). 새 경로는 word 데이터가 있을 때만 발현하는 순수 개선이고, 모든 실패 경로는 fail-soft(번인 실패해도 자막 텍스트는 정상). 파이프라인 기틀·출력 포맷 무침범.
