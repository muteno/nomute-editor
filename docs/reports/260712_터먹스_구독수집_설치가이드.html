폰 터먹스 — 구독 수집 설치

X·인스타·스레드를 폰(가정 IP)에서 30분마다 자동 수집 · 한 번만 세팅하면 끝 · 각 단계 끝나면 우측 체크
0 / 7 완료
1준비물 설치
터먹스 열고 아래 붙여넣기. 파이썬·크론(예약 실행기)·서비스 관리자 3개가 깔림.
pkg install python cronie termux-services
설치 끝나면 터먹스를 완전히 껐다가 다시 열어 — 다음 단계 명령이 그래야 인식돼.
2크론 켜기
재시작한 터먹스에서:
sv-enable crond
확인아무 에러 없이 조용히 넘어가면 성공.
3레포 확인
폰에 레포가 이미 있는지 확인:
ls ~/nomute-editor
파일 목록이 뜨면 (기사 공유 쓰던 폰) → 최신화만
cd ~/nomute-editor && git pull origin main
"No such file" 뜨면 → 새로 클론
git clone https://github.com/muteno/nomute-editor.git ~/nomute-editor
클론하다 아이디/토큰을 물어보면 = 푸시 권한 없는 새 폰이란 뜻 → 진행 멈추고 채팅으로 알려줘(토큰 발급 절차 안내).
4크론 등록 (한 줄 · 편집기 없이)
30분마다 자동 수집 예약을 통째로 등록:
(crontab -l 2>/dev/null; echo '*/30 * * * * bash ~/nomute-editor/scripts/phone_subs.sh >> ~/phone_subs.log 2>&1') | crontab -
등록됐는지 확인:
crontab -l
확인방금 넣은 */30 * * * * bash … 줄이 보이면 성공.
5안드로이드 설정 2개
• 설정 → 배터리 → Termux = 제한 없음 (백그라운드에서 안 죽게)
• 기억해둘 것: 폰을 재부팅하면 터먹스 앱을 한 번 열어줘야 크론이 다시 살아남. 평소엔 신경 안 써도 됨.
6즉시 1회 테스트
30분 안 기다리고 지금 바로 한 번 돌려보기:
bash ~/nomute-editor/scripts/phone_subs.sh
이렇게 뜨면 성공phone-subs 수집: x N건 · insta N건 · threads 0건
threads 0건 = 정상(아직 계정을 안 넣었으니까). x·insta에 숫자가 뜨면 경로 전체가 산 거고, 이후는 30분마다 자동.
7마무리 2개 — 채팅으로
㉮ 스레드 계정 — 팔로우할 스레드 계정명을 채팅으로 던져줘(한국/해외 구분해주면 좋음). 내가 목록에 커밋하면 폰이 다음 주기에 자동으로 물어감 — 폰에서 할 일 없음.

㉯ 카나리아 — GitHub → Actions → sns-trends → Run workflow(기본값 그대로) 1회 → 로그 끝줄 레딧 N · 블스 N 확인 → 채팅에 "됐다" 한마디. 그럼 레딧·블루스카이를 크론 상시 ON으로 승격.
🎉 전부 완료. 이제 폰이 30분마다 X·인스타·스레드를 집 IP로 긁어서 자동 반영한다. 폰에서 더 할 일 없음.

문제 생기면

x·insta도 0건으로 나와

일시적일 수 있어(상대 서버 레이트리밋) — 1시간 뒤 로그 확인: tail -20 ~/phone_subs.log. 계속 0건이면 채팅으로 로그 몇 줄 붙여줘.

push 실패 / 인증 에러가 떠

레포에 쓸 권한이 없는 상태. 기사 공유 쓰던 폰이면 원래 되던 거라 토큰 만료 가능성 — 채팅으로 알려주면 재발급 절차 안내할게.

크론이 안 도는 것 같아

① 터먹스가 켜져 있는지(알림바에 Termux 상주) ② sv status crond 결과 ③ tail -20 ~/phone_subs.log — 이 셋 확인. 재부팅 후엔 터먹스 한 번 열기.
