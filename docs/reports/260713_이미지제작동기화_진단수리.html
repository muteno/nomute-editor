1. 무슨 문제였나

① 지금 만든 게 '이전 제작' 목록에 간헐적으로만 제대로 쌓임

② 폰·PC 등 접속 기기마다 이력이 다르게 보임 — "시간은 같이 흐르는데 계정값이 다른 것 마냥". 항상 똑같이(동기화) 보여야 함

뿌리 = 썸네일 이력이 서버 공통 파일(thumb-hist.json) + 기기별 브라우저 저장(localStorage·IndexedDB) 두 곳에 나뉘어, 합칠 때 기기마다 어긋남.

2. 이번에 확실히 잡은 것

ⓐ 라벨 오분류 교정 — 기기 간 이름 일치 반영됨

전 (오분류)

서버가 파일 이름으로만 종류를 추측 → 릴스의 흰칸(box)·투명도(opa) 산출물을 '포스트'로 잘못 표기. 만든 기기(로컬)는 '릴스'라 부르는데 다른 기기(서버)는 '포스트'로 → 같은 제작이 기기마다 다른 이름.

후 (통일)

제작 설정(_src.json의 app)을 기준으로 앱명 통일(포스트/릴스/저작권/경고문). 어느 기기서 봐도 같은 이름.

ⓑ 중복 판정 안전망 — 주소 형식 통일 반영됨

같은 제작인지 판정하는 기준을 주소 문자열 통짜 비교에서 경로만 비교(호스트·캐시버스트 제거)로 바꿈. R2 전체주소든 상대주소든 같은 제작이면 한 번만 표시.

정직한 한계: 현행 데이터를 실측하니 서버 인덱스 400건에 중복 0건 — 카드뉴스·신규 썸네일은 이미 형식이 일치해 원래도 안 갈렸음. 즉 이 변경은 미래 형식 변화·관측 안 되는 기기별 옛 잔재를 막는 안전망이지, 지금 활성 중복을 없앴다고 단정하진 않음. (초기 "카드뉴스가 갈렸다"는 내 추정은 실측으로 정정)
