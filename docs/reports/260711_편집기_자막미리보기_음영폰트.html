3. 운영자 확인 포인트

  • 배치(§디자인 j): 신규 3행 = 자막 카드 안, 기존 게이지(배경/위치/크기) 바로 아래{음영 게이지 → 음영 색 칩 → 폰트 칩}. 라이브 미리보기는 위(소스 아래)에 상시라 조작-확인 인접. 이대로 좋으면 PR #1949 머지.
  • 머지 후 확인: 편집기 자막 카드에 음영/음영 색/폰트 3행 · 음영 게이지 드래그 시 미리보기 외곽선 두께 변화 · 그린 선택 시 외곽선 그린 · 손글씨 선택 시 미리보기 글씨체 변화 · 비율 칩(1:1 등) 클릭이 미리보기 캔버스에 반영.
  • 참고: 음영 기본 100%(readout = 기본) = 지금까지 뽑던 결과와 동일. ly 자막기 특유의 얇은 외곽선 느낌 = 50%.
  • 결정 요청 1건(평의회9 — 폰 동선): 자막 카드 아래쪽(음영 색·폰트)을 조작할 때 위의 라이브 미리보기가 화면 밖으로 스크롤된다(선존 동선 — 이번 3행 추가로 체감 커짐). 고치려면 미리보기를 스크롤 시 상단에 붙이는(스티키) 거동 변경이 필요한데, 화면 절반을 상시 점유하게 되는 트레이드오프라 임의 반영 안 했다 — 스티키로 갈까, 현행 유지할까? 스티키 = CSS 두 줄이라 즉시 가능.
  • 후속 기록(선존·비긴급): API opts 직렬화 캡(900/400자)이 JSON 비인식 절단 — 현재 최악 372/300자로 여유 크지만 키가 더 늘면 가드 필요(v2 평의회4).
