오버레이 생성기 — 큐 병합 · 지우개 · 자막 3줄 높이

운영자 요청 3건(같은 폼) 한 PR. 전부 viewer/thumb.html 단일 파일 · 흰=고정/백엔드/파이프라인 불변 = 렌더 회귀 0. 실제 헤드리스 브라우저 E2E 23개 통과.

요청 1

한 번에 만든 게 같은 큐 카드로 — 개별 + 전체 다운로드

전
헤더+자막(또는 저작권+안내문)을 한 번에 제작하면 큐 카드가 2개로 쪼개짐. 각 카드에 「전체」만.
후
한 번의 제작 = 한 큐 카드. 그 안에 변형별 개별 다운로드(헤더 흰칸·자막 OPA60·OPA30 …) + 「전체」 동시.
  • 원인: 헤더·자막이 각각 dispatchJob 호출 → 별도 잡. 근데 renderJob은 이미 한 잡 안 다중 변형(개별+전체)을 렌더함.
  • 방식: 여러 발사를 dispatchBatch가 한 잡으로 병합(변형 라벨에 역할 태그 접두) → 기존 검증된 멀티변형 카드를 그대로 재사용.
  • 단일 발사(헤더만/자막만)는 기존 경로 그대로 = 라벨·동작 회귀 0. 일부만 실패 시 성공분 표시 + 「일부 실패」 표식 · 재시도는 묶음째 재발사.
효율 새 렌더/컴포넌트 0(기존 .jvar-dl+.jsave 재사용) · 품질저하 위험 낮음 — 폴링/복원/#done 착지 영향 실측 무해(오히려 푸시 착지가 배치 전체 표시로 일관).
요청 2

지우개 버튼 — 부제·제목·자막 입력 비우기

전
붙여넣기 버튼만. 입력 지우려면 전체 선택→삭제.
후
붙여넣기 우측에 동일 크기 지우개(Lucide eraser) · 탭 = 즉시 비움 · 빈칸이면 흐리게·비활성.
  • 대상: 부제·제목·자막 3칸(운영자 지정). 안내문·포스트칸은 기존 단일 버튼 유지(syncGuide 토글 호환).
  • 배치: 지우개=최우측(우측 14px), 붙여넣기=좌측(46px)·6px 간격 · 비움 시 자간/맞춤법 리뷰도 동반 갱신(input 이벤트).
효율 공용 attachCopyPaste에 옵션 1개 · 품질저하 0 — 순수 UI 추가(동작 무해·아이콘 SVG 토큰 규격).
요청 3

자막 3줄 스크롤 제거 + 이미지 슬롯 높이 동기

전
3줄 콘텐츠(96px) > 칸 안높이(94px) = 2px 초과로 스크롤바. 게다가 요청2 지우개가 칸을 좁혀 좁은 폰선 4번째 줄로 줄바꿈.
후
칸 높이 96→104px(조금만 ↑) → 3줄 무스크롤 · 이미지 슬롯도 104px 동기.
  • 줄바꿈 회귀 차단: 자막 텍스트칸은 버튼이 1행 위 테두리로 떠 글자와 안 겹침 → 우측 여백 42px 유지(2버튼이어도 폭 안 먹음) = 360·390·412·480px 전부 3줄.
  • 단일행 입력(부제·제목)만 74px(글자 세로중앙=버튼과 겹침) · 슬롯은 ResizeObserver가 텍스트칸 높이로 자동 추적(폼 §4).
효율 CSS 2값(min-height) · 품질저하 0 — 4폭 실측 무스크롤+높이일치.
