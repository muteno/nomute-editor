장점 · 효율

  • 기기 바꿔도 설정 재입력 불필요 — 새 폰·PC서 접속하면 이미 잠금·PIN·설정이 그대로.
  • 잠금 껐다 켜기가 PIN 1번(전엔 껐다 켤 때마다 새 PIN 2번) — 조작 절반.
  • 기존 잠금 기능(자동 잠금·5회 오류 잠금·마스터 PIN 898900·제작 중 보류·최소화 보존) 전부 그대로 — 회귀 0.
  • 자동잠금 슬라이더는 드래그 중 서버 커밋을 디바운스(드래그 끝 1회만) — 커밋 폭주 방지.

리스크 · 운영자 판단 필요 확인

ⓐ 이 레포가 public이라, 서버에 저장된 PIN 해시가 외부에 노출됨

PIN이 4자리라 해시를 보면 역산이 가능하다. 단, 뷰어 접근 자체는 Cloudflare Access가 막으므로(PIN 알아도 로그인 못 함) 실제 침입은 안 되고, 이 잠금은 원래 "사생활 가림막" 성격이다. 그래도 public git 히스토리에 남는 게 찜찜하면 두 선택지: ① Cloudflare KV로 옮겨 비공개(운영자가 KV 설정 필요) ② 서버 검증 방식(오프라인 잠금해제 포기). 현 상태 유지도 가능(가림막 수용).

ⓑ 재입력할 때 방금 친 PIN이 화면에 떠오름(요청하신 연출)

보기 좋은 대신, 방금 오타 낸 PIN을 그대로 보고 다시 치면 "두 번 확인"의 오타 검출 효과가 사라진다. 요청하신 연출이라 그대로 뒀고, 원하시면 "마지막 1자리만 표시" 등으로 절충 가능.

지적받아 즉시 수정한 것:

  • 서버 저장 실패 시 다음 접속에서 PIN이 조용히 사라질 수 있던 구멍 → 저장 성공을 확인한 뒤에만 확정하도록 봉합.
  • 부팅 중 설정을 바꾸면 서버 로드가 덮어쓰던 경합 → 로컬 변경 우선 가드.
  • PIN 안 쓰는 기기의 AI썸네일 OFF 취향이 초기화되던 문제 → 서버 최초 판정으로 보존.
  • 재입력 오답이 스크린리더에 전달 안 되던 접근성 후퇴 → 시각은 흔들림만, 소리로는 고지.
  • PIN 변경 행이 버튼처럼 안 보이던 문제 → 알림/AI썸네일 행과 같은 모양으로 통일.
