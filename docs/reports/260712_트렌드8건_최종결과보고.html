1요구사항별 상태

8건 중 5건 완료 · 1건 보류(요청대로) · 2건 완료했으나 운영자 확인 필요(아래 3번).
#1필터 접었다 펼칠때 촤르륵완료·확인
트렌드 접기 섹션(.tgroup = AI요약·TOP10·플랫폼별)에 부드러운 높이 애니 배선(접기·펼치기 양방향·옛 브라우저 스냅 폴백). 단 대상 확인 필요: 검증 결과 피드 탭에 문자 그대로 "필터 아코디언"(.lp-filtersub)이 실존 — 운영자 "필터"가 이 피드 필터였는지, 내가 건 트렌드 접기 섹션이 맞는지 확정 필요(3번-③, 최대 리스크).
#21할때 좌측 네비 싱크·동시작동완료·확인
좌측 레일(.trail) 클릭 시 접힌 섹션을 펼치며(촤르륵) 스크롤·점등 동시작동. 단 요구 문언 "접을때→네비" 방향과 내 구현("네비→접힘")이 반대 = 원의도 확인 필요(3번-①).
#3플랫폼 라벨 색 조정완료
회신값대로 구글 = 레몬(--cat-eco #FFE13D), 나머지(유튜브·틱톡·인스타·X) 현행 유지. 구글이 유튜브와 같은 코랄이던 걸 분리.
#4TOP 슬라이드 20안 택1완료
선택하신 매거진 오버레이(1위 큰 히어로 + 커버 위 제목, 2·3·4위 하단 미니 3장). 순환·스와이프·6초 자동회전 전부 보존 — 다음거 넘어가고 순번 다시 돌아오는 것 실측 확인.
#5#4 확대 적용보류
"이미 패스" 하신 대로 미적용. 정상.
#6뉴스+쇼츠 카드 중구난방 해소완료 +보강
썸네일 쇼츠식·조회수 000만(코드 강제)·N일전·업로더 제거·제목 항상 커버 아래 통일. + 검증 중 발견한 크롭 버그 교정: 틱톡 인기·구독 틱톡·구독 릴스(세로 영상)가 16:9로 잘리던 걸 쇼츠와 같은 3/4로 통일("어디는 좌우 자름" 중구난방의 원인이었음).
#7AI 요약 페르소나·포맷완료·라이브대기
모델은 이미 opus 4.8 max라 그대로 — 포맷·뉘앙스만 손봄. 비서 페르소나("사장님," 톤)·요점별 줄바꿈·핵심 수치 강조·만단위 반올림. 뷰어는 줄간격 넓히고 *별표* → 강조색 굵게 렌더(줄바꿈 보존). ⚠ 지금 화면 브리프는 개선 전(stale): sns_brief.json이 수정 이전 생성분이라 원시숫자·한문단으로 보임. 새 포맷은 다음 트렌드 수집 run이 재생성해야 뜸(PVER로 캐시 1회 무효화됨). 만단위 반올림은 프롬프트 지시라 모델 준수 의존(#6 카드 조회수 코드강제와 다름).
#8예타풍 글래스모피즘 → 하단네비완료·확인
하단 네비에 예타풍 글래스(blur30·채도·깊은 그림자) 반영 = 유리감 강화. 솔기는 "전폭 바라 코너 이음새 없음"으로 재해석해 상단 광택을 전폭 페이드로 처리 — 단 검증가 지적: 첨부1이 가리킨 "도형"이 중앙 + 버튼(FAB)의 흰 링이었다면 미수정. + 광택값이 회신(.30) 대비 강함(코드 .08). 불투명도도 .30 vs .41 충돌. 육안 확인 필요(3번-②·④).

2이번 검증에서 추가로 잡은 것

#6 세로영상 크롭 16:9 버그 — .map(card)가 배열 인자를 크롭 플래그로 흘려 틱톡·구독 카드가 가로로 잘리던 것. 세 곳 교정 → 쇼츠와 3/4 통일. 헤드리스 실측: 틱톡·쇼츠 둘 다 3/4·16:9 잔존 0.
#7 별표 파서 하드닝 — 양끝 별표 안에 줄바꿈이 낀 병리 입력에서 통짜 볼드되던 미관 흠 제거(split과 동일 정규식으로 판정). 실측: 정상 강조 유지·개행 포함 통짜볼드 방지.

3운영자 확인 필요 4건 (우선순위 순)

① #1 "필터"가 어느 것 — 최대 리스크
"필터 접었다 펼칠때"를 나는 트렌드 접기 섹션(.tgroup AI요약·TOP10·플랫폼)으로 해석해 촤르륵을 걸었음. 그런데 검증 결과 피드 탭에 문자 그대로 "필터 아코디언"(.lp-filtersub)이 실존. 만약 그걸 뜻했다면 #1·#2가 엉뚱한 화면에 구현된 것.
추천: 트렌드 탭 섹션 헤더를 눌러 촤르륵이 원하던 그거면 OK. 아니라 "피드 필터"였으면 알려줘 — 거기로 옮김. (하위 소머리 .trend-sec까지 확장할지도 같이)
② #2 "동시작동"의 방향
요구 "접을 때 좌측 네비 싱크"는 접힘 → 네비 방향으로 읽히는데, 내 구현은 레일 클릭 → 섹션 펼침(네비 → 접힘). 접힘 자체는 레일을 안 건드리고, 레일 점등은 스크롤 위치가 담당.
추천: 지금 구현(레일 클릭으로 접힌 섹션 열기)이 실사용엔 자연스러움. "접을 때 레일도 반응"을 원하면 한 줄 더 추가.
③ #7 지금 브리프는 개선 전 — 라이브 재생성 대기
코드/프롬프트는 다 바꿨지만 현재 화면 브리프는 수정 이전 캐시(stale)라 원시숫자·한문단으로 보임. 새 포맷(만단위·줄바꿈·강조)은 다음 트렌드 수집 run이 sns_brief.json을 재생성해야 뜸.
추천: 다음 자동 수집 후 트렌드 AI 요약 육안 확인. 지금 바로 보고 싶으면 "브리프 한번 돌려줘" 하면 카나리아 1콜로 재생성해줌.
④ #8 솔기·불투명도
솔기를 "전폭 바라 코너 이음새 없음"으로 재해석해 광택을 전폭 페이드로 처리. 검증가 지적: 첨부1의 "도형"이 중앙 + 버튼 흰 링이었다면 미수정 · 광택값이 회신(.30)보다 강함(코드 .08) · 불투명도 .30 vs 예타값 .41 충돌.
추천: 하단 네비 육안 확인 — 아직 "도형 주변 남은 부분" 보이면 어디인지(+ 버튼? 바 위쪽?) 알려줘. 예타값 픽셀 정확 이식(불투명 .41·채도 1.95·테두리 .05) 원하면 그것도.
