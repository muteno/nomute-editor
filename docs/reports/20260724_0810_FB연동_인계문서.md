### 🔴 유일한 블로커 (운영자 액션 필요)
**nomute-bot(시스템유저)이 Nomute 페이지(id `809122025627220`)에 접근 권한이 없음.**
- 실측 로그: `프로필 실패 — 400/#10: requires 'pages_read_engagement'... or 'Page Public Content Access'`

- 페이지 토큰 전환 시도(`GET /{page}?fields=access_token`)도 동일 #10 = 봇이 페이지를 아예 못 봄.

## 2. 운영자가 해야 할 일 (이거 하나면 끝)

### A. 페이지가 비즈니스에 있는지 (90% 원인)
1. https://business.facebook.com/settings → **계정 → 페이지**
2. 목록에 **Nomute** 있는지 확인
3. 없으면 → **추가 → 페이지 추가**로 Nomute 페이지를 비즈니스 포트폴리오에 편입

### B. 봇한테 페이지 권한 부여
1. 비즈니스 설정 → **사용자 → 시스템 사용자 → nomute-bot**
2. **자산 탭 → 페이지 → Nomute** 할당
3. 권한 토글에서 **"콘텐츠 관리(Manage Page)" 전체 ON** (분석만 켜면 부족)

### C. ⚠️ 순서 주의
- A·B를 **먼저** 하고 → **그 다음 토큰 재생성**(할당 전에 만든 토큰은 옛 권한이라 안 붙을 수 있음)
- 새 토큰 → GitHub 시크릿 `FB_ACCESS_TOKEN` 교체
- 시크릿: https://github.com/muteno/nomute-editor/settings/secrets/actions

### D. 확인 방법
- fb-fetch 워크플로 수동 실행: https://github.com/muteno/nomute-editor/actions/workflows/fb-fetch.yml
- 로그에서 `프로필 실패 #10`이 사라지고 `팔로워 4952 · 시리즈 30일 · 게시물 10` 나오면 성공
- 또는 다음 세션에 "됐어" 하면 대행 실측

## 4. 아키텍처 (다음 세션이 알아야 할 코드 지도)

### 데이터 흐름
```
fb_fetch.py (수집) → viewer/fb_data.json → 뷰어 renderChan (채널요약 FB 탭)
                   ↘ topic_sample 저장 → fb_classify.py (LLM 분류) → topics 승격
```

### 파일
- **`.github/scripts/fb_fetch.py`** = FB 수집 정본. 토큰 해석(유저/페이지/시스템유저 3경로) + 인사이트(MET 딕셔너리 생존탐침) + 게시물 반응(권한 계단 폴백) + 주제 키워드 폴백 + topic_sample 저장.
- **`.github/scripts/fb_classify.py`** = 주제 LLM 분류(신규). `run_claude` SSOT(4계정 폴오버) · Haiku · `viewer/fb_cat_cache.json` 캐시(신규 제목만 LLM) · 6주제 배치 → topics 승격(기타 제외·n≥5 2개↑ 게이트) · fail-soft.
- **`.github/workflows/fb-fetch.yml`** = 30분 레인(FB 단독). FB_ACCESS_TOKEN·FB_PAGE_ID env.
- **`.github/workflows/insta-fetch.yml`** = 3h 브리프 레인. fb_fetch(스텝5) → insta_signals(6) → chan_brief(9) → **fb_classify(신규)** → 커밋. LLM 분류는 여기서만 돎(claude 설치 게이트 편승).
- **`viewer/index.html`** renderChan = 채널요약 렌더(topic·sig·tpost·fb_totals 등). FB/IG 토글 = `CHAN_SRC`.
- **`viewer/fb_data.json`** = FB 수집 산출(🔒 손편집 금지 · 스크래퍼가 재생성).
- **`viewer/fb_cat_cache.json`** = LLM 분류 캐시(봇 생성).

### 분류 상세 (주제별 반응)
- FB 헤드라인은 **편집 재작성**이라 뉴스 원본(candidates.json) 제목 매칭 실패(0% 정확·퍼지 60%) → **LLM 분류 채택**.
- 인스타는 키워드(category) + 수기라벨(cat_overrides)이라 LLM 0콜. FB는 수기라벨 없어서 LLM이 그 등가물.
- 비용: Haiku 배치+캐시 = 하루 새글 1~2개만 = 사실상 0(구독 쿼터 조각).
