발행본 — 버튼부 삭제 → 픽토그램 · 공유 ‘별자리 훑기’

프로필 ‘발행본’ 팝업 · viewer/index.html + nm-svg.js · 260703

공유·휴지통의 테두리 박스(버튼부)를 삭제하고 픽토그램만 남겼어(간격은 행 gap 그대로). 공유의 누름 액티브는 ↗상승 대신 별자리 훑기로 교체 — 클릭 순간 점·선이 살짝 죽었다가, 위→아래 3개 원(ㅇ)이 순서대로 강조색으로 점등되고 연결선이 따라 그려지면서 제자리로 돌아와(총 0.5초). 휴지통 액티브(눌림 축소)는 유지.

직접 눌러봐 — 실제 코드 그대로

7시간 전도보 순례 스님 8명 앗아간 트럭, 몰던 건 11살…
① 윗점 점등 → 선 드로우
② 왼점 점등 → 선 드로우
③ 아랫점 점등 → 복귀

공유 아이콘(작은 것·큰 것)을 탭 = 별자리 훑기 · 휴지통 탭 = 눌림 축소(유지)

전 / 후

Before — 현 라이브

  • 공유 = 초록 테두리+배경 플레이트(30px) · 휴지통 = 빨강 플레이트(28px)
  • 공유 누름 = ↗상승(ic-rise) · 휴지통 누름 = 피드백 미미
  • 행마다 색 박스 2개 — 잠금·제목보다 버튼이 먼저 보임

After — 이번 변경

  • 버튼부 삭제 → 픽토그램만(34px 히트영역 42px·간격 유지)
  • 공유 누름 = 별자리 훑기(3점 순차 강조색 점등+선 드로우 ≈0.5s)
  • 휴지통 누름 = 눌림 축소(scale .82) 유지 · hover 시 빨강
  • rest 무채색 → 눌렀을 때만 색이 들어와 ‘점등’이 또렷

경위 — 사실 이건 ‘복원+신설’이야

버튼부 삭제는 오늘 새벽 PR 1377로 이미 머지됐었는데, 직후 yeta v2(PR 1378) 머지가 옛 베이스 기준이라 이 부분을 조용히 덮어써 회귀됐어(스크린샷에 플레이트가 다시 보인 이유). 이번에 그 픽토그램 스펙을 복원하고, 그 위에 별자리 훑기 모션을 새로 얹었어.

장단점 · 효율 · 리스크

  • 장점행이 정돈되고(무채 크롬) 공유 모션이 아이콘의 의미(3점 연결)를 그대로 살림 — 시맨틱 모션 체계(ic-*) 5번째로 정식 등재(CII)
  • 효율순수 CSS 키프레임(GPU·JS 타이머 0) · 위임 1핸들러 data-motion="share" 옵트인이라 다른 버튼 무영향 · SHARE_SVG는 선만 2path로 나눠 렌더 동일
  • 리스크폰에서 navigator.share 시트가 바로 덮어 훑기가 짧게만 보임(복사 폴백·PC는 완주) · rest 무채색이 낯설면 상시 초록/빨강으로 한 줄 조정 가능
  • 품질저하없음 — 만료행(disabled)·✓복사 플래시·잠금 토글·행 클릭 동작 전부 불변, reduced-motion이면 모션 자동 꺼짐
