3탭 첫 블록 정렬 — 현재(#1068) vs 내 방법

실측 결론 — #1068이 필터→픽토그램·카드만→버튼·오버레이는 잘 했는데, 핵심 목표인 "스크랩 기준 3탭 첫 블록 일치"가 feed에서 8px 어긋났다(feed 145.6 / scrap·sns 153.6). 아래 토글로 직접 확인 —

📍 왜 어긋났나 — 방법의 차이

#1068 — 컨테이너마다 값 따로 (산발)

/* 스크랩·SNS만 손대고 feed는 빠뜨림 */
.scrapview { padding-top:10px; }
.snsview   { padding-top:10px; }
/* feed = 옛 값 그대로 ↓ */
.wrap      { padding-top:0; }
#list      { margin-top:2px; }  → feed 2px만
  • 3곳 값이 흩어져 있어 한 곳(feed)을 빠뜨림 → 8px 드리프트
  • 다음에 한 탭 손대면 또 어긋날 위험(같은 실수 반복)

내 방법 — 단일 토큰 공유 (SSOT)

/* 한 변수로 3탭이 공유 = 어긋날 수 없음 */
:root { --head-gap:10px; }
.wrap, .scrapview, .snsview {
  padding-top:var(--head-gap);
}
#list { margin-top:0; }  → 토큰이 단일 진실원본
  • 값이 한 곳 → 3탭이 구조적으로 항상 일치
  • §🎨 "값 SSOT" 원칙 그대로 — 미래 드리프트 0

미달 스펙 "스크랩 기준 3탭 일치"가 feed에서 8px 빗나감. 산발 padding이 원인 — 단일 토큰이면 안 생김.

제안 — "정렬 고쳐" 하면 위 내 방법(단일 토큰 --head-gap)으로 feed를 스크랩에 픽셀 일치 + 미래 드리프트 차단. 8px 한 곳만 손대는 게 아니라 3탭을 한 변수로 묶음.
