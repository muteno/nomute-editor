쉬운 설명 — 생성이 끝난 잡 카드 오른쪽에 + 한 장 더 필 버튼이 생겼다. 탭 1회 = 방금 그 설정(화풍·비율·무드·주문 전부) 그대로 1장 새로 발사, 새 잡 카드가 맨 위에 붙는다. 원본 카드·결과는 그대로. 실패하면 버튼이 ⚠ 실패 · 다시로 바뀌고 재탭이 곧 재시도다.

전 / 후

전 — 반복 생성 = 팝업 재진입 3탭
후 — 원탭 +1장(강조색 필 = .jretry 기하 계승)

장단점 · 효율성 · 품질저하 리스크

장점
  • 반복 생성 3탭 → 1탭(옵션 재설정 0 — 주문·무드 게이지까지 승계)
  • 실패 재시도(.jretry 빨강)와 색으로 의미 분리(초록 = 재생성)
  • 새 카드 방식 = 원본 결과·캐러셀 보존(genRefire in-place와 구분)
단점 · 리스크
  • 원탭 = 제미나이 1장 즉시 과금(확인창 없음 — 왕복 disabled·1장 고정으로 최소화)
  • 연속 다발 발사 시 h8 매칭 근사 한계(기존 명시 한계 그대로)
  • 품질: 발사 페이로드 형식·API 무접촉 = 0 변화
효율성
  • 변경 = thumb.html 3점(CSS 2룰·상수 1·renderJob 1블록) · API/워크플로 0바이트
  • 신규 상태·폴링 0 — 표준 잡 흐름(genJob) 재사용
  • payload 없는 구 카드 = 자동 미노출(.jretry 게이트 동일 = 죽은 버튼 0)
새 아이데이션
  • 길게 누르면 장수(1~4) 선택
  • 재생성 카드에 원본 잡 계보 배지
  • 기사 카러셀(+N장)에도 동일 원탭 이식
