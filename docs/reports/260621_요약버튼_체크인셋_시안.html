요약 버튼 · 체크박스 인셋 시안

현재는 체크박스가 글래스 알약 밖에 raw 사각으로 떠 있어(스크린샷). 이걸 요약 ↓ 글래스 알약 안으로 접어넣는 5가지 톤앤톤 시안. 라임 accent #0FFD02·글래스 rgba(255,255,255,.06)·blur 14px·34px 알약 = 뷰어 토큰 그대로 계승.

📍 탭해서 직접 만져봐 — 체크 영역을 누르면 토글, 요약 글자를 누르면 다운로드 플래시(초록)가 떠. 폰에서 손맛까지 확인용.

현재 (Before)
분리형 — 체크가 알약 밖

raw 사각 체크박스가 글래스 알약과 따로 놀아 '한 컨트롤'로 안 읽힘. 작은 사각이 톤에서 튄다.

요약 원문 ↗
시안 A
인셋 네이티브 체크 + 헤어라인

알약 안 왼쪽에 네이티브 체크(accent색)를 넣고 1px 디바이더로 구분. 구조 변경 최소 — 가장 안전한 한 수.

요약 원문 ↗
전/후 체크가 알약 안으로 들어와 한 컨트롤로 읽힘.
네이티브 체크는 OS마다 모양 편차(iOS/안드로이드).
시안 B · 추천
커스텀 글래스 체크박스

알약 안 왼쪽에 둥근 사각 커스텀 체크 — 끄면 은은한 테두리, 켜면 라임 그라데 채움 + 다크 체크마크 + 글로우. 뷰어 .sbtn.ok·토글칩과 가장 정합.

요약 원문 ↗
OS 무관 동일 모양 · 톤앤톤 완벽 · 켜짐 상태가 명확.
커스텀이라 CSS 몇 줄 추가(접근성 위해 숨김 input 유지).
시안 C
세그먼트 2존 알약

알약을 안쪽 헤어라인으로 둘로 — 왼쪽 존 탭=전체 토글(켜지면 존 배경 형광 틴트), 오른쪽 존 탭=다운로드. 탭 타깃이 큼.

요약 원문 ↗
탭 타깃 큼(폰 친화) · 토글/다운로드 영역 분리 명확.
알약이 약간 넓어짐 · 두 액션이 한 알약이라 첫 학습 필요.
시안 D
'전체' 라벨 복원형

체크 옆에 '전체' 글자를 되살림(현재 코드 주석: '전체' 글자 폐기). 켜지면 라벨이 형광 → 지금 뭘 받는지 글자로 설명. 폭은 늘지만 자기설명적.

전체· 요약 원문 ↗
자기설명적('전체'를 받는다는 뜻이 글자로).
폭이 가장 넓음 · 모달 헤더 공간 부담.
시안 E
액티브 = 알약 전체 강조

B의 커스텀 체크 + 켜지면 알약 전체가 형광 테두리·은은한 채움·글로우로 전환 → '전체 모드 ON'을 한눈에. 상태 신호가 가장 강함.

요약 원문 ↗
켜짐 상태가 가장 또렷(멀리서도 보임).
초록이 강해 다운로드 'ok' 플래시와 색이 겹칠 수 있음(시각 충돌 주의).

※ 전부 실제 뷰어 토큰으로 그렸지만 이건 시안(스냅샷)이라 라이브엔 아직 안 들어감. 고르면 viewer/index.html .dlbox/.dl-full에 반영(수정 모드 + 검증) → main 머지.
