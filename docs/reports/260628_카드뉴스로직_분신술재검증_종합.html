④ 그래서 진짜 해법 (재검증 수렴)

4-1. 최우선 측정 인프라부터 — "측정자 부재"가 근본

  • A/B 산출물 + 채점 루브릭을 cards/_ab/ 또는 docs/에 파일로 커밋 → 재현·반박·diff 가능. 룰을 7개 패치해도 본질이 안 변한 이유 = 측정자가 없어서.
  • fact_guard.py에 --coverage 역방향 모드 추가: 지금은 "출력−원문"(날조 차단)인데, "요약−카드 누락"(WHY 증발 차단)은 정반대 방향이고 토크나이저 90% 재사용. 기계 게이트를 새로 안 짜도 됨.

4-2. 자기모순 제거 (룰 충돌 봉합)

  • 지침 "줄≤4(5 금지)" ↔ 합성기 MEMORY "최대 5줄" ↔ 실측(5줄 렌더됨) 3중 불일치 → "줄≤5(6 금지)"로 통일(코드 0줄 변경).
  • ✳️ GPT·🎬 죽은 스펙이 01·00 양쪽에 살아있어 매 호출 모순 지침 주입 → 자동 경로만 INJECT-SKIP(수동 채팅 GPT엔 유효하니 삭제 X).

4-3. 정리·이전 룰 추가 말고 줄이기 (티어 비대칭 해소)

  • 진짜 문제는 개수가 아니라 ① 조각 KILLER(글자수)가 흐름 룰을 항상 이김 ② 소프트룰이 토큰 압축에 묻혀 작동 안 함.
  • 중복 ~15개 통합(RECOMMENDED 30개 중 절반이 금지패턴의 거울) · 기계검증 가능한 KILLER를 하드게이트로 이전(산문서 삭제 → 토큰↓ → 소프트룰 안 묻힘) · STOP 자가표 1개로 흐름룰 흡수.

4-4. 살아남은 미세 보강 (기존 강화 — 신설 아님)

ⓒ 첫카드가 제목 펀치라인 반복 금지(#6 확장) ⓑ 장소 리드 — 속보는 "기시감/긴장 프레임" 얹어 허용 마지막 카드 과부하 직격: 5W잔여·신규수치 금지·강조≤1 후반 칸 끝줄에 "다음 칸 회수할 씨앗 1톨"(천장 30% 여백 활용) 금지어 grep: 방점이 찍혔다·구조를 묻는다·체언종결 접속사(정작/그런데) 세트당 ≤2

⑥ 운영자 결정 필요 (기틀 = §기틀 보호)

  • 방향 승인: 위 「신설 최소화 + 측정 인프라 + 자기모순 제거 + 정리」로 갈지.
  • 통일 vs 획일: 클로징이 사건마다 "구조를 묻는다/평화는 못 갔다"로 동질화 = 통일의 목적(품질 하한)이 거꾸로 상투화로 뒤집힘. 통일 우선(하한 보장) vs 사건별 개성(편차 감수) — 룰로 못 박을 브랜드 톤.
