왜 버그가 아닌가 (검증 확정)

  • 두 룹브릭 도장이 현행 스크립트 RUBRIC_VER과 정확히 일치 → 최신 판정, stale 아님. (working tree = HEAD, diff 0 확인.)
  • breaking_judge 쓰기 로직상 도장은 AI가 명시적 Y/N을 반환했을 때만 찍힘 → breaking=false는 AI가 실제로 “N”을 반환한 실판정. 하드가드(김건희+판결류) 미발동 = 강제 아닌 순수 AI 판단.
  • 파편화(cross 11/9로 쪼개짐)는 무관 — group_judge가 이미 두 파편을 한 그룹으로 병합했고, cross 11만으로도 결과는 동일(grade 1). 신호 약화가 원인 아님.

2근본원인 — “발생/미발생” 이분법의 사각

두 룹브릭 모두 “사건이 실제로 발생했는가(피해 있음)” 축으로만 짜여 있다. 상어는 “나타났다 잇단 신고” = 목격·경보 단계(피해 0)라, ‘발생’ 게이트에도 ‘비뉴스’ 게이트에도 안 걸리고 그 사이 제3의 상태로 흘러 grade 1·breaking false로 침몰한다.

3해결책 — 두 룹브릭 짝 편집 (16인 수렴안)

긴급 공식이 breaking ∧ grade≥2 논리곱이라 한 벽만 고치면 다른 벽에 막힌다. 게다가 피드 카드 테두리는 raw breaking(grade 게이트 없음)을 읽어, breaking만 켜면 피드엔 🚨·수집함엔 무배지 비대칭이 생긴다. ∴ 두 룹브릭을 한 개념으로 반드시 짝 편집.

한 개념 = “임박·대규모 인명위협” (3중 좁은 가드)

① 대규모 군중·다중이용 인파 실재(해수욕장·경기장·축제·역·공항 등 수천~수만) AND ② 급성 물리위협(맹수·상어·유독가스·붕괴/폭발 임박 등) AND ③ 실제 대피/현장통제/폐쇄가 발동·진행 → 피해 0이어도 긴급 인정.

배제(하나라도 불명확하면 X): 일상 기상주의보(폭염·미세먼지)·안전 캠페인·점검·훈련·개별 소수 대상·장소 불특정 경고. 기존 게이트(🌐 해외군사·🔪 개별범죄·📈 증시·🔎 수사절차·badgeJunk·JUNK_HEAD) 전부 우선·무손상.

  • breaking_judge.py [속보 O]에 이 조항 1개 추가
  • gate_judge.py grade 가드에 예외 추가 (임박 대규모 위협 → grade 상향)

5운영자 결정 지점 — 긴급 티어

여기가 알고리즘이 못 정하는, 운영자 판단이 필요한 지점. 상어를 살릴 강도를 정해야 한다(gate_judge를 grade 2로 올릴지 3으로 올릴지).

권장  티어 A = grade 2 (🚨긴급 배지·핀까지)

🚨긴급 빨강 배지 + 신규 상단핀 + 랭킹부스트. 폰 푸시·자동 카드과금 없음(그건 grade≥3 게이트). 앱을 열면 “지금 중요”로 보이되 폰은 안 울림.

위원 5·16 디폴트 — 상어 “목격”은 미확인 정보라 폰 푸시+자동과금은 과함. 보수적·오발비용 낮음.

티어 B = grade 3 (배지 + 폰 앱푸시 + 자동요약 + 자동카드)

위 전부 + 폰 웹푸시 + 화면알림·토스트 + 자동 뉴스요약 + 자동 카드 Gemini 과금.
