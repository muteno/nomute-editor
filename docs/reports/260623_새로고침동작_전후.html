새로고침 동작 개선 — 전후 비교

웹앱 새로고침 2가지 고침 (① 보던 탭 유지 · ② 상단 로딩색 = 탭 고유색). 단일파일 viewer/index.html만 손봄. PR #853 머지·라이브 반영 완료.

📍 ① 새로고침 = 보던 탭 그대로

BEFORE
뉴스요약(feed)에서 새로고침하면 무조건 수집함(scrap)으로 튕김. 복원 화이트리스트에 scrap·sns만 있어 feed가 빠져 있었음.
AFTER
보던 탭(feed·scrap·sns) 그대로 복원. 저장값 없으면 기본 홈=수집함 유지(기존 디폴트 보존).

코드 한 줄: ['scrap','sns'] → ['feed','scrap','sns'] (복원 게이트에 feed 추가).

📍 ② 새로고침 상단 로딩색 = 탭 고유색

BEFORE
<meta theme-color>가 정적 #0a120d(고정 다크). 어느 탭이든 당겨서새로고침·주소창 로딩색이 똑같음.
AFTER
showTab()이 탭별 theme-color 갱신 → 로딩 영역이 보던 탭 고유색으로. 배너 글로우색과 매칭.

📍 장단점

  • 장점 · 보던 화면이 유지돼 새로고침 후 길 잃지 않음(특히 뉴스요약 읽다 새로고침해도 그대로).
  • 장점 · 로딩색이 탭 정체성과 일치 = 어느 탭에 있는지 로딩 중에도 직관적. 일관성↑.
  • 장점 · 두 변경이 한 쌍 — showTab이 색 저장 + 탭 저장을 같이 하니 복원 시 색·탭이 항상 동기.
  • 유의 · 기본 홈 디폴트(수집함)는 그대로 — 처음 방문·저장값 없을 때만 적용(의도 보존).
  • 유의 · theme-color 바 적용은 모바일 크롬 등 지원 브라우저 한정(데스크탑 일부 무시) — graceful, 미적용 시 기존 다크색.

📍 효율성

  • 경량 · 순수 클라이언트 JS(메타태그 1개 갱신 + 배열 1원소 추가). 네트워크·빌드·파이프라인 영향 0.
  • 비용 0 · LLM·렌더·워크플로 무관. no-cache 헤더라 하드새로고침 없이 즉시 라이브.
  • 재사용 · 색은 기존 배너 글로우 토큰값을 그대로 차용 — 새 색 창작 0, 드리프트 0.

📍 품질저하 리스크

항목판정
기존 디폴트(수집함 홈) 깨짐없음 — 저장값 없을 때 fallback 그대로
