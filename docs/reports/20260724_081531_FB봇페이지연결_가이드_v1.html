페이스북 봇 ↔ 페이지 연결

코드·토큰은 다 됐어. nomute-bot을 Nomute 페이지에 붙이기만 하면 FB가 인스타처럼 꽉 차.

지금 상태

✓
비만료 시스템유저 토큰 · 수집 코드 · 주제 LLM 분류 — 전부 완료·머지됨
✓
뷰어 타일(반응·댓글·공유·주제) — 데이터만 들어오면 자동 점등
✕
봇이 페이지에 안 붙어 있음 → 그래프 API가 #10 권한없음으로 막음. 토큰엔 권한이 다 있는데 페이지가 봇을 모름.

왜 코드로 못 푸나 = 시스템유저가 페이지 데이터를 읽으려면 메타가 "자산 할당"을 요구(인증 규칙). 버튼 클릭은 사람만 가능 — 그래서 이 3스텝이 필요해.

고치는 법 — 순서대로 3스텝

페이지가 비즈니스에 있나 확인

비즈니스 설정 → 계정 → 페이지 목록에 Nomute가 있는지 봐. 없으면 [추가 → 페이지 추가]로 편입.

비즈니스 설정 · 페이지 열기

이미 있으면 이 스텝은 건너뛰어도 돼.

봇한테 페이지 권한 주기 (핵심)

비즈니스 설정 → 사용자 → 시스템 사용자 → nomute-bot 선택 → [자산 추가] → 페이지 → Nomute 체크.

시스템 사용자 열기

권한 토글에서 "콘텐츠 관리(Manage Page)" 전체 ON — 분석만 켜면 부족해.

콘텐츠 관리 (Manage Page) — 이걸 켜라

⚠️ 순서 주의 — 그 다음 토큰 재생성

2번을 먼저 끝내고 → 같은 nomute-bot 화면에서 [새 토큰 생성]. 앱 = nomute-insights, 권한 4개 체크(아래 복사) → 생성.

할당 전에 만든 토큰은 옛 권한이라 안 붙어. 반드시 할당 → 그 다음 생성.

체크할 권한 4개 탭복사pages_read_engagement · pages_read_user_content · read_insights · pages_show_list

새 토큰 → GitHub 시크릿 FB_ACCESS_TOKEN 값 교체(덮어쓰기).
