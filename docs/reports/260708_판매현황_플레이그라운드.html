      재현 한계 각주

      · 구 판매현황 = 근사 재현 — 원 코드(죽은 코드) 미확보라 설명·통상 패턴으로 재구성. 실물과 다르면 스샷/코드 던져주면 갱신.

      · 상세 화면 = 260708 스크린샷 실측 발췌(색·모서리·간격 근사치) · 이 앱은 토큰 정본이 없어 계승/갱신 판정은 스샷 현행 대비만(기틀 ⚠ 딱지 해당 없음).

      · 데이터 전부 목업 시드(오늘 = 26.7.8 고정) — 실제 판매·저장 없음. 폼 업로드·검색·비교추가 = 화면 안 데모.

      · 일일입력 = 지정 폼(.xls) 업로드로 확정(운영자 260708) — 시트 스키마 = schedule{일자·요일·상품명·매수·금액·현금/카드·예매/현장}·grade{좌석등급·권종·단가}·payment{결제수단}. 업로드 존은 데모(실파싱은 실구현에서 · 첨부 실측값 37매/1,286,000원 사용).

      · 차트 = SVG 근사(스크린리더용 표 미포함) · 스파크라인은 각 프로그램 자체 스케일(모양 비교용) · 폰트 = 시스템 폴백(Pretendard 미임베드).

      · PC 프레임은 화면이 좁으면 비율 축소 표시(실폭 1280 기준) · 비교 선택은 최대 4개(색 구분 한계).
