? 한눈에 — 뭘 한 건가 (쉬운 설명)

운영자 요구: "이 레포에서 일할 때마다 항상 먼저 확인할 작업 규칙을 박아둬라." 그래서 매 세션 자동으로 읽히는 라우터(CLAUDE.md)에 4대 표준을 새겼다. 앞으로 모든 세션이 같은 절차·같은 품질로 일한다.

1
의도부터 확정 — 불분명하면 「지금 처한 문제 / 해결하려는 문제」를 되짚어 맞춘 뒤 시작.
2
결과는 HTML 보고서로 — 전/후·장단점·효율·리스크를 폰서 보이게(이 문서가 그 첫 사례).
3
적대적 10인 검증 후 머지 — 서로 다른 앵글의 Opus 4.8 10명이 토론·수렴.
4
작업 누적 — 완료 내역과 개선 우선사항을 docs/작업이력.md에 쌓아 다음 세션이 더 잘하게.

⇄ 전 / 후 비교

전 (Before)
  • 의도확인·10인검증·SSOT누적이 라우터 곳곳에 흩어짐
  • 전후비교 HTML 보고서 규칙 없음
  • 완료작업 누적 원장 없음 → 매번 방향 재탐색
  • "검증=10인"이 명시·기틀에만(상시는 5인)
후 (After)
  • §🎯 한 체크리스트로 묶음(얇은 포인터 + 신규 2종)
  • 실질 변경 → docs/reports/*.html + SendUserFile 규칙화
  • docs/작업이력.md 원장(작업로그 §1 + 개선 우선사항 §2)
  • 실질 코드·구조 변경은 10인 상시, 문구는 자가검증
