① 쉬운 설명 — 무슨 일이었나

긴급 기사를 자동으로 골라 서류(pending 파일)까지 다 만들어놓고, 마지막 접수(git push) 창구에서 매번 반려당하고 있었다. 반려 사유는 "책상 위에 정리 안 된 다른 서류(토큰 계측 기록 1줄)가 있다"는 것. 그런데 이 반려가 초록불(성공)으로 표시돼서 아무도 몰랐다. 자동픽이 픽 직전에 쓰는 AI 사건중복 검사가 토큰 사용량을 계측 파일에 한 줄 적는데, 그 파일은 같은 런에서 이미 커밋된 추적 파일이라 커밋 목록에 안 넣으면 "미정리 변경"으로 남아 git이 반입을 거부한다. 그래서 "직전 24시간 안에 픽이 하나라도 있으면"(=중복검사 AI가 돌면) 그 다음 긴급 픽은 전부 유실됐다.

③ 근인 — 런 안에서 벌어진 일

1
judge가 경중·속보 판정(claude 호출) → 계측 shard metrics/usage/<run>-judge-1.jsonl 생성
2
"Commit (breaking 확정)" 스텝이 candidates.json + metrics 커밋·push ✅ → 이 시점부터 shard는 추적 파일
3
자동픽 스크립트: 자격 건 발견 → AI 사건중복 검사(run_claude(source="autopick")) → 같은 shard에 계측 1줄 append
4
픽 성공 → pending 파일·원장 4경로만 git add·commit (metrics 미포함 → unstaged로 잔류)
✗
git pull --rebase 가 "You have unstaged changes"로 거부 — 4회 재시도 전부 동일(원인이 그대로니 재시도 무의미) → push 실패 → 분석 미발동 → 스텝은 ::warning+exit 0(green)으로 종료 = 은폐
