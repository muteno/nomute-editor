최소화 상태에선 이 피드(또는 기사 요약 화면)를 자유롭게 스크롤·조작할 수 있고, ▲ 복원 키가 항상 떠 있어.

이미지
① 우상단 ─ 누르면 → 촤르륵 접혀 ▲로 수렴
② ▲ 탭 = 다시 펼침 (입력 그대로)
③ ▲ 길게 누르면 "닫기" 떠서 → 한 번 더 탭 = 완전 종료

전 / 후

상황전후
편집 중 기사 다시 읽기모달 닫아야 함 → 편집 내용 소실최소화 → 편집 보존한 채 왕복
접기/펴기 동작없음▲로 촤르륵 수렴/역재생 (60fps GPU 합성)
다른 화면 이동 시—▲ 복원 키 항상 유지(요약리더 위에도 부유·닫기 전까지)
닫기X 버튼X 그대로 + ▲ 길게눌러 "닫기"(2단 확인=실수 종료 방지)

장점

  • 편집 무손실 왕복 — iframe(thumb 폼)을 파괴하지 않고 숨김만 → 입력값·상태 100% 보존(실측 검증).
  • 기존 UI 계승 — 최소화 버튼 = .tool-x(X와 동일 글래스·눌림), 복원 ▲ = accent 그린(우하단 특수버튼 색).
  • 레이어 충돌 0 — 복원 키가 요약리더(모달) 위에도 뜨도록 처리, 하단 nav/FAB/totop/토스트와 좌표·층 분리.
  • 발견성 — 접힌 직후 펄스 링 호흡 + 중앙 배치로 "여기 있다" 신호.

효율성

  • 애니메이션 = transform/opacity만(레이아웃·블러 트리거 0) → GPU 컴포지트 60fps.
  • 모달 본체(backdrop-filter blur34)엔 변형 안 줌 = 블러 재계산 '턱' 스내그 회피 → 블러 없는 가벼운 고스트가 모션 담당.
  • 추가 비용 = 순수 프론트(LLM·과금 0). 새 의존성 0. 코드 +156줄(단일 파일).

품질저하 리스크 & 완화

리스크완화
요약리더(모달) 위 부유 = 브라우저 top-layer 규칙상 까다로움실측 후 "열린 모달 안으로 재부모(reparent)" 방식 채택 → 모든 엔진에서 동작(Playwright 검증)
접힘 중 모달 본체 변형 시 블러 스내그고스트 클론으로 모션 분리(본체는 opacity만)
롱프레스 닫기 오발화550ms + 2단 확인(arm→재탭) + 드래그/스크롤 무시 + 2.6s 자동 해제
history(백버튼) 꼬임{tool} 칸 보존·재사용, 백버튼=재펼침, 완전종료서만 칸 회수
구형 브라우저WAAPI/pointer 미지원 시 애니 생략·즉시 동작(기능은 유지)
