편집 탭 — 발사 버튼·대기 멘트·음량 기본값 + 모자이크 확인

영상 스튜디오 편집 탭(viewer/edit.html) · 260712 · 질문 2건 답 + 수정 3건 · 기틀 토큰/컴포넌트 100% 계승(신규 색·raw 0)

• 질문 1 답 — "지금 무슨 단계인지 확인 가능해?" = 불가(현 구조)

단계 실측 = 불가. 편집 잡은 GitHub Actions 원샷 워크플로(edit-make)로 돌고, 뷰어(폰)가 받아볼 수 있는 산출은 끝에 커밋되는 video.json(성공)·error.log(실패) 둘뿐이야. 중간 단계(STT→의역→ffmpeg 보간→R2 업로드)를 알리는 채널이 없어서, 구 멘트 60fps 보간 중은 실측이 아니라 선택한 옵션으로 때려 맞춘 추정 라벨이었어(옵션 여러 개면 더 부정확). → 운영자 지시대로 멘트를 정직한 범용 편집 중 … (경과s)로 변경(클립 스캔만 잡 전체가 후보픽이라 클립 후보 찾는 중 유지).
  • 가능하게 만들려면(후속 제안·미구현): 워크플로가 단계마다 R2에 stage.json 마커 업로드 → 뷰어가 병행 폴. R2는 배포 대기 없이 즉시 읽혀서 실시간 단계 표기 가능 — 원하면 별도 승인으로 진행(잡당 R2 PUT 4~6회 추가·과금 영향 미미).

• 질문 2 답 — 모자이크 동작 로직 + 최근 구현 반영 확인

  • 타겟 지정 로직: 편집 탭에서 정하는 게 아니라 트래킹 화면에서 분석 1회 → 카드 선택이 정본 플로우야. 분석(track_analyze)이 얼굴 검출(YuNet)+동일인 군집(SFace)+피사체(YOLO 전신·사물)를 인물·피사체 카드로 깔아주면, 거기서 가릴 대상을 탭으로 고르고(카드마다 얼굴만|전신 토글) 렌더가 그 트랙만 가려. 얼굴이 한 번도 안 잡힌 인물은 카드가 안 생김(키잉은 '직접 지정' 탭이 보완 · 사물 모자이크 직접 지정은 후속 예정).
  • 근 2일 구현(260710~11) 반영 여부 = 트래킹엔 전부 반영돼 있음: 전신 폴백(뒤통수·측면도 퇴장까지 커버)·가림 범위 얼굴만|전신·키잉 얼굴 단위·직접 지정 전부 track.html+렌더에 살아 있고, 편집 탭 기본값(타원·115%·페더5·키잉0)도 최신과 일치. 핸드오프(설정 들고 트래킹으로)도 정상 배선 실측.
  • "여기 반영 안 된 듯"의 실체 = 편집 탭 기타 기능엔 대상 선택 UI가 원래 없음(모양·크기·페더 사전설정만) → 안내 문구가 "영상은 거기서 다시 지정"만 말해서 대상 지정 위치가 안 보였던 것. 문구를 거기서 영상 지정 → 분석 1회 → 인물·피사체 카드에서 가릴 대상 선택(얼굴만|전신)으로 보강함.
  • 참고: 기타 기능 칩은 복수 선택돼도 트래킹 렌더는 한 번에 한 모드(키잉>핀셋>모자이크 우선 전달) — 두 효과를 다 원하면 트래킹에서 모드 바꿔 두 번 렌더.
