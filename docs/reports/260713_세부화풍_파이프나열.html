2. 장단점 · 효율 · 품질 리스크

  • 장점 — 세부 화풍 발견성 즉시(탭 1회 절감 · 드롭다운 열기/닫기/스크롤 폐지) · 현 화풍의 선택 폭이 항상 가시 · 기존 워드칩(.geni-opt)·우측정렬(.geni-opts flex-end) 100% 계승 = 이 행만 이질감 0.
  • 단점·리스크 — 항목 6개 화풍(극화·수채)은 좁은 폰에서 2줄 랩(줄바꿈은 구분자+글자 묶음 단위라 어색한 고아 | 없음) · 행 세로 높이 소폭 증가 · 300ms 스왑 지연 동안 세부 클릭 불가(아웃 중) = 의도된 모션 비용.
  • 효율 — 신규 의존 0 · innerHTML 재구성은 화풍 전환 시에만(동일 화풍 geniApply 재호출 = on 토글만 · data-style 가드) · 선택박스 코드(geniDD*)는 '기타 화풍'용으로 존치 = 삭제 리스크 0.
  • 품질 저하 방지 — 발사 페이로드(style·sub 키)·서버 STYLE_SUB 계약·요약 리드백·미보유 세부 auto 강하 로직 전부 불변. reduced-motion = 애니 제거 등재. 연타 레이스(clearTimeout·A→B→A 복귀) 가드.

3. 새 아이데이션 (제안 · 미적용)

  • 무드 '직접' 게이지 아코디언처럼, 세부 화풍 각 항목에 길게 누르면 1줄 미리보기 설명(게키카 = 거친 펜선 극화체…) 팝 — 어포던스 부족 시에만.
  • 화풍 워드칩에도 동일 촤르륵을 확장해 '기타' 선택 시 기타 화풍명이 칩으로 승격되는 전환 모션 통일.
  • 세부 화풍 사용 빈도 로컬 누적 → 자주 쓰는 세부를 자동 다음 순서로 재배열(폐쇄 목록 내 정렬만이라 계약 무변).
