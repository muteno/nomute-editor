수집함 병합 기능

2026-06-25 · viewer/index.html · PR #953 · 운영자 요청(B 디자인)

한 줄 — 토큰 클러스터링에서 갈라진 같은 사건을 운영자가 길게 눌러 수동으로 묶고, 픽하면 하나로 종합 요약되게 한다.

왜 — 같은 사건이 갈라진다

수집함 클러스터링은 제목 토큰 교집합(겹침 3개+ 또는 자카드 0.5+)으로만 묶는다. 거대블록 방지로 임계를 일부러 보수화(2→3)했더니, 같은 사건이 측면 단어·조사·약어로 토큰이 안 겹쳐 따로 뜬다.

같은 베네수엘라 지진안 묶인 이유
① 강진·카라카스 시민대피
② USGS 사망 가능성
③ 주택·건물 붕괴		①↔③ 겹침 2개(베네수엘라·강진)뿐 → 임계 3 미달
② 베네수≠베네수엘라, 강진에≠강진(조사) → 겹침 0

임계를 낮추면 무관 기사가 줄줄이 엮여 거대블록이 재발 → 알고리즘은 못 건드린다. 그래서 운영자 수동 병합(2차 큐레이터 판단)으로 해결.

전 / 후

전

  • 같은 사건이 카드 3장으로 분리
  • 각각 픽 = 요약 3회 + 썸네일 9장(구독 쿼터·Gemini 3배)
  • 피드에 같은 사건 중복 요약
  • 합칠 방법 없음

후

  • 길게 눌러 3장 → 1장으로 묶음
  • 한 번 픽 = 요약 1회(멤버 종합)
  • 매체 보도수 합산(union)
  • "＋N 병합" 배지 탭 = 해제

어떻게 쓰나

1
카드 길게 누름(600ms) → 병합모드 진입. 그 카드 = ⚓ 기준(앵커)
2
나머지 합칠 카드 탭 = 체크(카드 전체가 선택면 = 시안 B · 초록 틴트+접힌 코너 ✓)
3
우상단 병합 박스("병합 N") 누름 → 묶임
4
해제 = 묶인 카드의 ＋N 병합 배지 탭 · 취소 = X·ESC·뒤로가기·빈곳 탭

구현 — 핵심은 "기존 메커니즘 재사용"

병합의 본질 = 앵커 카드의 cluster_members(같은 사건 타매체 url 목록)에 멤버 url들을 합치는 것 하나. 그러면:

  • 종합 요약 = 픽 파이프라인이 이미 cluster_members를 alt로 분석기에 넘겨 종합 fetch·요약함 → 자동으로 멤버들 종합
  • 매체 보도수 = 합쳐진 url의 union 크기로 표시
  • 픽 상태(PICKED 등) = 멤버 url 매칭도 기존 코드가 처리

→ 파이프라인·큐레이션 알고리즘 0 변경. 뷰어가 fold해서 소비만. 원본 데이터는 복제라 불변.

조각위치
저장(이 기기)localStorage nm_merges (앵커→멤버들)
foldrenderScrap의 applyMerges — 멤버 접고 앵커에 합산(복제)
UI롱프레스 진입 + 풀카드 토글 + position:fixed 병합박스

장 · 단점

장점단점 / 한계
  • 알고리즘 불변 = 거대블록 위험 0
  • 운영자 직접 선택 = 오병합 0
  • 요약·썸네일 3배→1배 = 구독 쿼터·Gemini 절감
  • 기존 요소 위치 0 이동
  • 저장이 이 기기만(localStorage) — 폰↔PC 미동기 (v1 의도 · 필요 시 후속 cross-device)
  • 수동이라 매번 손으로 묶어야
  • 모바일 롱프레스 = 600ms 학습 필요

효율성

  • 런타임 — applyMerges는 병합 0이면 즉시 반환(대부분). 있어도 pool 1회 순회 = 무시할 비용.
  • 비용 절감 — 같은 사건 N장을 1픽으로 = 요약 N→1, 썸네일 3N→3장. 운영자 핵심 페인(3배 픽) 해소.
  • 코드 — 신규 메커니즘 0, 기존 cluster_members·pickAlt 재사용 = 추가 표면적 최소.

품질 저하 리스크 — 점검

  • ✓ 알고리즘 오염 0 — 클러스터링·신호·랭킹 무변경(뷰어 소비만). 큐레이션 방향 불변.
  • ✓ 원본 불변 — 앵커는 복제본에 합산({...anchor}), CANDS·다음 렌더 오염 0(유닛테스트 확인).
  • ✓ 그룹 실종 방지 — 앵커가 별점·PASS로 pool서 빠지면 멤버를 접지 않음(통째 사라짐 차단).
  • ✓ 기존 동작 보존 — 평소 탭(본문→url)·픽·스와이프 그대로. 병합모드일 때만 탭=선택(위임 capture로 가드).
  • △ 실기기 롱프레스 — 표준 패턴(기존 대기열 롱프레스 600ms) 계승했으나, 다기종 터치는 운영자 라이브 확인 권장.

검증

  • 핵심 fold 로직 유닛테스트 12/0 — cross union·종합 alt 순서·복제 불변·앵커 부재/폴백/자기참조 엣지
  • check_refs 그린 — JS 구문·참조 정합·디자인 거울(blur baseline 104→106 갱신)
  • 실기기 인터랙션 = 머지 후 라이브(pages.dev)에서 운영자 확인 권장
