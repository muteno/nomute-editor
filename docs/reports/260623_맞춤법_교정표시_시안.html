맞춤법 틀렸을 때 — 입력창에서 어떻게 보여줄까

셋 다 자동 교정(즉시)은 그대로. 차이는 "뭘 고쳤는지"를 얼마나 보여주냐. 각 카드의 이미지 제작을 눌러봐 (샘플 = 속보 됬어요 함니다).

A · 현재 라이브

토스트만

텍스트가 슥 바뀌고 "N곳 교정함"만 잠깐. 뭐가 바뀐진 안 보임 = 놓치기 쉬움.

B · 추천

변경 칩 + 입력창 노란 글로우

교정 후 텍스트 바뀌고, 입력창이 노랗게 깜빡 + 아래에 바뀐 것만 칩으로(됬어요→됐어요). 한눈에 보이고, 그대로 제작 진행(안 막음).

C · 확인형

제작 전 미리보기 → 적용/그대로

제작 누르면 먼저 "이렇게 고칠게" 패널이 떠서 고를 수 있음. 통제력 ↑, 대신 한 번 더 누름(제작이 1탭 늦어짐).

됬어요→됐어요 함니다→합니다

* 모든 안에서 *별표* 강조는 보존되고, 검사기 죽으면 무보정 진행(제작은 안 막힘). 실시간 빨간 밑줄(타이핑 중)은 입력창(textarea)이 네이티브로 지원 안 해서 제외 — 위 방식이 현실적.
