다운로드 버튼전 흰 글래스(이미지 위에서 눈에 덜 띔) → 후 강조색 초록 글래스 = 뉴스 요약부 버튼과 통일(일관성·발견성↑). 크기·위치·블러·눌림 모션 동일.
강조 누락 동작전 "강조를 안 넣었는데 진행할까요?" 확인 팝업(진행 가능) → 후 팝업 폐지, 앰버 버튼 흔들림 + 안내, 진행 불가(강조 필수화).
색 구분빨강(#ff7a7a) = 필수값 미입력 / 앰버(--amber #ff9614) = 강조 누락. 같은 흔들림·같은 위치, 색만 달라 한눈에 원인 구분.
효율errStatus에 색 인자 1개 추가(중복 0) + .go.warn 1줄 + --amber 토큰(index.html SSOT 값 복사). 팝업 await 제거 = 탭 한 번에 즉시 피드백.
리스크낮음. 강조가 정말 필수가 되므로(이전엔 우회 가능) 의도와 일치 확인 필요 — 운영자 요구가 "진행 불가"라 반영. reduced-motion이면 흔들림 생략·색은 유지.
