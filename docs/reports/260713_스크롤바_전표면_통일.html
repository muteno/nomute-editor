스크롤바 전 표면 통일 — 제작·합성(cscroll 오버레이) 표준 이식

260713 10:23 KST 접수 · 운영자 "이 스크롤이 기본 — 프롬프팅·영상·트렌드(메뉴4) 다 같게" · 헤드리스 8페이지+프롬프팅 부착 전부 PASS · check_refs rc=0

1. 전 → 후

BEFORE — 표면 제각각

song·track·nb = 크롬 기본(투박) · edit = 아예 숨김 · index/프롬프팅 = 네이티브 10px 필

→

AFTER — 전 표면 동일(기준 = 제작·합성)

cscroll 오버레이: 다크 알약(24~64px)·스크롤 중 밝힘·자리 안 차지함·모바일 동일

2. 장단점·리스크

장점

  • 9개 표면 스크롤바 단일 표준(모바일 webkit이 네이티브 커스텀 무시하는 문제까지 원천 해소 = cscroll 설계 목적).
  • index = 오버레이라 자리 미점유 → 모달 열림 시 폭 점프 원천 0(구 gutter 방어의 상위 호환).
  • 규칙을 CLAUDE.md §디자인 z)로 명문 = 새 뷰어 누락 드리프트 차단.

리스크

  • 오버레이 바 = 드래그 불가(pointer-events:none·기존 thumb과 동일 트레이드오프 — 스크롤 자체는 불변). 중
  • index 첫 배포 직후 gutter 소멸로 본문 폭 +10px 1회 리플로(이후 고정). 낮음
  • JS 실패 환경 = 네이티브 thin 필 폴백(전 표면 동일 블록). 낮음

3. 아이데이션(미적용)

  • 부착 모드를 다른 내부 스크롤러(요약 요청 탭·메시지함 등)로 확장 — 이번 스코프는 지시된 4곳. 제안
  • 오버레이 thumb 드래그 지원(pointer-events 선택 활성). 설익음
