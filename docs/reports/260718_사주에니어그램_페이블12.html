3. 음양 축 × 오행 축의 겹침 (서술 문법)

  • 오행 = 기질 재료(오상 배속): 목=인(仁)·성장 / 화=예(禮)·표현 / 토=신(信)·중재 / 금=의(義)·결단 / 수=지(智)·사유.
  • 겹침의 실제 문법 = 물상: 같은 오행도 음양이 갈리면 "같은 재료, 다른 그릇" — 갑(대들보)vs을(덩굴), 병(태양)vs정(촛불), 무(산)vs기(밭), 경(무쇠)vs신(보석), 임(바다)vs계(이슬). 통변 성격 서술 = 물상 비유 1줄 + 경향 서술의 조합.
  • 경계: 신강신약·조후가 겹치면 같은 일간도 반대로 통변됨(예: 신약 갑목 = "고집" 아닌 "흔들리는 자존"). 일간 층 위에 보정 층이 얹히는 구조 전제.

③ 60갑자 압축 분류법 — 개별 암기 대신 2축 매트릭스 + 태그

축A. 일지 십성 5군 (60개 → 5군): 비겁좌(주체형) / 식상좌(표현형) / 재성좌(현실형) / 관성좌(규범형) / 인성좌(수용형). 성격 방향타.

축B. 십이운성 4단 강도 (12운성 → 4단, 통설 양간 순행 기준·음간 역행은 이견 有): 왕(건록·제왕 = 증폭) / 생(장생·관대·목욕 = 신선·유연) / 쇠(쇠·병·사·묘 = 내향·전문화) / 휴(절·태·양 = 리셋·잠재)。 성격 볼륨.

① 날개(인접 유형 혼합) ↔ 명리의 혼합 기질

날개의 핵심 구조 = "핵심 유형 1개 + 양옆 인접 유형만 가미 → 순수형이 아닌 혼합 표현, 평생 비교적 안정".

명리 후보구조판정근거
지장간(여기·중기·정기)한 지지 안에 주 기운(정기)+부 기운 혼재◐"주 기질+잠재 부기질"의 층 구조가 날개의 '가미' 논리와 동형. 단 혼합 상대는 계절·삼합 논리로 고정 — 성격 원(circle) 위 '인접'과 무관
월지 사령·투간잠재 기운 중 무엇이 활성화되나○(형식)"두 날개 중 어느 쪽이 발달했나" ↔ "지장간 중 무엇이 사령·투출했나" — '잠재 중 활성 선택'이라는 형식이 같음
간지 조합(같은 일간·다른 일지)갑자 vs 갑오 = 같은 핵심, 다른 받침○(형식)"같은 핵심 유형, 다른 날개 = 다른 맛" 서사와 동형. 60갑자 = '핵심×수식어' 격자
십성 주·부 패턴최강 십성+차강 십성◐혼합 서사는 되지만 부 십성은 아무거나 가능 — 인접 제약이 없음

안 되는 대응(명시)

  • 인접 토폴로지 ✕: 날개는 원 위 양옆 2개만 가능(9번은 8·1날개만). 명리엔 "이웃 기운만 섞인다"는 제약이 어디에도 없음 → 번지수 매핑(예: 4w5=특정 간지) = 억지, 금지.
  • 동역학 차이 ✕: 날개는 평생 안정 성향 / 지장간 '개고'(형·충으로 열림)는 사건적. 유비는 원국의 고정 구성까지만.
  • 사령·투간으로 "네 날개는 이쪽"을 계산하는 건 유비를 산식으로 승격하는 것 — v2 검증 전 금지(v1은 "너도 순수형이 아니라 혼합이야" 메타포까지만).

STEP 1  중심 점수 (원국 8자 십성 분포 × ②의 가중표)

  S_가슴 = Σ식상 + 0.7Σ재성 + prior(일간 화 +0.5)
  S_머리 = Σ인성 + 0.7Σ관성 + prior(일간 수·금 +0.5)
  ※ 일지 십성은 ×1.5 가중(앉은 자리=몸·본능 자리) · 동점 시 일간 오행 벡터 우선
STEP 2  중심 = argmax(S)
STEP 3  스탠스 점수 (A안 기세 + 십성 정편正偏 축)
  주장 = Σ(겁재+상관+편재) + 양간 보너스   |  순응 = Σ(정관+정인+식신)  |  후퇴 = Σ편인 + 음간 + 식상·재성 과소 보너스

① 일주 산식 검증 — 산식은 안전, 사고는 전부 "날짜→일수 변환"에서 난다

1. 그레고리력 경계: 책의 JDN 공식엔 율리우스력용 혼입 위험 — JS에선 JDN 자작 금지, Date.UTC를 일수 카운터로(프롤렙틱 그레고리력 = 1900~2100 정확). 2. KST 자정 기준 = 타임존 무접촉: 입력이 KST 민간 날짜라 시간대 변환 개입 금지. 고전 버그 = new Date(y,m-1,d)(로컬 자정)÷86400000 → 브라우저 KST면 하루 밀림. 반드시 Date.UTC만(러너 UTC/폰 KST 동일 결과). 3. 자시 경계 = v1 밖 선언: 23시 이후 = 다음 날로 보는 유파(야자시) 존재. v1은 시(時) 입력이 없으니 "자정 교체·민간 날짜 기준"을 계약 주석에 명문화 + 서사에 "밤 11시 이후 출생은 유파별 다음 날 일주" 1줄 고지(1954~61 KST=UTC+8:30 역사도 같은 축). 4. 치명 함정: JS % = 음수 나머지 — 1900~1969년생은 Date.UTC가 음수 ms → ((x%60)+60)%60 가드 필수. 주 사용자층이 정확히 이 위에 있다.

권고 정본 산식 (앵커 6개 교차검증 완료 · JDN 표현 (JDN+49)%60과 동치 실측):

const days = Math.floor(Date.UTC(y, m-1, d) / 86400000);
const cycleIdx = (((days + 17) % 60) + 60) % 60; // 0=갑자
const ganIdx = cycleIdx % 10, jiIdx = cycleIdx % 12;
