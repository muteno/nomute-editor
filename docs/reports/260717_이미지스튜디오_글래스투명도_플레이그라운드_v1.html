  • v1→v2 충실도 교정(운영자 "생각보다 비친다" 반영): v1은 단일 패널을 밝은 배경 바로 위에 얹어 실제보다 과다 비침이었어. v2 = 실제 모달 그대로 2겹{셸 --modal-glass(.64) + 내용부 오버레이 rgba(24,25,28,α)} + 뒤 ::backdrop 딤(.46)·블러 재현 → 실효 비침 = 셸 합산값(오버레이 α와 다름 · 슬라이더에 실효값 표기). 배경 피드는 근사(실제 feed_bg 아님).
  • 실제 코드 레버: #2 불투명 원인 = 내용부(.genihost/.toolfr)가 --modal-tabs-bg(.5)를 셸(.64) 위에 덮음. 운영자 지정 rgba(24,25,28,.50) = 바로 그 --modal-tabs-bg 정본값(계승) → 실효 비침 .82(살짝 비침·글자 또렷). 더 글래스로 = α를 0으로(=셸만=#3, 실효 .64).
  • ⚠ 프로스트 블러 주의: 내용부에 backdrop-filter를 새로 얹으면 내용 입력칸 타이핑마다 뒤 재샘플 → 모바일 번쩍(기틀 260701 금지). 프로스트는 뒤 ::backdrop(정적)이 전담 — 블러 슬라이더는 그 backdrop을 움직이는 것(내용부 아님 = 안전).
  • 헤더·탭 띠는 불투명 유지(.85/.5) — 타이틀·탭 가독. 조작 대상은 내용부 오버레이 α뿐.
  • 결정 = "복사" 값 회신 → 라이브 반영. .50·0(셸만)·.85는 계승(새 토큰 0) · 그 밖 = 갱신(토큰 절차).
