3인 적대적 아이데이션 + 종합으로 확립한 구현방안 · 260627

한 줄 요약 — 카드 글자수정(연필) 모달에 뉴스요약의 사진 첨부 버튼을 그대로 이식한다. 4:5 사진을 첨부하면 일러스트 대신 그 사진이 깔리고 글자 오버레이가 위에 얹힌 미리보기가 즉시 뜨고, 그 버튼을 한 번 더 누르면 게이지가 차며 합성된다. 합성은 이미 있는 정본 엔진 (recompose_card.py → card_news.py)이 처리 → 제미나이 호출 0 · 과금 0, 전/후 비교는 기존 버전 네비로 공짜로 따라온다.

📍 인터랙티브 목업 — 직접 눌러봐

아래는 제안 UX를 실제 토큰·컴포넌트로 재현한 거야. 우측 버튼으로 시나리오를 시뮬레이션해봐.

📍 확정 설계 (3안 만장일치)

  • 합성 위치 = 백엔드 정본 단일 (recompose_card.py가 이미 "임의 4:5 씬+텍스트" 합성용). 클라 캔버스는 미리보기 근사 전용 · 절대 업로드 안 함 — 웹폰트가 백엔드 NotoSansCJK와 미세히 달라 정본화하면 거짓 픽셀이 R2에 영구 저장되는 회귀.
  • 전송 = make-cards.js를 compose.js 패턴으로 흡수 (단일 엔드포인트). 사진 base64 → uploads/<id>/src.jpg 커밋 → dispatch에 scene·scene_sha 추가. 신규 함수·comp-make 직접호출 기각.
  • edit_one 주입 = regen 계산 직전 1지점 — EDIT_SCENE 있으면 그 사진을 scene_local로 복사 + regen=False 강제 → gemini_image 블록 스킵 = 제미나이 0.
  • 2단 버튼 = 첨부버튼 자체 (운영자 원문 직역). 게이지는 신규 CSS 창작 없이 기존 edLaunch firing→✓ 시퀀스를 첨부버튼에 재적용 → 컴포넌트 중복 0.
  • 4:5 검증 = 클라 ±2% tolerance + 어긋나면 기존 빈입력 shake/flash 가드 계승. 백엔드 ImageOps.fit(중앙 0.4)이 미세오차 흡수.
  • 전/후 = 코드 추가 0 — _save_versions가 v0=원본 일러스트·v1=사진 합성본 보존 → 기존 .slide-ver 전/후 네비 자동.
