무엇이 바뀌나 (쉬운 설명)

전에는 영상을 넣으면 받아쓰기(STT)가 끝나도 AI가 자막을 다 다듬을 때까지(1~5분) 화면에 아무것도 안 떴어. 이제는 받아쓰기가 끝나는 그 순간 편집기가 먼저 뜬다 — 기다리는 동안 받아쓴 자막을 미리 손볼 수 있고, 다듬은 완성 자막은 도착하는 대로 개요 탭에 조용히 채워진다(보던 화면 안 튕김·하던 편집 안 날아감).
영상 발사→받아쓰기 STT→편집기 즉시 등장 ←NEW→AI 다듬기 1~5분→완성 자막 개요 탭 합류

장단점 · 효율성 · 품질 리스크

장점
  • 다듬기 대기가 죽은 시간 → 편집 시간으로 전환(다듬기가 길수록 이득 큼)
  • best-effort 설계 — 조기 푸시가 어떻게 실패해도 최종 커밋이 포함 = 자막 유실 0
  • 롤백 한 줄(api/ly.js의 early_segs 필드 제거)
트레이드오프(수용)
  • 영상·파일 작업당 main 커밋·Pages 배포 +1회 — ly는 운영자 수동 발사라 희소(하루 몇 건) = 봇 트래픽(일 51커밋 실측) 대비 미미. 빌드 쿼터가 신경 쓰이면 알려줘
  • 조기 도착의 실효는 Pages 전파(약 1~2분)를 뺀 값 — 짧은 다듬기(약 90초)면 이득 미미, 긴 다듬기(3~5분)면 큼
  • 편집기가 미리 뜨는 만큼 「이미 완성?」 오해 여지 → 상태줄이 「받아쓴 자막 먼저 손봐도 돼 · 다듬는 중… Ns」로 계속 정직 표기
품질 저하 리스크
  • 다듬기 품질·자막 골격 = 무접촉(커밋 시점만 추가 · 총괄 평의회 기틀 판정 = 주변부)
  • 조기·최종 전사 데이터 바이트 동일 실측 = 두 화면이 다른 자막을 보일 가능성 0
