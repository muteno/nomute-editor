나노바나나 비율 재구성 도구 — 되냐? 된다, 이렇게

완성 이미지(예: 9:16 포스터)를 레이어 없이 → 구성 살리며 다른 비율(16:9 등)로

결론 = 조건부 GO. 만들 수 있고, 핵심 엔진은 레포에 이미 90% 깔려 있다(gemini_image()가 이미지 입력 ref_png + 목표비율 aspect를 받음). 다만 나노바나나엔 진짜 아웃페인팅(마스크)이 없어서 그냥 시키면 원본까지 재생성해 미묘하게 바뀐다. 그래서 알고리즘의 척추는 ─

패딩(PIL) → 나노바나나가 빈 곳만 채움 → 원본 픽셀 재부착(픽셀락)

이고, 한글 문구·로고는 절대 모델에 안 맡기고 격리(레포가 이미 쓰는 방식)한다. 착수 전에 공짜 실험 1판(v0)으로 "진짜 되나"부터 확인하는 걸 강권.

1실측으로 확인된 사실 4개

  • 엔진 이미 있음 — thumb_gen.gemini_image(prompt, image_size, aspect, ref_png)가 이미지 첨부(inlineData)+비율 지정(aspectRatio 4:5/16:9/9:16 등 10종)을 이미 지원. 신규 API 통합 0.
  • 마스크·아웃페인팅 공식 미지원 — 나노바나나 편집은 전부 전체 재생성 기반. "원본 건드리지 마" 프롬프트는 확률적 완화일 뿐 보장이 아님(공식 문서·업계 공통 확인). → 원본 보존은 후처리 재부착이 책임져야 함. Photoshop Generative Expand·Canva·Midjourney 전부 같은 사상(원본 불변 + 빈 곳만 생성 + N안 선택).
  • 한글 문구 = 최대 킬 리스크 — 레포 코드 주석에 이미 실측돼 있음("readable Korean text renders broken" · "1K는 한글 자모 뭉개짐"). 포스터는 곧 문구라서, 문구를 모델이 다시 그리는 순간 도구가 아낀 게 0이 됨. → 문구·로고는 PIL로 격리(레거시 선례 = recompose_card.py: 장면 보존+문구 재합성, 제미나이 0).
  • 비용은 사소 — 1K 한 장 ≈ $0.067(2K $0.101). 일 3~10회 써도 월 $3~24 수준. 진짜 비용은 지연(20~60초)과 러너 deps라서, 단순 배경은 공짜 PIL 경로로 돌리는 게 이득.

2제안 알고리즘 — 3층 라우팅

모든 이미지를 생성 AI에 넣지 않는다. 배경 복잡도를 보고 가장 싼 경로부터:

판정은 가장자리 픽셀 분산(단색 여부)으로 자동 — 네 직관("복잡한 배경 아니면 툴로 충분")이 그대로 라우팅 규칙이 된다.

32층(생성 경로) 상세 — 패드 → 필 → 픽셀락

1
패딩(전처리) — 타겟 비율 캔버스에 원본을 배치(피사체 치우침이면 반대쪽을 더 확장 — 레포 detect_subject_center 재사용), 빈 영역은 중립 그레이.
2
생성 1콜 — 그 캔버스를 ref_png로 첨부 + "회색 영역만 원본의 조명·원근·질감 이어서 채워라, 원본 변경 금지" 고정 프롬프트 + aspect=타겟.
3
픽셀락(후처리) — 결과 위에 원본 픽셀을 그대로 재부착(원본 100% 보존 보장), 경계 20~40px만 페더 블렌딩(레포 create_gradient_overlay의 알파 램프 재사용). 모델이 원본을 얼마나 바꿨든 무관해짐.
4
결정론 검증(공짜) — 비율 확인·디코드 검증·빈 밴드 검출(_band_fail 재사용)·이음새 톤 비교 → 미달이면 프롬프트 강화 재시도 1회 → 그래도 실패면 blur-pad 폴백(항상 결과는 나옴).

5장단점 · 품질저하 리스크

리스크정도방어
한글 문구·로고 재생성으로 깨짐최대모델에서 격리: 문구 영역이 확장부에 안 걸리게 패딩 설계 + 원본 문구는 픽셀락으로 보존. 문구를 새 영역까지 옮기고 싶으면 그건 PIL 재타이핑(0층). OCR·요소분해·클린플레이트 인페인팅은 과설계라 안 함(10인 만장일치).
이음새 위화감(색톤·조명)중페더 블렌딩 + 이음새 톤 자동 비교 → 초과 시 재시도. N안 생성 후 눈으로 픽(v2).
확장부 환각(워터마크·글자)중금지 프롬프트 + 결정론 검출 + 재시도.
인물 얼굴중얼굴은 원본 영역에 있으므로 픽셀락이 커버. 확장부에 새 인물 생성 금지 지시.
안전필터 차단·API 실패하기존 fail-soft(백오프·None) 상속 → blur-pad 폴백 → 최후엔 원본 그대로 반환(절대 안 죽음).

6효율성 (비용·재사용)

  • 과금: 복잡 배경만 Gemini(1K ≈ $0.067/장) — 라우팅으로 절반 이상은 공짜 PIL. 월 몇 달러 수준. 수동 발사 전용(workflow_dispatch·GEMINI_API_KEY 게이트)이라 자동 과금 0 불변, 카드 제미나이-0 잠금 무접촉.
  • 재사용(이미 있는 것): 렌더 gemini_image · R2 업로드 · 업로드 배관(compose.js의 base64→git 선커밋→SHA 가드 판박이) · 결과 그리드+폴링(이미지 생성 탭 그대로) · 피사체 검출·페더·빈밴드 검출 전부 기존 함수.
  • 신설(최소): resize_image.py(스크립트 1) · img-resize.yml(얇은 워크플로 1) · api/resize.js(compose.js 복붙 수준 1) · 뷰어 탭 1(결과 JSON은 free.json 모델). 러너 deps는 PIL만(cv2 안 씀 = 설치 세금 0).
