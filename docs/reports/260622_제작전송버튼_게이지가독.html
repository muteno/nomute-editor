버튼 진행모션 — 초록 풀필로 통일 (구성도 정본)

260622 · thumb·k·ly·comp.html(제작/생성) + index.html(전송/수정) · 정본 = 구성도/전송 버튼 상태머신 .send

🔎 운영자 피드백 2건 반영

① 제작 버튼: "게이지가 버튼 가득 차야 되는데 하단에만 라인 타고 가니 어색" → 버튼 전체 초록 채움으로.
② 전송 버튼: "검정으로 찰 게 아니라 구성도랑 동일하게" → 초록 채움으로(글래스 바탕이라야 초록이 보임).

💡 아래 버튼 직접 눌러볼 수 있음. 전/후 비교.

📍 1. 제작 버튼 (이미지 제작 · 생성)

전
하단 3px 라인만 좌→우
후
버튼 전체 초록 채움 → "전송됨 ✓"

썸네일 이미지 제작 + k·ly·comp 생성 버튼(.go) 전부 동일 적용. 채움 = background-size 0%→100% 좌→우(구성도 .send.is-fire와 동일 비주얼). 글자는 초록 채움 위 어두운색(--on-accent)이라 가독.

📍 2. 전송 버튼 (요약요청 전송 · 수정요청)

전
보내는 중…
검정 채움 → "전송" 글자 소멸
후
보내는 중…
초록 채움 → 글자 끝까지 읽힘

밝은초록 솔리드 버튼 → 글래스 바탕(accent-dim)으로 바꿔야 초록 채움이 보인다(구성도 .send idle과 동일). .asksend·.revsend 둘 다 적용.

📍 3. 전수 점검 (firing 버튼 패밀리)

버튼전후
제작/생성 .go ×4하단 3px 라인버튼 전체 초록 풀필
전송/수정 .asksend·.revsend검정 채움(글자소멸)글래스+초록 풀필
PICK .sc-pick이미 초록 풀필 = 정본 레퍼런스(무변경)
카드 슛 .mkbtn하단바(별 컴포넌트·요청 범위 외·무변경)

통일 규칙 = "눌림 = 초록이 버튼 전체를 좌→우로 채움 + 어두운 글자"(구성도 .send.is-fire). .sc-pick(PICK)이 이미 그 정본이라 무변경.

📍 4. 분석

전 → 후
제작: 하단 3px 라인 → 버튼 전체 초록 풀필.
전송: 검정 글자소멸 → 초록 풀필·가독.
장점
· 구성도 정본 .send 100% 계승 = 전 메뉴 일관.
· 채움이 또렷·풀버튼이라 진행감 확실.
· 모션 줄이기에도 색/라벨로 보임.
효율
· 순수 CSS(LLM 0·과금 0)·GPU 컴포지트.
· 0%(퍼센트)+명시 from = 부드러운 보간(0 길이면 끝에서 툭 튐 → 교정).
적용 범위
thumb·k·ly·comp .go + index .asksend·.revsend = 6곳. 제작 버튼은 "전송됨 ✓" 완료비트 유지.
⚠ 품질저하 리스크 — 낮음. 버튼 CSS만(동작·payload·polling 무손상)·국소 뷰어(:root·배너·파이프라인 무관)·check_refs(참조·JS구문·디자인거울) 통과·헤드리스 렌더로 좌→우 채움 실측. 전송 버튼 바탕이 밝은초록→글래스로 바뀜(의도 — 구성도 동일·PICK과 톤 일치).
