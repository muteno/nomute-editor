아이콘 버튼 누름효과 + 전송 제언

D1~D3 형태는 유지(stroke 2.0·SVG X·refresh SVG) — 누를 때 더 살아있게 6안. 그리고 전송버튼 개선 제언(네 상태머신 위에).

📍 아래 아이콘들 직접 눌러봐 — 효과별로 손맛 다름. 전송버튼도 끝까지 밟아봐.

① D1~D3 — 누를 때 액티브 효과 (6안)

버튼 = 닫기X · 새로고침 · 다운로드 · 복사(공통 아이콘). 아래로 갈수록/조합할수록 살아있음.

P1
현재 — 축소만

scale .85. 밋밋(지금 상태).

P2 · 기본 추천
눌림 + 밝기 + 글래스 틴트

scale .86 + 살짝 어둡게 + 누르는 순간 라임 글래스 틴트. 모든 버튼 공통 베이스.

힘있는 클릭감. 다른 효과의 토대.

P3
탭 리플(radial)

누른 지점에서 라임 물결이 퍼짐(머티리얼식).

촉각적·생동감↑. 작은 버튼엔 다소 화려.

P4
링 펄스(테두리 확산)

떼는 순간 라임 링이 한 번 퍼짐(요약 헤더 펄스와 통일).

요약 버튼 펄스와 같은 언어 = 통일감.

P5 · 강력 추천
시맨틱 아이콘 모션

아이콘이 자기 의미대로 움직임 — 새로고침=360° 회전 · 닫기=90° 회전 · 다운로드=↓바운스 · 복사=체크로 변신. 직관적이고 만족감 큼(P2 눌림 위에).

새로고침 빙글·복사 체크변신이 특히 좋음. 의미와 모션 일치.

P6
글로우 플래시

떼는 순간 라임 글로우+색이 확 켜졌다 꺼짐.

화려·주목도↑. 자주 누르면 과할 수.

💡 추천 조합 = P2(눌림 베이스) + P5(시맨틱 모션). 더 살아있게 원하면 거기에 P4(링 펄스)까지. 골라줘 — 예 P2+P5 또는 P2+P5+P4.

② 전송 버튼 — 개선 제언

네 상태머신(재확인→전송→완료)은 좋음. 거기에 내가 보탤 4가지(아래 직접 밟아봐).

라이브 — 눌러서 끝까지

눌러봐 — 첫 탭은 재확인(아직 전송 안 함)

  • 재확인 카운트다운 링 — 3초 타임아웃이 안 보였음 → 버튼 둘레 앰버 링이 소진되며 "곧 취소됨"을 시각화(오발사 방지 + 시간 인지).
  • 비동기 = indeterminate shine — 실제 생성은 분 단위 async인데 .72s 게이지는 "곧 끝"이라 거짓말. 끝 모를 작업은 좌→우 무한 shine "Picking…"이 정직(블루프린트 after+ §6과 일치).
  • 실패 = shake + red — 조용히 사라지지 말고 좌우 흔들림 + 빨강으로 즉시 인지 → 재시도 유도.
  • 완료 settle + 눌림 일관 — 체크 pop 후 안정 · 누름감은 아이콘버튼(P2)과 동일 토큰.

접근성: 재확인 aria-live · 전송중 aria-busy · reduced-motion이면 링/shine/shake 자동 생략.

📌 답 예: P2+P5 · 전송 4개 다 ㄱ. 고르면 viewer에 한 번에 반영(수정 모드+§✅ 검증).
