한눈에

컨트롤이 다 알약(도형) 안에 있던 걸 도형 없이 글자만으로. 포스트·릴스는 한 번에 하나만 보이고 탭하면 바뀜(썸네일 메뉴와 똑같은 방식). 기능·결과물은 그대로.

• 포맷 포스트|릴스
• 저작권 ON
• 안내문 OFF
포스트·릴스 둘 다 보임 · 모두 알약 박스 안

후 (AFTER)

• 포맷 릴스
• 저작권 ON
• 안내문 OFF
릴스 하나만 보임(탭→포스트) · 박스 없이 글자만 · 탭해서 직접 확인 ↑

동작 전→후

컨트롤전후
포맷[ 포스트 | 릴스 ] 둘 다 보임(알약)릴스 하나만 → 탭하면 포스트
저작권[ ON ] 알약ON 글자만(초록)
안내문[ OFF ] 알약OFF 글자만 → 탭하면 팝업(경고/AI/자유문구)

장단점 · 효율성 · 품질 리스크

  • 일관성↑ — 포맷 토글이 옆 썸네일 메뉴의 토글(#ovFmtTog)과 똑같은 컴포넌트(.ovfmt)를 그대로 계승. 화면마다 다르게 생기던 것 통일.
  • 군더더기 제거 — 알약 테두리·배경이 빠져 글자만 남으니 노뮤트 다른 화면(평면·검정)과 톤 맞음.
  • 효율 — 기존 토글 패턴 재사용 = 새 디자인 0 · 코드 라인·CSS(raw hex 2줄) 오히려 감소.
  • 품질저하 리스크 ≈ 0 — 발사 로직·백엔드(/api/thumb)·상태값(cpFmt·copyOn·guideMode) 전부 그대로. 흰칸=고정 불변 → 렌더 결과 회귀 없음.
  • 학습 포인트(소소) — 포맷이 한 번에 하나만 보이니, 처음엔 "다른 포맷은 어디?" 할 수 있음. → 탭하면 바뀌는 건 썸네일 메뉴에서 이미 쓰던 방식이라 금방 익숙(타이틀 툴팁도 안내).
