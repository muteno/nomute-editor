실측 (합성 얼굴 영상 · 8s · 1s 퇴장 재등장 포함)

  • 분석: 인물 1명으로 재군집(퇴장 전·후 트랙 2개 → 임베딩 병합) · 검출 스텝 2f(초당 15회)
  • 렌더 고퀄 5원칙: 갭 보간 · 3탭 이동평균 스무딩(지연 0) · 시간 마진 ±0.3s · 같은 인물 트랙 간 ≤1.2s 갭 브리지 · 공간 마진 1.45×/1.6×+위 10%(이마·머리카락)
  • 반전 토글: "선택 빼고 다 가리기" — 인터뷰이만 노출하는 뉴스 관례를 탭 하나로

장 / 단점

장점폰 완결 워크플로 · 드리프트 0 · 재등장 자동 재획득 · 분석 캐시로 재렌더 무한 · 과분할>과병합 편향(남남 병합 = 엉뚱한 얼굴 모자이크 원천 차단 — 같은 사람 카드 2장은 둘 다 선택하면 끝) · 모델 sha256 핀 + 7일 캐시
한계(정직)옆·뒷모습·초소형(<12px) 얼굴 미검출 가능(마진·브리지가 단기 가림은 커버) · VFR 폰영상 립싱크 미세 드리프트 가능(모자이크 위치는 무영향) · 수동 구간 보정 미구현(실패 시 재분석) · 길이 캡 3분 · 업로드 ≤30MB(큰 영상 = URL)

효율성

  • 60초 영상 기준: 분석 ~1분 + Pages 배포 1~3분 · 렌더 ~2분 — 러너 CPU만(과금 = Actions 분 + R2)
  • 렌더 모드 = 모델 다운·캐시 스킵(37MB 절약) · site-packages·모델·apt .deb 전부 7일 캐시(ffmpeg·한글폰트는 ly와 캐시 공유)
  • 결과 = stable 키 덮어쓰기 + ?v= 버스트 → 재렌더 반복해도 R2 고아·git 비대 0
