## 2. 사건 일지 (KST · 로그 원문 시각 = UTC라 +9 해석)

1. **낮 — "드라이브 동기화 멈췄어"**: 과거 기록 탐색 → 정본 문서(드라이브싱크 플레이북 · 260702 사고 때 박아둔 것) 발견 → **⓪ 10초 판별**(마지막 실행 N분 전 + 로그 꼬리) 신설·실측.
2. **판정**: Tasker·rclone 인증 = 무죄(2분 전 실행 + 16:44 카피 성공). 진범 = **15:45 `rclone copy` 구글 400 실패인데 v1.0 스크립트가 실패 배치도 SEEN(장부)에 등록 → 영구 실종**(260702와 동일한 SEEN 오염 — 유발원만 다름).
3. **복구·정밀화**: `--max-age 48h` 수동 카피로 구멍 메움(2파일 · 400 무재발). 사후 실측으로 400 원인 = **수백 파일 `.app` 폴더 통째 업로드**가 파일별 조회(`--no-traverse`) 쿼리를 폭발시킨 것 — 특수문자 아님.
4. **v1.1 패치**: 복사 **성공시에만** SEEN 등록 + `--no-traverse` 제거 + 실패 시 `--id` 고정 알림 1장. 폰 반영 확인(grep=1).
5. **v1.2 → v1.2.1 — 역방향 업로드 신설**: `drive-camera-up.sh`(첫 실행 = 기준선 → 이후 새 파일만 · 업로드 전 다운싱크 SEEN 선등록 = 에코 차단 · 다운싱크 말미 체인 = Tasker 무변경). 운영자 확정으로 **LTE 허용(`WIFI_ONLY=0`) · Shared 바닥 평평**. rclone 승인 질문 = **신규 승인 불필요**(scope=drive 실측 통과).
6. **저녁 — "다운로드가 어느 순간부터 안 됨"**: `bad interpreter: python3.13` = **pkg upgrade가 파이썬을 판올림(3.13→3.14)하면서 pip 도구(yt-dlp·gallery-dl)가 통째로 고아화**된 것. 낡은 패키지 목록(404) 때문에 재설치도 헛방 → `pkg update` → `pkg upgrade -y` → `pip install -U yt-dlp gallery-dl`로 부활.
7. **밤 — "업로드가 다시 안 되네?" = 오진 판정**: 실측 결과 업로드는 연속 정상(당일 스크린샷 7장 전부 자동 도착 · 20:39~20:43 런 로그 + lsf 교차 확인). 안 보인 이유 = **드라이브 웹 이름순 정렬에 PC 파일 사이로 묻힘**. "다운로드가 루틴을 끊는다" 가설 = 기각(딴 타이머·딴 장부 · 틱 실패 = 다음 틱 자동 복구 설계).
8. **폭탄 발견**: rclone 로그 NOTICE — **공용 client_id가 2026년 중 은퇴 예정**. 방치 시 어느 날 다운·업 전 파이프 일제 사망. 대비 절차 정본 등재(§4).

## 3. 오늘 확립된 법칙 (판례)

1. **실패도 '본 것' 처리하는 낙관 기록 = 조용한 영구 실종** — 상태 기록(SEEN)은 성공 확인 뒤에만.
2. **폴더 통째(수백 파일) 업로드가 파일별 조회를 깨뜨린다** — 맥 앱 같은 건 zip 1개로 올리는 게 안전(파일 1개 = 배치 1개).
3. **`pkg upgrade` 파이썬 판올림 = pip 도구 전멸**(`bad interpreter`) — 처방 3연타 = `pkg update` → `pkg upgrade -y` → `pip install -U yt-dlp gallery-dl`. rclone은 독립 바이너리라 무사.
4. **Termux 로그 시각 = UTC(KST−9)** — 15:45 사고가 로그엔 06:45로 찍힌다.
5. **"안 보임" ≠ "안 올라감"** — 드라이브 웹은 이름순 정렬이 기본이라 자동 업로드분이 묻힌다 → 정렬 = "마지막 수정 날짜"로.
6. **rclone 공용 출입증(client_id)은 시한부(2026 은퇴)** — 자체 client_id 발급이 예정된 과제(테스트 모드 게시 = 토큰 7일 만료 함정 주의).

## 5. 남은 일 (인수인계)

- **자체 client_id 발급**(운영자 GO 대기 · 15분 1회성 · 올해 안 시한부): https://rclone.org/drive/#making-your-own-client-id → 발급 후 `rclone config update gdrive: client_id <ID> client_secret <SECRET>` → `rclone config reconnect gdrive:`.
- 카메라 실사진 1장 교차 확인(스크린샷 축은 실증 완료 — 동일 코드 경로).
- Tasker 공유 다운로드(화면 안 뜨는 입구) 실사 1회.
- `~/bin/media-handler` 전문 백업(레포에 없는 유일한 폰 단독본) + "실패해도 ✅완료로 뜨는" 표시 결함 수리.
