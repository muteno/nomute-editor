✅ 3단계 (순서 중요!)

  1. 페이지를 비즈니스에 먼저 등록 (← 확정된 뿌리 원인)
    "자산 추가" 창에 "페이지" 유형이 안 뜨면 = 이 비즈니스에 페이지가 0개란 신호(있는 자산만 할당 가능).
    계정 → 페이지 → [추가] → 페이지 추가 → 개인계정이 관리하는 Nomute 등록. (⚠ 로그인한 개인 FB가 그 페이지 관리자여야 뜸 · 다른 비즈니스에 묶였으면 "이미 다른 포트폴리오" 표시)
    계정 · 페이지 열기 →
  2. 봇한테 페이지 담당 권한 주기 (← 지금 여기가 빠짐)
    시스템 사용자 → nomute-bot → [자산 추가] → 반드시 "페이지" 탭(❌"앱" 아님) → Nomute → "콘텐츠 관리(Manage Page)" 전체 ON → 저장.
    또는 계정 → 페이지 → Nomute → 사용자 추가 → nomute-bot → 콘텐츠 관리. (앱 할당창의 토글은 페이지랑 무관)
    시스템 사용자 열기 →
  3. 토큰 다시 만들어 시크릿 교체
    위 1·2를 먼저 끝낸 뒤 → nomute-bot 에서 새 토큰 생성(권한: pages_read_engagement · pages_read_user_content · read_insights) → GitHub 시크릿 FB_ACCESS_TOKEN 교체.
    GitHub 시크릿 열기 →
⚠ 순서 주의: 1·2(연결)를 먼저 하고 → 그 다음 토큰 재생성. 할당 전에 만든 토큰은 옛 권한이라 안 붙을 수 있어(이게 이번에 헤맨 진짜 이유).
