① 스레드 부계(가계정) 연결 폰 Termux

뭐냐 — 스레드는 인스타랑 같은 Meta라 GitHub 서버 IP를 차단한다. 그래서 스레드만은 서버 말고 네 폰(가정 IP)에서 30분마다 긁어(phone_subs.sh). 게스트(비로그인)로도 긁히지만, 부계로 로그인한 세션쿠키(THREADS_COOKIE)를 넣어주면 더 안정적으로·더 많이 보인다. 본계 말고 반드시 부계(자동화 감지 밴 리스크).

A. 부계 세션쿠키 뽑기 (PC 크롬이 제일 쉬움)

  1. https://www.threads.com 에 부계정으로 로그인.
  2. F12(개발자도구) → 상단 Network 탭 → 페이지 새로고침(F5).
  3. 맨 위 threads.com 요청 클릭 → Headers → Request Headers의 cookie: 줄 값을 통째로 복사. (최소한 sessionid=… 은 꼭 포함)
폰만 있으면: Kiwi 브라우저(안드로이드 · 크롬 확장/F12 지원)로 같은 과정. 단 sessionid는 httpOnly라 document.cookie(북마클릿)로는 안 나온다 → 반드시 Network/Application 패널에서 복사.

B. 폰 Termux에 쿠키 심기 (레포에 안 올라감 · 폰 로컬 시크릿)

echo "export THREADS_COOKIE='여기에_복사한_쿠키_붙여넣기'" >> ~/.nomute_phone_env
~/.nomute_phone_env = 크론이 매번 source 하는 폰 전용 비밀파일(재난문자 SAFETY_KEY랑 같은 자리). git 커밋 0. 넣고 나면 다음 30분 크론부터 로그인 상태로 수집.

C. 수집할 스레드 계정 등록 (누구 걸 긁을지 — 부계 로그인과 별개)

웹앱 → 트렌드 탭 → 계정 버튼 → 스레드 탭에서 대상 핸들 추가(권장). 최대 10개. 등록 즉시 다음 크론부터 수집.
확인: 폰에서 아래 실행 → threads N건 뜨면 성공.
cd ~/nomute-editor && bash scripts/phone_subs.sh; tail -3 ~/phone_subs.log

② 폰 클립보드/캡쳐 → 공유드라이브 (폰 → PC) 폰 Termux

지금은 Termux가 gdrive:Shared(ems1130g@gmail.com · 전 PC 저장경로가 여기로 자동 동기화)를 받기만 한다(drive-gallery-sync.sh). 여기선 같은 리모트에 폰이 자동으로 올려서 모든 PC가 받게 한다 = 정확히 반대 방향(텔레그램 우회 제거). 정본 스크립트 = docs/termux-drive-gallery-upsync.sh(갤러리) · docs/termux-drive-clipboard-upsync.sh(클립보드). 루프 차단·베이스라인·오프라인 아웃박스 내장.
⚠️ 딱 하나 걸림돌 — 쓰기 권한. 드라이브싱크를 scope=drive.readonly로 잡았으면 받기만 되고 올리기는 막힌다. 한 번만 아래로 scope drive(전체) 재동의:
rclone config reconnect gdrive:
브라우저 뜨면 전체 권한으로 동의. 이후 받기·보내기 둘 다 됨. (이미 drive 전체로 잡았으면 이 단계 건너뜀.)

설치 = 단계별 순서도가 정본

복사 버튼·성공신호까지 있는 순서도로 깔아 → docs/reports/260713_폰설치순서도_업싱크스레드.html(A 갤러리 · B 클립보드 · C 스레드). 두 스크립트를 ~/.termux/tasker/에 두고 Tasker에 문다: 갤러리=DriveSync 작업에 액션 추가(3분·잠금 공유) · 클립보드=ClipSync 작업 + "클립보드 변경" 이벤트(안드10+는 Tasker에 READ_LOGS 1회).

확인

rclone lsf -R gdrive:Shared/FromPhone; tail -5 ~/.upsync.log; tail -5 ~/.clipsync.log
UP=올림 · PASS(traveled/dup)=오고간·중복 스킵 · baseline=첫 등록. PC "내 드라이브 스트림"이면 G:\내 드라이브\Shared\FromPhone에 곧 나타남.

③ 윈도우 CapsLock 맥처럼 Windows

원하는 것: 짧게 탭 = 한/영, 길게 = 진짜 CapsLock(대문자 고정) — 맥 한글 키보드 그대로.
Shift + 알파벳 = 대문자는 윈도우 기본 동작이라 아무것도 안 해도 된다(자동). 우리가 흉내낼 건 CapsLock의 탭/길게 이중 역할뿐.

추천 = AutoHotkey v2 (탭/길게 이중 동작을 정확히 재현 · 무료)

PowerToys로도 CapsLock→한영 매핑은 되지만 탭/길게 구분은 안 된다 → 이 요구엔 부족. AutoHotkey가 정답.
  1. https://www.autohotkey.com → AutoHotkey v2.0 설치.
  2. 레포의 완성 스크립트 docs/pc-downloader/CapsLock_한영_맥방식.ahk 를 더블클릭 → 즉시 동작(트레이 'H').
  3. 부팅 자동 실행 = 시작프로그램 폴더에 넣기:
    Win+R → shell:startup
    열린 폴더에 위 .ahk(또는 바로가기)를 복사.

스크립트 전문 (파일 그대로 · 메모장 붙여넣고 이름.ahk로 저장해도 됨)

#Requires AutoHotkey v2.0
#SingleInstance Force

*CapsLock:: {
    st := A_TickCount
    KeyWait "CapsLock"                                  ; 손 뗄 때까지
    if (A_TickCount - st < 250)                         ; 250ms 미만 = 짧은 탭
        Send "{vk15sc138}"                             ; → 한/영 전환
    else                                                ; 길게 = 진짜 CapsLock 토글
        SetCapsLockState !GetKeyState("CapsLock", "T")
}
한/영이 안 바뀌면 Send "{vk15sc138}"를 Send "{RAlt}" 또는 Send "{vk15}"로 바꿔 시도(키보드·IME 편차). 길게 임계값은 250(ms) 숫자를 늘리/줄이면 됨.

대안 (참고)

  • CapsLockIME (github.com/iolate/CapsLockIME) — CapsLock→한영 + 키보드 LED 제어 전용 도구. 단 정식 릴리스 없음(reg/ahk 수동) · 탭/길게 이중동작은 위 AHK가 더 확실.
  • PowerToys Keyboard Manager — 단순 CapsLock→한영 매핑만. 탭/길게 구분 ✕.
