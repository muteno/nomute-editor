증상PC에서 픽토그램 버튼 클릭 시 버튼 윤곽(테두리)이 보이거나 전경색(배경)이 노출됨. 모바일은 정상.
원인.file.picking — JS가 클릭 시 라벨에 붙이던 강조 클래스(border-color:rgba(accent,.35) + background:rgba(accent,.08)). 파일창 닫힐 때(focus 복귀) 제거. PC는 별창이라 그 사이 페이지 뒤로 노출.
수정① CSS .file.picking 규칙 삭제 → 대신 ly 정본과 동일한 .file:active{background:transparent} 추가. ② .picking을 붙이고 떼던 JS(클릭 리스너 + focus 리스너 + handler 제거줄) 삭제.
정본 일치자막생성기 viewer/ly.html의 .file 동작과 100% 동일하게 통일(.picking 없음 · 누름 피드백 = :active svg 축소뿐).

3장점 · 효율성 · 품질저하 리스크

장점
  • PC에서 클릭 시 버튼 외형 변화 0 = 깔끔(증상 해결).
  • 썸네일(/1·/2)·카드뉴스(/5)·자막생성기 픽토그램 동작이 한 기준으로 통일(드리프트 해소).
  • 죽은 코드(클릭·focus 리스너) 제거로 bindFile 더 단순.
효율성순수 CSS/표식 변경 + 리스너 1쌍 제거 → 런타임 부담 미세 감소. 빌드·파이프라인·토큰 SSOT 무관(check_refs 통과).
기능 영향없음 — 파일 선택은 네이티브 <label>→<input type=file> 동작이라 그대로. .picking은 시각 강조 전용이었음(로직·상태·데이터 무관).
품질저하 리스크
  • 파일창 열린 동안의 "강조 표시"가 사라짐 — 단, 모바일에선 원래 안 보였고(전체화면), PC에선 그게 바로 문제였던 표시라 손실 아님.
  • 누름 즉시 피드백(:active svg 축소)은 유지 → 눌렀다는 감각은 그대로.
  • 회귀 위험 낮음: 단일 뷰어 파일·순수 시각 변경·정본(ly) 기준 통일.
