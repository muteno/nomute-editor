대기열 상태·버튼 지도

폰 공유·요약요청이 대기열(＋ > 대기열)에서 어떤 상태로 뜨고, 각 버튼을 누르면 어디로 가는지 — 실패 케이스와 이번에 추가한 재시도 포함. 실제 뷰어 토큰·CSS·아이콘 그대로 재현.

1눌러보는 대기열

대기열
← 왼쪽 대기열의 ↻ / ↗ 버튼을 눌러봐
액티브 모션과 “어디로 가는지”가 여기 떠
💡 재시도 ↻와 원문 ↗는 같은 레몬색(#FFE13D) — 색이 아니라 아이콘(회전/화살표)으로 구분해. 완료·뉴스요약은 터쿼이즈, 실패 글자는 빨강.

2버튼 범례

초록 ↗ — 뉴스요약 열기
분석 완료돼 피드에 뜬 기사. 누르면 뉴스요약 모달.
레몬 ↻ — 재시도
실패한 기사 다시 분석/요약. 이번 추가
레몬 ↗ — 기사 원문
처리중·자동재시도 때 원문 새 탭.
회색 ↗ — 비활성
원문 링크가 없어 갈 곳 없음(클릭 불가).

4이번 수정 — URL 실패 재시도

BEFORE — 재시도 없음

URL 공유 기사 분석 실패 → 원문 ↗만 뜸.
다시 분석할 방법이 화면에 없었음.

AFTER — 재시도 ↻

재시도 ↻ 노출 → 누르면 다시 분석.
수집함 Failed→✓ 재시도와 같은 경로.

한계 — 전문(붙여넣기) 실패는 원문 URL·전문이 뷰어에 안 남아 재시도를 못 붙였어(백엔드에만 있음). 이 경우만 회색 비활 유지 + “전문 다시 붙여넣어줘” 안내. 위 대기열의 전문 실패 행 ↗를 눌러보면 그 안내가 떠.
