## 만장일치 결론 (10/10)
1. **뷰/컨트롤러 분리가 정답** — 공통 `.pin-slot` CSS + 순수 렌더함수 `renderPinSlots(el, value, {reveal})`만 통일. **입력·보안·상태머신은 각자 유지**(어댑터). 로직 통합 금지.
2. **동그라미 마스킹으로 통일** → `*`/`translateY(3px)`/캔버스오차 **전량 소멸**, `place-items:center`로 정렬 자동.
3. **`%` gap/padding = 사이즈무관 비율** (flex % = 컨테이너 content-box 기준 = 닮음변환 불변). **`vw`·`space-between` 기각**(vw=뷰포트라 모달폭 같아도 뷰포트 다르면 깨짐).
4. **색 = 기존 토큰 재사용**(--mut 미입력·--fg 입력·--accent 방금·--danger 에러). 색 신설 0.
5. `.pin-cell` **raw accent 토큰화**(line 1155 `rgba(15,253,2,.08)`·1157 `.45` → `rgba(var(--accent-rgb),…)`) · `pinShake`→`lkShake` 단일 · `accent_raw` baseline 97→하향(ratchet).
6. **CII 등재 필수**(미등재가 5중복제 근원) · **단계적 표면별 PR**(빅뱅 금지·index.html 9286행 blast radius).

## 공통 스펙 (발행 `.pub-slot` 기준 = 정본)
> ⚠️ **260704 개정(운영자·5인 분신술)**: 아래 코드블록은 260703 원안. 실코드(viewer 정본 `.pin-slot`)는 이후 개정됨 — **방금 `.cur` 숫자 = 흰(`--fg`)·볼드(`--fw-x`)·키패드 폰트(inherit·모노 아님)** · **`.on`(입력됨) = `--mut`(미입력과 동일 톤·방금 자리만 부각)** · **크기 75%(12×18·15px)** · **화면잠금 키패드 배경 플래시 제거(scale `--press-m`+색만)**. 정본은 항상 viewer `:root`+`.pin-slot`(§🎨·CII 등재 260704).
```
.pin-slots { display:flex; justify-content:center; gap:5%(--pin-gap 선택); padding-inline:4.5%; width:100%; }
.pin-slot  { place-items:center; position:relative; font:400 20px mono; color:var(--accent); }  /* cur 숫자=형광 */
.pin-slot::before { content:''; width:56%; aspect-ratio:1; border-radius:50%; background:var(--mut); }  /* 미입력 원 */
.pin-slot.on::before { background:var(--fg); }        /* 입력됨(이전) = 흰 원 */
.pin-slot.cur::before { content:none; }               /* 방금 = 원 숨기고 숫자 노출 */
.pin-slots.bad .pin-slot::before { background:var(--danger); } .pin-slots.bad .pin-slot.cur { color:var(--danger); }
.pin-slots.shake { animation:lkShake .45s cubic-bezier(.36,.07,.19,.97); }
```
- 렌더: `renderPinSlots(el, value, {reveal:'last'})` — last-index=`.cur`(숫자), 이전=`.on`(흰원), 미입력=원. (②는 `reveal:'mask'`로 peek 700ms 옵션 · 운영자 결정으로 조회도 last 노출 허용)
- `.pub-slot`/`.lk-slot`은 `.pin-slot` **별칭 계승**(`.tool-x` 패턴).
