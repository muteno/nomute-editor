0걸려 있는 명령 정리 + 30초 확진
멈춰 있는 lsf부터 끊기: 하단 CTRL 누르고 c. 그다음 수다쟁이 모드로 30초 재실행 — 뭐에 막히는지 실토하게:
rclone lsf gdrive: -vv --contimeout 15s --timeout 30s 2>&1 | tail -30
403 · rateLimitExceeded · quota 반복
공용 출입증 스로틀 확정 → 이 가이드 그대로 진행.
invalid_grant · unauthorized · token 에러
토큰 사망 → 역시 이 가이드 그대로(6번 재인증에서 같이 해결됨).
dial tcp · i/o timeout · connection refused
폰→구글 네트워크 문제(VPN·사설 DNS·데이터 절약 모드 확인). 아래로 판별 — 200 나오면 네트워크 무죄 = 출입증 축:
curl -sS -m 10 -o /dev/null -w "%{http_code}\n" https://www.googleapis.com/discovery/v1/apis
1구글 클라우드 프로젝트 만들기 (폰 크롬)
console.cloud.google.com/projectcreate 열기 — 드라이브 쓰는 그 구글 계정으로 로그인. 프로젝트 이름 붙여넣고 [만들기]:
nomute-rclone
2Drive API 켜기
Drive API 페이지 열고 파란 [사용(Enable)] 버튼 한 번. (방금 만든 nomute-rclone 프로젝트가 상단에 선택돼 있는지 확인)
3동의 화면 — 외부 + 프로덕션 게시 ⚠핵심
OAuth 동의 화면 → 유형 = 외부(External) → 앱 이름 nomute-rclone·지원 이메일 = 본인 → 나머지는 전부 [저장 후 계속]으로 통과.
게시 상태를 반드시 "프로덕션(앱 게시)"으로 전환 — 테스트 상태로 두면 7일마다 토큰이 죽는 함정(레포 §2 판례). "확인되지 않은 앱" 경고 문구는 개인용이라 무시해도 됨. 메뉴 위치가 다르게 보이면 동의 화면의 [대상/Audience] 탭에서 [앱 게시] 버튼.
4출입증 발급 — OAuth 클라이언트 ID
사용자 인증 정보 → [+ 사용자 인증 정보 만들기] → OAuth 클라이언트 ID → 애플리케이션 유형 = 데스크톱 앱 → [만들기].
뜨는 팝업에서클라이언트 ID(…apps.googleusercontent.com)와 클라이언트 보안 비밀(GOCSPX-…) 둘 다 복사해 메모장에 잠깐 보관.
5폰 rclone에 심기 (Termux)
아래 붙여넣고 <ID>·<SECRET> 두 곳만 4번에서 복사한 값으로 교체 후 엔터:
rclone config update gdrive client_id <ID> client_secret <SECRET>
⚠ config update는 콜론 없이 gdrive — gdrive:로 쓰면 "invalid characters" 에러(260720 실증). 콜론은 reconnect gdrive: 같은 원격 경로 계열만.
6재인증 — 새 출입증으로 동의 1회
rclone config reconnect gdrive:
질문은 엔터(기본값)로 통과 → http://127.0.0.1:53682/auth… 링크가 뜨면 자동으로 브라우저가 열리거나, 안 열리면 링크 길게 눌러 복사 → 크롬에 붙여넣기 → 계정 선택.
"Google에서 확인하지 않은 앱" 화면이 나오면 정상 — [고급] → [nomute-rclone(안전하지 않음)으로 이동] → [허용] → "Success!" 뜨면 Termux로 복귀. scope 질문이 나오면 drive(전체) 유지.
7검증 + 밀린 것 회수
NOTICE 경고 없이 바로 목록이 뜨면 전환 성공:
rclone lsf gdrive:Shared | tail -5
싱크 한 바퀴 + 막혀 있던 최근 24시간 치 즉시 회수:
rm -f ~/.drivesync.last && bash ~/.termux/tasker/drive-gallery-sync.sh; echo rc=$?
rclone copy gdrive:Shared /sdcard/Pictures/DriveSync --max-age 24h --inplace -v
termux-media-scan -r /sdcard/Pictures/DriveSync
갤러리에 이미지 왔는지 확인하면 끝. (드라이브 데스크톱 쓰는 PC들은 자기 인증이라 손댈 것 없음 — 이 교체는 폰 rclone만)
🎉 전환 완료. 이제 공용 출입증 은퇴·스로틀과 무관해졌고, 혹시 또 죽어도 v1.4가 9분 안에 알림을 쏜다. 각 단계 막힌 데 있으면 그 화면 스크린샷 + 0번 출력 채팅으로.

문제 생기면

0번에서 curl도 200이 안 나와

폰→구글 API 네트워크가 막힌 것 — VPN·사설 DNS(1.1.1.1 등)·데이터 절약 모드·와이파이 공유기 차단 순으로 끄고 재시도. 이건 출입증 문제가 아니라서 1~7번 해도 안 풀림.

reconnect 링크가 크롬에서 안 열려

Success 리다이렉트는 폰 안(127.0.0.1)으로 돌아오는 구조라 같은 폰에서 열어야 함. 크롬 말고 삼성인터넷도 가능. 그래도 안 되면 그 화면 스크린샷 회신.
