해석 = ① 이름(Anonymous)을 안 넣으면 강제로 "Anonymous"를 박지 말고 이름을 빈 값(출력에 없음)으로 · ② 그 자리("Anonymous")는 볼드 빼고 이탤릭으로 표기. → 260706에 강제 텍스트로 바뀐 걸 플레이스홀더(힌트)로 복귀시키는 변경.

1. 전 / 후 — 편집 화면(저작권 문구)

BEFORE 기존

ⓒ 2026. Anonymous(threads).

"Anonymous"가 실제 텍스트(볼드·getName()="Anonymous")로 강제 삽입 · 탭하면 비었다가 이탈 시 다시 복원 · 발사 시 이름 필수(빈 채 발사 차단).

AFTER 변경

ⓒ 2026. Anonymous(threads).

"Anonymous"가 플레이스홀더(이탤릭·비볼드·회색 · getName()="") · 안 채우면 출력에 이름 없음 · 실제 이름 입력하면 그대로 강조 표시.

2. 전 / 후 — 실제 출력(제작 결과물 문구)

BEFORE 기존

이름 미입력 → (발사 차단 · "이름을 넣어주세요")ⓒ 2026. Anonymous(threads). all rights reserved.

이름을 안 넣으면 발사가 막히고, 넣더라도 기본 "Anonymous"가 결과물에 박힘.

AFTER 변경

이름 미입력 → 이름·괄호 생략(플랫폼만)ⓒ 2026. threads. all rights reserved.

이름 있으면 ⓒ 2026. 홍길동(threads). all rights reserved. · 이름·플랫폼 둘 다 없으면 ⓒ 2026. all rights reserved.

4. 장단점 · 효율성

장점
  • 운영자 요청 정확 충족 — 미입력=빈 출력 + 이탤릭·비볼드
  • 기존 검증된 메커니즘 재사용: .cph-ed:empty::before는 연도 칸(2026)이 이미 쓰는 패턴 = 신규 CSS·토큰 0(design 게이트 무영향)
  • 이름 자리를 "예시 빈칸(260705)"→"강제(260706)"로 오갔던 걸 힌트로 정리 — 어포던스 명확
  • 미리보기(JS)와 서버(PIL) 동일 로직 = WYSIWYG 일치
단점 / 트레이드오프
  • 이름 없이도 발사 가능 → 실수로 이름 비운 채 제작 가능(단, 저작권 자체는 유효 문구)
  • 편집화면 힌트는 이탤릭이지만 실제 출력 폰트는 정체(upright) — NotoSansCJK-Regular엔 이탤릭 글리프가 없어 출력까지 이탤릭은 별도 폰트 필요(요청 범위 밖 = 미적용)

5. 품질저하 리스크 (정직)

  • contenteditable :empty 신뢰성 — 브라우저가 빈 필드에 <br>를 넣으면 :empty가 깨질 수 있음 → blur 시 textContent='' 정리로 방어(연도 칸이 같은 패턴으로 이미 안정).
  • 스냅샷 하위호환 — 구 스냅샷에 cname:"Anonymous"가 저장돼 있으면 그 이름이 복원됨(구 사용자가 실제로 그 값을 썼다는 뜻이라 파괴 안 하는 게 맞음). 신규 미입력은 빈 값.
  • 발사 게이트 — 이름 가드만 제거 · 다른 필수 검증(저작권/안내문/헤더 중 하나는 있어야 발사)은 유지 = 완전 빈 발사는 여전히 차단.
