# 인계 — yeta(말벗 제타) 캐릭터 박스 클릭 무반응 · 얼굴 생성 (260703)

> 분신술 10인 적대 진단으로 밝혀낸 **"캐릭터 박스 눌러도 설명 안 뜨고 대화 안 이어짐"** 의 원인·수정 총정리 + 얼굴(프로필) 생성 파이프 기록. 미래 세션·운영자용 SSOT. 정본 코드 = `viewer/index.html`(yeta 블록) · `functions/api/yeta.js` · `.github/scripts/yeta_face.py` · `.github/workflows/yeta-face.yml`.

## 0. 증상 (운영자, 폰)
- 뷰어 프로필 메뉴 → yeta → 캐릭터 목록은 뜬다. 그런데 **캐릭터 박스(.ypick-row)를 눌러도 ① 소개(intro)가 안 나오고 ② 대화방 진입·대화가 안 됨.** = "완전 무반응"으로 체감.

## 1. 진단 방법
- **분신술 = 읽기전용(Explore) Opus 4.8 에이전트 10명 병렬 소환**, 각기 다른 앵글로 yeta 흐름을 적대 감사(READ-ONLY 계약 — 파일 변경 0, 메인만 수정).
- 앵글: ①클릭 이벤트 위임 ②draw API·에러 스왈로잉 ③R2 게이팅 ④뷰 토글 ⑤intro 렌더·캐시 ⑥JS 예외 ⑦진입 분기 ⑧**모바일 터치** ⑨roster 데이터 ⑩총괄.
- 라이브(apps.nomute.kr)는 Cloudflare Access가 감싸 에이전트/외부에서 직접 못 침 → **코드·데이터 정적 분석으로 원인 확정.**

## 2. 원인 (복합 — 4갈래)

### 🔴 주원인(폰) — `touch-action:manipulation` 누락 → click 미발화
- `.yeta-list`(캐릭터 목록)는 `position:absolute; inset:0; overflow-y:auto` = **스크롤 컨테이너**.
- 그 안의 버튼(`.ypick-row`·`.yl-dice`·`.yeta-who`·`.yeta-send`)에 `touch-action`이 없었다.
- iOS/모바일에서 스크롤 컨테이너 안 버튼은 손가락이 몇 px만 움직여도 탭이 **팬(pan) 제스처로 소비**돼 `click`이 아예 안 뜬다 → `yEnterChat` 호출 자체가 안 됨 = **완전 무반응**(PC는 마우스라 정상).
- 이 레포는 폰 탭이 안 먹던 다른 곳(`#bnav .bnav-i`·`.sc-item`·`.sc-fail`·`.ed-match`)을 **전부 `touch-action:manipulation`으로 하드닝**해 둔 관례가 있었는데, 신규 yeta UI(260703)만 그걸 빠뜨림.

### 🟡 UX — 실패 안내(miniToast)가 안 보임
- `yetadlg`는 `showModal()` = 브라우저 **top layer**. `miniToast`는 `document.body`에 `z-index:60`으로 붙어 → **토스트가 모달/백드롭 뒤에 깔려 안 보임.**
- 게다가 리스트 뷰에선 `#yetaLog`가 `display:none` → yLoad의 셋업 배너도 안 보임.
- 결과: R2 미설정·draw 실패 같은 **명확한 사유가 있어도 사용자 눈엔 "그냥 먹통"** 으로 보임. `.catch(()=>null)`이 원인도 삼킴.

### 🟢 방어 부재 — 스테일 persona → 죽은 대화방
- 세션(`sessions/main.json`)에 박힌 persona id가 현재 roster에 없으면(리네임·삭제) `yLoad`가 `yPersona()=undefined`인데도 `yView('chat')`을 강제 → `YCUR=null` 죽은 대화방(소개 안 뜸·`ySend`의 `!YCUR` 가드로 전송도 무반응).

### 🔵 잠재 — chars 캐시 staleness
- `functions/api/yeta.js`의 `chars` op이 roster를 `cf:{cacheTtl:300}`(5분) + raw.githubusercontent CDN 캐시로 반환 → **방금 편집한 intro/greeting이 몇 분간 지연 반영**(대화방이 열려도 소개칸이 빌 수 있음).

### ⚠️ 근본 배포설정 — `YETA_R2` 바인딩(대화 자체의 전제)
- `functions/api/yeta.js`: `chars`/`warm` 이후 모든 op(`get`·`draw`·`send`·`retry`)은 `if (!env.YETA_R2) return json({…,setup:true},501)` 게이트를 통과해야 함.
- **Pages에 비공개 R2 바인딩 `YETA_R2`가 없으면 draw가 501 setup** → (수정 전엔) yEnterChat이 토스트만 띄우고 조기 return → 소개·대화 둘 다 안 뜸.
- ⚠️ **얼굴용 공개 R2(`R2_*` 시크릿)와 대화용 비공개 `YETA_R2`(Pages 바인딩)는 완전히 다른 축.** mudi 얼굴이 보이는 것과 대화 안 되는 것이 동시에 성립하는 이유.

## 3. 수정 (코드 — PR #1461 · main 머지 완료)
| 갈래 | 수정 | 위치 |
|---|---|---|
| 🔴 폰 탭 | `touch-action:manipulation` 추가(+`.ypick-row`엔 `-webkit-touch-callout:none; user-select:none`) | `viewer/index.html` `.ypick-row`·`.yl-dice`·`.yeta-who`·`.yeta-send` |
| 🟡 UX | R2 미설정/실패여도 **대화방 진입 → 소개(intro) 표시 + 실패는 모달 안 배너**(`yShowIntroOnly`) — 스쳐가는 토스트 폐기 | `viewer/index.html` `yEnterChat`·신규 `yShowIntroOnly` |
| 🟢 방어 | 스테일 persona면 죽은 대화방 대신 **리스트 폴백** | `viewer/index.html` `yLoad` |
| 🔵 캐시 | chars `cacheTtl` 300→**60s** | `functions/api/yeta.js` |

> intro는 roster 데이터라 **서버(R2) 없이도 렌더 가능** → "설명조차 안 뜬다"는 이 수정으로 R2 유무와 무관하게 해소.

## 4. 남은 조치 — 운영자(배포설정, 코드로 해결 불가)
실제 **대화 주고받기**(메시지 전송·답장)를 켜려면:
1. Cloudflare **Pages → 프로젝트 → Settings → Functions → R2 bindings** → 변수명 **정확히 `YETA_R2`** 로 **비공개 세션 버킷**(`nomute-yeta`) 연결.
2. **Production 환경**에 걸렸는지 확인(Preview 전용이면 실서비스 미설정). 얼굴용 공개 `R2_*` 버킷과 **다른 변수명·다른 버킷**.
3. 전송까지 쓰려면 **`GH_TOKEN`**(Actions write) 시크릿도 Production에 존재해야(없으면 send 500). 세션 워크플로가 R2에 쓰는 짝 = GH Secret `YETA_R2_BUCKET`.
4. 재배포 후 검증: 폰에서 박스 탭 → 소개 배너 + 첫인사 버블 뜨고 답장 오면 정상. (또는 개발자도구 `POST /api/yeta {op:'chars'}` 응답의 `ready:true` 확인.)

## 5. 얼굴(프로필) 생성 파이프 — 기록
- **생성기** = `.github/scripts/yeta_face.py` (OpenAI **gpt-image-2** · 1:1 1024² 얼굴중심 · 한국 웹툰·살짝 판타지·착장 유지 안전가드). 워크플로 `.github/workflows/yeta-face.yml`(수동 dispatch·유료 종량제·운영자 승인 축).
  - 게이트 = GH Secret **`OPENAI_API_KEY_nomute`**(폴백 `OPENAI_API_KEY`). 저장 = 공개 R2(`R2_*` 5시크릿) `yeta_face/<id>.png` → roster `avatar` 주입 커밋. R2 없으면 git 폴백.
  - `force=1` = 기존 avatar도 재생성 / `only=<id>` = 한 명만(테스트·비용절감).
- **프롬프트** = 보강 확정 카드(수치·직업·이면·시그니처 착장)에 맞춰 `FACES` 10인 재작성(PR #1459).
- ⚠️ **타임아웃 함정**: 10장 × ~81초 ≈ 13분 + 재시도 → **15분 타임아웃에 걸려 step 취소·roster 커밋 skip = 무반영**(run#4). **`timeout-minutes` 30으로 상향**(PR #1460) 후 재실행(run#5) 완주 → 10인 avatar 전원 주입 완료.
- **확인/다운로드**: R2 공개 URL은 이 컨테이너 urllib이 프록시 403 → **`curl --cacert /root/.ccr/ca-bundle.crt`** 로 받아야 함(미래 세션 참고).
