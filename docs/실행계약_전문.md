> **목적**: 클로드 코드가 레포 최상위 `CLAUDE.md`를 *지속적으로 따르지 않는* 재발을 차단한다. 운영자가 요구사항을 던질 때마다 붙이던 「복사문구」를 **정본화**한 문서 — CLAUDE.md 마커 구간의 **【실행 계약 [E1]~[E12]】** 절이 요약 정본이고, 이 파일이 그 세부 전문 + **타 세션/레포에 그대로 붙여넣는 복사문구**다.

## 1. 전문 (E1~E12)

### 착수 게이트 — [E1]~[E7] (하나라도 누락 = 폐기·[E1] 재시작)

- **[E1] 기틀 정독**: 착수 *전에* 레포 최상위 `CLAUDE.md`와 그 파일이 참조시키는 정본(연쇄 참조 포함)을 **실제로 열어** 각 문장의 배경·목적을 파악한다([0][1]). CLAUDE.md까지만 읽고 값을 추측·창작 = 오류. **디자인을 건드리는 작업이면** 참조된 디자인 기틀 문서를 *개별로 모두* 정독한다([6] · nomute = `docs/디자인기틀_SSOT.md` → `viewer/index.html :root` → `docs/CII_컴포넌트계승인덱스.md`).
- **[E2] T0 기록**: 착수 시각을 기록한다(훅이 KST로 자동 스탬프 · 러너 UTC라 `TZ=Asia/Seoul` 강제). 이 T0가 평의회 타이머([E10])의 기준. 응답 본문에 "몇 분 뒤 완료" 같은 완료 시점 예고는 금지([11]).
- **[E3] 의도 정의**: 요구가 *어떤 맥락에서·어떤 의도로·어떤 목적을* 갖는지 정의한다(표면 문자 아님 = 왜 시켰나). "이렇게 이해했고 이렇게 간다"를 1줄로 명세하고 진행([1]·2조3항).
- **[E4] 최고품질·고효율 수행**: [E3] 정의대로 근본·전체를 해결한다(왜→방향→그 방향의 최선). 곁가지보다 **시킨 것 먼저** 끝낸다([1]).
- **[E5] 계획 표(10칸)**: 착수 시 수행 방안을 프로세스 순서표로 정의한다. 칸 = **①순서 ②내용 ③현황 ④목표 ⑤달성방안 ⑥예상소요시간 ⑦예상결과 ⑧리스크 ⑨완료여부 ⑩아이데이션**(2조5항 6칸의 260718 확장판). 기존에 작성된 코드의 품질을 저하시키지 않는다.
- **[E6] 지금 수행**: 요구사항은 이 세션에서 즉시 수행한다. 스케줄 예약·크론·백그라운드·모니터링·자가 체크인(send_later 포함)의 **임의 부착 금지**([8]) — 정말 필요하면 걸기 전에 채팅 대화로 물어 승인.
- **[E7] 토큰 누적 감시**: 세션 작업량을 누적 카운트한다. 누적 500K 도달 시 **압축에도 품질이 유지되고 다음 세션에 전달 가능한 중간 DRAFT**를 임시 작성하고 인계를 준비한다.

### 완료 — [E8]~[E12]

- **[E8] 완료 검문(선언 전 2관문)**:
  - **ⓐ 디자인 재확인**: 참조 디자인 정본대로 UIUX를 구성했는지 다시 확인. 아니면 작업 전량 폐기·[E1] 회귀. 세부 요소(픽토그램·버튼·텍스트박스·팝업·이미지) 도입 시 = **요소를 4분할해 나오는 기준점을 중심으로** 요소끼리 중앙(높이) 정렬·가로 정렬 일정 + **같은 요소 간 마진 균일** + **클릭 시 액티브** 적용(디자인 계약 3-4 = Playwright 픽셀 실측 의무).
  - **ⓑ 머지**: 무조건 MERGE한다([9][10]). 위험성이 중대한 경우에만 DRAFT 후 대화로 질의. '완료'는 main 머지 + `git show origin/main:<파일>` read-back 실측 후에만 선언([11]).
- **[E9] 결정 애매 = 플레이그라운드**: 디자인을 결정하기 애매한 부분이 생기면 **단일 파일 HTML 플레이그라운드**를 만든다.
  - 구조 = **대분류 5개(문제 수행에 가장 핵심적인 결정축) + 소분류 10개(보조)**. 의미 없는 값을 소분류에 넣지 않는다 · 하나의 요소를 잘게 쪼개 나열 금지([5]).
  - CLAUDE.md 참조 디자인 기틀(정본)의 요소 값이 얼마인지 파일 내에서 알 수 있게 하고, 그 값에서 **이탈하면 HTML 안에서 ⚠경고**.
  - 미리보기 = **상단 고정(모바일)** / **좌측(PC)**, 조정값 = 하단(모바일) / 우측(PC). 콘솔 조작마다 미리보기가 최상단으로 튕기지 않게 스크롤 보존.
  - 5요소 골격(실물 재현 미리보기·프리셋·미세조정·선택값 복사·한계 각주)·풍성한 옵션 = `docs/플레이그라운드_포터블.md` §3 정본 계승. 산출 = `docs/reports/{yymmdd}_{라벨}_플레이그라운드.html`(커밋 + 채팅 첨부).
- **[E10] 평의회(기틀 변경 시)**: CLAUDE.md·핵심 기틀 문서·토큰 구조·동작 로직·웹앱 전반 변경 = 병렬 수행 검증. 단순 국소 디자인 손질은 비대상([9] 억제).
  - 소환 = **서로 다른 관점의 OPUS 4.8 --EFFORT MAX 8인 병렬**(운영자 260718 정본 인원). 작업 완료 결과를 **적대적 관점으로 다각 검증** → 각 모델 독립 아이데이션 → 상호 검증 결과 열람 후 코멘트. 가능한 모든 모델 동시 돌입. **READ-ONLY**(분석·텍스트만 · 레포 변경 = 메인 세션만).
  - 타이머(T0 기준) = 작업 목표 10분 · **평의회 트리거 = T0+20분** · **런 = 첫 모델 돌입 후 10분** 경과 시 종료 · 특정 모델이 탐구를 요청하면 **그 사실을 볼드체로 사용자에게 알리고 최대 3분 추가 부여** · **14분 경과 = 강제 중지·수렴**.
- **[E11] 보고**: **최우선 2의 6단 골격이 유일 정본** — ① 질문(지시 원문 인용) ② 의도파악 ③ **결론: 성공/실패**(볼드 · 완벽 수행 아니면 '실패') **+ 3줄 초등학생 요약**(각 문장 불릿·쉬운 말·문장 끝나면 2단 줄바꿈) ④ 처리 결과(전→후 + [E5] 10칸 표 결과 + 머지 성공 여부 + 평의회 결과를 [E5] 형태로) ⑤ work to do ⑥ 💡 한 수 1개. (12-1)~(12-3) 항은 구분선 + 줄바꿈으로 읽기 쉽게.
- **[E12] 권한·질의 규율**: 불가피하지 않으면 사용자에게 **권한 수락을 요청·요구하지 않는다**([8][11] · 승인은 대화 문답만). 사용자에게 무언가 **물어볼 때는 전/후를 시각화해 비교**(정적 나열 금지 → 플레이그라운드/갤러리). UIUX 설계가 난해하면 예/아니오 팝업 말고 대화로 **신설 제안**([4]). 색은 **CLAUDE.md 참조 팔레트 정본 색만** 사용([4]·【레포 바인딩】).
