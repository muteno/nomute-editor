# 🔒 「디자인 = 루트 토큰 계승/갱신」 제1 핵심명령 — 타 레포 복제 전문

> **목적**: 노뮤트의 §🔒 제1 핵심명령(**모든 디자인 = 「루트 디자인 토큰」 [계승] 또는 [갱신], 둘 뿐**)을 다른 레포에 이식.
> **핵심 구조 = (A) 일반형 개념규칙(그대로 복제) + (B) 레포별 바인딩표(9슬롯 실측 치환).** 절을 융합 상태로 복붙하면 노뮤트 고유 경로(viewer/index.html·build_design_mirror·CII)가 dangling ref가 됨(분신술 앵글9). A는 verbatim, B는 그 레포 실측값으로.

---

## 복제 절차 (5단계 · 순서대로)

1. **A블록을 그 레포 CLAUDE.md(또는 동급 최상위 지침) 최상단에 verbatim 복제** — `{{슬롯}}`만 B표 실측값으로 치환.
2. **방향(SSOT)을 먼저 정한다** — 값 소스가 CSS `:root`인지(노뮤트), JSON/TS 토큰인지(Style Dictionary·Tailwind theme·styled-components ThemeProvider·SCSS `$var`). **후자면 CSS는 *생성물*이라 거울 방향이 반대** → B표 `{{TOKEN_SSOT}}`·`{{MIRROR}}`에 반영. 이걸 안 정하면 거울 모델 자체가 뒤집힘.
3. **스택별 폴백** — 거울 스크립트·게이트가 없는 레포는 `{{BUILD_CMD}}`/`{{GATE_CMD}}`/`{{LOCK_CMD}}`를 "N/A"로 두되 **개념 규칙 ①②③은 여전히 유효**(사람·에이전트 규율로 강제). "스크립트 없으면 규칙도 없다"가 아님.
4. **graceful-skip 함정 점검** — 노뮤트 게이트 스크립트를 복사했다면, 경로가 안 맞을 때 **조용히 통과(rc=0)**해 "지켜지는 줄 착각"할 수 있음(앵글9 실측). 복제 후 게이트가 실제 rc=1을 내는지 **1회 sanity 실측**(일부러 위반 하나 만들어 커밋 막히는지 확인).
5. **한글 고정명 중립화** — `구성도`·`CII` 등 한글 파일·디렉터리명은 그 레포 언어/관례로. (노뮤트 스크립트는 한글 경로를 자동 생성할 수 있으니 타 레포 오염 주의.)

## (A) 일반형 개념규칙 — verbatim 복제 (`{{슬롯}}`만 치환)

> ## 🔒 제1 핵심명령 — 모든 디자인은 「루트 디자인 토큰」 계승 또는 갱신, 둘 뿐 (레포 절대명령 · 최상위 불변)
> ⛔ **이 레포 UI의 *모든* 디자인 값(색·간격·radius·blur·곡선·모션·그림자·크기·타이포 등)은 「루트 디자인 토큰」을 [계승]하거나 [갱신]한다 — 둘 중 하나로만. raw 값 임의 창작·제각각 재설계 절대 금지.** 레포 **제1 핵심·절대명령.** 「루트 디자인 토큰」 = **값 SSOT = `{{TOKEN_SSOT}}`**. 컴포넌트는 토큰이 아니라 `{{COMPONENT_INDEX}}` 정본을 계승(별개 축).
> ⚠️ **우선순위 = 디자인 *내용* 축 한정**: 이 절은 "raw 금지·계승/갱신 2택"의 최상위 선언일 뿐, 그 레포의 **라우팅·실행절차·기틀검증을 대체·완화하지 않는다**(별개 축=중첩). 값·절차·예외·기틀판정의 실체 정본이 따로 있으면 **구체 충돌 시 그쪽이 우선**.
> - **① [계승] = 항상 디폴트(안 물음)**: `{{CONSUME}}`로 가져다 쓴다. **정확 토큰이 없어도 가장 가까운 토큰을 자동 계승**(안 물음). 재설계·raw 창작만 금지.
> - **② [갱신] = {{APPROVER}} 승인 필수 · 2티어**: 값 미세조정=승인 후 편입 / 토큰 신설·삭제·구조 변경=기틀 → 확인+교차검증까지가 승인의 일부. 승인 후에만 편입(→ `{{BUILD_CMD}}` → `{{COMPONENT_INDEX}}` 등재 → `{{GATE_CMD}}` → `{{LOCK_CMD}}` 락 갱신).
> - **③ 근접 토큰조차 없어 [갱신]이 불가피할 때만 묻는다**: 「[갱신](신설)할지 / 어떤 근접 토큰을 [계승]할지」 두 선택지 제시 후 답 받고 진행. ⚠️ 정확히 안 맞는다고 바로 묻지 마 — 근접 계승은 자동.
> - **⛔ 예외(= 이 명령 대상 아님 · 정당 raw)**: 기하·수학값(중앙 -50%·원 50%·빈 진행바 0)·애니 키프레임 안무 %값·1회성 광학 보정 px·OS/서드파티 강제(`env()`·벤더 프리픽스)·컴포넌트 로컬 스택 z-index·아직 토큰 없는 축(그림자 등)·표면별 독립 팔레트·자기완결 템플릿(토큰 참조 불가 → 값 복사=계승). 이들은 매 px·%·그림자마다 안 물음.
> - ⚙️ **강제 경계(정직)**: 기계가 하드차단하는 것과 소프트룰(모델 준수 의존)을 명시 구분하라. **게이트 초록 ≠ 규칙 완전 준수**(통과를 이행으로 착각 금지).

## 기계게이트 이식 (선택 · 강력 권장 = "진짜 지켜지게"의 핵심)

분신술 앵글7·14 결론: **문구(현저성)만으론 과거처럼 안 지켜진다. 진짜 강제 = 토큰 승인 락.**

- **토큰 승인 락** = `{{TOKEN_SSOT}}`의 **토큰명 집합**을 lock 파일에 스냅샷 → 게이트가 소스 vs lock 대조 → **락에 없는 신토큰이면 rc=1**(커밋 차단). 락 갱신은 별도 명령(`{{LOCK_CMD}}`)으로만 = 승인의 명시 행위 + lock diff가 PR에 남아 오너 가시화.
- **노뮤트 구현 정본** = `shared/build_design_mirror.py`의 `_root_token_names()`·`build_lock()`·`check_lock()` + `design-tokens.lock`. 이식 시 `{{TOKEN_SSOT}}` 파싱만 그 레포 형식(CSS `:root` → 정규식 / JSON → 키 목록 / Tailwind → theme 평탄화)으로 교체.
- **한계(정직)**: 완벽한 강제는 아님(에이전트가 락도 갱신 가능) — 그러나 ①조용한 토큰 추가 차단 ②lock diff 가시화 ③CI에서 lock 변경 시 승인 라벨 요구(후속) 로 "승인의 *부재*를 기계가 잡는" 최선 근사.
- **못 잡는 것(정직)**: 승인 여부 자체·물어봤는지·토큰 없는 축(그림자·radius·간격)의 raw = 여전히 소프트룰. 그래서 A블록 마지막 "강제 경계 정직" 줄이 필수.
