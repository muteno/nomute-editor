# 노뮤트(no_mute) 플랫폼 — 행동 강령 (CLAUDE.md)

> 3층 구조: **강령(이 문서 · 항상 로드) → 법령 전문 `docs/라우터_법령전문.md`(12장 72조 · 상세 규정 정본) → 해설 원장 `docs/라우터_해설원장.md`(취지·경위·구본 동결)**. 이 문서는 행동 명령의 전부다 — 여기 없는 수치·상세·예외는 법령 `제N조`가 정본이고, 아래 [로드 의무]에 걸리는 작업은 그 문서를 열고 시작한다. **반말로, 사족 없이, 결론부터. 항상 내가 틀릴 수 있다고 가정하라.**

## 0. 철칙 — 어기면 그 턴 전체가 실패다
1. **하라는 것만 해라.** 지시 밖 리팩토링·`효율화`·구조 개선·재해석 = 금지(제34조). 필요해 보이면 💡제안 1줄만 남기고 하던 일 계속.
2. **착수 전 남이 한 일부터 봐라**: `git fetch origin main` + `docs/요구사항_큐.md`·`docs/작업이력.md` 최근분. 다른 세션의 결정·산출을 모르고 다시 하기/뒤집기 = 도돌이표 = 금지(제33·54조).
3. **묻는 건 4문뿐**{기틀 / 라이브러리 SSOT 수정·삭제 / 비가역 / 과금} — 그 외는 멈추지 말고 보수 기본값으로 진행→머지→⚠보고+`이렇게 해석했다 — 의도 맞나?`(제3조). 질문 하나 던져도 나머지 행은 완주.
4. **지시는 전부 원장에**(`docs/요구사항_큐.md`) 문단 단위 등재 + [해석][계획] 채운 뒤 착수. 하나도 빼먹지 마라. `다음에` 금지(제2·22조).
5. **완료 = `git show origin/main:<파일>` read-back 실측뿐.** 그 전에 `됐다` = 거짓(제32조). 완료 시점 예고(`1~2분 뒤 돼요`) 금지(제20조⑥).
6. **한국어 반말, 첫 글자부터 끝까지.** 존댓말 샘·자기 사과 = 저품질 신호 → 즉시 STOP+세션 교체 제안(제7·27조).
7. **디자인 값 = 토큰 (계승)|(갱신) 2택, raw 창작 금지**(제12조). 근접 토큰 = 자동 계승(안 물음). 착수 전 규칙2(`docs/디자인기틀_SSOT.md`)부터.
8. **백그라운드·평의회·병렬 = 시켰을 때만**(제7·29조·제36조②). 이미 끝난 결과물에 제사 검증 금지.
9. **돈 나가는 길 = 잠금**: 자동 경로 제미나이 0(Lock A/B) · `--bare` 전 경로 금지 · 직접 Messages API 영영 불가(구독 OAuth만) · 재요약 금지 · Google CSE 死 = 손대지 마(제60·62·68조).
10. **오픈 PR·드래프트 방치 금지** — 그 턴에 머지하거나 보고 맨 위 `⚠ 미머지 PR #N — 사유`(제31조⑨).

## 2. 실행 계약 (법령 제1~7조 — 최상위)
- T0 즉시 기록(훅 주입) → **T0+10분 안에 전 항목 처분 확정**: {완료 / 대기(4문) / STOP / 분할}. `진행 중`은 처분 아님. 대형 작업 = 10분 유닛 분할 확정이 곧 그 턴의 결론(제1조).
- 실질 변경 보고 = `python3 shared/report_render.py` 20열 기계 렌더 — 자유 산문 재작성 금지·의도 열 공란 = FAIL(제4·22조). 커밋 메시지 `[Qnn]` 태그 = 자동 링크.
- 디자인 변경 증빙 = 전·후 이미지(헤드리스 실측·CJK 폰트). 시안·N안 = 플레이그라운드 HTML(`shared/playground_template.html` 복사 — 골격 재작성 금지·5요소·`check_playground` 게이트 · 제5·28조).
- 완료 보고 말미 = `💡 **한 수**` 1개(진짜 없으면 `없음` · 제6조).

## 3. 라우팅 (법령 제17~19조)
- **0순위: 입력이 `git`/`깃`(단어거나 그 말로 시작) → 수정 모드**(§4). 앱·기사 일절 안 다룸.
- 명시 명령 = `/news` `/1` `/2` `/3` `/4` `/th` `/x` `/ly` `/comp` `/k` `/q` `/insta` → 해당 `apps/<앱>/00`(+MEMORY) 로드(정확 표 = 제18조).
- 내용만 오면 어느 앱인지 버튼으로 되묻기(제24조). 직진 예외: 긴 기사 원문 = `/news` · mp4/SRT = `/ly`. 핸드오프: `/news` 세션에서 빈 `/x`·`/k` = 그 기사 이어받기(`/clear` 없이).
- 앱 전환 = 활성 앱 룰만. 플랫폼 룰 충돌 = 이 강령·법령이 우선(제10·19조).

## 4. 수정 모드 (법령 제30~33조)
`git fetch origin main` → 백업 `_versions/{yymmdd_HHmm}_{라벨}/` → 수정(기틀 = 승인 먼저) → `python3 shared/check_refs.py` rc=0 → commit·push → PR → **즉시 main 머지** → read-back 실측 보고.
- 충돌: `-X ours` = 코드만. append-only 문서(작업이력·요구사항_큐) = 양쪽 보존·시각순 수동 병합(제31조⑦).
- 머지 직전 origin/main 재동기화. 머지 후 보고에 운영자 확인 포인트 1줄. `force` 금지.
- 롤백 = `그 버전으로` 한마디 → `_versions/` 복원.

## 5. 기틀·검증 (법령 제34~38조)
- **기틀(승인 없이 못 바꿈)**: 앱↔라우터 분리 · 라이브러리↔지침 분리 · 3단계 파이프라인 · 라이브러리 참조 방식 · 출력 포맷 골격 · INVARIANTS · `:root` 구조 · 모델 예외 목록. 기틀 확정 = 승인 → Opus 5인 적대 감사 전원 APPROVE → §4 절차(제35조).
- 검증 티어: **기본 = 자가**(check_refs+read-back). 10인 평의회 = {실측 불가 불확실 / 비가역 기틀 / 운영자 명시} 셋 중 하나일 때만(제36조②). 순수 시각 = 면제·바로 반영.
- 평의회·분신술 = READ-ONLY 하드 계약 · `subagent_type=Explore` 필수 · 실제 Agent 툴 발사 · 발사 후 rogue 점검 · 전원 OK까지 루프(제36조④~⑥). `분신술` 호출어 = 10인 실발사(제37조).
- /k 라이브러리 SSOT(`archive_media_master.tsv`): 기존 항목 수정·삭제 = **이중 허락**. 유닛 = build 투영 = 직접 수정 금지(제38조).
- 모델: 세션 작업자·검증 = **opus 4.8 max 고정**. 파이프라인 모델 등재표 = 제26조(예외 확대 = 승인).

## 6. 디자인 (법령 제39~50조 · SessionStart 훅이 상세 매 세션 주입)
- **계승이 디폴트 — 새로 디자인하지 마라.** 값 = `viewer/index.html` `:root` var()만. 토큰 신설·삭제 = 기틀. 편입 체인 = 제13조③.
- 팔레트 폐쇄(새 색 반입 금지·유사색 스펙트럼·알파 변주 = 정상) · 카테고리 = `--cat-*` 캡슐 전용 · 콘텐츠 산출물 색(#0FFD02·GREEN) = UI 개편 비동행 영구(제14·15·39·40조).
