- 설명할 때 어려운 용어를 쓰지 말고 쉬운 용어로 풀어 쓰거나 비유(대유법)를 쓴다. 사용자 이해 상한선 ≈ '카나리아' — 그보다 어렵게 가면 못 알아듣는다. 사용자는 지금 이 일에 전념하고 있지 않아서 빨리 보고 바로 이해되어야 한다.

- (260714 운영자 확립) **완료 보고 = 아래 3단 구조 그대로**: ① `[Q. NN]` 지시 원문 인용(+수신 시각) ② `[A. NN]` 처리 결과(+응답 시각) 끝에 `→ 결론: ` 한 줄 ③ 말미 💡 한 수 1개. NN = 지시 원장(docs/요구사항_큐.md) Q번호와 1:1 · 연속 지시는 `[Q. 05~08]`처럼 묶음 허용 · 시각 = UTC+9.
- ② [A. NN]의 내용 순서 = {내가 지시를 어떻게 받아들였는지(문단별 카테고라이징) → 각각의 처리 결과 → 현재 상태}. 이 문단만 봐도 전체가 파악되게 쓴다.
- 정본 예시(260714 실물 보고 그대로):
```
[Q. 05~08]
"ㄱ 믿어볼게" / "오지게·전략 뿅뿅·초등학생도" / "토큰 아끼지 말고 다 갈겨" / "채널 요약 만드는 거다·SNS 덮어씌우지 말고"
(2026-07-14 00:35~00:40)

[A. 05~08] (2026-07-14 01:00)
넷 다 반영 — 신설·품질 바(서사+전략 실브리프로 실측)·데이터 풀탑재·SNS 무접촉(신규 파일 축 분리). 원장 Q05~Q08 ✅.
→ 결론: 완료 — 라이브 브리프 생성까지 실측 끝

💡 한 수
브리프가 매일 갱신되니 어제 브리프를 프롬프트에 참고로 동봉하면 연재가 된다 — "어제 말한 회복세가 사흘째 이어진다"처럼 전날과 이어 말해서 채널 일지처럼 읽히고, 같은 말 반복도 줄어. chan_brief.sh에서 직전 chan_brief.json의 text를 [어제 브리핑(참고 — 반복 말고 이어서)]로 프롬프트에 한 블록 추가하면 끝(코드 3줄). 원하면 "연재 ㄱ"로.
```
