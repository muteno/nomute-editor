"내가 틀렸을 수 있다"를 기본값으로 **주장 전에 검증**(파일·실행 확인, 근거 제시) — 추측을 사실처럼 말하지 않는다(§수정 모드 "검증 안 된 '됐다'는 거짓"과 동일 정신). 측정으로 뒤집히면 변명 없이 정정. 사용자 전제도 틀려 보이면 근거로 반박.
- **능동 제안 = 디폴트, 침묵이 해명할 일.** 분기·교착·"근데" 신호 끝에 「한 수」 1개(없으면 "없음"; 짜내면 노이즈). 의도 안 좁혀지면 "이거 맞아?"로 목표부터 확정.
- **비전문가 보정:** 답만 말고 선택지(A/B/C) 펼쳐 제일 나은 걸 기본값으로 진행(고르라고 멈추지 마 — 사용자는 선택자면 됨). 한 창 ≤2.
- **정직 표식:** 추측·곁가지는 던지되 ✅확인/🟡추정/💡제안/🌱설익음 딱지 필수(표식 없으면 최약등급). '됐다 거짓'은 단정에만, 제안엔 안 걸림. 몰입 중이면 보류, 'ㄴ' 하면 즉시 접어라.

- **✂️ 수정 모드 절차(반드시 이 순서):** ⓪ **편집 전 `git fetch origin main` → 최신 main 기준으로 시작**(오래된 복제본 위 작업 금지) → ① 고칠 파일 현재본을 `_versions/{yymmdd_HHmm}_{라벨}/`에 **백업** → ② 수정 → ③ **`python3 shared/check_refs.py` 통과 확인**(참조 경로 실존·파일명↔내부 버전 일치 기계 점검; 실패면 고치고 재실행) 후 `git add·commit·push`(작업 브랜치) → ④ **PR 생성 → 즉시 `main`에 머지**(draft로 방치 금지 = 이게 '라이브 확정') → ⑤ `git fetch` 후 **`git show origin/main:<파일>`로 원격 실내용을 직접 읽어 검증·보고**(해시 + 핵심 변경 줄 실측; 화면·로컬 파일만 보고 '됐다' 금지). ⚠️ **기틀 변경이면 ② 전에 사용자 확인.** ⚠️ **머지 직전 main이 그새 앞섰거나 squash 머지로 작업브랜치가 갈렸으면 `git merge origin/main -X ours`로 흡수(또는 rebase)·충돌 확인 후 머지; 진짜 충돌이면 STOP+사용자 보고, `force`는 안 씀(흡수 머지로 해결).** ⚠️ **`-X ours`는 *코드 충돌* 한정 — `docs/작업이력.md` 등 append-only 누적문서는 `-X ours`가 다른 세션 추가분을 *조용히 덮어* 데이터 손실(260628 실측: '멜론 차트 가수 등재' 항목이 사라질 뻔) → 그런 파일은 `-X` 없이 `git merge origin/main` 후 충돌 마커를 *수동 정리*해 양쪽 항목 모두 보존(시각순 정렬).**
- **⚠️⚠️ 반영 보장 (불변·최우선):** 사용자가 변경을 요청하면 **무조건 `main`에 반영 + 실측 검증까지가 '완료'다.** ① 완료 선언은 **`git show origin/main:<파일>`로 원격 실내용을 직접 읽어 확인한 뒤에만** 한다(화면 표시·로컬 파일만 보고 '됐다' 금지). ② 머지가 막히면(force 차단·non-ff·충돌) **우회책으로 끝까지 반영**(작업브랜치에 `git merge origin/main -X ours` 흡수 후 머지 등 · ⚠️ 단 append-only 문서[작업이력]는 §수정모드 ⑤ 단서대로 `-X ours` 금지·수동 보존). ③ 그래도 안 되면 **'됐다' 절대 금지 → 즉시 STOP + 사용자에게 보고.** *검증으로 확인 안 된 완료 보고는 거짓이다 — 이 원칙이 모든 편의·속도보다 우선.*
- **⚠️ 동시 세션 동기화 (사고 방지 핵심):** `main`이 단일 진실원본 — 클라우드는 세션마다 `main`을 새로 복제한다. 그래서 **각 수정은 위 절차로 그때그때 `main`에 머지**(브랜치·draft에 방치하면 새 세션이 옛 룰을 봄). 시스템 파일 수정은 가급적 **한 번에 한 세션**; 부득이 동시면 머지 직전 최신 main 재동기화 필수. **이미 열려 있던 세션은 옛 복제본** → 최신 룰 쓰려면 새로 열거나 `git fetch`로 재동기화.
- **롤백** = 사용자가 "그 버전으로" 한마디 → 해당 `_versions/` 폴더에서 복원.
