## 🗂 저장소·백업 원칙 + ✂️ 수정 모드 (운영 고정)
- **단일 원본 = 이 GitHub 레포(`muteno/nomute-editor`).** 모든 수정은 여기서만. OneDrive·로컬·다운로드 사본은 폐기(동기화 안 함).
- **⛔⛔ 수정 모드 트리거 = `git`(대소문자 무관 `git`/`GIT`/`Git`) 또는 `깃`.** 입력이 이 단어 하나거나 이 말로 시작하면 → **앱(뉴스 등) 절대 시작 금지.** 즉시 **✂️ 수정 모드**로 전환해 시스템 파일(라우터 CLAUDE.md·apps/* 지침·라이브러리 등)을 편집한다. 기사·이슈·카드뉴스는 이때 일절 안 만든다. (`git` 뒤에 지시가 붙어 있으면 그게 수정 지시.)
- **✂️ 수정 모드 절차(반드시 이 순서):** ⓪ **편집 전 `git fetch origin main` → 최신 main 기준으로 시작**(오래된 복제본 위 작업 금지) → ① 고칠 파일 현재본을 `_versions/{yymmdd_HHmm}_{라벨}/`에 **백업** → ② 수정 → ③ **`python3 shared/check_refs.py` 통과 확인**(참조 경로 실존·파일명↔내부 버전 일치 기계 점검; 실패면 고치고 재실행) 후 `git add·commit·push`(작업 브랜치) → ④ **PR 생성 → 즉시 `main`에 머지**(draft로 방치 금지 = 이게 '라이브 확정') → ⑤ `git fetch` 후 **`git show origin/main:<파일>`로 원격 실내용을 직접 읽어 검증·보고**(해시 + 핵심 변경 줄 실측; 화면·로컬 파일만 보고 '됐다' 금지). ⚠️ **기틀 변경이면 ② 전에 사용자 확인.** ⚠️ **머지 직전 main이 그새 앞섰거나 squash 머지로 작업브랜치가 갈렸으면 `git merge origin/main -X ours`로 흡수(또는 rebase)·충돌 확인 후 머지; 진짜 충돌이면 STOP+사용자 보고, `force`는 안 씀(흡수 머지로 해결).**
- **⚠️⚠️ 반영 보장 (불변·최우선):** 사용자가 변경을 요청하면 **무조건 `main`에 반영 + 실측 검증까지가 '완료'다.** ① 완료 선언은 **`git show origin/main:<파일>`로 원격 실내용을 직접 읽어 확인한 뒤에만** 한다(화면 표시·로컬 파일만 보고 '됐다' 금지). ② 머지가 막히면(force 차단·non-ff·충돌) **우회책으로 끝까지 반영**(작업브랜치에 `git merge origin/main -X ours` 흡수 후 머지 등). ③ 그래도 안 되면 **'됐다' 절대 금지 → 즉시 STOP + 사용자에게 보고.** *검증으로 확인 안 된 완료 보고는 거짓이다 — 이 원칙이 모든 편의·속도보다 우선.*
- **⚠️ 동시 세션 동기화 (사고 방지 핵심):** `main`이 단일 진실원본 — 클라우드는 세션마다 `main`을 새로 복제한다. 그래서 **각 수정은 위 절차로 그때그때 `main`에 머지**(브랜치·draft에 방치하면 새 세션이 옛 룰을 봄). 시스템 파일 수정은 가급적 **한 번에 한 세션**; 부득이 동시면 머지 직전 최신 main 재동기화 필수. **이미 열려 있던 세션은 옛 복제본** → 최신 룰 쓰려면 새로 열거나 `git fetch`로 재동기화.
- **롤백** = 사용자가 "그 버전으로" 한마디 → 해당 `_versions/` 폴더에서 복원.
- **실행** = 폰/PC 어디서든 *이 레포를 열고 기사 던지기*(기사 던지면 라우터가 뉴스 에디터로 진입). 로컬 `,` 트리거는 폐지.

## 🖼 미디어 첨부 입력 (플랫폼 공통 · 260607)
이미지·영상 첨부를 받는 앱(`/1`·`/2`·`/comp`·`/ly`·`/news`·`/k`)은 **첨부 경로를 추측·하드코딩하지 말고** `shared/attach.py`의 `latest_attachment()`로 가져온다. 접속 환경마다 첨부 저장 위치가 다르거나 없어서다(아래 4환경 실측). 각 앱 지침·메모리의 `/mnt/user-data/uploads/...` 표기는 **레거시 예시** → 이 함수 결과로 대체.

| 접속 환경 | 첨부 디스크 경로 | 대화로그(jsonl) base64 |
|---|---|---|
| 안드로이드 **모바일 앱** | `~/.claude/uploads/<세션ID>/` ✅ | ✅ |
| 안드로이드 **웹앱** · **PC-웹** · **데스크탑(Win)** | ❌ 없음 | ✅ |
| (구) Claude.ai 코드실행 | `/mnt/user-data/uploads/` | — |

- **동작**: 디스크 우선 탐색(`/mnt/user-data/uploads`·`~/.claude/uploads`) → 없으면 **현재 세션 jsonl(`~/.claude/projects/**/*.jsonl`)의 timestamp 최신 image base64 디코드**. 어느 환경이든 최신 첨부 1장을 로컬 경로로 반환. (배치 다장 = `session_images()`.)
- **호출**: `import sys; sys.path.insert(0,'shared')` → `from attach import latest_attachment` → `path, src = latest_attachment()`. 로직 정본은 `shared/attach.py` **하나**(라우터엔 코드 안 박음 = 얇게 유지).
- ⚠️ **영상**: 디스크 떨어지는 환경(모바일 앱)에서만 가능 — jsonl 폴백 불가(실측 확정: 영상은 대화로그에 base64 미포함). 디스크 부재(웹·PC웹·데스크탑)면 영상 접근 불가 → 영상 URL(yt-dlp)·SRT/STT 텍스트·모바일 앱으로 우회. `latest_attachment(kinds=VID_EXT)`는 디스크에서만 잡고 그 외엔 None.
- 이 규칙은 **입력(첨부)** 한정. 출력 경로(`/mnt/user-data/outputs/`)·산출물 전송은 무관(정상 작동).
