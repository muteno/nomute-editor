# 🏦 우리은행 고객정보 1만7천건, 외주업체 손에서 샜다

## 🧷 한줄 요약
우리은행이 2024년 NFT 플랫폼 사업 때 외부 개발업체에 넘긴 고객 연계정보·닉네임 1만7551건이, 사업이 끝난 뒤에도 업체가 임의 보관하다 직원 과실로 지난달 30일 외부에 유출됐다.

## 📰 Fact (확인된 사실)
- 우리은행은 3일 고객 공지에서 외부 개발업체가 임의로 보관하던 개인정보 1만7551건이 업체 직원 과실로 유출됐다고 밝혔다.
- 유출 정보는 온라인 개인 식별용 암호화 값인 연계정보(CI)와 고객 닉네임이다. 전화번호·주민등록번호·계좌·비밀번호는 포함되지 않았다.
- 경위: 2024년 9월 우리은행이 대체불가토큰(NFT) 플랫폼 구축 프로젝트를 하며 외부 개발업체에 정보를 공유 → 프로젝트 종료 후에도 업체 직원이 임의 보관 → 개발자 플랫폼에 공유하며 외부 유출.
- 우리은행은 지난달 30일 유출을 인지한 즉시 업체를 통해 접근을 차단하고, 개인정보보호위원회에 신고한 뒤 홈페이지에 공지했다.
- 은행 측 설명: 현재까지 확산·악용 사례 없음 / 닉네임은 임의 별칭이라 ID·로그인 계정이 아님 / 연계정보는 다른 정보와 결합하지 않는 한 단독으로 개인 식별 불가.
- 은행은 보이스피싱·스미싱 주의(출처 불명 전화·문자 URL 클릭 주의)를 당부하고, FDS(이상금융거래탐지시스템) 적용·개발업체 전수조사·피해 시 보상 방침을 밝혔다.
- 출처: 연합뉴스(임지우 기자) / 파이낸셜뉴스·헤럴드경제·뉴스핌·이데일리·이투데이 교차 확인.

## 🔎 Inference (추론·해석)
- '외부 업체 과실' 프레임은 직접 유출자를 정확히 특정하지만, 애초에 은행이 민감 식별값을 외주에 넘기고 사업 종료 후 회수·파기를 확인하지 않은 관리 책임은 그대로 남는다 [확신 중간].
- 연계정보(CI)는 단독 식별이 어려워도 다른 경로에서 유출된 정보와 결합하면 특정 개인을 겨누는 연결 키가 될 수 있어, '악용 사례 없음'이 곧 '위험 없음'을 뜻하지는 않는다 [확신 중간].
- 은행이 유출 안내와 동시에 보이스피싱·스미싱 주의를 당부한 것 자체가, 결합·표적 악용 가능성을 완전히 배제하지 못한다는 방증으로 읽힌다 [확신 낮음].

## 🧭 다각도 해석
- **소비자·이용자**: 연계정보가 단독으론 식별 불가라 해도, 내 식별값이 외주 사슬을 타고 개발자 플랫폼까지 흘러간 사실 자체가 신뢰를 흔든다 — '피해 없음' 통보로 닫힐 문제가 아니다.
- **보안·위탁 구조**: 진짜 구멍은 유출한 직원 한 명이 아니라, 사업 종료 뒤에도 외주업체에 원본이 남아 회수·파기 검증이 안 된 위탁 데이터 관리 체계다.
- **규제·감독**: 개인정보보호위 신고와 전수조사는 사후 절차일 뿐, 금융권 외주 위탁 시 데이터 파기 의무와 점검을 사전에 강제하지 않으면 같은 사고가 반복될 여지가 있다.
