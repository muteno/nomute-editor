# 트래킹 (핀셋·모자이크) — 두뇌 v1

> 진입 = 뷰어 `+ > 영상 > 트래킹` 탭(`viewer/track.html`). 운영자 확정: **통합 플로우** — 영상 분석 1회 → 인물 확정 화면에서 {핀셋(이름표) / 모자이크} *거기서* 선택 → 렌더 N회(결정 경위 = `docs/작업이력.md`). 존재 이유 = 프리미어 수동 마스킹{시작점 잡고 → 끝까지 트래킹 → 어긋나면 손수 교정} 대체: **앱에서 업로드 → 자동 해결**(초상권 모자이크가 주 용례).

## 1. 구조 (ly 파이프 미러 · LLM 0콜)

### 1) 흐름
- a) **analyze**: 뷰어 폼{URL 또는 파일 ≤30MB} → `functions/api/track.js`{업로드 = 일회용 `up-<id>` 브랜치} → `track-make.yml` `mode=analyze` → `apps/track/track_analyze.py`{얼굴 검출+트랙+군집} → `viewer/track_out/<id>/tracks.json`+`crops/p*.jpg` 커밋 → 뷰어 폴링(5s·error.log 우선·SPA HTML 가드).
- b) **render**: 인물 확정 화면에서 모드·대상 선택 → `/api/track` `{render:{id,mode,targets,invert,names,colors,opts}}` → `mode=render` → `apps/track/track_render.py` → 결과 mp4 R2(`track_res/<id>/…`) → `video.json{url,mode,ts}` 커밋 → 뷰어 폴{ts 신선도 게이트 = 과거 결과 오인 차단}.
- c) **분석 1회 = 렌더 N회**: 원본은 analyze 때 R2 `track_src/<id>.<ext>` 보관(R2 미설정 = ≤30MB git 폴백 · 둘 다 실패 = `src_note:"src-lost"` → 렌더가 명확 에러). 재렌더(다른 인물·다른 모드)는 재업로드·재분석 0.

### 2) 모델·알고리즘 (전부 로컬 = 토큰 0 · 과금 = Actions 분 + R2뿐)
- a) 검출 = **YuNet**(232KB) · 정체성 = **SFace 임베딩**(37MB) — 둘 다 opencv 내장 API(`FaceDetectorYN`/`FaceRecognizerSF`), onnxruntime·torch 불요. 다운로드 = `setup.sh`가 `media.githubusercontent.com`(⚠ raw.githubusercontent는 LFS 포인터 131바이트 함정 — 실측) + **sha256 핀**. 캐시 = `~/.cache/nomute-track`(actions/cache 7일 · 렌더 모드는 `TRACK_MODE=render`로 모델 스킵).
- b) 트래킹 = 검출 스텝(초당 12회) 간 그리디 IoU 매칭 → **검출 기반 = 드리프트 0**(매 검출이 재접지 — 상관 트래커의 누적 어긋남 원천 부재). 트랙 종료 = 0.7s 미검출. 오탐 컷 = 3회 미만 검출 트랙.
- c) 군집 = 트랙 대표 임베딩(고신뢰·대형 top6 정렬크롭 평균 — **정렬크롭은 원본 프레임 + 좌표 업스케일**{축소 프레임 정렬 = 고해상 소스 저질 임베딩 → 과병합 꼬리위험 · 평의회3}) 코사인 ≥ `SIM_MERGE 0.48` 그리디 병합 — **과분할 > 과병합 편향**(동일인 카드 2장 = 무해{둘 다 선택하면 끝} / 남남 병합 = 엉뚱한 얼굴 모자이크 = 치명). 퇴장→재등장·컷 전환도 임베딩이 같은 사람으로 재결합(합성 E2E 실측: 1s 퇴장 후 1명 재결합).
- d) 렌더 고퀄 6원칙: ① 갭 보간(깜빡임 0) ② 3탭 이동평균 스무딩(지연 0) ③ 시간 마진 ±0.3s(한 프레임 노출도 실패 → 과잉 커버) ④ 같은 인물 트랙 간 ≤1.2s 갭 브리지 ⑤ 공간 마진 가로 1.45×·세로 1.6×·위 10% 시프트(이마·머리카락) ⑥ **가장자리 접촉 끝단 = 속도 외삽 패딩**(0.3s→0.8s · 화면 이탈/진입 슬리버{검출 불가 부분 얼굴 조각·평의회I 실측 4~15프레임 노출}를 이동 방향 그대로 추적 커버).
- e) 모자이크 = 픽셀레이트 + **조절 옵션(운영자 요청)**: `opts{pxw·pxh(블록 가로/세로 분할 수 3~20 — 프리미어 수평/수직 블록 동일 개념 · 상한 20 = 얼굴당 ~14블록 = 재식별 방지 바닥 · 기본 9) · size(커버 배율 0.75~2.5 — 하한 0.75 = 하단 시프트 구속{0.4+0.8×0.75≥1} = 커버 ≥ 검출박스 전 변 · 기본 1.0) · feather(가장자리 페더 0~40px · 기본 0) · shape(rect/ellipse)}`.
  - 가) **커버 보증 = 코어-강제 마스크(평의회 A·G 봉합)**: 코어(마진박스 · *참 중심* 기준 = 화면 가장자리 클램프에도 후퇴 0) 마스크를 블러 후 `maximum(core)`로 내부 1.0 복원 — **페더는 코어 바깥 링 전용**(구 선확장-내축 방식은 소형 얼굴{폭<83px}에서 중심까지 반투명 = 노출이었음·실측 봉합). 블록 절대 하한 8px(구 렌더 익명성 바닥 유지).
  - 나) **타원 = 코어 내접 모양 선택**: 검출박스 모서리(대부분 배경·목)는 타원 밖 — size 하한 0.75가 얼굴 타원 커버를 보증. **완전 박스 커버가 필요하면 네모**(정직 경계).
  - 다) 기본값(9·9·1.0·0·네모) = 솔리드 완전 커버 · **커버 영역은 구 렌더와 동일하나 타일 분할은 종횡 독립(세로 = 높이 기준 · 구 폭 기준) = 픽셀 비동일**(세로 블록이 소폭 굵어짐 = 안전 방향 · 평의회F 정직 정정). UI = 뷰어 STEP2 모자이크 전용 게이지(ly 옵션 게이지 계승 · pend 영속 · 편집 즉시 재저장). 핀셋 = PIL{코너 브래킷+리더선+필 이름표 · 스타일 = 구 목업 캔버스 계승 · 폰트 = fonts-noto-cjk(fc-match) · **CJK 폰트 하드게이트**(`assert_cjk_font` — 없으면 두부 번인이 성공으로 위장 = ly-make 두부 게이트 동일 원리·사일런트 폴백 금지)}. 인코딩 = ffmpeg rawvideo 파이프 → libx264 crf19 + 원본 오디오 aac(+faststart). **출력 키 = stable**(`track_res/<id>/<mode>.mp4` 덮어쓰기 + `?v=` 캐시버스트) — epoch 접미 = 재렌더(주 루프)마다 R2 고아·git mp4 무한 누적(ly stable+`?v=` 불변 계승).
- f) 좌표 불변: **분석·렌더 모두 cv2 디코드 + `CAP_PROP_ORIENTATION_AUTO=1`** — 폰 세로영상 회전 메타를 양쪽 동일 처리 = 좌표 공간 일치. 한쪽만 바꾸면 모자이크가 통째로 어긋난다(절대 한쪽만 수정 금지).

### 3) 상한·에러 표면
- a) 길이 캡 = `TRACK_MAX_SEC 180`(메타+실측 이중) · 인물 카드 캡 = 12명(등장시간 순 · 초과 = `meta.dropped` 정직 표기) · 업로드 = ≤30MB(큰 영상 = URL·yt-dlp).
- b) analyze 실패 = `/tmp/track_err.txt` → 워크플로 failure 스텝이 `error.log` 커밋(뷰어 즉시 표시). render 실패 = **fail-soft**: `video.json{error}` rc 0(ly_burn 동일 — 분석 산출 보호).

## 2. 기틀 (§기틀 보호 대상 — 운영자 확인 없이 변경 금지)
- a) 통합 플로우 3스텝(분석→확정{모드 선택}→렌더) · 분석/렌더 잡 분리 · 분석 1회=렌더 N회(R2 src 보관).
- b) 과분할>과병합 편향(SIM_MERGE 하향 = 남남 병합 위험 → 운영자 확인) · 렌더 5원칙의 마진 축소.
- c) tracks.json/video.json 스키마·render 페이로드 `opts` 키 이름(뷰어↔api↔py 3자 계약) · PALETTE 순서(뷰어 track.html ↔ track_render.py 1:1 짝).
- d) 주변부(자유): 문구 · 픽셀 블록 크기 · 카드 UI 세부.

## 3. 알려진 한계·후속 (정직)
- a) YuNet은 옆·뒷모습·초소형(<12px) 얼굴을 놓칠 수 있음 — 마진·브리지가 단기 가림은 커버하나 장시간 뒤통수는 미커버. 후속 = 사람(전신) 검출 병행 폴백.
- b) 사물 모자이크(번호판·로고 등) = 2단계(SAM2 포인트 프롬프트 — 인물 확정 화면에 `사물 지정` 축 추가 예정).
- c) 수동 보정(특정 구간 박스 수정) 미구현 — 실패 시 현재 답 = 재분석 또는 프리미어. 후속 = STEP2 스크럽 보정.
- d) 완료 웹푸시 미연결(뷰어 폴만) — 필요 시 notify_fail 패턴 이식.
- e) **VFR(가변 프레임레이트) 폰 영상 = A/V 미세 드리프트 가능**: rawvideo 파이프가 상수 fps로 재클럭(cv2 평균 fps 기준)이라 심한 VFR 소스는 립싱크가 어긋날 수 있음(모자이크·이름표 위치는 프레임 인덱스 기준이라 무영향 · 평의회4). 후속 = 프레임별 PTS 보존 먹싱.
- g) **완전 이탈 마지막 수 프레임의 뒤통수/모자 가장자리 조각 = 부분 커버**(실측: 얼굴 부위는 외삽 패딩이 끝까지 커버 · 잔여는 이목구비가 이미 화면 밖으로 나간 뒤의 비식별 머리 가장자리 8~40px 조각 ~0.15s). 재군집·경계 커버는 합성 E2E 실측 기준 — **실사(각도·조명 변화) 재진입 재군집은 미실측**(2단 임계 0.48/0.60이 흡수 설계 · 현장 과분할 시 카드 2장 = 둘 다 선택으로 해소).
- f) 렌더 error.log 감시는 *내용 변화* 기준선 — 같은 id 동일 문구 재실패는 미검출(하드 타임아웃이 백스톱 · ly 계승 트레이드오프).
