## 1. 구조 (ly 파이프 미러 · LLM = 분석 캡션 보조 1콜뿐 · 렌더/검출/군집 0콜)

> **엔진 모듈 계약(UI 무의존) = `apps/track/MODULES.md`** — 트래킹(M1)·모자이크(M2)·피사체 키잉(M2-b)·크로마키(M3 `track_chroma.py`)·세그 채움(M4 `track_maskfx.py` = 픽셀 실루엣 추종 모자이크·가면) 입출력 정본. UI 개편 시 이 계약만 보고 배선(운영자 모듈화 지시 — 경위 = `docs/작업이력.md`).

### 1) 흐름
- a) **analyze**: 뷰어 폼{URL 또는 파일 ≤30MB} → `functions/api/track.js`{업로드 = 일회용 `up-<id>` 브랜치} → `track-make.yml` `mode=analyze` → `apps/track/track_analyze.py`{얼굴 검출+트랙+군집} → `viewer/track_out/<id>/tracks.json`+`crops/p*.jpg` 커밋 → 뷰어 폴링(5s·error.log 우선·SPA HTML 가드).
- b) **render**: 확정 화면에서 모드·대상 선택 → `/api/track` `{render:{id,mode,targets,invert,names,colors,opts}}`{키잉 = `{id,mode:'keying',keep,extra,opts}` · §1.5 / 실루엣 = `{mode:'maskfx',keep,keepP,extra,fill,preset,opts}` · 크로마키 = `{mode:'chroma',opts}` · §1.6} → `mode=render` → `apps/track/track_render.py` → 결과 mp4(키잉·크로마키 = MOV+webm) R2(`track_res/<id>/…`) → `video.json{url,mode,ts}`{키잉·크로마키 +`preview`·`note`} 커밋 → 뷰어 폴{ts 신선도 + 모드 일치 게이트 = 과거·타 모드 결과 오인 차단}.
- c) **분석 1회 = 렌더 N회**: 원본은 analyze 때 R2 `track_src/<id>.<ext>` 보관(R2 미설정 = ≤30MB git 폴백 · 둘 다 실패 = `src_note:"src-lost"` → 렌더가 명확 에러). 재렌더(다른 인물·다른 모드)는 재업로드·재분석 0.
- d) **카드 캡션 + 동일인 병합 힌트 (분석 보조 LLM 1콜 · 운영자 승인 = LLM 0콜 기틀 해제 · 경위 = `docs/작업이력.md`)**: analyze 직후 `.github/scripts/track_caption.sh`가 claude -p{**opus 4.8 · effort max**(운영자 지정 · §모델 d 등재) · `--safe-mode`·Read만 · 폴오버 SSOT 경유}로 크롭들을 보고 {카드별 한 줄 묘사(`people[].cap`·`subjects[].cap` — 프롬프트 14자 지시·파서 20자 하드캡) + *확실한* 동일인 pid 쌍(`meta.same_hint`)}을 tracks.json에 additive·원자 교체 병합 → 뷰어 카드가 크롭 아래 회색 캡션 + 좌상단 `#N 같을 수도` 무채색 글래스 배지로 표시(없으면 미표시 = 구 tracks.json 하위호환). 불변: **전면 fail-soft — 어떤 실패도 exit 0·error.log 금지**(error.log = 분석 폴 치명 신호 · 워크플로 캡션 축 3스텝도 `continue-on-error` 필수 = 스텝 하드킬이 failure() error.log를 못 만들게) · 묘사만(실명·신원·유명인 추정 절대 금지) · 병합 힌트는 표시 전용(SFace 군집·렌더 대상 무개입 — 사용자가 두 카드 다 선택하면 끝) · 모델이 얼굴 비교를 거부하면 same=[] = 정직. 게이트 = `TRACK_CAP`+워크플로 입력 `cap`(§파이프라인 e 카나리아: 기본 OFF 머지 → dispatch cap=1 실측 → 기본값 승격). ⚠️ 프라이버시(정직): 캡션 활성 = 얼굴·피사체 *크롭 이미지가 Claude(Anthropic)로 전송*됨 — 실명 미출력 ≠ 이미지 미전송(운영자 인지 하 승인). cap=1이면 tracks.json은 {분석 직후 조기 커밋 → 캡션 병합 2차 커밋} 2단 — 첫 세션은 캡션 없이 볼 수 있고 재진입 시 표시(수용 트레이드오프).

### 2) 모델·알고리즘 (전부 로컬 = 토큰 0 · 과금 = Actions 분 + R2뿐)
- a) 검출 = **YuNet**(232KB) · 정체성 = **SFace 임베딩**(37MB) — 둘 다 opencv 내장 API(`FaceDetectorYN`/`FaceRecognizerSF`), onnxruntime·torch 불요. 다운로드 = `setup.sh`가 `media.githubusercontent.com`(⚠ raw.githubusercontent는 LFS 포인터 131바이트 함정 — 실측) + **sha256 핀**. 캐시 = `~/.cache/nomute-track`(actions/cache 7일 · 렌더 모드는 `TRACK_MODE=render`로 모델 스킵).
- b) 트래킹 = 검출 스텝(초당 12회) 간 그리디 IoU 매칭 → **검출 기반 = 드리프트 0**(매 검출이 재접지 — 상관 트래커의 누적 어긋남 원천 부재). 트랙 종료 = 0.7s 미검출. 오탐 컷 = 3회 미만 검출 트랙.
- c) 군집 = 트랙 대표 임베딩(고신뢰·대형 top6 정렬크롭 평균 — **정렬크롭은 원본 프레임 + 좌표 업스케일**{축소 프레임 정렬 = 고해상 소스 저질 임베딩 → 과병합 꼬리위험 · 평의회3}) 코사인 ≥ `SIM_MERGE 0.48` 그리디 병합 — **과분할 > 과병합 편향**(동일인 카드 2장 = 무해{둘 다 선택하면 끝} / 남남 병합 = 엉뚱한 얼굴 모자이크 = 치명). 퇴장→재등장·컷 전환도 임베딩이 같은 사람으로 재결합(합성 E2E 실측: 1s 퇴장 후 1명 재결합).
  - 가) **pid 봉합 = 전신 must-link post-merge(260710)**: 같은 *연속 전신(person) 트랙*에 투표(≥2)로 연결된 얼굴 트랙들의 people은 임베딩 유사도 미달이어도 병합(`merge_people_by_body` · 측면·소형 얼굴의 pid 분열 봉합). **SIM_MERGE 하향이 아님** — 별도 고정밀 증거 축이라 과분할>과병합 기틀 안에서 작동. 오병합 방어(평의회2 보강 260710) = {증거 자격 = 카드 필터와 동일(SUBJ_MIN_DETS+SUBJ_MIN_AREA · 원경 잡음 전신 배제)·긴 트랙 우선 결정적 순서 · veto① 동시존재(두 그룹 얼굴 세그 시간중첩 >0.2s = 그 전신 트랙 증거 통째 폐기) · veto② 임베딩 강반대(*멤버 쌍별* cos <0.05 = 짝 거부 — 그룹 평균 희석 차단) · veto③ 무임베딩 보수화(양쪽 그룹 emb 부재 = 투표 하한 2→4 상향)}.
- d) 렌더 고퀄 7원칙: ① 갭 보간(깜빡임 0) ② 3탭 이동평균 스무딩(지연 0) ③ 시간 마진 ±0.3s(한 프레임 노출도 실패 → 과잉 커버) ④ 같은 인물 트랙 간 ≤1.2s 갭 브리지 ⑤ 공간 마진 가로 1.45×·세로 1.6×·위 10% 시프트(이마·머리카락) ⑥ **가장자리 접촉 끝단 = 속도 외삽 패딩**(0.3s→0.8s · 화면 이탈/진입 슬리버{검출 불가 부분 얼굴 조각·평의회I 실측 4~15프레임 노출}를 이동 방향 그대로 추적 커버) ⑦ **얼굴 미검출 구간 = 전신 폴백(260710 "종료지점까지")**: pid 연결 전신 트랙(`people[].body`)에서 머리 영역을 추정(`head_from_body` — 캘리브레이션 `hr` 비율 중앙값·클램프·1.25× 안전 팽창 = 과잉 커버 편향)해 얼굴 kf와 단일 스트림으로 병합 → 뒤통수·측면·몸만 구간도 퇴장 시점까지 이어 커버. 값 유도 = 렌더 시점(분석은 body 원본+hr만 저장 = 공식 튜닝에 재분석 0 · "분석 1회=렌더 N회" 유지). 얼굴 kf ±0.25s 우선 = 실측·추정 교대 지그재그 차단.
  - 가) **가림 범위 scope(운영자 260710 "얼굴만 선택 기능")**: 모자이크·핀셋 대상 카드마다 `얼굴만 | 전신` 토글 — `scopes:{pid:"body"}` 페이로드(미기재 = face 기본). face = 얼굴 + ⑦ 머리 추정 폴백 / body = 전신 박스 가림(전용 마진 1.10/1.08 — 얼굴 마진 1.45/1.6은 전신 박스에 과잉) + 얼굴 스팬 병행(전신 미검출·클로즈업 폴백). 토글은 `body` 있는 카드(v3 분석)만 노출 = 구 분석 하위호환.
- e) 모자이크 = 픽셀레이트 + **조절 옵션(운영자 요청)**: `opts{pxw·pxh(블록 가로/세로 분할 수 3~20 — 프리미어 수평/수직 블록 동일 개념 · 상한 20 = 얼굴당 ~14블록 = 재식별 방지 바닥 · 기본 9) · size(커버 배율 0.75~2.5 — 하한 0.75 = 하단 시프트 구속{0.4+0.8×0.75≥1} = 커버 ≥ 검출박스 전 변 · **기본 1.15** · 260711) · feather(가장자리 페더 0~40px · **기본 5** · 슬라이더 step 1) · shape(rect/ellipse · **기본 ellipse** · 260711 3분류)}`.
  - 가) **커버 보증 = 코어-강제 마스크(평의회 A·G 봉합)**: 코어(마진박스 · *참 중심* 기준 = 화면 가장자리 클램프에도 후퇴 0) 마스크를 블러 후 `maximum(core)`로 내부 1.0 복원 — **페더는 코어 바깥 링 전용**(구 선확장-내축 방식은 소형 얼굴{폭<83px}에서 중심까지 반투명 = 노출이었음·실측 봉합). 블록 절대 하한 8px(구 렌더 익명성 바닥 유지).
  - 나) **타원 = 코어 내접 모양 선택**: 검출박스 모서리(대부분 배경·목)는 타원 밖 — size 하한 0.75가 얼굴 타원 커버를 보증. **완전 박스 커버가 필요하면 네모**(정직 경계).
  - 다) 기본값(9·9·1.15·5·타원 · 260711 3분류 — 구 9·9·1.0·0·네모) = 얼굴 타원 커버(모서리 제외 · 완전 박스 커버 = 네모 선택) · **커버 영역은 구 렌더와 동일하나 타일 분할은 종횡 독립(세로 = 높이 기준 · 구 폭 기준) = 픽셀 비동일**(세로 블록이 소폭 굵어짐 = 안전 방향 · 평의회F 정직 정정). UI = 뷰어 STEP2 모자이크 전용 게이지(ly 옵션 게이지 계승 · pend 영속 · 편집 즉시 재저장). 핀셋 = PIL{코너 브래킷+리더선+필 이름표 · 스타일 = 구 목업 캔버스 계승 · 폰트 = fonts-noto-cjk(fc-match) · **CJK 폰트 하드게이트**(`assert_cjk_font` — 없으면 두부 번인이 성공으로 위장 = ly-make 두부 게이트 동일 원리·사일런트 폴백 금지)}. 인코딩 = ffmpeg rawvideo 파이프 → libx264 crf19 + 원본 오디오 aac(+faststart). **출력 키 = stable**(`track_res/<id>/<mode>.mp4` 덮어쓰기 + `?v=` 캐시버스트) — epoch 접미 = 재렌더(주 루프)마다 R2 고아·git mp4 무한 누적(ly stable+`?v=` 불변 계승).
- f) 좌표 불변: **분석·렌더 모두 cv2 디코드 + `CAP_PROP_ORIENTATION_AUTO=1`** — 폰 세로영상 회전 메타를 양쪽 동일 처리 = 좌표 공간 일치. 한쪽만 바꾸면 모자이크가 통째로 어긋난다(절대 한쪽만 수정 금지).

## 1.5 키잉 — 선택 피사체만 남기는 알파 렌더 (운영자 승인 · 요구 = {피사체 기본 조회 · 빼기 가능 · 누락 직접 지정} · 결정 경위 원장 = `docs/작업이력.md`)
- a) **흐름**: analyze가 `subjects[]`(전신·사물 카드)까지 뽑음 → STEP2 `키잉` 모드에서 남길 피사체 다중선택(+`누락 피사체 직접 지정` = 원본 스크럽+탭 → `{t,x,y}` 정규좌표 · **양방향 전파 = 어느 장면에서 찍어도 앞뒤 전체 커버**) → 렌더 = 선택 제외 전 픽셀 투명 → **마스터 ProRes 4444 MOV(프리미어 네이티브 알파) + 프리뷰 VP9 webm(뷰어 체커보드 재생)** → R2 `track_res/<id>/keying.mov`·`keying_preview.webm`(파일 직접 업로드 — GB급이라 bytes 적재 금지·타임아웃 900s).
- b) **피사체 조회(분석 확장)** = YOLO11n `cv2.dnn`(onnx 레포 커밋본 `apps/track/yolo11n.onnx` · 추가 pip 0 · 53ms/f 실측) 공칭 초당 4회{2→4 = 260710 전신 폴백 편차 절반 · 검출 스텝 양자화 · 5분 영상 +약 64s} → 화이트리스트 선필터 + 클래스별 NMS → 그리디 IoU 트랙 → 얼굴 트랙과 연결 투표{얼굴 중심이 전신 상단 55% 안·2표 이상 — 투표 프레임에서 얼굴/전신 비율 `cal` 샘플 동시 축적 = `hr` 캘리브레이션 원천} = 같은 pid 재결합·얼굴 크롭 재사용 + **pid 확정 전신 세그는 `people[].body`로도 투영**(subjects 캡 적용 전 원천에서 = 캡과 디커플 · 렌더 ⑦ 폴백 소비). 얼굴 없는 인물(뒷모습)·사물(차량·동물 등 `SUBJ_CLS`)도 카드. 상한 12(`subj_dropped` 정직 표기 · 중도 실패 = `subj_note:"partial"`) · 사물 재등장 = 카드 분리(과분할 무해 철학 동일).
- b-2) **키잉 얼굴 단위(운영자 260710)** = `keepP:[pid]` 별도 배열(keep 정수 계약 불변) — people 카드(`pf`/`pb` = 첫 양호 *얼굴* 프롬프트 · subjects pf 산식 미러)를 키잉 그리드에도 노출('얼굴' 라벨·`#p` 힌트). SAM2 프롬프트 = pb 1.3× 팽창(`FACE_KEY_EXPAND` — 머리카락·두상 포함 유도). 캡 4객체 = keep+keepP+extra 합산 · 예산 가드는 패스 prompts 수 기반이라 자동 합산. pb 없는 구(v2) 분석에서 keepP = 명시 에러(재분석 안내).
- c) **마스크 엔진** = SAM2.1 tiny(공식 ckpt·Apache) × ultralytics 비디오 전파 — 실측(4vCPU=러너 동급): 512입력 627ms/f(1객체)·1030ms/f(3객체)·마스크 오차 0.35% / 1024는 3937ms/f = 불가 판정. **키잉 캡 = 90초·피사체 4개·긴 변 1920px**(분석 300s 캡과 별개 · `90s ≈ 25분`은 30fps·단일 패스 기준 — 멀티패스 총량은 **발사 전 예산 가드**(`KEY_BUDGET_SEC` = 실측 단가 × Σ패스 프레임 ≤ 27분)가 사전 거절 + 루프 30분 백스톱), 세그 15fps + 마스크 hold + 페더(기본 3px = 512 업스케일 계단 완화).
- d) **멀티패스 불변**: SAM2는 전파 시작 후 새 객체 추가 금지(가드 실측) + 박스·포인트 혼합 = 같은 객체로 접힘(`_prepare_prompts` cat) → **프롬프트 프레임 그룹(±0.5s)별 패스 분리·박스/포인트 패스 분리·마스크 합집합 + 패스마다 `inference_state` 리셋**{라이브러리 init_state가 `len>0`이면 스킵하는 가드라 리셋 없인 두 번째 패스부터 프롬프트가 조용히 무시됨 — 평의회 적발·봉합}. 프롬프트 = 첫 *양호* 등장 프레임(`pf` = 면적 중앙값 70% 이상 첫 검출 — 진입 슬리버 프롬프트 방지). **직접 지정 = 순패스(탭→끝) + 역패스(탭→0 · `[0,t0)` 역재생 트림) — 구간이 0.7초 이하인 쪽은 생략(탭=끝이면 역만 — 순패스 0프레임 사망이 역패스까지 무산시키는 것 방지 · 초단편은 순 폴백)** — 역트림은 `reverse` 필터가 전 프레임을 램에 버퍼라 **512급 스케일 필수**(원해상 = GB급 폭발 · SAM2 입력이 어차피 512라 마스크 품질 동급 · 합성에서 원해상 업스케일 · 예산 가드에 자동 포함).
- e) **무거움 격리 불변**: torch+ultralytics는 **키잉 렌더 잡에서만** 설치(`TRACK_HEAVY=1` · pydeps/모델 캐시 키 h-접미 분리) — 분석·모자이크·핀셋 경로는 캐시·설치 비용 불변. `track_render.py`는 keying일 때만 `track_keying` lazy import.
- f) 실패 = 기존 fail-soft 그대로(video.json{error} rc0) + 정직 가드 3종{전 구간 세그 실패(실마스크 0) = 명시 에러 · 마스크 치수 불일치(ffmpeg↔cv2 회전 파리티 90/270 발산) = 명시 에러 · 예산 초과 = 발사 전 거절}. 마스터 업로드 실패(R2 미설정·30MB 초과) = `note:"master-lost"` + 프리뷰만 — 뷰어가 정직 표시·저장 버튼 숨김.

## 1.6 실루엣 채움(M4)·크로마키(M3) — 모드 4·5번째 버튼 (운영자 260712 배치 승인 · 엔진 정본 = `MODULES.md`)
- a) **실루엣(`maskfx`)** = SAM2 마스크가 실루엣 그대로 피사체를 따라가며 그 영역만 채움 — 선택 축 = **키잉과 동일**(인물 카드 = 얼굴 · 피사체 카드 = 전신/사물 · 직접 탭 = 양방향 전파 · `keyLike()` 공용) + 채움 세그{`모자이크`(블록 픽셀레이트·전 구간 고정) ↔ `가면`(내장 프리셋 칩 3종 = 스마일·가림[실루엣 재단 완전가림]·하트 — `apps/track/assets/masks/*.png` · 경로 아닌 프리셋명만 전송 = py `MASK_PRESETS` 화이트리스트와 이중)} + 페더 게이지(기본 8). 캡 = 키잉 동일(90s·keep+keepP+extra 합산 4객체·예산 가드) · 러너 = `TRACK_HEAVY`(keying과 동일 스택 — 워크플로 contains에 maskfx 편입). 위임 = `track_render.run_maskfx` → `track_maskfx.run`(로컬 in→out) + src 회수·R2/git 업로드·`video.json{url,mode,fill,n,frames}`(mp4 번인 = 모자이크 소비 계약).
- b) **크로마키(`chroma`)** = 특정 색 빼기 — **대상 선택 없음**(카드 그리드·카운터 숨김 · 색만). 노브 = 색상 칩{그린·블루 — 뷰어는 키워드만 전송·hex 해석 = api(`CKC` 매핑 · 디자인 게이트 hex 0)} + 강도(similarity ×100 게이지) + 테두리(choke −4~4) + 부드럽게(feather 0~10) · `despill .5`·`blend .05`·`edge high` = api 고정 주입(운영자 `테두리` 우선 — 노브 최소화). 캡 = 300s(키잉 90s와 별개 · `durOver` 301 게이트). 러너 = 경량(ffmpeg 단독·TRACK_HEAVY 아님). 위임 = `track_render.run_chroma` → `track_chroma.run` + 키잉 산출 계약 미러(ProRes4444 MOV 마스터+VP9 webm 프리뷰 · `video.json{url,preview,mode,note,kind,opts}` · master-lost 동일) — 뷰어 결과 = 키잉과 같은 체커보드 알파 표시.
- c) **위임 관례**: 두 모듈 다 실패 = `die(SystemExit(JSON))` — track_render fail-soft는 Exception만 잡으므로 위임 래퍼가 `except SystemExit`로 메시지 회수 후 RuntimeError 변환(미변환 = 실패 사유가 video.json에 못 실림). 실측(260712 배선) = UI 실클릭 payload → 위임 실행: maskfx 가면 720p 8s = 85s · chroma edge:high 720p 8s = 28s.
