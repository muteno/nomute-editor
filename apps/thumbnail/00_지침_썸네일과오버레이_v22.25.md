## §공통 원칙 — 포맷 미지정 처리 (F04/F16/F17 공유)

## 절대 규칙

1. **(통합) 코드 파일 불변** — `nomute_overlay.py` / `nomute_compose.py` / `nomute_copyright.py` / `nomute_reels2.py` 네 파일의 **내용 수정 금지, import만 허용.** 다음 모두 금지: ⓐ 내용 수정, ⓑ "개선"·"최적화"(파일 내용 한정), ⓒ 에러 시 코드 고치기 — **에러는 고치지 말고 그대로 사용자에게 보여라.** 실행 방법은 명시된 그대로만. **사용자 직접 교체는 예외.**

2. **실행 전 `*` 강조 페어 검증 (F13) — 강화.** `generate` 호출 전 각 라인을 점검하고, 짝이 어긋나면 렌더하지 말고 **어느 줄·어느 위치**인지 지목 후 교정한다:
    - **ⓐ 줄별 `*` 개수 = 짝수.** 홀수면 여는/닫는 별 한쪽 누락 → 스킵 확정. 해당 줄·위치 지목.
    - **ⓑ 여는 별·닫는 별은 같은 줄 안에서 1쌍 완결.** 강조 구간이 줄 경계를 넘으면(L1에 여는 별 / L2에 닫는 별) **양쪽 다 스킵** → 한 줄 안에 닫히게 재배치.
    - **ⓒ `parse()` 자가 대조.** 의도한 강조가 🟢('h') 세그먼트로 정확히 잡히고, **빈 강조(`''`) 및 일반('n') 세그먼트의 `*` 노출이 0**인지 확인.
    - **근거**: 정규식 `*[^*]+*`는 줄 안에서 1쌍이 정확히 닫혀야만 강조. **별 1개를 써도 위 조건 위반 시 스킵된다.** (실측 확인 — 닫는 별 누락·홀수개·줄 경계 갈림·강조 안 별 전부 스킵 재현)
    - 자동 환기: §오버플로우 가드의 **페어 가드 출력**이 measure 루프 끝에 박혀, 렌더/B안 작성 직전 강제 노출된다.

3. 입력 텍스트 자동 변환 화이트리스트 4종 (`...`→`⋯`, `•`→`·`, `이,`→`李,`, `윤,`→`尹,`). 그 외 원문 보존이 디폴트. (F11)

4. 정형 사이클 thinking 즉시 실행. 깊은 사고는 분석/디버깅/엣지/룰 패치/오버플로 B안 + 사용자 명시 트리거에 한정.

5. **맞춤법: 감으로 판단, 웬만하면 수정 + 알림 디폴트.** 의도 명백한 케이스(강조 묶음 내부, 자모 분리, 밈/신조어, retro, 짧은 구절 압축)만 원문 유지 + 사유 보고. 룰 강박 X. (F12)

6. 입력 텍스트의 사실/내용 진위 검증 비실행 (사용자 명시 요청 시 예외).

7. 룰 개선 제안 출력 금지 (사용자 명시 요청 시 예외). 자가 보고 1줄은 디버깅용 유지.

8. **(구 10번) 버전 갱신 프로토콜 — 회귀 방지.** 기능을 바꿀 때 **변경 규모에 따라 발동 강도 분기**:

    **［풀 프로토콜］** — 다음 중 하나면 발동:
    - 지침/메모리 **버전 넘버 갱신**
    - 지침 **구조적 변경 / 재작성**
    - **기능 추가 또는 삭제**

    → **(a) 매니페스트 전수 대조**: §0 인벤토리(F01~F17) 전체를 체크리스트로 훑는다.
    → **(b) 변경 4분류 출력**: 이번 갱신을 `추가 / 변경 / 보존 / 삭제`로 명시. **삭제가 있으면 강조 + 사용자 승인 필수.**
    → **(c) C-등급 보존 선언 의무**: 카피라이트(F16)·경고문(F17)은 변경 대상이 아니어도 "보존 확인" 라인 필수.
    → **(d) 기능 제거 ≠ 기본값**: "효율성/정합성 개선"이 기능 제거를 의미하지 않는다. 개선하며 기능을 빼는 게 낫다고 판단되면 **임의로 빼지 말고** `[제거 제안] 기능 X → Y 대체 시 이점/손실` 형태로 승인받는다.

    > **［경계 판단］** 풀 vs 경량이 애매하면 **풀 쪽**(보수적).
    > **［규칙 서술 통합 vs 기능 변경］** 같은 기능의 반복 서술을 한 곳으로 합치는 건 *기능 삭제가 아니다*(보존). 동작이 바뀌거나 기능이 빠지면 그때부터 추가/변경/삭제. 통합 시에도 (c) C-등급 보존확인은 출력.

    **［경량 발동］** — **단일 파라미터 in-chat 튜닝** (sweep 한 칸, opacity 조정 등 **구조 무관 잔작업**):
    → 영향받는 **F만 짧게** 언급 (전수 대조·4분류 **생략**)
    → **C-등급 보존 확인 1줄은 유지**

## 시그니처 캐시

매 사이클 시작 시 `nomute_*.py` view 안 함. 메모리 시그니처 즉시 사용:

### overlay (F01~F03, F13)
- `generate(fmt, lines, out, opacity=None, tracking=None, lm_offsets=None)` — lines는 `*강조*` 포함 list[str], lm_offsets는 라인별 좌측 마진 오프셋 list[int] 또는 None
- `draw_t(d, x, y, t, f, c, tp, us)` — t=`*` 제거 plain, tp=tr/1000*fs
- `parse(t) → list[(tag, text)]` — tag=`'h'`(강조 🟢, 색 15,253,2) / `'n'`(일반 흰색). 정규식 `*[^*]+*` 기준 분절. **페어 검증(F13)·실제 렌더 양쪽이 이 함수로 강조를 판정** — 같은 줄 1쌍 안 맞으면 별이 'n'으로 새거나 빈 'h'(`''`) 발생.
- `SPECS[fmt]` keys: `w, h, fs, lh, tr, lm, ty, logo_rect, stroke, grad`

### compose (F05)
- `prepare_background(bg, W, H, fmt, adj_offset_x=0, adj_offset_y=0, adj_scale=1.0) → (rgba, case, method, crop)`

### copyright (F16/F17)
`nomute_copyright.py`는 함수 import가 아니라 **CLI 스크립트**. 호출법:

- 검증: 스크립트 내부 numpy alpha bbox 자체 출력 (`OK: ... text_x=...`)

### lm_offsets 사용법·영향 범위 (F03)
- `lm_offsets=[0, -15]` → L1은 기본 lm, L2는 15px 좌측 이동. 양수=우측, 음수=좌측. 리스트 길이 < lines 수이면 나머지 라인은 기본 lm. None이면 전 라인 기본 lm.
- **영향받는 것**: 텍스트 x 시작점, 로고-텍스트 좌측 정렬 밸런스, 해당 라인 가용폭. **자동 연동**: 섀도우(텍스트 이동 시 자동 추종). **영향 없는 것**: 로고 위치(logo_rect 고정), 그라디언트, 합성(compose), 저작권(copyright), measure() 폭 측정.

### measure() — 픽셀 검증 정규 정의 (PYEOF 표준)
> **정의 권위는 여기.** 표준 bash의 PYEOF 안에 들어가는 measure는 **이 정의의 인라인 사본**이다(heredoc은 외부 정의를 끌어오지 못하므로 PYEOF 안에 물리적으로 포함돼야 실행됨). 변경 시 이 정의 한 곳만 고치고 사본에 반영.

```python
def measure(text, sp, font, tr):
    plain = text.replace('*', '')
    img = Image.new('RGBA', (5000, 300), (0,0,0,0))
    draw_t(ImageDraw.Draw(img), 100, 100, plain, font,
           (255,255,255), tr/1000*sp['fs'], sp['stroke'])
    bbox = img.getchannel('A').getbbox()
    return (bbox[2] - 100 + 5) if bbox else 0
```

**view 허용**: 시그니처에 없는 새 함수, 파일 교체 정황, 디버깅 시.
**view 금지**: 정형 사이클, 시그니처 내 함수·CLI 호출법만 사용 시.

### B안 제시 룰 — lines별 분기 강제

자간 한계 초과 시 사용자에게 B안 제시. **현재 라인 수에 따라 옵션 종 수 분기**:

|---|---|
| 1줄 오버플로 | **3종**: 축약 / 유의어 / 2줄 분할 |
| 2줄 오버플로 | **3종**: 축약 / 유의어 / 3줄 분할 |
| 3줄 오버플로 | **2종 ONLY**: 축약 / 유의어 (줄 추가 절대 금지) |

> 오버레이 텍스트 최대 3줄 한계. "B안 3종" 관성으로 lines=3에 4줄 옵션 섞는 것 금지.

### 가드 출력 (PYEOF measure 루프 끝) — LINES 가드 + 페어 가드 (F08 / F13)

measure 루프 끝에 **두 가드를 함께 출력**한다. 실행결과에 박혀 B안/렌더 직전 자동 환기.


## 릴스 헤더형 (형태2 — F18)

> **강조(`*`) 없는 reels 입력 전용** 신규 형태. 상단 헤더(로고+부제+제목) + 하단 영상 흰칸. 베이스 이미지에 텍스트 2줄+흰칸만 얹는다(오버레이/합성과 별개 경로). `nomute_reels2.py` (절대규칙1 불변·import만). **실측 검증: 첨부 원본과 픽셀차 0.**

### 트리거 (출력분기 ⑥)
- **이미지 X + reels + 강조(`*`) 없음** → 형태2.
- 강조(`*`)를 쓰면 기존 reels 오버레이(⑤)로 분기 — 형태2는 제목 줄이 **자동 초록**이라 `*` 마크업 불필요.
- **줄수 가드 (260613 사용자 확정)**: 입력 실효 줄수 ≠ 2(1줄·3줄 이상)면 **추측 렌더 금지 — `AskUserQuestion` 버튼으로 즉시 확인**(라우터 §선택 질문=버튼+알림): ㉠ **"오버레이형 의도"** → reels 오버레이 경로(⑤와 동일 출력 2종 — 강조 없으면 전체 흰색) / ㉡ **"헤더형 맞음(누락/초과)"** → 부제+제목 2줄 교정본 받아 ⑥ 진행. 자유 답변은 'Other'.

### 입력 → 출력
- 입력 2줄: **1줄 = 부제(흰 255,255,255), 2줄 = 제목(초록 15,253,2)**. **매핑은 위치 기계적 — 입력 순서 = 화면 순서(1줄=위, 2줄=아래). 의미 판단(헤드라인성·따옴표·임팩트)으로 줄 스왑 금지.** 부제/제목 라벨은 색·크기 스펙명일 뿐 의미 재배치 근거가 아니다. (260613 실측 회귀: 따옴표 줄을 '제목답다'고 아래 초록으로 내림 → 사용자 의도 위반 → 재렌더. 이 줄로 차단.)
- **텍스트 = 원문 그대로 (따옴표 가드)**: 따옴표·문장부호 포함 입력 문자열을 그대로 렌더 — **인용부호를 장식·마크업으로 보고 벗기기 금지**(변형은 화이트리스트 4종만, 절대규칙 3). ⑥은 ①~⑤와 달리 measure 루프 에코가 없어 누락이 안 드러남 → **표준 bash의 `TEXT:` 에코(원문 가드)로 입력↔렌더 문자열을 대조한 뒤에만 보고의 "따옴표 OK"를 적는다**(에코 대조 없는 "따옴표 OK"는 오보). (260613 실측 회귀: 따옴표 벗기고 "따옴표 OK" 오보 → 재렌더. 이 가드로 차단.)
- 출력: **2160×3840 PNG(2K·SCALE2·260619; 설계 1080×1920 기준)** **2종 동시(v22.24)** — **①(기본 1번) `_reels2_nobg.png` = 흰칸 없음**(베이스 그라데이션 그대로 + 텍스트 2줄) / **② `_reels2.png` = 흰칸(y590~1431, 영상 자리) 포함**. 두 종의 텍스트·자간 sweep은 픽셀 동일(실측). 베이스 = `assets/reels2_base.png`(그라데이션+로고).
- 한 종만 원하면 말로 지정("흰 배경 없는 것만"/"흰칸 있는 것만") → 표준 bash의 해당 ①/② 부분만 실행. (구 v22.22~23 "nobg=요청 시 옵션" 룰은 "기본 2종"으로 대체 — 흰칸 버전 명세는 ②로 보존.)

### 시그니처 (nomute_reels2.py)
- `render(sub, title, base_path, out, sub_fs=66, sub_y=270, title_fs=90, title_y=385) → {out, sub_tr, title_tr}`
- `_draw_line(d, text, y, font, color, fs)` — 가운데 정렬. 폭이 AVAIL 이내면 단일 렌더(자간0·레거시 동일), 초과 시 글자별 tracking으로 자간만 좁힘. 적용 tr 반환.
- `_fit_tr(text, font, fs)` — 자간 0→-1…로 AVAIL 안에 드는 첫 tr (하한 TR_MIN=-45).
- `_line_width(text, font, tr, fs)` — tracking tr(1/1000 em) advance 합 폭.
- 고정 스펙: FONT `NotoSansCJK-Bold.ttc` index=1 / GREEN=(15,253,2) / WHITE=(255,255,255) / 흰칸 BAND=(590,1431) / **MARGIN=60·AVAIL=960·TR_MIN=-45**.

### 좌우 마진 + 자간 sweep
