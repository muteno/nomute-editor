# 편집 (피사체 모자이크) — 두뇌 v1

> 진입 = 뷰어 이미지 도구(`이미지`)의 **`편집` 탭**(thumb.html app='8'). 형제 탭 = 썸네일·카드뉴스·제작·합성(구 리사이즈). 스킬 명령어 없음(뷰어 전용).
> 존재 이유 = **단일 이미지에서 특정 피사체(얼굴·인물·사물)에 정확히 묻는 모자이크**. 핵심 = "모자이크를 어떻게 하느냐보다 *어떤 피사체에 딱 묻느냐*"(운영자 260712) → 검출 정확도 + 지정 정밀이 관건. 아키텍처 = `apps/track` 파이프를 단일 이미지로 미러(영상 기계 전량 제거).

## 1. 구조 (track 파이프 미러 · LLM 0콜 · 순수 OpenCV/PIL)

### 1) 흐름 (분석 1회 = 렌더 N회)
- a) **analyze**: 뷰어 폼{이미지 파일 ≤30MB · JPG/PNG/WEBP} → `functions/api/imgedit.js`{업로드 = 일회용 `up-<id>` 브랜치} → `imgedit-make.yml` `mode=analyze` → `apps/imgedit/img_detect.py`{YuNet 얼굴 + YOLO11n 피사체 검출} → `viewer/imgedit_out/<id>/boxes.json` 커밋 + 원본 R2 `imgedit_src/<id>.<ext>`(폴백 = git `outdir/src.<ext>`) → 뷰어 폴링(error.log 우선).
- b) **render**: 확정 화면에서 박스 탭 지정 → `/api/imgedit` `{render:{id,targets,opts,precise}}` → `mode=render` → `apps/imgedit/img_mosaic.py` → 결과 jpg R2 `imgedit/<id>/out.jpg`(폴백 git) → `result.json{url,ts,precise}` 커밋 → 뷰어 폴.
- c) 원본은 analyze 때 보관 → 재렌더(다른 피사체·다른 옵션·정밀 토글)는 재업로드·재검출 0.

### 2) 검출 = track 함수 재사용 (`img_detect.py`)
- a) `track_analyze`를 **import**(안전 = `__main__` 가드) → `load_models`(YuNet)·`load_subj_net`(YOLO11n)·`yolo_detect`·`SUBJ_CLS`·`kst_now` 그대로 사용. 영상 기계(VideoCapture·트래킹·SFace 군집·전신 봉합·ffmpeg)는 미사용 = **이미지 1콜**.
- b) 출력 `boxes.json` = `{meta:{w,h,src_url,src_ext,made}, items:[{id, kind:"face|person|object", label, box:[x,y,w,h], score}]}`. 큰 것 우선·캡 24. 얼굴은 YuNet, 인물·사물은 YOLO11n(fail-soft — YOLO 실패 시 얼굴만).
- c) **EXIF 회전 = 검출·렌더 동일 로더 필수**(`load_image_bgr` = PIL `exif_transpose` → BGR). 한쪽만 바꾸면 좌표 어긋나 모자이크가 통째로 밀린다(track `CAP_PROP_ORIENTATION_AUTO` 대응).

### 3) 모자이크 2티어 (`img_mosaic.py`) — 운영자 260712 승인
- a) **기본(박스/타원)** = `track_render.mosaic_region` **재사용**(코어-강제 커버 픽셀레이트 · pxw/pxh 블록 분할·size 커버 배율·feather 페더·shape rect/ellipse). 경량(opencv) · "딱 묻는" = 검출 박스/내접 타원 수준.
- b) **정밀(SAM2 실루엣)** = `precise:true` → `ultralytics SAM(sam2.1_t.pt)` image predictor 1콜로 피사체 실루엣 마스크 → 그 윤곽에만 픽셀레이트(`mosaic_by_mask`). 헤비 스택(torch+ultralytics · track 키잉과 동일 모델·캐시 키 분리). **SAM2 실패 = 박스로 fail-soft 폴백**(정밀이 안 되면 기본이라도 나옴).
- c) 기본값 = pxw/pxh 9 · size 1.15 · feather 5 · shape ellipse(track 모자이크 기본값 계승). API·py 이중 클램프(pxw/pxh 3~20·size 0.75~2.5·feather 0~40).

## 2. 기틀 (§기틀 보호 — 운영자 확인 없이 변경 금지)
- a) 2모드(analyze/render) 단일 워크플로 · 분석 1회=렌더 N회(원본 R2 보관) · 검출/렌더 = track 함수 재사용(중복 포팅 금지 = 드리프트 차단 · SSOT = apps/track).
- b) boxes.json/result.json 스키마 · render 페이로드 `{targets,opts,precise}` 키(뷰어↔api↔py 3자 계약).
- c) 정밀 티어 헤비 격리(캐시 키 `IMG_HEAVY` 분리 = 경량 경로 오염 0) · EXIF 동일 로더 불변.
- d) 주변부(자유): 검출 캡·DET_LONG·픽셀 블록·페더 기본값·SUBJ_CLS 확장(= apps/track SSOT).

## 3. 상한·에러
- a) 업로드 ≤30MB · JPG/PNG/WEBP(매직바이트 게이트) · 검출 카드 캡 24 · 타깃 캡 32.
- b) analyze 실패 = `/tmp/imgedit_err.txt` → 워크플로 failure() → `error.log` 커밋(뷰어 즉시). render 실패 = **fail-soft** `result.json{error}` rc0(네이티브 크래시만 스텝 실패 → failure()가 result.json도 error로 덮음 = 스테일 성공본 오표시 차단).

## 4. 한계 (정직)
- a) 정밀(SAM2) = 512 마스크 엣지 계단(페더 완화 · 로토 대체 수준) · 박스 프롬프트가 의도보다 크게 물 수 있음(상체까지) — track 키잉 한계 계승.
- b) 검출 못 한 피사체는 지정 불가(v1 = 검출 박스 탭만 · 임의 지점 지정은 후속 = track 키잉 `직접 지정` 포인트 프롬프트 이식).
- c) 겹친 피사체(얼굴+전신 중복 박스)는 둘 다 카드 = 원하는 것만 탭(과분할 무해 철학).
- d) 라이브 E2E는 Actions·R2·SAM2 러너 의존 = 세션 내 검증 불가(운영자 라이브 실행으로 확정).
