> 뷰어 `+ → 영상` 헤드리스 경로의 **모델 분기 정본**. `[모델: X]` 마커(`functions/api/k.js`가 부착)가 가리키는 절을 `prompts/k-make.md` 절차가 적용한다. 라인업·기본값 = 운영자 확정(260709 — Kling 최신 검색 반영 + Veo + Seedance · 기본 = Kling Omni).
> **편집은 이 문서에서만** · 모델 추가/삭제 = ① 여기 `## <id> —` 절 ② `functions/api/k.js` `K_MODELS` ③ `viewer/k.html` `K_MODELS` 3면 동기 — `shared/check_refs.py check_k_models()`가 정합을 커밋 전 기계 강제(한쪽만 고치면 rc=1).
> 검증 태그: **kling = 실측 정본**(지침 00 + CASES ⭐ 승격 체계) / **veo·seedance = 🟡 리서치 초안**(260709 웹 검색 — 실측 0). 초안 모델은 실사용 피드백이 오면 `CASES.md` 적립 + 이 문서 태그 승격, 버전·스펙 수치는 사용 시점 재확인 대상(엔진 업데이트 잦음).

## 공통 (모든 모델 불변)
- 15초 서사 원칙(지침 §2)·물리 성립(§3-2)·연출 문법(§3-1)·라이브러리 모듈 조립(§10 행 단위 Grep) = **모델 무관 공통** — 방언은 표현 형식이지 설계 로직이 아니다.
- 산출 골격(k-make.md 헤딩 계약 `⚙️/🎬/샷/🚫/🖼/📌`) **불변** — 이 프로필이 정의하는 건 {⚙️ 항목 구성 · 샷 블록 문장 방언 · 테일/네거티브 처리 · 참조 표기}뿐.
- 모델이 미지원하는 `[설정]` 항목 = 무효 처리 + `## 📌 안내`에 `○○: 이 모델 미지원 — △△로 대체` 1줄(조용한 무시 금지).
- 레퍼런스 이미지(`## 🖼 레퍼런스`) = 전 모델 유효(생성은 어차피 Gemini) — **첨부 방법 안내만** 모델별로 다름(각 절).

## kling — Kling 3.0 Omni (기본 ★ · 실측 정본)
- **정본 = 지침 00 전체** — 한국어 본문 + 영문 테일 3종 + `@` 단독·정체명사 + `### 🚫 네거티브` 블록 + ⚙️ Kling 화면 순서, 전부 현행 그대로(이 절은 포인터·규칙 중복 금지).
- 버전 노트(260709 검색 실측): **3.0 계열이 최신** — 3.0(26-02 출시·네이티브 오디오·4K·15s) + **3.0 Turbo·Omni 업그레이드(26-06-17)**{Turbo = 고속·저가(오디오 포함 초당 과금) · Omni 편집 3–15s·4K}. 3.1 없음.
- **Turbo 활용** = ⚙️ 해상도 2단계의 `검증` 단계 후보로 표기(검증 = Turbo/720p 반복 → 확정 = Omni 1080p) — §9-9 크레딧 2단계와 동일 정신.

## veo — Google Veo 3.1 (🟡 초안 · 260709 검색)
- **스펙**: 클립 **4/6/8s**(장편 = 분막 또는 extend) · 720p/**1080p** · 16:9·**9:16** · **네이티브 오디오·대사 동기**(동기 사운드 원조) · 이미지 참조(재료/첫 프레임) O · 진입 = Google Flow·Gemini API.
- **방언 = 영어 시네마틱 산문 단락**(샷당 1문단): 요소 순서 {Camera(샷 크기·앵글·무브 — 프론트로딩) → Subject → Action → Setting(장소·시간대) → Style · Audio}. 대사 = 인라인 따옴표 — 한국어 대사 기술상 가능하나 품질 미실측 = 짧게(§6 다이어트 준용) + 📌에 재실측 고지. 오디오 = 문장 서술(묘사형) + 필요 시 문단 끝 `Audio:` 큐.
- **네거티브 = 전용 `negativePrompt` 파라미터**(API/Flow) — `### 🚫 네거티브` 블록을 **영어 명사구 나열**로 산출(문장·`no ` 접두 금지 — Veo 관례). 샷 블록 안 `Avoid:` 문단은 **생략**(본문 회피는 긍정 서술로 — 지침 §2 정신).
- **참조 표기**: `@` 문법 **없음** — 샷 본문에 텍스트 앵커(같은 인물·의상 반복 서술)로 정체성 고정, `### 🔗 첨부 순서` 대신 📌에 `🖼 이미지를 Flow 재료(ingredients)/첫 프레임으로 첨부` 1줄.
- **⚙️ 구성(항목 순서)**: 모델(Veo 3.1 — Flow/Gemini API) · 클립 길이(4/6/8s — 초과 서사 = `클립C-샷N` 분막) · 비율(16:9/9:16) · 해상도(720p/1080p) · 오디오(네이티브 — 프롬프트 안 서술로 제어) · negativePrompt(🚫 블록 붙여넣기) · 수량.

## seedance — ByteDance Seedance 2.5 (🟡 초안 · 260709 검색 · 공개 = 26-07 초 예고 — **가용성 실측 필요**, 미가용이면 2.0 스펙 폴백)
- **스펙**: **네이티브 최대 30s 단일 생성**(경쟁 5–15s 대비 최장) · **4K(10-bit)** · 참조 입력 최대 50(이미지·오디오·영상 — 2.0은 12) · **동기 오디오 공동생성** · 로컬 리드로(부분 재생성) · 프롬프트 준수율 개선 주장(+20%).
- **방언 = 짧은 샷 나열 영어 산문**: `Shot N (Ns): ...` 번호 나열, 샷당 2~3문장{카메라 프론트로딩 → 주체·행동(능동동사) → 오디오 1줄 묘사}. 대사 = 따옴표 인라인(한국어 품질 미실측 = 짧게 + 📌 고지).
- **네거티브**: 전용 필드 미확인 — `### 🚫 네거티브` 블록은 산출하되 📌에 `필드 미노출이면 본문 회피 서술로 흡수` 1줄.
- **참조 표기**: `@` 문법 없음 — 🖼 블록 순서 = 업로드 순서를 📌에 고지(엔진 한도 50이나 우리 파이프는 ≤7 유지 = k_refgen 가드 불변).
- **길이 특례**: ≤30s = **분막 불필요**(15s 초과여도 단일 생성 설계 허용 — 지침 §1 분막 룰의 모델별 상한 치환) · 30s 초과만 `클립C-샷N` 분막.
- **⚙️ 구성**: 모델(Seedance 2.5 — 미가용 시 2.0) · 길이(≤30s 단일) · 비율 · 해상도(≤4K — 검증은 낮게) · 오디오(공동생성 — 프롬프트 서술 제어) · 참조 업로드 순서 · 수량.
