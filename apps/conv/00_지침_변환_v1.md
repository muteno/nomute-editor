# 변환(conv) 지침 v1 — 영상 다각도 변환(60fps 보간 · 비율 크롭/여백 · 트림 · 해상도 · 음량 통일)

> 진입 = 뷰어 `+ > 영상 > 변환` 탭(트래킹 우측 — 운영자 지정 배치). LLM 0콜·모델 0 = 순수 ffmpeg(과금 = Actions 분 + R2). 골격 = 트래킹(track) 미러 — 업로드/URL → Actions → R2 → video.json 폴링.

## 1. 동작 (운영자 확정 스코프 — 4기능 260710 버튼 확답 + 여백·음량 260710 2차 확답)
- a) **흐름**: URL(yt-dlp) 또는 파일{**R2 직업로드 ≤2GB**(`api/upload` 32MB 조각 · `up_src/*` 일회용 = 잡 끝 삭제 · 공용 `viewer/upload.js`) · R2 바인딩 없으면 ≤30MB up-<id> 브랜치 폴백(회귀 0) · 260711} → `conv-make.yml` 단일 잡 → `conv_run.py`가 ffmpeg 한 파이프{트림 → 비율 크롭 → 스케일 → fps} → h264 crf19+aac(+faststart) → R2 `conv_res/<id>/out.mp4`(stable 키+`?v=` 버스트) → `viewer/conv_out/<id>/video.json` 커밋 → 뷰어 폴이 결과 표시+`.dlbtn` 저장(api/dl 프록시).
- b) **프레임**: 유지 · **60fps 보간**(`minterpolate=fps=60` 기본 프리셋 = mci·obmc — 실제 중간 프레임 계산) · 30 · 24(다운 = `fps=` 필터 · 원본이 목표 이하면 무변환).
- c) **비율 = 자르기(crop)/여백(pad) 2방식**(`fit` · 비율 선택 시 뷰어 [자르기][여백(검정)] 세그): 원본 · 9:16 · 1:1 · 4:5 · 16:9.
  - 가) **자르기(기본)**: 잘리는 축 팬 위치 `pos`(0=좌/상 · 0.5=중앙 · 1=우/하). 뷰어 = 위치 슬라이더 + **위치 미리보기 드래그 프레임**(파일 선택 시 · objectURL — 키잉 스크럽 오버레이 계승). URL 소스 = 슬라이더만(프리뷰 불가 = 정직).
  - 나) **여백(검정 · 260710)**: 안 자르고 목표비 캔버스 **중앙** 배치 + 남는 축 검정 레터박스(운영자 = `1:1을 위아래 검정 채워 9:16 · 영상은 가운데`). `pos` 미적용 = 위치 슬라이더·미리보기 숨김. 캔버스 긴 변 = `min(1920, res캡)`에 목표비 정확 스냅(9:16 캡 = 1080×1920) · 원본은 캔버스 안 contain(업스케일 없음) · 패드는 fps 필터 *뒤*(보간이 검정 여백 픽셀을 계산하지 않게 = 예산 산식 불변) · 말미 `setsar=1`(contain 짝수화의 미세 SAR 제거 = 정사각픽셀).
- c-1) **음량 통일(`audio:'norm'` · 260710)**: 완성본 후처리 — 오디오만 {L/R 모노합 → loudnorm 2패스 −14LUFS·**TP−1.5dB**·LRA11} 재인코딩(aac 192k), 비디오 스트림 copy(재인코딩 0 = 몇 초). TP −1.5 = 운영자 하드 제약 `0dB 절대 안 넘음`의 마진(다이내믹 리미터·AAC 오버슈트 최대 ~1dB 흡수 — 버튼 문구 −1dB보다 안전 측 · 평의회4). **로직 SSOT = `shared/audio_norm.py`**(통합 편집기·ly 수렴 예정 — 자체 loudnorm 재구현 금지). **스킵(원본 유지 + `video.json.note` 사유 = 침묵 금지)** = 오디오 없음 · 오디오 확인 실패 · 소리 감지 안 됨(디지털 무음·좌우 완전 역위상 = 모노합 상쇄) · 소리가 거의 없음(−55LUFS 미만 룸톤 = 히스 증폭 방지) · 처리 실패 · 시간 초과. **측정만 실패 = 단일패스 폴백**(스킵 아님) = `근사(1패스)` 표기.
- d) **구간 트림**: 시작~끝 초(입력 옵션 `-ss`/`-t` = 재인코딩이라 프레임 정확). **해상도**: 원본 · 1080p · 720p(긴 변 캡 다운스케일만 — 업스케일 안 함) · 짝수 정렬 = 크로마 시프트 방지.
- e) **대기 화면** = 운영자 픽(260710 플레이그라운드 p2 스캔라인 ★): 원본 미리보기(muted 저속 루프 · 업로드 시) + 스크림(8,15,11,.48)+blur5 + 스캔 빔 1.4s + 프리뷰 안 좌하단 `변환 중‥ (NNNs)` — 트래킹 렌더 대기와 동일 문법.

## 2. 캡·예산 (기틀 — 완화 = 운영자 확인)
- a) 일반(크롭·트림·스케일·fps다운) = **트림 후 유효 길이 300초**(트래킹 분석 캡 선례) · 입력 긴 변 **3840px(4K)** 캡(260711 상향 · 8K 거절). **4K급 출력(캔버스 픽셀 > FHD 2배 — 긴 변 판별은 세로 폰영상 오분류라 픽셀 기준 · 평의회4·5) = 120초 캡**{`CONV_4K_MAX_SEC` · 픽셀 4배 = 인코딩 폭발이라 별도 예산 · 60i×4K는 기존 예산식이 자동 제한(약 20초)}. **pad 캔버스 캡(res 결측) = min(3840, max(1920, 소스 긴 변))** — 소스≤1920은 종전 1080×1920 스냅과 동일(회귀 0 · 평의회3·5). **URL 소스는 1920 페치 유지**(전 URL 잡 4K화 = 비용 — 4K는 파일 업로드로).
- b) **60fps 보간 = 120초 캡(운영자 260710 승인) + 발사 전 예산 가드**: 실측 단가(4vCPU=러너 동급 · 260710) minterpolate 기본 270ms/출력프레임@1080×1920 → `INTERP_S_PF 0.30 × 출력프레임 × 해상도비 ≤ INTERP_BUDGET 1500s`. **1080×1920 풀 ≈ 83초 · 720p급 = 120초 풀** — 초과 = "해상도를 720p로 낮추거나 구간을 잘라줘" 정직 거절(키잉 예산 가드 문법). 상위 프리셋(bidir·aobmc·vsbmc) = 약 2배 비용(515ms/f 실측)이라 채택 안 함 — 재검토 시 실측 먼저.
- c) ffmpeg 백스톱 timeout 1900s(R2 산출 업로드 480s) · 스텝 33분 캡(잡 58분 — R2 선스텝 8분 포함 재산정 260711 — 하드캔슬 전 failure() error.log 표면화 — track 문법) · URL 소스 확보 스텝 8분 캡(yt-dlp 행 차단). 뷰어 폴 = 예산 1900s(하드 ×2 = 3800s ≥ 잡 캡+배포 — 뷰어가 잡보다 먼저 포기 안 함) · **재개 게이트 = 24h**(pend 유지 = `완성되면 다음에 열 때 이어받아` 실현 · track 동일).
- d) 예산 가드 마진의 정직 경계(평의회2): 단가 상수 0.30은 실측 0.27의 +11% — 러너 CPU 편차(±30%)는 가드 단독으론 못 덮고 **FF_TIMEOUT 1900s(명목 +40%)가 실백스톱**(초과 = 정직 타임아웃 메시지·무음 스텝킬 아님). 극꼬리(ffmpeg 1900 근접 완주 + R2 행 동시)는 제네릭 error.log = 수용 트레이드오프.

## 3. 알려진 한계 (정직)
- a) 보간 = 모션 보상 근사 — 빠른 회전·큰 오클루전에서 워블/고스팅 가능(원리 한계 · RIFE급 ML 보간은 러너 CPU에서 비현실 = 후속감).
- b) URL 소스 = 위치 드래그 프리뷰 불가(서버만 치수 인지) · 사이트 봇 차단 시 yt-dlp 실패 = 파일 업로드 우회 안내.
- c) VFR 폰 영상 = fps 필터가 공칭 fps 기준 재클럭 — 심한 VFR은 미세 A/V 드리프트 가능(트래킹 3-e 동일 축).
- d) 크롭 pos = 전 구간 고정(키프레임 팬 없음 — 후속 = 구간별 팬).
- e) 완료·실패 웹푸시 연결(260711 — `conv-make.yml` 완료 푸시 스텝 · 편집기 문법 · 탭 = `/conv.html`(pend 24h가 결과 복원) · 구독 OFF/VAPID 미설정 = 조용히 생략).
- f) 아나모픽(SAR≠1) 소스 = 비회전만 표시 공간 정규화(scale+setsar 선행 — 크롭 축이 프리뷰와 일치) · **회전+SAR 동시 소스는 미정규화 잔여**(희귀 — 폰·yt-dlp 경로는 정방픽셀) · 레거시 tags.rotate 전용(displaymatrix 없는) 구형 파일 = 회전 미인지 가능(트래킹 동일 축).
- g) 음량 통일 = 오디오 aac 2세대(본인코딩 160k → 후처리 192k) — 음성 콘텐츠에서 비가청 수준(수용 트레이드오프 · 후처리 = 측정 정확성 우선). 여백(pad) 모드 = 영상 위치 중앙 고정(팬 없음 — 필요 시 후속). 고크레스트(순간 피크가 큰) 소스 = 피크 제약이 우선이라 −14까지 못 올라갈 수 있음(운영자 요구 = `0dB 안 넘게`가 상위 · 평의회4) · 매우 큰 소스는 −14로 크게 내려감(통일이 목표) · 스테레오 음악은 모노합으로 공간감 감소(뉴스 인터뷰 무해 = 의도 동작).
- h) **통합 영상 편집기(260710 평의회 10인 · 운영자 2단계 확정)**: 이 도구는 P1에서 신설될 `편집기` 탭으로 점진 이관 예정{레시피/카테고리 골격 = 플레이그라운드 시안 후 확정 · 기존 탭 병존 → 검증 후 진입로 도먼트}. 그전까지 본 지침이 정본.

## 4. 3자 계약 (뷰어 ↔ api ↔ py — 키 이름 변경 = 기틀)
- a) 페이로드 `opts{fps:'keep|60i|30|24', ar:'orig|9:16|1:1|4:5|16:9', fit:'crop|pad'(비율 선택 시 · 기본 crop), pos:0..1(crop만), t0:s, t1:s, res:'orig|1080|720', audio:'keep|norm'}` — api 화이트리스트 클램프 + py 실측 dur 재클램프 = 이중 방어.
- b) `video.json{url, w, h(=패드 시 캔버스 치수), fps, dur, size_mb, ts(KST), note?(음량 통일/스킵 사유 — 있을 때만)}` · 실패 = `error.log` **즉시 보고**(conv id = 단발·분석 스텝 없음 = 스테일 잔존 없음 — track 렌더의 기준선 폴과 의도적 상이 · 평의회5).
- c) id = `YYMMDDHHMMSS-hex6`(KST · pick.js 규칙) · 업로드 = R2 `up_src/<id>.<ext>`(페이로드 `r2key` → 워크플로 입력 `r2_src` · 잡 끝 `aws s3 rm`) 또는 폴백 `uploads/<id>/src.*`(잡 끝 git rm) · R2 키 = `conv_res/<id>/out.mp4`.
