## [PHASE 0] 입력 감지 & 자동 분기
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

사용자 입력을 감지하여 자동으로 분기한다:

### A) mp4/영상 파일이 업로드된 경우 → STT 파이프라인 실행

**즉시 아래 순서를 자동 실행한다. 사용자에게 되묻지 않는다.**

> ⚡ **빠른 시작(환경 캐싱 = 정답)**: **클라우드 환경의 "Setup script"에 `bash apps/ly/setup.sh`를 등록**하면, large-v3-turbo 모델을 **처음 한 번만 받고 파일시스템 스냅샷에 캐시**한다 → 이후 세션은 모델이 디스크에 있는 채 시작(**재다운로드 없음**, ~7일 만료나 스크립트 변경 시에만 재빌드). 이게 매 세션 재다운로드를 없애는 정답이다(설정은 클라우드 UI / turbo 1.6GB라 Setup script ~5분 제한에 안전). ⚠️ 환경에 안 넣었으면 **폴백**: `/ly` 진입 시 `setup.sh`를 백그라운드 실행(이 경우 매 세션 받음). SessionStart 훅(레포 `.claude/settings.json`)은 **캐시 안 됨** → 비권장.

```
STEP 0-1. ffmpeg로 오디오 추출
STEP 0-2. Whisper STT 실행 (large-v3-turbo·로컬·키 불필요)
STEP 0-3. STT 결과를 PHASE 1~3(자막 변환)으로 전달
```

#### STEP 0-1: 오디오 추출

```bash
ffmpeg -i "[업로드된 파일 경로]" -ar 16000 -ac 1 -f wav /home/claude/stt_audio.wav -y
```

#### STEP 0-2: Whisper STT 실행 (large-v3-turbo·로컬·키 불필요)

**모델 = `large-v3-turbo` 고정.** large-v3의 95%+ 정확도를 유지하면서 1.6GB·다운로드/추론 4배 빠름(한국어 양호 — 실측: 시위 클립도 `재선거`까지 정확, lang ko 0.98). 환경 Setup script 캐싱으로 재다운로드 없이 시작.

```python
from faster_whisper import WhisperModel

# large-v3-turbo = 정확도≈large-v3·1.6GB·4배 빠름(기본). 최고 정확도가 꼭 필요하면 "large-v3"로 올릴 수 있음.
model = WhisperModel("large-v3-turbo", device="cpu", compute_type="int8")
segments, info = model.transcribe(
    "/home/claude/stt_audio.wav",
    language=None,                      # 자동 감지 (한국어 → ko)
    beam_size=5,
    vad_filter=False,                  # ⚠ 이 환경 VAD 오작동(전 구간 과필터→0개) 실측 → 끔
    condition_on_previous_text=False,  # 반복 환각 루프 억제
)
result = [{"t": f"{int(s.start)//60:02d}:{int(s.start)%60:02d}", "text": s.text.strip()}
          for s in segments]
print(f"🎙️ Whisper STT(large-v3-turbo): {len(result)}개 세그먼트 ({info.language} {info.language_probability:.2f})")
```

- ⚠️ **품질/비용**: large-v3-turbo는 1.6GB·4배 빠르고 정확도는 large-v3의 95%+(자막용 체감 차이 거의 없음). 재다운로드는 환경 Setup script 캐싱으로 회피. 현장 소음 영상은 일부 오인식 가능 → 불명확 구절은 자막에 플래그하고 사용자 교정받는다. (최고 정확도가 필요하면 STEP 0-2 모델을 `large-v3`로 올림.)
- **언어**: `language=None` 자동 감지(한국어→ko). 외국어는 원문 그대로 트랜스크립션되고, 번역·의역은 PHASE 2에서. 숫자 아라비아 표기·비음성 정리는 PHASE 2~3 [표기 규칙]에서 처리.

#### STEP 0-3: STT 결과 파싱 후 PHASE 1로 전달

- 「🎙️ STT 완료: X개 세그먼트 (Whisper large-v3-turbo)」 출력 후 자막 변환 시작

## [절대 규칙 - 위반 시 작업 실패]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 내용 누락 절대 금지. 원본의 모든 문장, 모든 구절을 빠짐없이 자막으로 변환한다.
2. 의미 단위 삭제·병합 금지. 원본의 모든 의미 단위를 빠짐없이 포함한다. 단, 수식어·부사·반복 표현 정리는 의역 범위 내 허용이며, 이로 인한 분량 감소는 정상이다.
3. **1박스 = 1문장 원칙 (NEW v2.0 핵심 규칙)**: 한 자막 박스는 한 문장만 담는다. 문장 구분은 마침표(.), 물음표(?), 느낌표(!) 기준. 한 박스에 두 문장 이상을 묶지 않는다.
4. 한국어 자막은 한 줄당 공백 포함 **11자 이내 권장, 최대 13자**. 13자 초과 시 반드시 줄을 나눈다.
   - 기준: "미군 241명이 죽었다." = 한 프레임에 보이는 최대치 (13자).
   - 한 자막 박스는 **최대 3줄**을 넘지 않는다 (한국어 기준).
5. 자막 출력 모드를 준수한다. 기본값은 **통합 모드** (박스마다 영한 통합 박스 + 한국어 단독 박스 = 코드블록 2개 출력). 모드별 상세 규칙은 [STEP 3: 포맷팅 & 출력] 참조.

## [박스 단위 규칙 — v2.0 핵심]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 1박스 = 1문장 원칙

- **문장 구분 기준**: 마침표(.), 물음표(?), 느낌표(!).
- **두 문장 한 박스 묶기 금지**: STT/SRT 한 세그먼트에 두 문장 이상이 묶여 있으면 반드시 분리하여 각각 별도 박스로 출력.
- **한 문장 내 임의 분할 금지**: 콤마, 접속사(and, but, so 등)로 한 문장을 임의 분할하지 않는다. 한 문장이 길어도 한 박스 안에서 줄바꿈으로 처리.
- **목적**: 영상 편집 시 자막 박스 단위 = 화면 표시 단위. 한 박스 = 한 자막 화면.

■ 예외

- **한국어 원본 모드**는 1박스=1문장이 아니라 **짧은 구절 단위로 잘게 분할**한다(아래 [모드 정의] 한국어 원본 모드 참조). 화자의 호흡·의미 단위(5~13자)에서 끊고, 조각마다 개별 코드블록으로 복사 가능하게 출력. 마침표가 없는 연속 발화도 자연 호흡 지점에서 분리한다.

■ 카운팅 변화

- 원본 세그먼트 수와 최종 자막 박스 수가 다를 수 있다.
- 두 문장 묶인 세그먼트가 있으면 박스 수 > 원본 세그먼트 수.
- 카운팅 선언: 「원본 구절 수: X개 → 자막 박스 수: Y개」

## [모드 정의 & 전환 규칙]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 3가지 출력 모드

| 모드 | 조건 | 설명 |
|------|------|------|
| **통합 모드** (기본값) | 외국어 소스 | 박스마다 코드블록 2개 출력: ① 영한 통합 박스(영문+한국어) ② 한국어 단독 박스. 영상 편집 시 영한 같이 / 한국어만 그때그때 선택 복사. |
| **분리 모드** | 사용자 요청 시 | 외국어 원문: 코드블록 밖 / 한국어 의역: 코드블록 안 |
| **한국어 원본 모드** | 소스가 한국어일 때 자동 | 번역 없이 한국어만 출력. **짧은 구절(5~13자) 단위로 잘게 끊어, 조각마다 개별 코드블록**(영상 편집 시 조각별로 탭 한 번 복사). |

■ 코드블록 설계 철학

- **코드블록 안 = 복사 영역**: 영상 편집 앱에 그대로 붙여넣는 자막 텍스트만 포함.
- **코드블록 밖 = 인덱싱 영역**: 박스 번호 `(N/총개수)`, 타임코드 — 사용자가 어디에 넣을지 확인용. 복사 대상 아님.
- **통합 모드의 2박스 구조**: 한 자막 박스(인덱싱 단위)에 코드블록 2개 — 위는 영한 통합본, 아래는 한국어 단독. 두 코드블록의 한국어는 완전 동일. 사용자가 그 박스 자막을 영상에 넣을 때 영문도 같이 띄울지(영한 박스 복사) / 한국어만 띄울지(한국어 박스 복사)를 클릭 한 번으로 선택.

■ 모드 전환 우선순위

1. **소스 언어 감지가 최우선**: 소스가 한국어이면 한국어 원본 모드 자동 적용. 통합/분리 전환 트리거보다 우선.
2. **한국어 원본 모드에서 통합/분리 트리거 발동 시**: 소스가 한국어이므로 전환 불가. "이 영상은 한국어 원본이라 통합/분리 모드가 적용되지 않아"라고 안내.
3. **통합 ↔ 분리 전환**: 외국어 소스에서만 유효.

■ 전환 트리거

- **분리 모드 전환**: "분리해줘", "따로 빼줘", "한국어만 복사하게", "영문 밖에"
- **통합 모드 복귀 (기본값 복귀)**: "합쳐줘", "같이 넣어줘", "한 박스에", "코드블록 하나로", "통합", "기본으로", "원래대로"

## [의역 철학 - 이것이 핵심]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 기본 원칙: "직역의 정확도 + 의역의 임팩트"

- 직역하면 딱딱해지고, 과하게 의역하면 원문을 왜곡한다.
- 정확한 지점: 원문의 의미를 100% 유지하되, 한국어로 '때렸을 때' 타격감이 있는 표현을 쓴다.
- 번역이 아니라 "한국어로 다시 말하기"에 가깝다.

■ 압축 우선 원칙

- 원문의 수식어·부사·반복을 과감히 덜어낸다.
- 의미가 유지되는 한, 짧을수록 좋다.
- "I say tonight that" → "경고한다" (O)
- "a vicious group of very hard, terrible people" → "잔인하고 끔찍한 놈들" (O)
- "I say tonight that" → "오늘 밤 여러분에게 말씀드립니다" (X, 너무 김)
- "잔인하고 매우 강경하며 끔찍한 사람들로 구성된 사악한 집단" (X, 직역병)

■ 어순 보정 원칙 (영한 자막 최대 난관)

- 영어와 한국어는 어순이 반대다. 영문 자막과 한국어 자막이 같은 박스에 함께 뜬다(통합 모드).
- 한 박스 = 한 문장이므로 영문 한 문장과 한국어 한 문장이 의미적으로 1:1 대응되어야 한다.
- 직역 어순으로 영문 흐름을 그대로 쫓아가면 한국어가 어색해진다.
- 해결법: 한국어를 자연어순으로 재배치하되, 영문 한 문장과 의미 단위가 일치하도록 유지.

  BAD (영어 따라가기):
  영문: "the US military began major combat operations in Iran"
  한국어: "미합중국 군대가 시작했습니다 대규모 전투 작전을 이란에서"

  GOOD (한국어 자연어순):
  영문: "the US military began major combat operations in Iran"
  한국어: "미군은 이란을 상대로 대규모 전투를 개시했다"

■ 끝맺음 원칙

- 문장 끝은 짧고 단정적으로 끊는다.
- "~했다", "~이다", "~한다", "~된다"가 기본.
- 피하는 끝맺음: "~하게 되었습니다", "~라고 할 수 있겠습니다", "~인 셈이죠"
- 감정을 실을 때만 "~거다", "~뿐이다", "~않겠다" 허용.

■ 톤 스위칭 원칙

- 화자의 대상에 따라 톤이 자동으로 전환되어야 한다.
- 자막은 화자의 말투를 반영한다. 뉴스 앵커가 번역한 게 아니라, 화자가 한국어로 말하는 것처럼.

  | 맥락         | 톤       | 예시                     |
  |------------|--------|------------------------|
  | 사실 서술/역사   | 담백 사실체 | "미군 241명이 죽었다."       |
  | 선언/결의      | 단언체    | "숭고한 임무다."             |
  | 경고/위협      | 반말 명령체 | "무기를 내려놔라."            |
  | 국민/대중 향    | 존칭     | "집 밖에 나오지 마십시오."      |
  | 감정 토로      | 구어체    | "더 이상 못 참는다."         |
  | 자기 과시      | 자신감체   | "세계 최강이다. 반드시 이긴다."   |

- 한 영상 안에서 톤이 자연스럽게 전환되는 게 정상.
- **판단 애매 시 기본값: 담백 사실체(중립).**
- **한국어 원본의 경우**: 화자의 원래 말투를 최대한 유지. 반말이면 반말, 존댓말이면 존댓말.

## [말투 세부 규칙]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 핵심: "짧고 담백하게 때리기"

- 꾸미지 말고 덜어낸다. 실제로 말하듯 자연스럽게.
- 은어 허용: "놈들", "박살 냈다", "밀어 버린다", "걷어찼다" — 맛깔나는 구어 OK.
- 혐오·차별 뉘앙스 표현은 금지.
- 자음 축약 (ㅇㅇ, ㄹㅇ, ㅋㅋ 단독) 지양. 글자로 풀어 쓴다.

■ 금지 패턴

- 원문보다 길어지는 의역 금지.
- 불필요한 강조 부사 남발 금지 ("진짜 무조건 싹 다 완전히" 류).
- 설명조 금지. 자막은 내뱉는 말이지 설명하는 글이 아니다.
- 번역투 금지: "~하는 것이다", "~라고 할 수 있다", "~에 대해서"
- 과한 인터넷 말투 금지: "~거임", "~잖아 ㄹㅇ?"

■ 의역 허용/불허 가이드

| 구분       | 허용                            | 불허                      |
|----------|-------------------------------|-------------------------|
| 수식어 삭제   | "very good friends" → "우방"    | 핵심 의미 삭제                |
| 함축 압축    | "I say tonight that" → "경고한다" | 의미 변형 ("경고한다" → "협박한다") |
| 어감 강화    | "obliterated" → "박살 냈다"       | 없는 뉘앙스 추가 ("신나게 박살 냈다") |
| 어순 재배치   | 한국어 자연어순으로                     | 원문 의미 뒤바꿈               |
| 존칭/반말 전환 | 화자 대상에 따라                      | 일관성 없는 랜덤 전환            |

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
## [표기 규칙]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 숫자·단위 표기

- 숫자는 **아라비아 숫자 표기 원칙**: "이백사십일 명" (X) → "241명" (O)
- 큰 단위는 한글 혼용 가능: "3억 달러", "14만 명"
- 서수는 맥락에 따라: "제1차", "첫 번째", "1번째" 모두 허용 (가독성 우선)

■ 통화·기호 표기

- **원문 통화 기호 유지 우선**: $500만, €200, £30억
- 기호가 한국 독자에게 생소할 경우 한글 병기: "500만 바트", "200만 루피"
- 원(₩)은 생략 가능 (한국 독자 기본 통화): "3,000만 원" 또는 "3,000만"

■ 외국어 원문 줄 바꿈

- 통합/분리 모드에서 외국어 원문은 가독성을 위해 **5~7단어 내외에서 줄 바꿈**한다.
- 의미 단위(구·절)가 끊기지 않는 지점에서 줄을 나눈다.
- 한 문장이 길어 줄바꿈 후에도 영문이 3줄 이상이 되면, 한국어 의역 압축으로 한국어 줄 수를 줄여 박스 균형을 맞춘다.

■ 비음성 태그 처리

- STT에서 감지된 비음성 태그 — (웃음), (박수), (환호), (한숨), (감탄) — 의 자막 변환 규칙:
  - **단독 비음성 세그먼트**: 자막 박스로 출력한다. 코드블록 안에 태그 그대로 표기.
    ```text
    (웃음)
    ```
  - **대사 + 비음성 혼합 세그먼트**: 대사만 자막으로 변환하고, 비음성 태그는 삭제한다. (영상에서 이미 보이므로)
  - **연속 비음성 태그**: 같은 태그가 연속 2회 이상이면 1회만 출력한다.
  - **(불명확) 태그**: 문맥으로 추정 가능하면 추정 내용을 자막으로 변환. 추정 불가면 해당 세그먼트를 "[불명확]"으로 표기하되, 자가검증에서 별도 보고한다.

## [작업 프로세스 - 3단계]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ STEP 1: 전수 파싱 + 문장 단위 재분할

- SRT의 경우: 모든 시퀀스 번호와 타임코드를 하나도 빠짐없이 읽는다.
- STT 결과(Whisper segments)의 경우: 마침표/물음표/느낌표 기준으로 **문장 단위 재분할**한다.
  - 한 세그먼트에 두 문장 이상이 있으면 분리해서 각각 별도 박스로 처리.
  - 한 문장 안의 콤마·접속사는 분할 기준 아님.
- 이 단계에서 원본 세그먼트 수 + 최종 박스 수를 카운팅한다.
- ★ 카운팅 결과를 출력 맨 위에 반드시 선언한다:
  「원본 구절 수: X개 → 자막 박스 수: Y개」
  (Y가 X와 다르면 그 이유를 한 줄로: "5번 세그먼트 두 문장 분리됨" 등)

■ STEP 2: 의역 변환

- 원문 한 문장을 한국어 한 문장으로 옮긴다.

  [의역 체크포인트 - 매 박스마다 자문]
  ① 이 한국어만 보고 의미가 통하는가?
  ② 원문 의미를 왜곡하지 않았는가?
  ③ 한 줄 13자를 초과하지 않는가? (11자 이내 권장)
  ④ 한 박스 한국어가 3줄을 초과하지 않는가?
  ⑤ 한 박스 안 영문 1문장 ↔ 한국어 1문장 의미 대응이 자연스러운가?
  ⑥ 화자의 톤/대상에 맞는 말투인가?

- 한국어 원본인 경우: 번역 없이 한국어만 출력하되, **글자 수 제한과 줄 바꿈 규칙**은 동일하게 적용한다.

■ STEP 3: 포맷팅 & 출력

**[통합 모드] (기본값 — 외국어 영상)**

박스마다 코드블록 2개를 출력한다. 인덱싱(박스 번호 + 타임코드)은 두 코드블록 위에 한 번만.

(현재순번/전체개수) 시작시간

```text
외국어 원문
(5~7단어 내외에서 줄 바꿈)

한국어 의역
(한 줄 11자 권장, 최대 13자)
(최대 3줄)
```


```text
한국어 의역 (영한 박스의 한국어와 정확히 동일)
(같은 줄바꿈 유지)
```

**[분리 모드]** (사용자 요청 시)

(현재순번/전체개수) 시작시간

외국어 원문
(5~7단어 내외에서 줄 바꿈)

```text
한국어 의역
(한 줄 11자 권장, 최대 13자)
(최대 3줄)
```

**[한국어 원본 모드]** (한국어 영상/한국어 STT) — **짧은 조각 + 조각마다 개별 코드블록**

- 연속 발화를 **짧은 구절(5~13자)로 잘게** 끊는다. 한 조각 = 한 코드블록(복사 단위). 조각이 길면 2줄까지.
- 인덱스 `(N/총개수) 시작시간`은 각 코드블록 위에 한 번.

(1/N) 시작시간
```text
짧은 구절 1
```
(2/N) 시작시간
```text
짧은 구절 2
```
(…조각마다 반복)

**[타임코드 변환 규칙]**
- SRT 타임코드 → 간결한 형태로 변환, 밀리초는 버린다.
- 영상 길이 판단: **마지막 타임코드(SRT 마지막 시퀀스의 종료 시간, 또는 STT 마지막 세그먼트의 타임스탬프)** 기준.
- 영상 길이 1시간 미만: **M:SS** (예: 00:01:23,456 → 1:23)
- 영상 길이 1시간 이상: **H:MM:SS** (예: 01:23:45,678 → 1:23:45)
- Whisper STT 결과: MM:SS → M:SS로 변환 (00:03 → 0:03)
- 타임코드가 없으면 추정 시간으로 표기하되, 「타임코드 추정」이라고 맨 위에 명시

## [자가 검증 - 출력 완료 후 필수 실행]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

출력을 모두 마친 뒤, 아래 체크리스트를 **현재 모드에 해당하는 항목만** 점검한다:

**[공통 — 모든 모드]**
□ 원본의 핵심 정보/의미 단위가 빠짐없이 포함되었는가?
□ 누락된 문장이나 구절이 없는가?
□ **1박스 = 1문장 원칙을 지켰는가? (마침표/물음표/느낌표 기준)**
□ **한 박스에 두 문장 이상 묶인 케이스가 없는가?**
□ 한 문장을 콤마/접속사 기준으로 임의 분할한 케이스가 없는가?
□ 한국어 자막이 한 줄 13자를 초과한 것은 없는가?
□ 한 박스 한국어가 3줄을 초과한 것은 없는가?
□ 톤 스위칭이 자연스럽게 적용되었는가?
□ 숫자·단위가 아라비아 숫자 원칙에 맞게 표기되었는가?
□ (불명확) 태그가 남아있다면 별도 보고했는가?

**[통합 모드 추가]**
□ 각 박스마다 영한 통합 박스 + 한국어 단독 박스 = 코드블록 2개 출력됐는가?
□ 두 코드블록의 한국어 내용·줄바꿈이 정확히 동일한가?
□ 박스 번호와 타임코드(인덱싱)는 두 코드블록 위에 한 번만 표시됐는가?
□ 영한 박스 안에서 영문 1문장 ↔ 한국어 1문장 의미 대응이 자연스러운가?

**[분리 모드 추가]**
□ 모든 외국어 자막에 한국어 의역이 포함되어 있는가?
□ 한국어 코드블록 안에 영문이 섞이지 않았는가?

**[한국어 원본 모드 추가]**
□ 화자의 원래 말투(반말/존댓말)가 유지되었는가?

→ 불일치 발견 시: 누락분을 즉시 보충 출력한다.
→ 전부 통과 시: 「✅ 검증 완료: 원본 X개 → 자막 Y개, 1박스=1문장 원칙 준수」 출력

- Claude 경향: 정확하지만 딱딱해지기 쉬움. "~하게 되었다", "~을 수행하다" 류의 문어체 주의. → "수행하다" → "한다", "제거하게 되었다" → "없앴다"로 구어체 전환 의식할 것.
