## [260605] 쇼잉 도구 오류 — `Read` vs `SendUserFile`

**무슨 일이 있었나 (한 줄)**
카드 `_final_` 6장을 채팅에 띄울 때 `Read` 툴을 썼는데, 그건 에디터(나)만 보는 검수용이라 **사용자 화면엔 아무것도 안 떴다.** 원격 실행 환경(Claude Code 웹/클라우드)에선 `SendUserFile`로 보내야 사용자에게 보인다. 한 사이클(쇼잉) 헛돌았음.

**왜 헷갈렸나**
`03_자동화_레퍼런스.md`의 프리뷰 절차가 "`Read {temp.jpg}` → 채팅 렌더"로 적혀 있음. 이건 데스크톱/로컬 클라이언트 기준이고, 원격 실행 환경엔 안 맞는다.

### ✅ 고칠 곳 (3군데)

**1) `03_자동화_레퍼런스.md` — "프리뷰" 섹션 4단계**
- 현행 4번: `Read {temp.jpg}` → 채팅 렌더
- 수정안: 4번을 「**`SendUserFile`로 전 카드 한 번에 전송(+캡션)**」으로 교체.
- 섹션 상단에 ⚠️ 한 줄 추가: "원격 실행 환경(웹/클라우드) 쇼잉 = `SendUserFile` 필수. `Read`는 에디터 자가 검수 전용(사용자 화면에 안 뜸). 데스크톱/로컬 클라이언트만 Read 렌더 가능."
- (1~3단계 = files.list 한 콜 + curl 다운로드는 그대로 유지. **마지막 '표시' 단계만 교체.**)

**2) `CLAUDE.md` — §2 ② 발사 단계 '쇼잉' 문구**
- 현행: "…전 카드 합성본(_final_)을 이 채팅에 전부 띄우고 + ③ 첫 질문…"
- 수정안: "띄우고" 뒤에 짧은 포인터 삽입 → "(원격=`SendUserFile`로 전송 / `Read`는 검수용 · 상세 03)".

**3) `PROJECT_MEMORY.md` — 결정 로그**
- 추가: "(260605) 쇼잉 도구 확정 — 원격 실행 환경 = `SendUserFile`로 사용자 전송, `Read`는 자가 검수용(사용자 화면 미표시). 03 프리뷰 절차 갱신. 다음 세션 혼동 방지."

### 성격 판정
주변부(절차·문구) 갱신이라 **기틀 변경 아님** (라이브러리↔지침 분리·3단계·참조 방식·출력 포맷 골격·INVARIANTS 무관) → 사용자 사전 확인 불요. 단 시스템 파일이므로 `깃` 수정 모드 절차(백업→수정→push)로 실행.
