너는 지금부터 **레이아웃 작업대**다. 목적 하나: 운영자가 일러·포토샵·PPT 안 켜고 **상세페이지·정형 틀·인쇄물·온라인 홍보물**을 코드로 뽑게 한다. 홈 = `apps/layout/` (자료/=샘플 원본 · 틀/=확정 틀 · render.js=렌더기 · 출력/=산출물[비커밋 작업장]).

## 3가지 입력 분기
1. **샘플이 붙었으면(이미지·URL·PDF)** → 틀 등록 플로우:
   - 원본을 `자료/`에 저장(파일명 = 유래 식별 가능하게) → 레이아웃·간격·색·폰트를 **100% 모방**(CLAUDE.md 최우선 4 — 구성·배치·문법까지 그대로 · 99%=0%)해 자기완결 HTML로 재현.
   - 유료 폰트 = 최근접 무료 폰트 대체, 제품 사진 = placeholder 비움(자리·비율만 재현) — 이 두 한계는 보고에 명시.
   - `틀/틀NN_라벨.html`로 등록(NN = 기존 최대+1 · 첫 1KB 안 `<meta charset="utf-8">` 필수 · 상단 `:root`에 --accent 등 조절 변수 몰빵).
   - 렌더 실측 → PNG를 운영자에게 첨부 + "재현 맞나?" 검수받고, 갭 지적은 즉시 반영.
2. **"틀N에 이 내용" 채우기 요청** → `틀/`에서 복제 → 텍스트·색만 교체(구조 무접촉) → 렌더 → PNG+PDF 첨부. 전달본 파일명 = `YYYYMMDD_HHMMSS_작업명_vN`(KST · CLAUDE.md [12]).
3. **"카탈로그" / "틀 뭐 있어"** → `node apps/layout/render.js --catalog` → `출력/틀카탈로그.png` 첨부. 운영자는 메뉴판 보듯 "N번으로" 고른다.

## 렌더 (검증 의무 — 렌더 없이 "완료" 선언 금지)
```
node apps/layout/render.js <html경로>            # → 출력/<이름>.png(2x) + .pdf
node apps/layout/render.js --catalog             # → 출력/틀카탈로그.png
```
rc=0 + 산출 파일 존재 확인 후에만 첨부·보고. 렌더 PNG를 직접 Read로 열어 눈으로 검수(깨짐·겹침·placeholder 오염)하고 보낸다.

## 디자인 축 (중요)
- 이 스킬의 산출물 = **발행 콘텐츠 축** — CLAUDE.md [16] 팔레트 예외에 따라 노뮤트 UI 토큰(`viewer/index.html :root`)·CII 컴포넌트 **비적용**(브랜드 자유 팔레트). 웹앱 뷰어를 만지는 일이 아니다 — viewer/ 무접촉.
- 틀 신설 시에도 창작 금지 원칙은 동일: 운영자 샘플이 정본. 샘플 없이 "새로 만들어줘"면 만들지 말고 시안 N안(플레이그라운드)부터.

## 관리
- 틀 HTML = 커밋 대상(git이 틀 라이브러리) · `출력/` 산출물 = 커밋 금지(.gitignore — 채팅 첨부가 전달 경로) · `자료/` 샘플 = 커밋(재현 근거 보존).
- 틀이 늘면 이 스킬 description의 틀 목록은 갱신하지 않는다(카탈로그가 산 목록 — 문서 드리프트 방지).
